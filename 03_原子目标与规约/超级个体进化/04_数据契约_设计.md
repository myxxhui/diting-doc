## 一、存储拓扑

| 数据类 | 主存 | 索引 | 归档 |
|--------|------|------|------|
| 评测集 / 任务 / 报告 | PostgreSQL + 对象存储（dataset 文件） | OpenSearch | OSS |
| 反馈（显式 + 隐式） | PostgreSQL（按月分区）| OpenSearch | OSS |
| 知识条目 + 嵌入 | PostgreSQL + 向量库（Milvus / pgvector / OpenSearch） | OpenSearch | OSS |
| 复盘报告 | PostgreSQL + 对象存储 | OpenSearch | OSS |
| 版本注册 | PostgreSQL（WORM） | — | OSS |
| 外部动作记录 | PostgreSQL（WORM） | OpenSearch | OSS（永久） |
| 用户成长档案 | PostgreSQL（按 user 分片） | — | 用户控制 |

## 二、表结构（PostgreSQL）

### 2.1 评测中心

```sql
CREATE TABLE eval_datasets (
  id TEXT NOT NULL,
  version TEXT NOT NULL,
  description TEXT,
  size INT NOT NULL,
  storage_ref TEXT NOT NULL,
  approved_by TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (id, version)
);

CREATE TABLE eval_tasks (
  id UUID PRIMARY KEY,
  kind SMALLINT NOT NULL,                 -- 1=OFFLINE, 2=ONLINE_AB, 3=CANARY
  target_artifact_type SMALLINT NOT NULL,
  target_artifact_id TEXT NOT NULL,
  target_version TEXT NOT NULL,
  baseline_version TEXT,
  dataset_id TEXT,
  dataset_version TEXT,
  config JSONB,
  status TEXT NOT NULL DEFAULT 'pending',
  report_id UUID,
  started_at TIMESTAMPTZ,
  ended_at TIMESTAMPTZ
);

CREATE INDEX idx_eval_tasks_status ON eval_tasks(status);
CREATE INDEX idx_eval_tasks_target ON eval_tasks(target_artifact_type, target_artifact_id, target_version);

CREATE TABLE eval_reports (
  id UUID PRIMARY KEY,
  task_id UUID NOT NULL REFERENCES eval_tasks(id),
  artifact_type SMALLINT NOT NULL,
  artifact_id TEXT NOT NULL,
  version TEXT NOT NULL,
  metrics JSONB NOT NULL,                 -- [{name, baseline, candidate, delta, passed, note}, ...]
  summary TEXT,
  approver TEXT,
  approval_status TEXT NOT NULL DEFAULT 'pending',
  generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_eval_reports_artifact ON eval_reports(artifact_type, artifact_id, version);
CREATE INDEX idx_eval_reports_generated ON eval_reports(generated_at DESC);
```

### 2.2 反馈

```sql
CREATE TABLE feedback_explicit (
  id UUID NOT NULL,
  user_id TEXT NOT NULL,
  subject_type TEXT NOT NULL,
  subject_id TEXT NOT NULL,
  kind SMALLINT NOT NULL,
  rating INT,
  comment TEXT,
  context JSONB,
  at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (id, at)
) PARTITION BY RANGE (at);

CREATE INDEX idx_feedback_explicit_subject ON feedback_explicit(subject_type, subject_id);
CREATE INDEX idx_feedback_explicit_user_at ON feedback_explicit(user_id, at DESC);

CREATE TABLE feedback_implicit (
  id UUID NOT NULL,
  user_id TEXT NOT NULL,
  subject_type TEXT NOT NULL,
  subject_id TEXT NOT NULL,
  kind SMALLINT NOT NULL,
  payload JSONB,
  at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (id, at)
) PARTITION BY RANGE (at);

CREATE INDEX idx_feedback_implicit_subject ON feedback_implicit(subject_type, subject_id);
CREATE INDEX idx_feedback_implicit_user_at ON feedback_implicit(user_id, at DESC);
```

### 2.3 知识库

```sql
CREATE TABLE knowledge_entries (
  id UUID PRIMARY KEY,
  type SMALLINT NOT NULL,                 -- 1=FACT, 2=RULE, 3=PLAYBOOK, 4=GLOSSARY
  subject_ids TEXT[] NOT NULL DEFAULT '{}',
  content TEXT NOT NULL,
  embedding_ref TEXT,                     -- 向量库引用（vec_id）
  source_refs TEXT[] NOT NULL DEFAULT '{}',
  confidence DOUBLE PRECISION NOT NULL,
  valid_from TIMESTAMPTZ,
  valid_to TIMESTAMPTZ,
  decay_policy TEXT NOT NULL DEFAULT 'medium', -- fast/medium/slow/none
  version TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'active',  -- active / decayed / archived
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_knowledge_subject ON knowledge_entries USING GIN (subject_ids);
CREATE INDEX idx_knowledge_type_status ON knowledge_entries(type, status);
CREATE INDEX idx_knowledge_valid ON knowledge_entries(valid_from, valid_to);
```

### 2.4 复盘

```sql
CREATE TABLE retrospective_reports (
  id UUID PRIMARY KEY,
  scope SMALLINT NOT NULL,
  period TEXT NOT NULL,
  user_id TEXT,
  summary TEXT,
  attributions JSONB,                     -- [{factor, contribution, evidence}, ...]
  story_ref TEXT,                         -- s3://...
  knowledge_entries_created UUID[] NOT NULL DEFAULT '{}',
  generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_retro_scope_period ON retrospective_reports(scope, period);
CREATE INDEX idx_retro_user_generated ON retrospective_reports(user_id, generated_at DESC);
```

### 2.5 版本管理（WORM）

```sql
CREATE TABLE version_entries (
  artifact_type SMALLINT NOT NULL,
  artifact_id TEXT NOT NULL,
  version TEXT NOT NULL,
  status SMALLINT NOT NULL,               -- enum VersionStatus
  approver TEXT,
  approved_at TIMESTAMPTZ,
  eval_report_id UUID REFERENCES eval_reports(id),
  rollout_plan JSONB,
  signature TEXT NOT NULL,
  previous_prod_version TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (artifact_type, artifact_id, version)
);

CREATE INDEX idx_version_status ON version_entries(artifact_type, artifact_id, status);

-- WORM
CREATE OR REPLACE RULE no_delete_version_entries AS ON DELETE TO version_entries DO INSTEAD NOTHING;
-- UPDATE 仅允许 status / approver / approved_at / signature 字段（应用层校验）

CREATE TABLE version_status_transitions (
  id UUID PRIMARY KEY,
  artifact_type SMALLINT NOT NULL,
  artifact_id TEXT NOT NULL,
  version TEXT NOT NULL,
  from_status SMALLINT NOT NULL,
  to_status SMALLINT NOT NULL,
  actor TEXT NOT NULL,
  reason TEXT,
  at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_vst_artifact_at ON version_status_transitions(artifact_type, artifact_id, at DESC);
```

### 2.6 外部动作（WORM）

```sql
CREATE TABLE external_action_records (
  request_id UUID PRIMARY KEY,
  idempotency_key TEXT NOT NULL,
  kind SMALLINT NOT NULL,
  target_endpoint_id TEXT NOT NULL,
  payload JSONB NOT NULL,
  requester_module TEXT NOT NULL,
  requester_user_id TEXT,
  correlation_id UUID,
  status SMALLINT NOT NULL,
  reason TEXT,
  response JSONB,
  gate_decision_id UUID,
  audit_entry_id UUID,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  finished_at TIMESTAMPTZ,
  UNIQUE (idempotency_key)                -- idempotency 强制
);

CREATE INDEX idx_eareq_status_created ON external_action_records(status, created_at DESC);
CREATE INDEX idx_eareq_kind_created ON external_action_records(kind, created_at DESC);
CREATE INDEX idx_eareq_correlation ON external_action_records(correlation_id);

-- WORM
CREATE OR REPLACE RULE no_update_external_action AS ON UPDATE TO external_action_records DO INSTEAD NOTHING;
CREATE OR REPLACE RULE no_delete_external_action AS ON DELETE TO external_action_records DO INSTEAD NOTHING;
```

### 2.7 用户成长档案

```sql
CREATE TABLE user_profiles (
  user_id TEXT PRIMARY KEY,
  preferences JSONB,
  habits JSONB,
  growth_metrics JSONB,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE user_profile_history (
  id UUID PRIMARY KEY,
  user_id TEXT NOT NULL,
  field TEXT NOT NULL,                    -- preferences / habits / growth_metrics
  before JSONB,
  after JSONB,
  actor TEXT NOT NULL,                    -- user / system
  at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_uph_user_at ON user_profile_history(user_id, at DESC);
```

## 四、配置数据（动态配置中心）

```yaml
super_evo:
  eval_center:
    require_eval_pass_for:
      - canary
      - prod
    auto_run_offline:
      schedule: "0 2 * * *"               # 每天凌晨 2 点
    metrics_thresholds:
      research_card_fact_score: 0.85
      advisory_actionability_score: 0.7

  knowledge_base:
    decay:
      fast:  { half_life_days: 7 }
      medium:{ half_life_days: 30 }
      slow:  { half_life_days: 180 }
      none:  { half_life_days: 0 }
    dedup_threshold: 0.92                 # 嵌入相似度
    rag:
      top_k_default: 8
      hybrid_weight: { dense: 0.6, sparse: 0.4 }
      rerank_enabled: true

  external_action:
    idempotency_ttl_sec: 86400            # 24h
    retry:
      max_attempts: 3
      backoff: exponential
    audit_required: true                  # 永远必须

  rollout:
    default_steps:
      - { traffic_pct: 0.05, dwell_minutes: 60 }
      - { traffic_pct: 0.20, dwell_minutes: 120 }
      - { traffic_pct: 0.50, dwell_minutes: 240 }
      - { traffic_pct: 1.00, dwell_minutes: 0 }
```

## 五、保留与归档

| 表 | 主存 | 索引 | 归档 |
|-----|------|------|------|
| eval_datasets | 永久 | — | OSS |
| eval_tasks / eval_reports | 1 年 | 1 年 | OSS |
| feedback_explicit / implicit | 1 年（按月分区） | 90d | OSS |
| knowledge_entries | 永久 | 永久 | OSS（按版本快照） |
| retrospective_reports | 永久 | 永久 | OSS |
| version_entries | 永久（WORM） | — | OSS |
| version_status_transitions | 永久 | — | OSS |
| external_action_records | 永久（WORM） | 1 年（OpenSearch） | OSS（永久；retention lock） |
| user_profiles | 用户控制；删除请求 = 立即删除 | — | 用户控制 |
