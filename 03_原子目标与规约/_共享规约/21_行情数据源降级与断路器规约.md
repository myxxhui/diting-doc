## 一、根因（同会话实测 2026-05-23）

| 接口 | 实测 | 结论 |
|---|---|---|
| `akshare.stock_zh_a_hist` → `push2his.eastmoney.com/api/qt/stock/kline/get` | ❌ 100% `RemoteDisconnected` | 本机网络对该端点 TCP 握手后立即 RST；连续 7 次复测全失 |
| `akshare.stock_zh_a_daily` → 新浪历史接口 | ❌ `KeyError: 'date'` | akshare 0.x 适配漂移，字段已变 |
| `akshare.stock_zh_a_spot_em` → 全市场 spot | ❌ 同 `RemoteDisconnected` | 同端点族被阻断 |
| **`qt.gtimg.cn/q={codes}`**（腾讯实时） | ✅ **0.17s / 4 只批量** | 最稳；无需 UA；支持任意只数 |
| **`hq.sinajs.cn/list={codes}`**（新浪实时） | ✅ **0.37s / 4 只批量** | 须 `Referer: https://finance.sina.com.cn` |
| **`push2.eastmoney.com/api/qt/ulist.np/get`**（东方财富实时 list） | ✅ **2.42s / 4 只批量** | 与 push2his 同 host 不同 path，未被阻断 |
| **`web.ifzq.gtimg.cn/appstock/app/fqkline/get`**（腾讯日线 K） | ✅ **0.72s / 1 只 × 5 日** | 取代失败的 push2his K 线 |

**核心结论**：
1. 不是网络全断，是 **eastmoney `push2his.*` 端点族**在本机被阻断（可能 HTTP/2 握手 / 证书 / GFW 节点）。
2. **不要再依赖 akshare 封装**：①封装层版本漂移（`stock_zh_a_daily` 字段名变 `date` → 无）；②封装层把所有路径堆到 eastmoney。
3. **直接 HTTP 调用腾讯/新浪/eastmoney push2** 三源稳定可用。
4. **行情数据源应抽象到「源 + 端点」二维**，不能继续按"调用 akshare 的哪个函数"组织。

## 二、对外接口契约

模块：`apps/common/market_quote/`（**新建**，由 D4 / D3 / D0 / D2 共享，不再放 `exit_engine/data/`）

```python
# 仅签名，不写实现
class MarketQuoteClient:
    def get_realtime(self, symbols: list[str]) -> dict[str, RealtimeQuote]: ...
    def get_recent_kline(self, symbol: str, days: int = 30) -> list[Kline]: ...
    def health(self) -> dict[str, SourceHealth]: ...

@dataclass(frozen=True)
class RealtimeQuote:
    symbol: str
    close: float            # 当日收盘价（盘中 = 最新价；盘后 = 收盘价）
    prev_close: float
    change_pct: float       # (close - prev_close) / prev_close
    volume: int             # 成交量（股）
    timestamp: datetime     # 数据时间戳（行情源给的）
    source: str             # "tencent" / "sina" / "eastmoney_list"
    is_stale: bool          # 数据时间戳与当前时间差 > 闭市阈值时 True

@dataclass(frozen=True)
class Kline:
    date: date
    open: float; high: float; low: float; close: float
    volume: int
    adjust: str             # "qfq" / "none"

@dataclass(frozen=True)
class SourceHealth:
    source: str
    status: str             # "ok" / "tripped" / "degraded"
    last_ok_at: Optional[datetime]
    consecutive_failures: int
    tripped_until: Optional[datetime]
```

**异常约定**：
- 单只标的拉取失败 → 不抛异常，返回 dict 中**缺失该 symbol**（调用方自行决定降级）
- 全部源失败 → 仅写 WARNING 日志，返回空 dict
- 配置错误（symbol 非 6 位、days ≤ 0）→ 抛 `ValueError`，不重试

## 三、数据源矩阵（按优先级）

| 优先级 | 源名 | 端点 | 类型 | 批量 | 超时 | UA/Referer | 备注 |
|---|---|---|---|---|---|---|---|
| **P1** | `tencent` | `http://qt.gtimg.cn/q={codes}` | 实时 | ✅ 无上限 | 3s | 无 | 字符串解析；最稳 |
| **P2** | `sina` | `http://hq.sinajs.cn/list={codes}` | 实时 | ✅ ≤ 50 只/批 | 3s | `Referer: https://finance.sina.com.cn` | 字符串解析 |
| **P3** | `eastmoney_list` | `https://push2.eastmoney.com/api/qt/ulist.np/get?secids={codes}&fields=f12,f14,f2,f3,f4,f6` | 实时 | ✅ ≤ 100 只/批 | 5s | 标准 UA | JSON；`f2 / 100 = 最新价` |
| **K1** | `tencent_kline` | `https://web.ifzq.gtimg.cn/appstock/app/fqkline/get?param={ex}{sym},day,,,{days},qfq` | 日线 K | ❌ 逐只 | 5s | UA + `Referer: https://gu.qq.com/` | 取代失败的 push2his |
| **K2** | `sina_kline` | `https://finance.sina.com.cn/realstock/company/{ex}{sym}/hisdata/klc_kl.js` | 日线 K | ❌ 逐只 | 5s | UA | 兜底；JSONP，须正则提取 |

**`codes` 编码规则**：
- 沪市 `60xxxx` / `68xxxx` / `5xxxxx` → 前缀 `sh`；用于腾讯/新浪
- 深市 `00xxxx` / `30xxxx` → 前缀 `sz`
- 东方财富前缀：沪市 `1` / 深市 `0`
- 标准化代码（无前缀 6 位）由调用方传入，**内部统一封装** `_resolve_exchange(symbol) -> Literal["sh","sz"]`

## 四、字段映射 → 标准化结果

### 腾讯 `qt.gtimg.cn` 返回示例

```
v_sh601138="1~工业富联~601138~67.16~65.50~66.11~1947455~...~2026-05-22 15:30:00";
```

| 索引 | 含义 | 标准字段 |
|---|---|---|
| `[1]` | 名称 | `name`（可选） |
| `[2]` | 代码 | 校验用 |
| **`[3]`** | **当前/收盘价** | `close` |
| `[4]` | 昨收 | `prev_close` |
| `[5]` | 开盘 | — |
| `[6]` | 成交量（手 → ×100 转股） | `volume` |
| `[30]` | 时间戳 `YYYY-MM-DD HH:MM:SS` | `timestamp` |

> 字段位置在腾讯接口长期稳定（2018 至今）；以 `~` split 即可，**不要用 regex**。

### 新浪 `hq.sinajs.cn` 返回示例

```
var hq_str_sh601138="工业富联,66.110,65.500,67.160,...,2026-05-22,15:00:00";
```

| 索引 | 含义 | 标准字段 |
|---|---|---|
| `[0]` | 名称 | — |
| `[1]` | 开盘 | — |
| `[2]` | 昨收 | `prev_close` |
| **`[3]`** | **当前/收盘价** | `close` |
| `[8]` | 成交量（股） | `volume` |
| `[30] + " " + [31]` | 日期 + 时间 | `timestamp` |

### 东方财富 `push2.eastmoney.com` 返回示例

```json
{"data": {"diff": [{"f2": 6716, "f3": 252, "f4": 166, "f12": "601138", "f14": "工业富联"}]}}
```

| 字段 | 含义 | 转换 | 标准字段 |
|---|---|---|---|
| `f2` | 当前价 ×100 | `/100` | `close` |
| `f3` | 涨跌幅 ×100 | `/100` | `change_pct` |
| `f4` | 涨跌额 ×100 | `/100` | （由 close 推） |
| `f12` | 代码 | — | `symbol` |
| `f6` | 成交额 | — | （可忽略） |

**`is_stale` 判定**：A 股交易时段 09:30~11:30 / 13:00~15:00；超出时段且 `timestamp` 与当前时间差 > 30 分钟 → `is_stale = True`（D4 SP1/SP2 启动期可接受 stale；扩展期改严）。

## 五、降级链与超时策略

### 实时报价 `get_realtime(symbols)`

```
       ┌────────────────┐
input → │ tencent (P1)   │ ──成功── 返回（成功 ≥1 只即可）
       │ 批量；3s        │
       └───────┬────────┘
               │ 失败 or 部分缺失
       ┌───────▼────────┐
       │ sina (P2)      │ ──成功── 合并结果
       │ 批量；3s        │
       └───────┬────────┘
               │ 仍缺失
       ┌───────▼────────┐
       │ eastmoney_list │ ──成功── 合并结果
       │ 批量；5s        │
       └───────┬────────┘
               │ 全失败
            返回空/部分
```

**关键规则**：
1. **不要串行重试同一源**（之前 _MAX_RETRIES=3 是错的，端点被阻断时重试只浪费时间）。
2. **每源单次调用**；失败立即降级到下一源。
3. **批量请求**：4 只一次性发，不要 N 次 RTT。
4. **总预算 ≤ 11s**（3 + 3 + 5），实际正常 < 0.5s。

### 日线 `get_recent_kline(symbol, days)`

```
tencent_kline (K1, 5s) → 失败 → sina_kline (K2, 5s) → 失败 → 返回 []
```

K 线接口**逐只调用**；如需多只，调用方循环；建议 `asyncio.gather` 并发 ≤ 4。

## 六、断路器（Circuit Breaker）

每个源（tencent / sina / eastmoney_list / tencent_kline / sina_kline）独立维护状态：

| 状态 | 进入条件 | 行为 | 离开条件 |
|---|---|---|---|
| `ok` | 初始 / 半开探测成功 | 正常调用 | 连续失败 ≥ `fail_threshold`（默认 5） |
| `tripped` | 连续失败达阈值 | **直接跳过该源**（不调用、不等待） | 距 `tripped_at` ≥ `cool_down_sec`（默认 60s） → `half_open` |
| `half_open` | 冷却到期 | 单次试探调用；成功 → `ok`，失败 → 续 `tripped` 一个周期 | 单次结果决定 |

**实现**：
- 状态存内存 `Dict[str, SourceState]`；进程级即可，不必持久化。
- 同会话冷却 60s + 阈值 5，可避免对死端点反复重试拖时间。
- 探针接口 `health()` 返回当前各源状态供 D0 副驾驶展示。

## 七、缓存与并发

| 项 | 策略 |
|---|---|
| **实时报价缓存** | Redis `quote:rt:{symbol}` TTL 60s；命中即返回；冷启动 / TTL 过期 / 强制刷新跳过 |
| **K 线缓存** | Redis `quote:kline:{symbol}:{days}` TTL **闭市 1h / 开市 30s**；闭市定义见 §四 |
| **并发** | 批量实时一次请求；K 线 `asyncio.gather` ≤ 4；**禁止** for 循环串行调 K 线 N 次 |
| **重入幂等** | 同 symbol + 同时间窗 → 结果一致 |
| **强制刷新** | `get_realtime(symbols, bypass_cache=True)` 跳过 Redis；用于 D4 SP1 触发时 |

### 9.4 性能预算

| 场景 | 预算 | 实测要求 |
|---|---|---|
| 4 只批量实时（缓存命中） | < 50ms | Redis O(1) |
| 4 只批量实时（缓存 miss · P1 命中） | < 500ms | tencent < 200ms + Redis 写 |
| 4 只批量实时（P1 fail + P2 命中） | < 4s | 3s 超时 + sina < 400ms |
| 4 只批量实时（全源失败） | < 12s | 3 + 3 + 5 + buffer |
| 单只 K 线 30 日 | < 1s | tencent_kline < 800ms |

### 9.5 强制不变量

- 4 只真持仓在**所有 5 个真实源中至少 1 个可用**的本机环境下，`updated == 4`。
- 断路器跳闸后，下次调用**不浪费时间**等失败源（< 100ms 跳过）。
- 任何源解析失败**不**导致其他源被跳过。
