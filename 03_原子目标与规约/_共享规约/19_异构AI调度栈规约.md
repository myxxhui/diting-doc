## 二、双模型职责分工总览

| 维度 | 远程超级大模型（**Claude Opus 4.7 · 全替**）| 本地微调小模型（Qwen-14B + LoRA）|
|---|---|---|
| **任务定位** | 脑力：零样本通识推理、复杂判断、跨领域综合 | 体力：结构化抽取、模板化清洗、JSON 实体识别 |
| **唯一模型** | **Anthropic Claude Opus 4.7**（`claude-opus-4-7`）；Lighthouse-Alpha 五场景**不并行** GPT-4o / Claude 3.5 | — |
| **典型场景** | The Scorer（policy/mapping）/ The Critic / The Architect / The Timer | ETL LLM Engine + Scorer `industry_space` + The Mapper |
| **调用频率上限** | 日 ≤ **200** 次（启动期）/ **400** 次（扩展期）（Opus 单价更高）| 日 ≤ **50,000** 次（吞吐约束）|
| **单次成本** | ¥ **0.25 ~ ¥ 1.20**（Opus 4.7 估算；按 tokens 计费）| ¥ 0（自有 GPU 折旧 + 电费）|
| **延迟** | P95 ≤ **5s**（含网络）| P95 ≤ **2s/文档** |
| **可访问数据** | **不**接触持仓 SoT、决策日志原始数据、PII | 全数据（含 PII，但输出前必须脱敏）|
| **审计要求** | prompt + response 留存 **90 天** | input + output 留存 **30 天** |
| **降级路径** | API 不可用 → 切到过去 7 天缓存（同主题）| GPU 不可用 → 任务回退到 Kafka DLQ |

## 五、数据隔离与脱敏规则

### 5.1 远程大模型不可见数据清单

| 数据类型 | 原因 |
|---|---|
| **持仓 SoT**（`my_holdings.yaml`）| 隐私（投资偏好）+ 防 prompt injection 套取 |
| **决策日志原始数据**（含用户标注、心理状态）| 隐私 |
| **用户 PII**（姓名、手机号、邮箱）| 法律合规 |
| **券商 API 返回的原始持仓快照** | 同上 |

### 5.2 脱敏规则

| 数据 | 给远程大模型前的处理 |
|---|---|
| 标的代码 | ✅ 可直接传（公开信息）|
| 标的财务数据 | ✅ 可直接传（公开信息）|
| 用户持仓比例 | ❌ 脱敏为 "high / med / low"；不传具体数字 |
| 监控字典 source URL | ✅ 可直接传 |
| 历史决策结果（pass/fail）| ✅ 可传"匿名化的策略类型 + 结果"；不传具体盈亏数字 |
| 财报原文 | ✅ 可直接传（已公开）|

### 5.3 检测

| # | 检测项 | 命令 |
|---|---|---|
| DS1 | 远程调用 payload 不含 `my_holdings` 字样 | runtime guard + 单测 |
| DS2 | 远程调用 payload 不含手机号格式（13/14/15/17/18/19 开头 + 9 位）| 正则单测 |
| DS3 | 决策日志原文不入远程 | grep 单测 |

## 七、调用 SDK 契约

### 7.1 统一 abstract（Python）

```python
# diting-src/engines/ai_dispatch/dispatcher.py

class AIDispatcher:
    def call(
        self,
        scene: Literal["scorer", "critic", "mapper", "architect", "timer", "etl"],
        sub_scene: Optional[str],   # 如 "policy_tier" / "industry_space"
        prompt_template_id: str,
        prompt_vars: dict,
        max_tokens: int = 2000,
        budget_check: bool = True,  # False 仅用于单测
    ) -> DispatchResult:
        """
        统一入口；自动按 §三路由表选模型；自动按 §4.2 检查预算；自动审计；自动回流 D5。
        """
        ...

@dataclass
class DispatchResult:
    call_id: str
    model_name: str
    response_json: dict       # 结构化输出（小模型抽取场景）
    response_text: str        # 自由文本（大模型推理场景）
    tokens_used: int
    cost_yuan: float
    latency_ms: int
    routing_decision: dict    # 显式留痕路由原因（如"sub_scene=policy_tier → remote_opus_4_7"）
    audit_logged: bool
```

### 7.2 强制约束

| # | 约束 |
|---|---|
| SDK1 | 业务代码**必须**通过 `AIDispatcher.call()` 调用大/小模型；**禁止**直接 `openai.ChatCompletion.create()` 或 `qwen.generate()` |
| SDK2 | 每次 call 必须传 `scene` + `prompt_template_id`（用于审计与回流） |
| SDK3 | budget_check=False 仅允许 `tests/` 单测使用；业务代码 grep 必须 0 命中 |
| SDK4 | DispatchResult.routing_decision 必须非空（防穿透：路由必须留痕）|
| SDK5 | DispatchResult.audit_logged=False 时**业务路径不得继续**——否则违反 §六 审计要求 |
