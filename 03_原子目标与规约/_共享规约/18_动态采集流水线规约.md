## 三、组件契约

### 3.1 物理采集层契约

| 组件 | 输入 | 输出 | 部署形态 | 失败动作 |
|---|---|---|---|---|
| **Playwright 浏览器集群** | `crawl_task_config:*`（Redis 热加载）| `sniffer_raw_text` topic | K8s CronJob × 3（卖方研报 / 政策 / 海外科技）| 单源 5 次连续失败 → 标 `source_degraded`，告警；不阻塞另两源 |

| **Tushare/聚宽 API** | 同上 | 同上 | 备份 CronJob | 双源都失败 → DLQ + 告警 |

## 四、消息总线（Kafka topics）

| topic | 用途 | partition | retention | producer | consumer |
|---|---|---|---|---|---|
| `sniffer_raw_text` | 嗅探原文流（结构化 + 非结构化）| 6 | 7 d | Playwright/AkShare | NLP 聚类 / ETL LLM Engine |
| `sniffer_clusters` | 题材聚类簇 | 3 | 30 d | NLP 聚类 | The Critic |
| `monitor_dict_updates` | The Architect 输出的监控字典 update | 3 | 90 d | The Architect | Redis monitor:* writer + 审计 |
| `probe_metrics_p5p6p7` | P5/P6/P7 探针每日观测值 | 6 | 365 d | AkShare + Playwright | D3 探针 + ClickHouse |
| `timer_signal` | The Timer 三段窗口事件 | 3 | 90 d | The Timer | D4 SP5 + D0 alerts |
| `monitor_dict_dlq` | 监控字典写入失败死信 | 1 | 30 d | The Architect writer | 人工复审 |

### 4.1 消息 envelope 契约

所有 topic 消息须遵守：

```yaml
KafkaEnvelope:
  event_id: str               # UUID v4
  event_type: str             # 与 topic 对齐的枚举
  produced_at: datetime
  producer_service: str       # diting-deep-strike / diting-co-pilot 等
  producer_version: str       # commit sha
  trace_id: str               # 跨服务追溯
  payload: object             # 业务体（按 topic schema）
  schema_version: str         # 业务体 schema 版本，违反则消费端拒绝
```

### 4.2 schema 演进规则

| 变更类型 | 处理 |
|---|---|
| 新增可选字段 | 不升 schema_version；老消费端忽略 |
| 新增必填字段 | 升 minor 版本（如 v1.2 → v1.3）；老消费端按"缺失字段 → 默认值"处理 |
| 修改字段类型 | 升 major（v1 → v2）；并发 Cancellation 期 30 天双写 |

## 五、配置中心（Redis schema）

### 5.1 `monitor:*` 命名空间（监控字典消费端 · 共享规约 20 详细 schema）

```
monitor:{symbol}:dict:{field_id}      # JSON value（MonitorMatrix.monitor_matrix[] 一项）
monitor:{symbol}:dict:_meta           # 该 symbol 监控字典 meta：count / last_updated / source thesis_card_id
monitor:dict:stale_index              # ZSET：score=last_hit_at；底部为 stale 候选
```

**TTL 规则**：无 TTL（不过期）；GC 由 D3 §6A.3 "死字段 GC" 月度 cron 处理。

### 5.2 `crawl_task_config:*` 命名空间（爬虫热加载）

```
crawl_task_config:research            # 研报采集任务配置（JSON）
crawl_task_config:policy              # 政策门户采集任务配置
crawl_task_config:overseas            # 海外科技采集任务配置
crawl_task_config:macro:hs_codes      # HS Code 监控清单（List）
crawl_task_config:_version            # int，每次改配置 +1，爬虫每 5 min 比对
```

**热加载约束**：

| # | 约束 | 检测 |
|---|---|---|
| HR1 | 改配置 5 分钟内生效（无须重启爬虫）| 单测：`SET crawl_task_config:research.enabled=false` 后 5 min 内爬虫停采 |
| HR2 | 旧任务正在运行的不强中断（current run 跑完后退出）| 集成测 |
| HR3 | 版本回退（version 减少）拒绝 | runtime guard |

### 5.3 `sniffer_dedup:*` 命名空间（去重）

```
sniffer:dedup:{YYYYMMDD}              # SET，元素是 md5(url + content[:500])
```

**TTL**：3 天。

## 七、降级与失败处理

### 7.1 降级矩阵

| 故障源 | 第一降级 | 第二降级 | 最后兜底 |
|---|---|---|---|
| Playwright 集群整体不可用 | 单源切其他两源 | 全部停 + 监控告警；用昨日缓存 | 标 `BLOCKED` 不准出 |
| AkShare API 不通 | 切 Tushare | 切聚宽 | DLQ + 告警 |
| Kafka 集群不可用 | 直写 SQLite 本地缓冲 | 缓冲满 → 停 producer + 告警 | 不允许丢消息 |
| Redis 配置中心不可用 | 用本地 `crawl_task_config.yaml` 兜底 | 仅采 active 持仓核心字段 | 告警 |
| ETL LLM Engine 不可用 | 任务回退到 Kafka 死信队列 | 人工触发重处理 | raw_text 不消费 → 30 天后过期 |

### 7.2 DLQ 处理 SLA

| topic DLQ | 处理 SLA | 责任 |
|---|---|---|
| `monitor_dict_dlq` | 24 h 内人工复审 | The Architect 维护者 |
| `sniffer_raw_text` 失败 | 7 d 内重处理 | 数据采集团队 |
| `timer_signal` 失败 | 12 h 内人工复审 | The Timer 维护者 |
