## §1 本规约定位

把「**断言一项事实 / 关系 / 数字之前，必须有 ≥ 2 个独立可信源相互印证；否则降一档处理或拒绝写库**」固化为**跨维度共享 gate**。

**触发场景**（典型，凡触及以下 5 类断言必走本 gate）：

| 场景 | 典型断言 | 历史教训 |
|---|---|---|
| **持股 / 关联关系** | "X 公司持有 Y 公司股权" | 利欧股份持有宇树科技 → 实为假相关 |
| **市占率 / 渗透率** | "Z 公司浸没式液冷市占率 50%" | 缺 as_of_date / source_url → 老数据 |
| **物理因果** | "夏季高温 → 风冷宕机 → 倒逼液冷" | 真因实为单机柜功耗密度，非外部温度 |
| **行业普遍性结论** | "To-B 硬件 Q1 一定惨淡" | 过度概括，需公司级合同负债数据 |
| **政策 / HS Code / 标签** | "HS 85176239 = 光收发模块对美出口" | 缺 `verified_at`，HS 编码版本会变 |

**模块边界**：
| ✅ 本规约 | ❌ 其他规约 |
|---|---|
| 跨维度断言的多源验证 gate + 4 档处理矩阵 + DLQ 落库 | 单维度内部规则（归各维度 step）|
| 证据置信度衰减规则（含 `verified_at` 过期）| evidence_chain 字段定义（归 D2 step_03）|
| LLM 输出 `source_urls[]` 强制留痕 | LLM 调用本身（归共享规约 19 异构 AI 调度）|

## §2 4 档处理矩阵（核心 schema）

```yaml
FactClaim:
  claim_id: str                    # 雪花 ID
  claim_type: enum                 # 5 类（见 §1 表）
  claim_text: str                  # ≤ 200 字
  subject_symbol: str | null       # 涉及标的代码（如 002837）
  subject_entity: str | null       # 涉及实体（公司 / 机构 / 政策）
  proposed_by:
    service: str                   # 哪个服务/agent 提出（如 D2.thesis_generator）
    model_name: str | null         # 若 LLM 提出，记录模型
    prompt_template_id: str | null
  sources: list[SourceCitation]    # 至少 1 条；目标 ≥ 2 条独立源
  verification_status: enum        # 4 档（见下）
  confidence_after_gate: float     # [0, 1]，gate 后置信度
  verified_at: datetime
  expires_at: datetime | null      # 数据时效（HS Code / 市占率默认 90 天）
  dlq_reason: str | null           # 若进 DLQ，记录原因

SourceCitation:
  source_url: str                  # 必填，HTTPS
  source_type: enum                # primary | secondary | aggregator | llm_generated
  source_publisher: str            # 如 "巨潮资讯" / "国家发改委" / "ArXiv"
  raw_excerpt: str                 # ≥ 50 字原文摘录（RED 强制）
  fetched_at: datetime
  independence_score: float        # [0, 1]，与其他源的独立性评分（同集团媒体 ≤ 0.3）
```

**4 档判定矩阵**：

| # | verification_status | 触发条件 | 置信度上限 | 后续动作 |
|---|---|---|---|---|
| 1 | `verified_strong` | ≥ 2 个 `independence_score ≥ 0.7` 的独立源 + raw_excerpt 齐 | 1.0 | 写库 + 可用于 thesis 卡 |
| 2 | `verified_weak` | 仅 1 个 primary 源 OR 2 个非独立源（同集团媒体）| 0.55 | 写库但 thesis 卡 `valuation_anchor` 不可引用 |
| 3 | `pending_more_evidence` | 0 个 primary 源 OR 仅 LLM 生成无 URL | 0.20 | 落 `fact_claims_pending` 表 + 后台 24h 再爬 |
| 4 | `dlq_rejected` | 多源相互矛盾 OR 检测到伪造（如 URL 404 / 引用失效） | 0 | 落 `fact_claims_dlq` + 告警 |

**独立性评分（independence_score）**：

| 关系 | independence_score |
|---|---|
| 同 publisher | 0.0 |
| 同集团（人民日报系 / 财联社系 / 同花顺系） | 0.3 |
| 一为 primary（监管/官方）一为 secondary（媒体转载） | 0.7 |
| 两个完全独立的 primary 源 | 1.0 |
| 一为 primary，一为 LLM 生成（含 URL） | 0.4 |

## §3 与现有 D2 evidence_ref 的关系

| 现有机制 | 本规约关系 |
|---|---|
| D2 evidence_chain（≥ 3 条） | **不冲突**：evidence_chain 是支撑 thesis 整体逻辑的证据集；本 gate 是对**每一条具体断言**（含 evidence_chain 内每条 evidence）的多源校验 |
| evidence_ref RED 强制 raw_excerpt | **本规约升级**：除 raw_excerpt 外，还要求 `independence_score` 计算 + `expires_at` 时效管理 |
| D2 The Critic 物理证伪（物理底线 + 产能弹性） | **互补**：The Critic 判「物理可证伪性」（这个断言是否原则上能验证）；本 gate 判「实际证据是否足够」（已经收集到的源是否够多够独立）|

**调用关系**：D2 thesis_generator → 对每条新断言调 `fact_gate.verify(claim)` → 返 `FactClaim`（含 `verification_status`） → 若 `dlq_rejected` 拒绝写 thesis_cards / 若 `pending_more_evidence` 标记 thesis 卡 `confidence ≤ 0.2`。

## §4 4 类常见反模式（启动期必拦）

| # | 反模式 | 例 | 判定 |
|---|---|---|---|
| R1 | **持股 / 关联无 primary 源** | LLM 说 "A 公司持有 B 公司"，未给 primary 源（巨潮披露公告 / 工商登记） | dlq_rejected |
| R2 | **物理因果 LLM 编造** | "夏季高温倒逼液冷" 无 PUE / 单机柜功耗 primary 数据 | dlq_rejected |
| R3 | **市占率无 as_of_date** | "X 公司市占率 50%" 但 source 未注明数据时点 | verified_weak + 标 `as_of_date_missing` |
| R4 | **行业普遍性结论** | "To-B 硬件 Q1 一定惨淡" 推断到具体公司，未取该公司合同负债 | pending_more_evidence |

**实现**：runtime guard（写 thesis_cards / 写 monitor_dict / 写 market_phase 前调用）+ CI grep（禁止业务代码绕过 fact_gate）。

## §5 时效管理（expires_at）

| claim_type | 默认有效期 | 续期触发 |
|---|---|---|
| 持股 / 关联关系 | 永久（公司间股权关系稳定） | 季报 / 重大资产重组公告触发重验 |
| HS Code 政策标签 | 1 年 | HS 编码版本更新（每 5 年） |
| 市占率 / 渗透率 | 90 天 | 季报 / 行业研究院更新 |
| 物理量数据（PUE / CAPEX） | 1 季度 | 公司业绩说明会 / IR 披露 |
| 一致预期数据 | 14 天 | 券商研报更新 |

**过期处理**：`expires_at < now()` 的 claim 自动降一档 → 触发后台重验 worker。
