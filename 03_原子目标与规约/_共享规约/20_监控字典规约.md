## 二、MonitorMatrix 完整 jsonschema

```yaml
# diting-src/data/sniffer/schemas/monitor_matrix.schema.yaml
$schema: "https://json-schema.org/draft/2020-12/schema"
title: MonitorMatrix
type: object
required: [thesis_card_id, target_company, symbol, generated_by, monitor_matrix]
properties:
  thesis_card_id:
    type: string
    pattern: "^thesis_[0-9]{6}_[a-zA-Z0-9_]+_[0-9]{8}$"
  target_company:
    type: string
    minLength: 2
    description: "自然语言名称，含括号代码，如 '中际旭创 (300308)'"
  symbol:
    type: string
    pattern: "^[0-9]{6}$"
  generated_by:
    type: object
    required: [model_name, prompt_template_id, generated_at, tokens_used]
    properties:
      model_name:
        type: string
        enum: ["claude-opus-4-7", "claude-opus-4-20250514"]
      prompt_template_id:
        type: string
        pattern: "^the_architect_v[0-9]+$"
      generated_at: {type: string, format: date-time}
      tokens_used: {type: integer, minimum: 1}
  monitor_matrix:
    type: array
    minItems: 1
    maxItems: 10
    items:
      type: object
      required: [field_id, probe_id, metric_name, data_source_type, specific_target, alert_threshold, alert_threshold_struct, polling_frequency, mapped_logic_chain_nodes, status, created_at]
      properties:
        field_id:
          type: string
          pattern: "^field_[a-z0-9_]+$"
        probe_id:
          type: string
          enum: [P5, P6, P7]
        metric_name:
          type: string
          minLength: 4
        data_source_type:
          type: string
          enum: [STRUCT_DATA_API, WEB_SCRAPING]
        source_api:
          oneOf: [{type: string, pattern: "^akshare\\.|tushare\\.|jq\\."}, {type: "null"}]
          description: "STRUCT_DATA_API 时必填"
        source_url:
          oneOf: [{type: string, format: uri}, {type: "null"}]
          description: "WEB_SCRAPING 时必填"
        specific_target:
          type: string
          minLength: 4
          description: "如 'HS Code: 85176239 (光通信设备), 目的地: 美国'"
        keywords:
          type: array
          items: {type: string, minLength: 2}
          maxItems: 20
        alert_threshold:
          type: string
          minLength: 10
          description: "自然语言（人审阅）"
        alert_threshold_struct:
          type: object
          required: [operator, value, window_days]
          properties:
            operator: {type: string, enum: [gt, lt, mom_pct, yoy_pct, sum_pct]}
            value: {type: number}
            window_days: {type: integer, minimum: 1, maximum: 365}
        polling_frequency:
          type: string
          enum: [daily, monthly_after_release, weekly]
        mapped_logic_chain_nodes:
          type: array
          minItems: 1
          maxItems: 10
          items: {type: string, pattern: "^node_[a-z0-9_]+$"}
        status:
          type: string
          enum: [active, stale, archived]
        created_at: {type: string, format: date-time}
        last_hit_at:
          oneOf: [{type: string, format: date-time}, {type: "null"}]
allOf:
  # MA2: alert_threshold + alert_threshold_struct 双形式
  - if: {properties: {monitor_matrix: {items: {properties: {data_source_type: {const: STRUCT_DATA_API}}}}}}
    then: {properties: {monitor_matrix: {items: {required: [source_api]}}}}
  - if: {properties: {monitor_matrix: {items: {properties: {data_source_type: {const: WEB_SCRAPING}}}}}}
    then: {properties: {monitor_matrix: {items: {required: [source_url]}}}}
```

## 三、字段语义详解

### 3.1 `probe_id` 与 D3 探针映射

| probe_id | 探针名 | data_source_type 倾向 | 典型 specific_target |
|---|---|---|---|
| **P5** | 政府招标物理量探针 | WEB_SCRAPING（ccgp + 三大运营商）| "中国政府采购网 / 三大运营商电子采购平台" |
| **P6** | 海关 HS Code 出口量探针 | STRUCT_DATA_API（AkShare macro_china_customs）| "HS Code: 85176239 (光通信设备), 目的地: 美国" |
| **P7** | 产业产能利用率探针 | STRUCT_DATA_API（AkShare + 国家统计局）| "光模块行业产能利用率 / 大宗能源消耗" |

### 3.2 `alert_threshold_struct.operator` 语义

| operator | 含义 | 示例 |
|---|---|---|
| `gt` | 大于阈值 | `产能利用率 > 0.85` |
| `lt` | 小于阈值 | `产能利用率 < 0.65` |
| `mom_pct` | 环比涨幅 | `海关月度环比 > 0.30` |
| `yoy_pct` | 同比涨幅 | `单价同比 > 0.10` |
| `sum_pct` | 窗口累计与基准比 | `30 天中标累计 / 上一年总营收 > 0.20` |

### 3.3 `mapped_logic_chain_nodes` 与 thesis 卡逻辑链

监控字典必须能反向追溯到 D2 thesis 卡的某个或多个逻辑链节点：

| node_id 示例 | 含义 |
|---|---|
| `node_supply_demand_mismatch` | 供需错配节点 |
| `node_capacity_elasticity` | 产能弹性节点 |
| `node_overseas_demand` | 海外需求节点 |
| `node_policy_driver` | 政策驱动节点 |

> 监控字典字段命中 → 反向标记对应逻辑链节点状态变化（→ D3 持仓监控状态机）。

## 四、生产端契约（The Architect）

### 4.1 6 条硬约束（MA1~MA6）

| # | 约束 | 不通过动作 |
|---|---|---|
| MA1 | **jsonschema 必须 validate 通过** | 写 Redis 前拒绝；落 DLQ `monitor_dict_dlq`；告警 The Architect prompt 异常 |
| MA2 | **alert_threshold + alert_threshold_struct 双形式** | 缺一拒绝；防止 D3 探针无结构化字段无法自动判定 |
| MA3 | **probe_id ∈ {P5, P6, P7}** | 与 D3 §6A.1 探针 ID 映射严格对账；非枚举值拒绝 |
| MA4 | **HS Code / source_url / keywords 至少一项非空** | 三者全空 = 无法触发任何探针 → 拒绝 |
| MA5 | **mapped_logic_chain_nodes 非空且引用真实 node_id** | 监控字典必须能反向追溯到 D2 thesis 卡某个逻辑链节点；否则 D3 探针告警时无法关联 thesis |
| MA6 | **status 初值为 active；GC 流程见 §七** | 90 天无 last_hit_at 更新 → status=stale；架构师月度复审 |

### 4.2 生产时序

```
The Architect 大模型生成 JSON
  → 走共享规约 19（**Claude Opus 4.7**，scene="architect"）
  → 收到 response_text → json.loads
  → jsonschema.validate（MA1）
  → 若失败 → 落 DLQ + 告警
  → 若通过 → 检查 MA2~MA6
  → 若任一不通过 → 拒绝写入 + 告警
  → 若全部通过 → 写 Redis `monitor:{symbol}:dict:{field_id}` + 推 Kafka `monitor_dict_updates`
  → 写审计 ai_call_logs_remote（共享规约 19 §六）
```

## 五、消费端契约（D3 P5/P6/P7 探针）

### 5.1 5 条硬约束（MC1~MC5）

| # | 约束 | 检测 |
|---|---|---|
| MC1 | **消费前必须读 _meta** | 检查 `monitor:{symbol}:dict:_meta` 是否存在；不存在 → 跳过该 symbol（不阻塞） |
| MC2 | **轮询频率严格遵守 polling_frequency** | P5 daily / P6 monthly_after_release / P7 monthly |
| MC3 | **触发 alert 必须写 last_hit_at** | 触发后更新 Redis `monitor:{symbol}:dict:{field_id}.last_hit_at = now`；防 GC 误删 |
| MC4 | **alert 事件必须含 evidence_ref** | D3 探针触发后产 `events:monitor:health_change` 事件须含 `monitor_field_id` 字段供 D4 SP3/SP5 审计回溯 |
| MC5 | **不修改 monitor:* 字段语义** | 消费端只读 + 更新 last_hit_at；其余字段不可写 |

### 5.2 探针消费时序

```
D3 探针每日 cron 启动（按 polling_frequency）
  → 读 `monitor:*:dict:_meta` 列表
  → for each symbol:
       for each field_id where probe_id == self.probe_id:
         读 alert_threshold_struct
         调对应 source（AkShare / Playwright）
         按 operator + value + window_days 判定
         若触发 → 写 last_hit_at + 产 events:monitor:health_change（含 evidence_ref）
         若不触发 → 仅更新 last_hit_at（标已轮询）—— 注意：实际 GC 用最近**alert 命中**时间
```

## 六、Redis 命名空间与 TTL

| 命名空间 | value 类型 | 用途 | TTL |
|---|---|---|---|
| `monitor:{symbol}:dict:{field_id}` | JSON | 单字段监控字典 | 无 TTL（GC 见 §七）|
| `monitor:{symbol}:dict:_meta` | JSON `{count, last_updated, source_thesis_card_id}` | 索引 + 元信息 | 无 TTL |
| `monitor:dict:stale_index` | ZSET（score=last_hit_at unix）| 死字段候选池 | 无 TTL |
| `monitor:dict:_dirty_queue` | LIST | The Architect 写入待审计的 field_id 队列 | 24 h |

## 七、死字段 GC

### 7.1 GC 流程

| 步骤 | 动作 | 频率 |
|---|---|---|
| 1 | 扫 `monitor:dict:stale_index`，找 score < `now - 90d` 的 field_id | 月度 cron（每月 1 日 02:00）|
| 2 | 标记 `monitor:{symbol}:dict:{field_id}.status = stale` | 同 step 1 |
| 3 | 产 stale 列表 → D5 飞轮回流（→ The Architect 字典生成质量训练样本）| 同 step 1 |
| 4 | 架构师月度复审：决定 reactivate / archive / 重生成 | 月度（架构师工作日）|
| 5 | reactivate → status = active；archive → DELETE Redis + 写历史表 | 复审后 |

### 7.2 GC SLA

| 指标 | 启动期 | 扩展期 |
|---|---|---|
| 死字段比例 | ≤ 20% | ≤ 10% |
| GC 复审延迟（stale → reactivate/archive）| ≤ 30 d | ≤ 7 d |

## 八、监控字典质量度量

| 指标 | 公式 | 启动期门槛 |
|---|---|---|
| **active 字典覆盖率** | active 标的中"有 ≥ 1 个 active monitor 字段"的标的占比 | ≥ 80% |
| **死字段比例** | `count(status=stale) / count(status in [active,stale])` | ≤ 20% |
| **字典命中率（30 天）** | `count(last_hit_at >= now-30d) / count(status=active)` | ≥ 50% |
| **The Architect 生成成功率** | `count(jsonschema_pass) / count(all_generated)` | ≥ 95% |
| **The Architect 生成成本** | 月度 Architect 大模型成本 | ≤ ¥ 150/月（启动期） |
