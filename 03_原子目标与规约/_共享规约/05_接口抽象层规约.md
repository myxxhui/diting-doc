## 一、设计原则

- 上层工作流只依赖抽象，不依赖具体供应商、具体模型、具体运行环境。
- 任何外部系统接入都必须通过统一 Port/Adapter 边界，不能把 SDK 直接扩散到业务层。
- 接口抽象既服务产品可替换性，也服务岗位训练中的工程深度验证。

## 二、必须存在的接口家族

| 接口家族 | 解决的问题 | 典型实现 |
|----------|------------|----------|
| **Data Source Port** | 行情、公告、新闻、行业、主营构成等输入接入 | `AkshareAdapter`、`TushareAdapter`、自建抓取器 |
| **Inference Gateway Port** | 商业 API 与自托管推理服务统一接入 | `OpenAIGateway`、`vLLMGateway`、`TritonGateway`、`TGIGateway` |
| **Research Workflow Port** | 检索、总结、候选生成、工具调用、人工复核统一编排 | `LangGraphWorkflow`、规则流、异步任务编排器 |
| **Session Transport Port** | WebSocket、SSE、任务状态同步、会话恢复 | `WebSocketSessionHub`、`SSETransport` |
| **Runtime Sandbox Port** | browser/computer-use runtime、安全隔离、资源限制 | `BrowserRuntime`、`ComputerUseRuntime`、`SandboxAdapter` |
| **Model Ops Port** | 实验记录、模型注册、服务发布、漂移检测 | `MLflowAdapter`、`KServeAdapter`、`BentoMLAdapter` |
| **External Action Port** | 通知、工单、执行、外部 API 或第三方系统 | `NotificationAdapter`、`BrokerAdapter`、`WebhookAdapter` |

## 三、关键边界要求

### 1. 推理网关边界

- 必须支持商业 API 与自托管推理的统一请求模型。
- 必须暴露延迟、吞吐、并发、失败率、缓存命中等观测字段。
- 必须允许按模型、租户、任务类型切换后端，实现岗位 1 的性能治理和岗位 3 的发布治理。

### 2. 会话与长连接边界

- 所有长连接链路必须有 `session_id`、`resume_token`、`heartbeat`、`reconnect_policy`。
- 必须能区分用户断线、服务端重启、runtime 崩溃、浏览器关闭等不同恢复场景。
- 会话恢复失败时必须有降级路径，而不是直接丢失上下文。

### 3. runtime 与安全边界

- browser/computer-use runtime 必须与主业务隔离。
- 必须有资源限制、文件系统边界、网络边界、进程边界和审计日志。
- `seccomp`、`nsjail`、`eBPF` 等能力可以按阶段引入，但接口层必须为其预留统一策略注入点。

### 4. 模型运维边界

- 训练、评测、注册、部署、回滚不能散落在各模块内部，必须通过统一的 Ops Port 暴露。
- `MLflow`、`Kubeflow/Airflow`、`KServe/BentoML`、feature store、drift detection 都应挂在这一层，而不是直接耦合到模块逻辑。

## 四、接口稳定性规则

- 接口一旦被 L4 或代码使用，就必须明确输入、输出、错误模型和版本策略。
- 任何破坏兼容的改动都必须更新 DNA、L4、L5，并在必要时补 ADR。
- 不允许在未定义抽象前直接把第三方 SDK 调进核心模块。

## 五、与岗位能力的对应

- **岗位 1**：重点在 `Inference Gateway Port` 的性能、资源和推理治理。
- **岗位 2**：重点在 `Research Workflow Port`、`Model Ops Port` 的指标、门禁和发布决策。
- **岗位 3**：重点在 `Model Ops Port` 的实验、注册、部署和漂移治理。
- **岗位 4**：重点在 `Session Transport Port` 与 `Runtime Sandbox Port` 的长连接、恢复和隔离能力。

## 六、落地要求

每个接口家族都必须至少具备以下四类产物：

1. 统一接口定义。
2. 至少一个真实实现。
3. 至少一个测试或仿真实现。
4. 错误处理、观测字段和版本说明。
