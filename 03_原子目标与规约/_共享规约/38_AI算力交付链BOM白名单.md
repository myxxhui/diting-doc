## 判决树（BOM 纳入/排除规则）

```
┌─ 该业务是否是「AI 算力基础设施」或「AI 模型服务」的直接物理/软件组成？
│   ├─ 是 → 继续
│   └─ 否 → 排除
│
├─ 该业务的主要下游客户中，AI/大数据是否 >50%？
│   ├─ 是 → 继续
│   └─ 否 → 检查: 该业务技术路线是否以 AI 需求为核心驱动力？
│       ├─ 是 → 继续 (如 800G 光模块)
│       └─ 否 → 排除
│
├─ 是否有 ≥1 家 A 股上市公司在该细分环节具备实质性业务？
│   ├─ 是 → 纳入候选
│   └─ 否 → 标注「当前无 A 股标的」· 不纳入
│
├─ 排除哨兵检查:
│   1. 纯消费品 (如 AI 手机) → 排除
│   2. 纯下游应用且无技术壁垒 (如 AI 客服 SaaS) → 排除
│   3. 基础设施但非 AI 专用 (如普通 IDC、通用光纤) → 排除
│   4. 被证监会 ST/*ST 标注 → 排除
│   5. 近 12 个月被证监会立案调查 → 排除
│   → 任意触发 → 排除
```

## 代表性标的数据模型（YAML 格式）

每个 BOM 节点在 `z0_policy_keywords.yaml` 的 `curated_bom_nodes` 中须包含 `representative_stocks` 字段：

```yaml
- node_id: ai_chip
  name: AI训练/推理芯片(GPU/NPU/ASIC)
  tier: 核心
  layer: L2
  representative_stocks:
    - symbol: "688256"
      name: 寒武纪
      rationale: 国产AI训练芯片龙头，思元系列已落地互联网/运营商
    - symbol: "688041"
      name: 海光信息
      rationale: 深算系列DCU对标CUDA生态，国产替代核心标的
    - symbol: "300474"
      name: 景嘉微
      rationale: GPU国产替代先行者，JM系列已适配LLM推理
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `symbol` | string | A 股 6 位代码 |
| `name` | string | 公司简称 |
| `rationale` | string | 纳入理由（约 20 字），注入 Prompt 作为 LLM 参考锚点 |

### 标的来源标记

LLM 输出时须为每只标的标记来源：

| `stock_source` 值 | 含义 | 前端徽章 |
|-------------------|------|---------|
| `"curated"` | 来自 YAML 白名单的代表标的 | 🏷 代表标的 (purple) |
| `"llm"` | LLM 自行增补的标的 | 🤖 LLM 增补 (gray) |

### 降级策略

| 级别 | 触发条件 | 行为 |
|------|---------|------|
| A: LLM 全量生成 | 正常 | LLM 以代表标的为锚点，增补至 3-5 只并完整打分 |
| B: 混合模式 | LLM 部分标的缺分 | `_clean_output()` 过滤掉不合格标的；代表标的自动保留 |
| C: 纯代表标的最低集 | LLM 全部失败 | 全部回退到代表标的，带入低置信度标记 |

## 边界案例手册

所有边界争议的裁决必须记录在此，新增节点时必须先回答类似问题。

| 案例 | 问题 | 裁决 | 理由 |
|------|------|------|------|
| edge_01 | 光模块 vs 通用光纤 | 光模块纳入，通用光纤排除 | 光模块（800G/1.6T）的技术迭代 100% 由 AI 数据中心需求驱动。普通光纤是通信基建用途 |
| edge_02 | 工业富联 (AI 服务器 ODM) | **纳入** (AI 服务器节点) | 其为英伟达/微软等生产 AI 服务器，是算力基础设施的直接物理组成部分；虽无自有 IP，其代工壁垒（产能/良率/客户认证周期 ≥ 12 个月）符合高壁垒判据 |
| edge_03 | PCB 纳入范围 | 纳入「高速 PCB/IC 载板」；排除通用 PCB | AI 服务器对高多层 (≥20 层)、超低损耗 PCB 有特殊要求；沪电股份/深南电路在此领域为全球主要供应商。通用 PCB 不纳入 |
| edge_04 | 液冷/散热 | **纳入** | AI 服务器功率密度 700-1500W，是传统服务器 3-5 倍。液冷从可选项变为必选项。通用空调/风机排除 |
| edge_05 | 晶圆代工 (中芯国际/华虹) | 纳入「先进封装与 Chiplet」的补充备注 | 晶圆代工是半导体制造上游基础环节，不是 AI 算力的直接组成部分。中芯国际若未来直接参与 AI 芯片制造（如 CoWoS 封装），则纳入此节点 |
| edge_06 | AI 电源 | **纳入** | AI 服务器单机柜功耗可达 50-100kW，BBU/PSU 需求数量和功率等级倍增。麦格米特/欧陆通已进入英伟达/AMD 供应链 |
| edge_07 | 连接器/线缆 | 暂不纳入 | 高速连接器受益但非 AI 独占需求，且 A 股纯正标的较少 |
| edge_08 | 算力租赁 (服务) | 纳入「AI 服务器与系统集成」配套 | 算力调度/租赁平台算力是算力资源的社会化分发；但纯平台运营模式壁垒较低 |
| edge_09 | 交换机芯片 | **纳入** | 盛科通信已量产 400G 交换芯片，800G 在研；AI 集群对交换芯片速率要求极高 |

## LLM 行为约束（Prompt 硬约束）

以下规则写入 LLM System Prompt，禁止在运行时被覆盖：

```
1. 你必须在白名单所列节点范围内进行标的映射。
2. 你不得擅自新增或删除任何节点。
3. 如果你认为白名单外有值得考虑的新节点，
   请在输出的 suggested_additions 字段中列出，由架构师审批。
4. 每个标的必须输出完整的 scoring_detail。
5. 每个因子必须附带至少 2 条可验证的 evidence。
6. 概念标签信息仅用于 policy_bond 加分判断，
   不可用于限制候选范围。
7. 排除规则检查必须逐项执行。
```

## LLM 输出校验

服务端在收到 LLM 输出后，须执行以下校验，任意一项不通过则拒绝落库：

| 校验项 | 规则 |
|--------|------|
| 节点合法性 | 输出中的所有 node_id 必须在白名单 v1.0.0 的 `node_id` 列表中 |
| scoring_detail 完整性 | 每个标的必须包含 moat/growth/profit/localize 四项 + composite |
| evidence 完整性 | 每个因子必须包含 evidence (≥2 条) |
| exclusion_check 完整性 | 每个标的必须包含 exclusion_check 子字段 |
| composite 值域 | composite = moat×0.30 + growth×0.25 + profit×0.20 + localize×0.15 + [0 或 0.2] |
