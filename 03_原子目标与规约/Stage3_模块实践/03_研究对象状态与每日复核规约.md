## 1. 目标与边界

**目标**：在 A 轨短线场景下，明确 **Module B（信号层）** 与 **持仓/执行（Module E 及 OMS）** 的职责：开仓时锚定风险与止盈结构；持仓期间用新数据做 **去留复核**，止损调整 **默认只紧不松**；**不**默认用每日新表整体覆盖既有止损/止盈阶梯。

| 层级 | 职责 |
|------|------|
| **Module B** | 每日输出 `technical_score`、`risk_rules_json`、`take_profit_json` 等（见 [02_B模块策略 §8](./02_B模块策略_策略实现规约.md)）；**不**持久化持仓、不单独决定「是否平仓」的最终状态机 |
| **Module E / 执行** | 读取持仓与当日 B 输出，按本规约合并止损、累计「低于阈值」日数并触发复核结果；落库与下单在其职责内完成 |
| **本规约 + diting-core** | 提供 **纯函数级** 合并与复核规则（`diting.position_lifecycle`），供热路径调用；配置键与 [dna_a_track_position_lifecycle](../_System_DNA/core_modules/dna_a_track_position_lifecycle.yaml) 一致 |

## 2. 开仓锚定（与 B 的 `a_track_short.risk` 一致）

开仓成交或确认开仓时，以 **当日** B 输出写入持仓记录至少包含：

- 入场价 `entry`
- 初始止损价 `stop`（由 `risk_levels` / `risk_rules_json` 解析）
- 风险额 **R（风险单位）** = |entry − stop|（多头为 entry − stop）
- 止盈阶梯（如 1R、2R 对应价），来自 `take_profit_r_multiples` 与当日字段

后续 **不**因「仅是新的一天」而自动重算整组止盈档；若需调整须在策略或产品层显式开关（见 §5）。

## 3. 持仓期：止损合并（只紧不松）

每日可根据最新 B 建议止损价 `suggested_stop` 与持仓已有 `current_stop` 合并：

- **多头**：更紧 = 止损价 **更高**（更接近现价）。合并：`max(current_stop, suggested_stop)`（在止损位于 entry 下方的常规定义下）。
- **空头**：更紧 = 止损价 **更低**。合并：`min(current_stop, suggested_stop)`。

若 `tighten_only_default: true`，**禁止**用更宽的 `suggested_stop` 覆盖已有更紧止损。

实现见 `diting.position_lifecycle.merge_stop_tighten_only`（[Ref: 03_研究对象状态与每日复核规约]）。

## 4. 持仓期：信号复核（去留）

用当日 B 的 `technical_score`（或等价主分）与配置阈值比较：

- 低于 `min_technical_score_for_hold` 计为 **不合格日**。
- 连续 `consecutive_days_below_before_exit` 个不合格日 → 复核结论为 **建议退出**（由 E/策略最终下单）。

缺失当日信号时行为由 `treat_missing_daily_signal_as` 定义（默认等同「低于阈值」）。

实现见 `diting.position_lifecycle.update_exit_streak`。

## 5. 止盈阶梯与每日 B

默认 **`daily_b_resets_tp_ladder_default: false`**：每日全量 B 输出 **不**用于整体替换开仓时落库的 1R/2R 等结构。新开仓仍使用 **当日** B 表；已持仓标的的止盈管理以初始锚定 + 显式规则（如移动止盈、决策中枢输出）为准。

## 7. Module E 最小数据接口（只读）

Module E（及同进程编排层）要对 **已持仓** 做合并止损、累计复核日数、硬止损等，必须能拿到 **持仓与行情** 的**只读**输入；**Module B 不**承担账户/持仓真相，也**不**作为 last price 的唯一来源。与 [05_接口抽象层规约](../_共享规约/05_接口抽象层规约.md) 中的 **BrokerDriver**、**MarketDataFeed**（或项目内等价抽象）对齐实现。

### 7.1 只读持仓 API（Read-only positions）

**用途**：列出当前未平仓持仓，并读取每条持仓上用于风控与复核的字段（可与 OMS/本地表同源）。

**最小字段集**（名称可与实现一致，语义须覆盖）：

| 字段语义 | 说明 |
|----------|------|
| `symbol` | 标的代码 |
| `side` | 多 / 空（或 long / short） |
| `quantity` | 持仓数量（或可用张数） |
| `entry_price` | 开仓锚定入场价 |
| `current_stop_price` | 当前有效止损价（经合并后的结果） |
| `exit_review_streak`（可选） | 连续低于技术面阈值日数，若由 E 持久化则读出 |

**约束**：本接口为 **只读**（查询）；**写入**（更新止损、streak、平仓）仍在 E/OMS/执行链路内完成，不通过「假装的写」混在只读 API 中。

### 7.2 只读 last price（或等价标记价）

**用途**：**盘中**规则依赖「当前价 vs 入场/止损」时调用，例如硬止损触发、跌破止损线即市价处理。

**语义**：按标的返回 **最后一笔成交价、盘口中间价、或交易所标记价** 之一（产品在 L3 或部署配置中**固定一种**并文档化），记为 `last_price`；**只读**。

**与日频复核的关系**：若产品仅做 **每日一次** 的 B 快照复核（`technical_score` + 止损合并），**可以**不依赖逐笔行情，仍以**当日 B 输出 + 持仓快照**为主；一旦启用 **盘中** 硬止损或实时触发，`last_price` 为**必需输入**。

### 7.3 与 Module B 的衔接方式

对已持仓标的：由编排层用 `symbol` 将 **只读持仓** 与 **当日 B 输出（或 L2 快照）** 做 join，再调用 `diting.position_lifecycle` 与 RiskShield；**B 不负责 join**。
