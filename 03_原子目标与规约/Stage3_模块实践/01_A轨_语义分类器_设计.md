<a id="design-stage3-01-goal"></a>
## 本步目标

实现 Module A 语义分类器：基于标的代码、申万行业、营收占比等输入，输出领域标签（农业/科技/宏观/未知）与置信度；单模块实现 + 与下游联动 + 全链路验证可达。

<a id="design-stage3-01-points"></a>
## 设计要点

- **输入**：标的代码、申万行业分类、营收占比数据（来源见 11_ 数据采集与输入层规约）；**扩展**：L2 标的主营构成表（见 [12_右脑数据支撑与细分规约](../_共享规约/12_右脑数据支撑与Segment规约.md)），供输出主营占比列表。
- **输出**：Domain Tag、置信度分数 (0.0–1.0)；**主营占比列表**（segment_shares：repeated { 细分标识、营收占比、是否主营 }，与标的主营构成表一致，见 12_、[04_ ClassifierOutput 扩展](../_共享规约/04_全链路通信协议矩阵.md)）；**须写入 L2 或共享存储**（表 classifier_output_snapshot 或约定协议），供 Module B/C 按约定读取，见 [01_A轨_语义分类器_实践·上部分·数据流与 L2 规约](../../04_阶段规划与实践/Stage3_模块实践/01_A轨_语义分类器_实践.md#l4-stage3-01-part1-dataflow)。
- **标的池**：当前全 A 股（universe）；执行前通过 [11_](./11_数据采集与输入层规约.md) 约定的 **`get_current_a_share_universe()`** 获取列表，或由调用方传入该列表；对**全部 N 只**全量分类，与 Module B 使用同一接口或同一传入列表（同批一致见 09_）。
- **分类规则**：农业（农林牧渔/营收占比）、科技（电子计算机通信/研发投入）、宏观（有色金属石油石化/大宗商品营收）；其余 未知
- **不做预测**：仅做分类，为 Module B/C 提供 Tag
