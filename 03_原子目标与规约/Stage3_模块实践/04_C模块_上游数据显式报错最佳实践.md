# C 模块 · 上游数据显式报错最佳实践

> 设计目标：C 模块所有策略/逻辑输出**依据实际生产工作流**；**上游无数据时显式输出「上游无数据」**，禁止虚假或兜底返回。

## 一、设计原则

1. **显式报错**：segment_list、segment_signals 任一缺失时，输出 `is_supported=False`，且 `reasoning_summary` 明确写明「上游无数据」及对应来源。
2. **禁止 stub 冒充生产**：`MOE_STUB_SEGMENT_SIGNALS=1` 仅用于本地联调，**不得**作为生产依据；生产必须 `stub=0` 且使用真实右脑输入：**`DITING_TRACK=b`** 时以 L2 **`segment_signal_cache`** 为准；**`DITING_TRACK=a`** 时 Module C 合并 **`a_track_signal_cache`**（与 B 轨缓存并列，见 [06_A轨_生产级数据要求_设计](../Stage2_数据采集与存储/06_A轨_生产级数据要求_设计.md)、[07_行业新闻与标的新闻分离存储_设计](../Stage2_数据采集与存储/07_行业新闻与标的新闻分离存储_设计.md)）。
3. **不兜底**：无数据时不伪造、不猜测、不返回中性兜底；仅返回「上游无数据」类说明。

## 二、上游数据与报错文案

| 缺失数据 | 报错文案 | 数据来源 |
|----------|----------|----------|
| segment_list 为空 | 上游无数据：标的主营构成（segment_list）为空；来源 L2 symbol_business_profile 或 Module A segment_shares | 采集 ingest_business_profile |
| segment_signals 为空 | 上游无数据：细分垂直信号（segment_signals）为空。**B 轨**：L2 `segment_signal_cache`，需 `refresh_segment_signals_for_symbols`。**A 轨**：合并后仍空时，常见为 `a_track_signal_cache` 无数据或 `news_content` 无正文（须 scope 迁移与 refresh） | Stage2-06 信号层 + A 轨 `a_track_refresh` |
| 全部细分无有效信号 | 上游无数据：全部细分无垂直信号。**B 轨**：`segment_signal_cache` 无该标的 `segment_id` 或条目无方向。**A 轨**：主营细分无 B 轨缓存且 A 轨合并未提供可用信号 | 按 `DITING_TRACK` 区分 |
| 主营一票否决 | 主营细分无利好或利空（认知边界） | 正常逻辑，非上游缺失 |
| 利好与主营未对齐 | 利好与主营未对齐（alignment_score < 0.3） | 正常逻辑，非上游缺失 |

## 三、实现要点

1. **alignment.py**：`compute_alignment_and_aggregate`、`should_reject_by_cognitive_boundary` 的 `reasoning_parts` / 返回原因均使用上述显式文案。
2. **experts.py**：`unified_opinion` 在 `reject` 分支中，`reasoning_summary` 必须含「上游无数据」或等价语义，不得用「无」「为空」等模糊表述。
3. **run_module_c_local.py**：
   - 默认 `MOE_STUB_SEGMENT_SIGNALS=0`（生产）；`make run-module-c` 改为传 0 或由环境变量控制。
   - `stub=0` 时：**B 轨**从 `segment_signal_cache` 读；**A 轨**再合并 `a_track_signal_cache`（`merge_a_track_into_segment_signals`）。若无数据则保持空或仅部分填充，C 输出「上游无数据」或认知边界文案。
   - `stub=1` 时打印醒目警告，标注「仅供联调，不作为生产依据」。

## 四、验收

- [ ] segment_list 为空时，输出含「上游无数据：标的主营构成…」
- [ ] segment_signals 为空时，输出含「上游无数据：细分垂直信号…」
- [ ] 全部细分无信号时，输出含「上游无数据：全部细分无垂直信号…」
- [ ] 生产环境禁止 stub；stub 仅用于显式联调
