**定位**：在现有规约与实现基础上，对 B 模块策略与打分机制做**深度审查**，给出「更高、更准、更稳」的改进方向与可落地方案。

**实施状态**：以下「优先可落地方案」已**全部严格实现、默认开启**（code 与 scanner_rules.yaml / DNA 一致）：趋势 ADX 过滤、趋势部分确认(2/3)、反转 RSI 扩展区间、放量止跌加成、突破 ATR 归一化、多池加权融合、输出截面分位(technical_score_percentile)。详见 [02_B模块策略_策略实现规约](./02_B模块策略_策略实现规约.md) 与 diting-core/config/scanner_rules.yaml。

**B 轨视野**：当前策略与打分为 A 轨（短周期技术面）服务；若需让**具备长期价值的标的进入 B 轨视野**，需在 Module B 或独立 B 轨预筛中补充「长期价值发现」能力（如长期动量 120d/250d、或基本面质量标签）。缺口与可选方案见 [06_B轨需求与实现缺口分析](../../06_追溯与审计/B轨需求与实现缺口分析.md)#四、Module-B-与-B-轨视野。

## 一、当前设计优点（保留）

| 维度 | 现状 | 评价 |
|------|------|------|
| **多池分离** | 趋势/反转/突破（+ 可选动量）独立计算，取 max + 共振加成 | 符合「多策略、可解释」；下游可按 strategy_source 区分 |
| **连续子分** | 每条件 0–50、两条件 cap 100，避免 0/40/80 离散 | 比离散档位更细，利于排序与阈值微调 |
| **确认根数** | 趋势 3 根、突破 2 根满足才给分 | 降低单日噪音，减少假信号 |
| **YAML 驱动** | 阈值、开关、优化项全配置化 | 可调参、可 A/B 而不改代码 |
| **可选优化与过滤器** | 趋势强度、急跌阴跌、突破 3 日确认、流动性/波动率/板块 | 已预留扩展，按需开启 |

## 二、薄弱点与可改进方向

### 2.1 趋势池

| 问题 | 说明 | 改进方向 |
|------|------|----------|
| **无趋势强度（ADX）** | 仅用均线多头 + MACD，不区分「趋势强弱」；震荡市也可能短暂满足均线 | **可选**：引入 ADX(14)，当 ADX &gt; 25 时趋势池得分才生效或乘以 min(1, (ADX-10)/25)，弱趋势降权 |
| **确认根数全有全无** | 3 根中 1 根不满足即整池 0，过于严格 | **可选**：部分确认也给分，如「满足根数/3 × 原得分」或「2 根满足给 70% 分」 |
| **单周期** | 仅日线，易受单日波动影响 | 中长期可考虑周线或双周期一致（日线+周线同向）作为加分项（实现成本较高，可作 Phase2） |

### 2.2 反转池

| 问题 | 说明 | 改进方向 |
|------|------|----------|
| **RSI 阈值过严** | RSI&lt;20 才给 RSI 子分，多数时间为 0 | **可选**：RSI 在 [20, 35] 区间给**连续子分**，如 50×(35-RSI)/15，与超卖深度成正比 |
| **缺量能确认** | 止跌仅 close&gt;MA5，无「放量止跌」 | **可选**：当日或前一日 volume &gt; 1.2×SMA(vol,5) 时给小幅加成（如 ×1.1 cap 100） |
| **布林与 RSI 重叠** | 超卖 + 触及下轨 常同时出现，两子分易同高 | 可保持现状；若希望区分「纯超卖」与「触及下轨」可拆权（如 RSI 子分 60%、布林 40%） |

### 2.3 突破池

| 问题 | 说明 | 改进方向 |
|------|------|----------|
| **放量倍数固定** | 2×SMA(vol,20) 为硬门槛，不同标的波动率差异大 | **可选**：用「成交量在全市场或同板块分位」替代固定倍数，或 2× 与 5× 之间线性插值子分 |
| **突破幅度与波动未挂钩** | 突破 2% 即满子分，小盘高波易满分 | **可选**：突破幅度除以 ATR(14) 或 20 日波动率，使「有效突破」与波动率成比例（如 0.5×ATR 以上才给满） |
| **假突破无惩罚** | 突破后若很快回落，历史上无「失败记录」反馈 | 已有 breakout_hold_days 可选；可补充「近 5 日曾突破又跌破」的降权（需额外状态或查历史） |

### 2.4 多池融合与总分

| 问题 | 说明 | 改进方向 |
|------|------|----------|
| **仅取 max** | 总分 = 最高池 + 共振加成，次高池仅影响共振 | **可选**：加权融合，如 score = 0.7×max + 0.2×second + 0.1×third，或按历史胜率加权（需回测） |
| **无截面排序** | 输出为绝对分，下游若做 Kelly 需自行分位 | **可选**：输出 technical_score 的同时输出 technical_score_percentile（全市场分位 0–1），便于仓位与阈值按分位设 |
| **共振阈值固定** | 40 分、二档 60 分为固定值 | 可保持；若标的数很多，可改为「前 20% 分数」等相对阈值（需规约约定） |

### 2.5 与市场/板块环境

| 问题 | 说明 | 改进方向 |
|------|------|----------|
| **大盘环境未用** | 未考虑指数（如沪深300）趋势；熊市里趋势池易假阳性 | **可选**：当指数 MA20 &lt; MA60 时，趋势池得分 ×0.7 或仅当指数多头时趋势池才生效（需指数数据源） |
| **板块强度未接** | sector_strength 默认 1.0，未接 Module A 或行业数据 | 按现有规约接入板块/行业 technical_score 均值比即可 |
| **波动率仅惩罚** | 高波时整体乘 penalty，未区分「高波突破」与「高波震荡」 | 可保持；更细可做「高波 + 突破」不惩罚、「高波 + 趋势」惩罚（实现复杂，可选） |

### 2.6 稳定性与可解释性

| 问题 | 说明 | 改进方向 |
|------|------|----------|
| **无历史命中反馈** | 各池权重与阈值未根据回测/实盘命中率迭代 | 建议：用 Alphalens/Qlib 做因子 IC、分层收益、换手，定期调 multi_pool_bonus、各池阈值 |
| **同一标的重复触发** | 连续多日同一标的趋势 100 分，无「去重」或冷却 | **可选**：L2 或下游记录「最近 N 日已入选」同一 symbol 时，对得分 ×0.9 或仅取「新入选」 |
| **边界数值敏感** | 如 RSI=19 与 21 差异大 | 已用连续子分缓解；可保持 |

## 三、优先可落地的「更准」方案（建议顺序）

以下按**实现成本低 → 中**、**收益高**排序，便于分阶段做。

1. **趋势池：ADX 过滤/调制（可选）**  
   - 增加配置项 `trend_adx_enabled`、`trend_adx_min`（如 25）。  
   - 当 ADX(14) &lt; trend_adx_min 时，趋势池得分 ×0.6 或 0；否则不变。  
   - **目的**：弱趋势时降权，减少震荡市假金叉。

2. **反转池：RSI 扩展区间连续子分（可选）**  
   - 配置 `reversion.rsi_oversold`（20）保留；增加 `reversion.rsi_soft_ceiling`（如 35）。  
   - 当 RSI ∈ [20, 35] 时，RSI 子分 = 50×(35−RSI)/15；RSI&lt;20 仍按现有公式。  
   - **目的**：温和超卖也能拿到部分分，提高反转池覆盖与排序能力。

3. **突破池：突破幅度与 ATR 挂钩（可选）**  
   - 配置 `breakout.atr_scale_enabled`、`breakout.min_atr_multiple`（如 0.5）。  
   - 价格突破子分 = 在现有公式基础上 × min(1, (close−max_high_prev) / (min_atr_multiple×ATR))。  
   - **目的**：突破幅度相对波动率归一化，减少「小波动假突破」拿满分。

4. **输出截面分位（可选）**  
   - 在 scan_market 输出中增加 `technical_score_percentile`（全市场 0–1）。  
   - 下游阈值可按「取前 10%」而非「&gt;70 分」设，更稳定。

5. **多池加权融合（可选）**  
   - 配置 `fusion_mode`: `max`（当前）| `weighted`。  
   - `weighted` 时：score = w1×s1 + w2×s2 + w3×s3（可配置 w1,w2,w3），再 cap 100。  
   - 权重可由回测得出或先用 0.6/0.25/0.15 等经验值。

6. **趋势确认根数部分给分（可选）**  
   - 配置 `trend_partial_confirm`：当 3 根中满足 2 根时，趋势池得分 ×0.7。  
   - **目的**：在「更准」与「更多候选」之间折中。

## 五、总结

| 结论 | 说明 |
|------|------|
| **当前机制** | 多池、连续子分、确认根数、YAML 驱动、可选优化与过滤器已成型，**符合预期**，可继续作为主方案。 |
