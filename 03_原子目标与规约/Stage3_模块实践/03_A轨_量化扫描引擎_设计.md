<a id="design-stage3-03-goal"></a>
## 本步目标

实现 **Module B 量化扫描引擎（执行层）**：按 [Stage3-02 产出的策略规约与配置](./02_B模块策略_策略实现规约.md) 加载策略（含 scoring、optimization、filters、long_term）、执行全市场扫描（TA-Lib 指标、**多池判定**（含可选第四池动量）、长期动量分位、过滤）、写入 L2 quant_signal_*；产出 QuantSignal（含 second_pool_id、second_pool_score、pool_scores、long_term_score、long_term_candidate（B 轨候选）、liquidity_score、volatility_ratio，规约 §3.7、§4）供 Module C/D 消费。**策略语义与打分公式由 02 定义，本步不重新定义。**

<a id="design-stage3-03-points"></a>
## 设计要点

- **策略来源**：策略池条件、TA-Lib 映射、technical_score 计算方式见 [02_B模块策略_策略实现规约](./02_B模块策略_策略实现规约.md)；本步从 `config/scanner_rules.yaml`（或 02 约定路径）加载配置，**不硬编码策略语义**。
- **标的池**：当前全 A 股（universe）；执行前通过 [11_](../_共享规约/11_数据采集与输入层规约.md) 约定的 **`get_current_a_share_universe()`** 获取列表，或由调用方传入；对**全部 N 只**全量向量化扫描，与 Module A 使用同一接口或同一传入列表（同批一致见 09_）。不写死数量。
- **Scanner**：technical_score_threshold、sector_strength_threshold 从配置加载（与 dna_module_b.scanner 一致）。
- **输入**：市场数据 OHLCV（L1）；**Module A 的 Domain Tag**：从 L2 表 `classifier_output_snapshot` 或约定共享存储按 batch_id/最新批次读取（见 [03_A轨_量化扫描引擎_实践·ClassifierOutput 输入约定](../../04_阶段规划与实践/Stage3_模块实践/03_A轨_量化扫描引擎_实践.md#l4-stage3-03-classifier-input)）。
- **输出**：QuantSignal 列表（含 technical_score、strategy_source、sector_strength、second_pool_id、second_pool_score、pool_scores、long_term_score、long_term_candidate（B 轨候选）、liquidity_score、volatility_ratio，见 02 规约 §4）；L2 表 quant_signal_snapshot、quant_signal_scan_all。

<a id="design-stage3-03-dna-keys"></a>
## 本步落实的 _System_DNA 键

- `core_modules/dna_module_b.yaml`（strategy_pools、scanner、**scoring**、**optimization**、**filters**、integration_packages）；本步**消费**02 产出的、与 dna_module_b 对齐的 YAML。

<a id="design-stage3-03-deps"></a>
### 依赖与构建（部署配套）

- **逻辑填充期借鉴组件**：本步实现依赖 [01_顶层概念/05_谛听优先借鉴的十大开源选型](../../01_顶层概念/05_谛听优先借鉴的十大开源选型.md) 中 **TA-Lib**、**VectorBT**；Qlib/Alphalens 归属 Stage3-02 策略层，本步可选依赖（如回测脚本）。
- **要求**：diting-core 的 Dockerfile 及 requirements 中**显式声明并安装** TA-Lib、VectorBT 等；TA-Lib 需系统层 C 库 + Python 绑定。构建后 `make test` 及本步单测须在镜像内可运行。
- **验收**：L4 本步准出时须包含「依赖已写入 Dockerfile/requirements」「镜像内 make test 与本步单测通过」的验证。

<a id="design-stage3-03-integration-talib"></a>
### 逻辑填充期开源接入点：TA-Lib（Phase1 必选）

- **实践重点**：扫描引擎的**全部技术指标**统一基于 TA-Lib 计算；指标列表与 API 映射以 [02_B模块策略_策略实现规约](./02_B模块策略_策略实现规约.md) 为准。
- **验收要点**：至少一个策略池的指标 100% 来自 TA-Lib；单测用固定 OHLCV 与已知结果对比，数值误差在约定范围内。

<a id="design-stage3-03-integration-vectorbt"></a>
### 逻辑填充期开源接入点：VectorBT（Phase1 必选）

- **实践重点**：全市场粗筛、向量化；输入 L1 OHLCV、输出候选池（技术分数 0–100）；与 02 产出的阈值配置对应。
- **验收要点**：给定数据规模下跑通粗筛，输出符合下游 Module C 契约；性能可约定（如 &lt; 30 分钟）。

<a id="design-stage3-03-deploy-chain"></a>
### 可部署单元与构建部署

Module B 为**独立可部署单元**（见 [09_ 可部署单元清单](../_共享规约/09_核心模块架构规约.md#可部署单元清单)）。须支持**单独运行**、Dockerfile（路径与 `global_const.deployable_units.module_b.dockerfile_path` 一致）、**镜像与版本管理**、**一键构建**（如 `make build-module-b`）、**一键部署**。具体约定见 [03_A轨_量化扫描引擎_实践](../../04_阶段规划与实践/Stage3_模块实践/03_A轨_量化扫描引擎_实践.md#l4-stage3-03-deploy-chain) §6 与 `global_const.deployable_units.module_b`。
