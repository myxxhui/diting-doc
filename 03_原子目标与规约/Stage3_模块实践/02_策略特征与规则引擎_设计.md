<a id="design-stage3-02-goal"></a>
## 本步目标

将 **Module B 的策略层**独立成步：负责**策略池定义、打分策略设计、策略实现规约、配置 schema（scanner_rules.yaml）、验证与优化**。本步**不运行** QuantScanner、不写 L2 表 `quant_signal_*`；产出供 [Stage3-03 量化扫描引擎](./03_量化扫描引擎_设计.md) 消费。

<a id="design-stage3-02-scope-boundary"></a>
## 范围边界（与 01 / 03 严格区分）

| 维度 | Stage3-01 语义分类器 | Stage3-02 B 模块策略（本步） | Stage3-03 量化扫描引擎 |
|------|----------------------|------------------------------|------------------------|
| **所属** | Module A | Module B（策略层） | Module B（执行层） |
| **产出** | ClassifierOutput、L2 classifier_output_snapshot | 策略实现规约、scanner_rules.yaml 及 schema、验证报告 | QuantSignal、L2 quant_signal_* |

<a id="design-stage3-02-points"></a>
## 设计要点

- **策略池**：趋势（均线/MACD）、反转（RSI/Bollinger）、突破（高点/成交量）；可选第四池动量（20 日收益分位+MA20）；各池 score_range [0,100]。可执行条件、TA-Lib 映射、打分公式见 [02_B模块策略_策略实现规约](./02_B模块策略_策略实现规约.md)。
- **配置与 YAML**：`scanner_rules.yaml`（或等价）的 **schema** 与键名由本步定义；与 `dna_module_b.strategy_pools`、`scanner`、**scoring**、**optimization**、**filters**、**long_term**（中长期价值发现 / `long_term_candidate`）对齐；禁止在引擎代码中硬编码策略语义。**scanner** 节点含通过阈值及可选收紧变量 `pass_require_lead_pool_min`、`pass_require_second_pool_min`（未设置或 null 则无额外约束），见 [02_B模块通过组合与设计符合度分析](./02_B模块通过组合与设计符合度分析.md)#五。**long_term** 节点含 `enabled`、`lookback_days`（如 120）、`score_threshold`（截面分位阈值），见 [01_需求与产品范围](../../03_原子目标与规约/平台与产品/01_需求与产品范围.md)。
- **打分策略**：每条件 0–50 连续子分、两条件相加 cap 100，多池共振（含可选二档）、趋势/突破确认根数、反转止跌、可选趋势强度/急跌阴跌/突破 3 日确认/动量池等，见 [02_B模块策略_策略实现规约](./02_B模块策略_策略实现规约.md)；scoring、optimization、filters 均由 YAML 驱动。扫描输出含 second_pool_id、second_pool_score、pool_scores、**long_term_score**（长期动量截面分位）、**long_term_candidate**（B 轨候选）等供下游与 Router/VC-Agent 使用。
- **验证与优化**：Alphalens 质检（IC、分层、换手）、Qlib 回测、参数调优流程；本步定义「策略池/因子上线准入」条件；未通过质检的策略池不得进入生产配置（由 03 加载的 YAML）。

<a id="design-stage3-02-alphalens"></a>
### 逻辑填充期接入点：Alphalens（策略质检）

- **归属**：策略池/因子上线前的质检归属**本步**（Stage3-02），不归属 03 执行层。
- **实践重点**：每个策略池（或新因子）上线前经 Alphalens：IC、分层收益、换手率、tear sheet；准出规则（如 IC 均值 > 阈值、分层单调性）由本步设计文档或策略实现规约定义。
- **触发时机**：策略池或因子配置变更、定期复检时触发；与 Qlib 配置联动。
- **验收要点**：至少一个策略池走通「因子 → Alphalens → tear sheet → 准出判断」或占位约定；脚本/单测可自动跑 Alphalens 并解析结果。

<a id="design-stage3-02-qlib"></a>
### 逻辑填充期接入点：Qlib（回测与配置驱动）

- **归属**：策略回测与配置驱动归属**本步**（Stage3-02）；03 仅消费本步产出的配置与规约。
- **实践重点**：YAML 配置驱动回测、数据/因子管道、与策略实现规约的因子一致；回测结果用于策略调优与准入判断。
- **验收要点**：修改 YAML 可重放回测；数据/因子管道有单测或小规模集成测；本阶段可占位。

<a id="design-stage3-02-exit"></a>
## 准出

- 策略实现规约定稿，与 [02_B模块策略_策略实现规约](./02_B模块策略_策略实现规约.md) 一致。
- `scanner_rules.yaml`（或等价）schema 定义完成（含 strategy_pools、**long_term**（enabled、lookback_days、score_threshold）、**scanner**（含 technical_score_threshold、sector_strength_threshold、pass_require_lead_pool_min、pass_require_second_pool_min）、scoring、optimization、filters），可被 Stage3-03 加载；键名与 dna_module_b 对齐。
- 至少一个策略池完成 Alphalens 质检流程（或占位约定达成）；Qlib 回测可占位。
- L5 [l5-stage-stage3_02](../../05_成功标识与验证/02_验收标准.md#l5-stage-stage3_02) 可更新。
