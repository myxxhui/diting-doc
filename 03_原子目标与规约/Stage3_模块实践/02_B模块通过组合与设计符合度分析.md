> - **原子能力**: [02_B模块策略_策略实现规约](./02_B模块策略_策略实现规约.md)、[09_核心模块架构规约](../_共享规约/09_核心模块架构规约.md)（Module B 机会丰度）
> - **阶段**: [Stage3_模块实践](../../04_阶段规划与实践/Stage3_模块实践/)

**目的**：枚举「总分≥70 通过」的所有池分数组合，分析当前策略/打分/放通逻辑是否合适、是否存在「组合差却放通」的风险，并对照 Diting 总体设计与 B 模块职责评估是否符合或超出预期。

---

## 一、当前公式与通过条件（约定）

| 项 | 公式或条件 |
|----|------------|
| **加权融合** | 总分 base = 0.7×s1 + 0.2×s2 + 0.1×s3（s1≥s2≥s3 为三池得分降序） |
| **共振加成** | 若至少 2 池得分 ≥ 40，则 总分 = min(100, base + 10) |
| **通过** | 总分 ≥ 70 且 sector_strength ≥ 阈值 |

因此有两类通过方式：**无共振即过**（base≥70）、**靠共振加成才过**（base∈[60,70) 且至少 2 池≥40）。

---

## 二、能通过的所有组合类型（摘要）

枚举方式：三池得分在 0,5,10,…,100 上离散，s1≥s2≥s3；代码见 `diting-core/scripts/analyze_b_module_pass_combinations.py`。

### 2.1 无共振即通过（base ≥ 70）

- **组合数**：约 **835 组**（离散步长 5）。
- **典型**：
  - **单池满分**：(100, 0, 0) → base=70，**刚好过**。
  - **一强两弱**：(100, 任意, 任意) 只要 base≥70 即过，例如 (100, 35, 0)=70.5。
  - **双强**：(90, 35, 0)=70；(80, 70, 0)=70；等。
  - **三池都贡献**：如 (80, 65, 50)=78.5。

**结论**：只要「最高池」足够高，即可单独把总分撑过 70；或两池都较强（如 80+70、90+35）也可。

### 2.2 靠共振加成才通过（base ∈ [60, 70)，至少 2 池 ≥ 40）

- **组合数**：约 **220 组**。
- **条件**：base ≥ 60 且 base + 10 ≥ 70，且至少 2 池 ≥ 40。
- **典型**：
  - (85, 50, 0)：base=69.5，+10=79.5。
  - (75, 75, 20)：base=69.5，+10=79.5。
  - (70, 70, 65)：base=69.5，+10=79.5。
  - **边界**：base=60 时需如 (86, 40, 0) 或 (75, 75, 0) 等，即「一池很强或两池都较强」且存在第二池≥40。

**重要**：仅「两池刚过 40」(40, 40, 0) 时 base=36，+10=46 **不通过**。共振只能让 base 在 [60, 70) 的组合过线，无法让「双弱」过线。

### 2.3 你关心的两种说法

| 说法 | 是否成立 |
|------|----------|
| 一个指标 100 分就能通过 | **成立**。 (100,0,0) → 总分=70，刚好过。 |
| 两个指标都到 70 分能通过 | **成立，但依赖共振**。 (70,70,0) base=63，无共振不过；两池都≥40 触发 +10 后 63+10=73，**通过**。 |

## 三、是否存在「组合差却放通」？

### 3.1 风险点识别

- **单池 100、其余 0**：技术面仅一种信号极强（如纯趋势或纯突破），其余无支持。  
  - **是否算「差」**：取决于产品意图。B 的职责是「机会丰度」、做**初筛候选**，下游还有 Module C（专家）+ Module D（Quant∩Expert 投票）。单池满分在规约上被允许，且 09_ 明确「技术面得分>70 的候选池」。
- **base 刚过 60、靠共振 +10 到 70**：例如 (86,40,0)、(75,75,0)。  
  - 特征：**至少有一个池很强（86 或 75）或两池都较强（75+75）**，第二池≥40 仅用于触发加成。  
  - **不会**出现「两个都只是 40 分左右」就过线的情况（(40,40,0)=46）。

### 3.2 结论（组合质量）

- **不会出现「很多组合出的策略筛选的标的不行却放通」的宽口径漏洞**：  
  - 无共振时，必须 base≥70，即至少有一个池把加权分撑到 70（单池 100 或双池如 80+70）。  
  - 有共振时，base 也必须在 [60,70)，即至少「一强（≈86+）或双中强（≈75+75）」且另一池≥40；**双弱（两池仅 40）无法过线**。
- **可能的设计争议点**：  
  - **单池 100 即过**：若希望「至少两个维度都有一定支持」才放行，当前设计不满足，需加约束（见下节「可选的收紧」）。  
  - 当前 Diting 与 09_ 未要求「必须多池同时达标」，只要求「技术面得分>70」，故单池 100 放通与现有规约一致。

## 四、对照 Diting 总体设计与 B 模块职责

### 4.1 B 模块职责（09_、L1）

| 来源 | 表述 |
|------|------|
| **09_** | Module B：**机会丰度 (Opportunity Abundance)**；每日对全 A 股向量化扫描，输出**技术面得分>70 的候选池**。 |
| **L1 / 核心公式** | $Alpha = (Quant \cap Router) \times Kelly$；Quant 为技术面信号，与 Router(Experts) 取交后经 Kelly 仓位。 |
| **判官规则** | Quant **Pass** + Expert **Strong Buy** = 有效信号；B 只负责「Quant 是否 Pass」（得分>70），不单独决定下单。 |

因此：B 的职责是**宽进、可解释**的初筛，把「技术面有机会」的标的交给 C/D，由专家与投票做二次过滤与仓位决策。不要求 B 自己做到「高精度、低假阳」，而是「不漏机会、可解释、下游可纠偏」。

### 4.2 当前设计是否符合预期

| 维度 | 结论 | 说明 |
|------|------|------|
| **阈值 70** | ✅ 符合 | 与 09_「技术面得分>70」一致。 |
| **加权融合 0.7/0.2/0.1** | ✅ 符合 | 主池主导、次池辅助，兼顾「单信号极强」与「多信号中等」；strategy_source 仍为最高池，可解释。 |
| **共振 +10（≥2 池≥40）** | ✅ 符合 | 多维度弱确认给加分，但不足以让「双弱」过线，边界清晰。 |
| **单池 100 即过** | ✅ 与规约一致 | 规约未要求「至少两池达标」；若产品希望收紧，需显式改规约并加实现。 |
| **通过组合数量与质量** | ✅ 合理 | 无「双弱放通」；过线组合要么一强要么双中强/三池协同，符合「机会丰度」的宽进定位。 |

### 4.3 是否「完全充分符合或超出预期」

- **完全充分符合**：是。当前策略、打分、总分计算与放通条件与 09_、02_B 规约一致；通过组合集合无逻辑漏洞，且与「机会丰度 + 下游投票」的分工匹配。
- **是否「超出预期」**：在「可解释、可配置、多池+共振」上已优于「单一阈值」的简单设计；若希望进一步「更保守」或「更激进」，可在同一套公式下通过**调阈值、调权重、或加「最低主池分」等约束**实现，无需推翻当前设计。

## 五、可选收紧（已实现，配置驱动）

若产品/风控希望**避免「单池 100、其余全 0」即过**，可在保持当前公式前提下通过**配置变量**启用准出约束（二选一或组合使用）：

| 选项 | 配置键 | 规则 | 效果 |
|------|--------|------|------|
| **A. 主池下限** | `scanner.pass_require_lead_pool_min` | 通过条件增加：最高池 s1 ≥ 配置值（如 85） | 未设置（null）则不做约束；设置后主池低于该值不通过。 |
| **B. 次池下限** | `scanner.pass_require_second_pool_min` | 通过条件增加：次高池 s2 ≥ 配置值（如 25） | 未设置（null）则不做约束；设置后直接排除 (100,0,0)，要求至少两维度有分数。 |

**实现与默认**：在 `quant.py` 中根据 `get_pass_tightening_params()` 读取 `scanner_rules.yaml` 的 `scanner.pass_require_lead_pool_min`、`scanner.pass_require_second_pool_min`；**未设置或为 null 时使用当前默认行为（无额外约束）**，设置后按上述规则在计算 `passed` 时生效。

## 六、总结表

| 问题 | 结论 |
|------|------|
| 一个指标 100 或两个指标 70 能否通过？ | 能。单池 100→70 过；两池 70→63+10=73 过。 |
| 所有能通过的组合是否都合理？ | 是。无「双弱（如两池 40）」过线；过线均为「一强」或「双中强/三池协同」。 |
| 会不会很多「组合差却放通」？ | 不会。base 或 ≥70 或 ∈[60,70) 且至少一强/双中强。 |
| 相对 Diting 与 B 职责，设计是否充分符合预期？ | 是。与 09_、02_B 规约一致，且符合「机会丰度 + 下游投票」的职责划分；可选收紧已记录供后续产品决策。 |
