## 3. TA-Lib 指标与策略池映射

### 3.1 指标清单与 TA-Lib API

| 指标 | TA-Lib 函数 | 参数 | 用途 |
|------|-------------|------|------|
| 均线 MA | `talib.MA(close, timeperiod)` | timeperiod=5,10,20,60 | 趋势池：多头排列 |
| MACD | `talib.MACD(close, fastperiod=12, slowperiod=26, signalperiod=9)` | 返回 macd, signal, hist | 趋势池：DIF=macd, DEA=signal；水上金叉：macd>0 且 macd>signal |
| RSI | `talib.RSI(close, timeperiod=14)` | 0–100 | 反转池：RSI<20 或 RSI<30（超卖） |
| 布林带 | `talib.BBANDS(close, timeperiod=20, nbdevup=2, nbdevdn=2)` | 返回 upper, middle, lower | 反转池：价格触及下轨（close ≤ lower 或接近） |
| 最高价 N 日 | `talib.MAX(high, timeperiod=N)` | N=20 | 突破池：close > 前 N 日最高 |
| 成交量均线 | `talib.SMA(volume, timeperiod=20)` | 20 | 突破池：volume > 2×SMA(volume,20) |
| ATR | `talib.ATR(high, low, close, timeperiod=14)` | 14 | 波动率 regime、突破幅度归一化 |
| ADX | `talib.ADX(high, low, close, timeperiod=14)` | 14 | 趋势池：弱趋势降权（ADX<25 时得分×0.6） |
| N 日最低 | `talib.MIN(close, timeperiod=N)` | N=60 | 趋势强度：60 日区间分位 |

### 3.2 策略池条件（可执行布尔表达式）

| 策略池 | 条件（与 dna_module_b 语义一致） | 实现要点 |
|--------|----------------------------------|----------|
| **趋势 Trend** | ① 均线多头排列：**MA5 > MA10 > MA20 > MA60**；② MACD 水上金叉：DIF > 0 且 DIF > DEA。 | **最近 3 根 K 线均满足**两条件时，取最后一根连续子分；否则 0。 |
| **反转 Reversion** | ① RSI(14) **< 20**（超卖）；② 收盘价 ≤ 布林下轨（close ≤ lower×1.01）；③ 止跌：**close > MA5**。① 或 ② 命中；与 ③ 同满足时给高分。 | RSI 默认 20；配置可放宽为 30。下轨容差 1.01。 |
| **突破 Breakout** | ① 收盘价 > 前 20 日最高（不含当日）：close > MAX(high, 20)[-2]；② 成交量 > 2×SMA(volume, 20)，量比 cap 5.0。 | **最近 2 根 K 线中至少 2 根**满足双条件时取最后一根连续子分。 |

### 3.3 technical_score 计算（0–100，连续子分 + 多池共振）

**打分规则**：每池**每条件 0–50 连续子分、两条件相加 cap 100**；取**多池中最高分**为 technical_score 主值，strategy_source 为最高分对应池；**多池共振**：若至少两池得分 ≥ 40，则 technical_score = min(100, 最高分 + 10)。得分为 [0, 100] 内实数。

#### 趋势池（Trend）

| 子分 | 计算方式 | 说明 |
|------|----------|------|
| **均线子分（0–50）** | 满足「MA5>MA10」「MA10>MA20」「MA20>MA60」的条数 count ∈ {0,1,2,3}，子分 = (count / 3) × 50 | 3 条均满足 50，2 条约 33，1 条约 17，0 条 0 |
| **MACD 子分（0–50）** | 当 DIF>0 且 DIF>DEA 时：50 × min(1, (DIF−DEA) / (scale×close))，scale 默认 0.01；否则 0 | 柱状图强度归一化 |

**趋势池得分** = min(100, 均线子分 + MACD 子分)。当 `trend_confirm_bars=3` 时，**最近 3 根 K 线均满足**两条件才取该得分；若 `trend_partial_confirm_enabled=true`（默认），**2/3 根满足**时得分 × trend_partial_confirm_ratio（默认 0.7）；否则 0。**ADX 弱趋势降权**（默认开启）：当 `trend_adx_enabled=true` 且 ADX(14) < trend_adx_min（默认 25）时，趋势池得分 × trend_adx_penalty_ratio（默认 0.6）。

#### 反转池（Reversion）

| 子分 | 计算方式 | 说明 |
|------|----------|------|
| **RSI 子分（0–50）** | RSI < rsi_oversold（20）时：50×(20−RSI)/20；20 ≤ RSI ≤ rsi_soft_ceiling（35）时：50×(35−RSI)/15；RSI>35 为 0 | 扩展区间 [20,35] 连续子分，生产默认开启 |
| **布林下轨子分（0–50）** | 当 close ≤ lower×1.01 时：50 × min(1, (lower×1.01 − close) / (lower×0.01))；否则 0 | 触及越深分数越高 |
| **止跌加成** | 当 reversion_require_above_ma5 为 true 且 close > MA5 时，与 RSI/布林同满足则子分按规约叠加；否则仅 RSI/布林子分 | 减少接飞刀 |

**反转池得分** = min(100, RSI 子分 + 布林下轨子分)。**放量止跌加成**（默认开启）：当 reversion_volume_bounce_enabled=true 且 volume > reversion_volume_bounce_ratio×SMA(vol,5)（默认 1.2）时，得分 × reversion_volume_bounce_bonus（默认 1.1）cap 100。

#### 突破池（Breakout）

| 子分 | 计算方式 | 说明 |
|------|----------|------|
| **价格突破子分（0–50）** | 当 close > max_high_prev 时：50×min(1, (close−max_high_prev)/(0.02×max_high_prev))；**ATR 归一化**（默认开启）：当 breakout.atr_scale_enabled=true 时，该子分 × min(1, (close−max_high_prev)/(min_atr_multiple×ATR(14)))，默认 min_atr_multiple=0.5 | 突破幅度与波动率挂钩，减少假突破满分 |
| **放量子分（0–50）** | 当 volume > 2×SMA(vol,20) 时：50 × min(1, (volume − 2×vol_sma) / (3×vol_sma))，量比 > volume_cap_multiple（默认 5）按 50；否则 0 | 2 倍均量 0 分，5 倍 50 分 |

**突破池得分** = min(100, 价格突破子分 + 放量子分)。当 `breakout_confirm_bars=2` 时，**最近 2 根 K 线中至少 2 根**满足双条件才取该得分。

#### 多池融合与共振

- **融合模式**（默认 weighted）：`optimization.fusion_mode` 为 **weighted** 时，technical_score = w1×s1 + w2×s2 + w3×s3（s1≥s2≥s3 为排序后三池分），fusion_weights 默认 [0.7, 0.2, 0.1]，再 cap 100；为 **max** 时取最高池分。strategy_source 始终为最高分对应池。
- **多池共振**：当至少两池得分 ≥ multi_pool_min_score（40）时，在融合得分基础上 + multi_pool_bonus（10），再 cap 100。
- **通过组合与设计符合度**：所有「总分≥70」的池分数组合枚举及是否「组合差却放通」的分析见 [02_B模块通过组合与设计符合度分析](./02_B模块通过组合与设计符合度分析.md)。

#### 可选：趋势强度（价格在 60 日区间分位）

- **开关**：`optimization.trend_position_strength_enabled`（默认 false）。为 true 时，趋势池得分乘以 **(0.5 + 0.5 × position_pct)**，其中 position_pct = (close − MIN(close,60)) / (MAX(high,60)−MIN(low,60)) 在区间 [0,1] 内，避免除零；高位趋势给满权、低位趋势折半。
- **目的**：区分「刚金叉但仍在低位」与「金叉且接近前高」，提升趋势池质量。

#### 可选：反转池急跌/阴跌权重

- **开关**：`optimization.reversion_acute_slow_enabled`（默认 false）。为 true 时，以 5 日收益率 ret_5d 分档：**急跌**（ret_5d < −0.08）权重 1.2、**阴跌**（ret_5d > −0.03）权重 0.6、否则 1.0；反转池得分 = min(100, 原始得分 × 权重)。
- **配置**：reversion_acute_threshold（−0.08）、reversion_slow_threshold（−0.03）、reversion_acute_weight（1.2）、reversion_slow_weight（0.6）。

#### 可选：突破 3 日有效确认

- **开关**：`optimization.breakout_hold_days`（默认 0 表示不启用）。为 3 时，仅当「存在某日满足突破条件，且其后 3 根 K 线收盘均 ≥ 当日突破位×breakout_hold_ratio（默认 0.98）」时，突破池才给分；否则突破池 0。
- **配置**：breakout_hold_days（3）、breakout_hold_ratio（0.98）。

#### 可选：动量池（第四池，Pool 4 Momentum）

- **开关**：`optimization.pool_4_momentum_enabled`（默认 **true**，与生产配置一致）。为 true 时增加第四池：① 20 日收益率在全市场或同板块分位数 ≥ momentum_percentile_threshold（默认 0.80）；② close > MA20。两条件各 0–50 连续子分，相加 cap 100。
- **与三池关系**：取**四池**最高分作为 technical_score 主值，strategy_source 增加 MOMENTUM（4）；多池共振仍按「≥2 池 ≥ multi_pool_min_score」计算，池集合含动量池。

#### 可选：多池共振二档

- **开关**：`optimization.multi_pool_tier2_enabled`（默认 false）。为 true 时：至少两池 ≥ multi_pool_min_score 时 +multi_pool_bonus（如 10）；**且**至少两池 ≥ multi_pool_tier2_threshold（默认 60）时再 +multi_pool_tier2_bonus（如 5），总分 cap 100。
- **配置**：multi_pool_tier2_threshold（60）、multi_pool_tier2_bonus（5）。

#### 配置项（scanner_rules.yaml / DNA）— 基础

- **scoring**：trend.macd_hist_scale_ratio（0.01）；reversion.rsi_oversold（20）、reversion.rsi_soft_ceiling（35）；breakout.price_scale_ratio（0.02）、volume_cap_multiple（5.0）、atr_scale_enabled（true）、min_atr_multiple（0.5）。
- **optimization**：trend_confirm_bars（3）、trend_adx_enabled（true）、trend_adx_min（25）、trend_adx_penalty_ratio（0.6）、trend_partial_confirm_enabled（true）、trend_partial_confirm_ratio（0.7）；reversion_volume_bounce_enabled（true）、reversion_volume_bounce_ratio（1.2）、reversion_volume_bounce_bonus（1.1）；fusion_mode（weighted）、fusion_weights（[0.7,0.2,0.1]）；output_score_percentile（true）；以及 multi_pool_min_score、multi_pool_bonus 等（见 scanner_rules.yaml）。

### 3.4 sector_strength（板块强度）

- **默认**：`filters.sector_strength.enabled` 为 false 时，`sector_strength` 固定为 `1.0`（不参与相对强弱，确认/预警仍只由 `technical_score` 与收紧条件决定）；输出字段 **`industry_mapped`** 为 `null`（不适用）。
- **生产实现（同行业截面）**：`filters.sector_strength.enabled: true` 且 `mode: technical_ratio_to_sector_mean`（默认）时：自 L2 表 **`industry_revenue_summary`**（与 Module A 同源，采集任务写入）**批量**查询 `symbol -> industry_name`；**同一扫描批次内**按 `industry_name` 分组，对组内标的的 **technical_score**（已含多池、流动性/波动率调制后的最终分）取算术平均 `μ_sector`，令  
  **`sector_strength = technical_score / max(μ_sector, ε)`**（ε 为小正数防止除零）。**>1** 表示高于同行业当批平均分，**<1** 表示低于同业平均。有行业名且参与截面均值时 **`industry_mapped = true`**。
- **无行业映射**：无行业名或 L2 无该 `symbol` 行时，使用 **`filters.sector_strength.unmapped_sector_strength`**（默认 `1.0`，与历史「无映射恒为 1.0」一致）作为 `sector_strength` 系数，并 **`industry_mapped = false`**；便于与「有行业截面比」区分，避免误读为「恰好等于行业均值」。
- **可观测性**：`scanner_run_metrics.extra` 与 `last_scan_pipeline` 含 **`sector_strength_unmapped_count`**、**`sector_strength_mapped_count`**；板块强度开启时 **`last_scan_pipeline.unmapped_sector_strength`** 为当前配置值。实现见 `diting-core/diting/scanner/industry_map.py`、`diting/scanner/quant.py`（`_sector_strength_ratios`）。
- **可选扩展（未默认开启）**：对 `technical_score` 乘以 `min(1.5, sector_strength)` 做最终分调制时，须在规约与本表显式开关；当前默认 **仅作过滤与输出字段**，**不**回乘 technical_score。
- **阈值**：`scanner.sector_strength_threshold`（如 1.0～1.1）；确认档需 `sector_strength >=` 该阈值（与 `confirmed_passed` / `alert_passed` 中的板块条件一致）。

### 3.5 可选：流动性过滤与调制

- **开关**：`filters.liquidity.enabled`（默认 false）。为 true 时：计算 20 日日均成交额（或日均换手率）得到 liquidity_score ∈ [0,100]（可相对全市场分位线性映射）；**过滤**：liquidity_score < liquidity.min_score 的标的不进入候选；或**调制**：final_score = technical_score × (liquidity_score/100)。
- **配置**：liquidity.min_score（0）、liquidity.as_filter（true 时低于 min_score 直接剔除）。

### 3.6 可选：波动率 Regime 调制

- **开关**：`filters.volatility_regime.enabled`（默认 false）。为 true 时：计算标的或全市场 ATR(14)/close 分位；当分位 > volatility_regime.percentile_threshold（默认 0.90）时，对 technical_score 乘以 volatility_regime.penalty_ratio（默认 0.8），再 cap 100。
- **目的**：高波/混沌期降权，与「市场混沌期弃单」对齐。

### 3.7 最终 technical_score 与下游可用字段

- **计算顺序**：① 可选预筛（见 §3.8：粗筛分位、冷却、Classifier）→ ② 各池子分：趋势在 `evaluate_trend` 后乘 **index_regime_trend_mult**；**突破/反转**在单池得分后分别乘 **index_regime_breakout_mult** / **index_regime_reversion_mult**（见 §3.8 指数波动应力）→ ③ 融合最高分 + 多池共振（含可选二档）→ ④ 可选 × 趋势强度系数 → ⑤ 可选 × 波动率系数 → ⑥ 可选 × sector_strength 系数 → ⑦ 可选 × 流动性系数 → cap 100。
- **输出供下游**：除 technical_score、strategy_source、sector_strength 外，增加 **second_pool_id**、**second_pool_score**、**pool_scores**、**technical_score_percentile**；**long_term_score**、**long_term_candidate**；**market_regime**（本批基准指数环境快照，与 `product_signals.emit_market_regime_per_row` 一致）；便于决策中枢/Kelly 与 Module C 按策略维度、分位与大盘环境一致决策；中长期候选与产品范围见 [01_需求与产品范围](../../03_原子目标与规约/平台与产品/01_需求与产品范围.md)。

### 3.8 可选：两阶段粗筛、指数 regime、信号冷却、Classifier 门控

本节描述执行层在 **全量 TA-Lib 多池** 之前或并行读取 L2 的可选能力，键名与 `scanner_rules.yaml` / `dna_module_b.yaml` 一致。**生产默认**：粗筛、指数 regime、Classifier 门控为 **开启**；粗筛分位阈值为 0 时不额外剔除标的；`allowed_primary_tags` 为空时门控 **全放行**；`signal_cooldown_days` 默认为 **1**（见 §3.8 冷却小节）。

#### 两阶段粗筛（`optimization.coarse_screen`）

- **目的**：先在同一批 universe 上算 **20 日收益截面分位**、**20 日日均成交额截面分位**（与 liquidity 同源口径），仅对同时满足分位下标的标的进入 `evaluate_pools`（TA-Lib 全池），降低算力。
- **开关**：`coarse_screen.enabled`（默认 false）。
- **参数**：`min_momentum_percentile`、`min_liquidity_percentile`（[0,1]，0 表示不约束该项）；若动量下限 &gt;0，则与 `pool_4_momentum_enabled` 解耦，仍会计算 20d 收益用于分位；若流动性下限 &gt;0 且 `filters.liquidity.enabled` 为 false，仍会计算 20d 日均成交额用于粗筛分位。

#### 指数 regime（`filters.index_regime`）

- **目的**：用 **基准指数**（如沪深300）日线 **MA_short &gt; MA_long** 判定多头；非多头时对 **趋势池** 乘 `bear_trend_pool_mult`（默认 0.72）→ `index_regime_trend_mult`。另可选 **波动应力**（`stress_vol_enabled`）：对指数序列算 ATR(14)/close，若当前值在**近 stress_lookback_bars 根**内的分位 ≥ `stress_atr_percentile`，视为高应力：对 **突破池** 乘 `stress_breakout_mult`（默认 0.88）、对 **反转池** 乘 `stress_reversion_mult`（默认 1.06）。实现单次拉取指数 OHLCV（`index_regime.compute_index_regime_modifiers`），避免重复查询。
- **开关**：`index_regime.enabled`。无 L1 DSN 或数据不足时不施加乘子（保持 1.0）。

#### 信号冷却（`optimization.signal_cooldown_days`）

- **目的**：读 L2 **`quant_signal_scan_all`**，在 `cooldown_days` 个**日历日**窗口内，若某标的 **时间最新的一条** 仍为 `confirmed_passed`（默认）或 `passed`（由 `signal_cooldown_confirmed_only` 决定），则**跳过**本轮 TA-Lib，减少重复信号。
- **与「任意历史通过」的区别**：按 **每个标的最新一行** 判断；若最新一次扫描已**未通过**，则**不冷却**，允许行情或补数后当天重算。
- **有效时间**：`GREATEST(created_at, COALESCE(updated_at, created_at))`；表列 **`updated_at`** 由初始化脚本维护，补数或修正若更新该列，冷却窗口以较晚时间为准。
- **参数**：`signal_cooldown_days`（默认 1；0 表示关闭）；`signal_cooldown_confirmed_only`（true 时仅看 `confirmed_passed`）。

#### Classifier 门控（`filters.classifier_gate`）

- **目的**：读 L2 **`classifier_output_snapshot`** 最新一行 `primary_tag`，仅保留 `allowed_primary_tags` 列表中的标的；**无快照数据的标的剔除**（严格门控）。
- **开关**：`classifier_gate.enabled`（默认 false）；`allowed_primary_tags` 为空时等价于全放行（不启用过滤）。

#### 因子质检门禁（脚本，非运行时依赖）

- **`scripts/factor_quality_smoke.py`**：若已安装 **Alphalens + pandas + numpy**，则跑通 **get_clean_factor_and_forward_returns** 合成数据管线；未安装依赖则 **exit 0**。用于 CI/发布前门禁。

#### L2 数据治理（运维脚本）

- **`scripts/prune_quant_signal_scan_all.py`**：按 `--days`（默认 90）删除 `quant_signal_scan_all` 中早于保留期的行；`--dry-run` 仅统计。需 `PG_L2_DSN`。

### 3.9 性能与并行（`optimization` 同级 `scanner_performance`）

- **目的**：全市场扫描时降低 **evaluate_pools** 墙钟时间；TA-Lib 多释放 GIL，适合 **线程池**并行。
- **配置**：`parallel_workers`（0=串行）、`min_symbols_for_parallel`（达到该数量且 workers&gt;0 才并行）、`metrics_log_json`（是否打结构化 `scanner_run_metrics` 日志）。
- **实现**：`quant.py` 对通过粗筛/冷却/门控后的下标列表使用 `ThreadPoolExecutor` 调用 `_score_row`；`scanner_metrics.ScannerRunMetrics` 记录各阶段毫秒耗时。

### 3.10 下游契约（`product_signals`）

- **`emit_market_regime_per_row`**：为 true 时，每条 QuantSignal 附带 **`market_regime`** 字典（benchmark、ma_bullish、trend_pool_mult、stress_vol、index_atr_ratio、breakout_mult、reversion_mult），与 Module C / Router 对齐大盘环境语义。
- **`emit_scanner_metrics_log`**：为 true 时，`QuantScanner.last_scan_metrics` 与日志行 `scanner_run_metrics` 供运维与压测对比。

### 3.11 L2 可观测性：规则指纹与评估来源（`scanner_rules_fingerprint` / `evaluation_source`）

- **目的**：复盘「同一交易日多批扫描结果为何不同」时，除数据与截面分位外，能区分 **配置版本** 与 **是否本轮重算**。
- **`scanner_rules_fingerprint`**：对 `diting-core/config/scanner_rules.yaml` **原始文件字节**做 SHA-256，取十六进制**前 16 位**，写入 L2 表 `quant_signal_snapshot`、`quant_signal_scan_all` 同名列；与单次扫描的 `scanner_run_metrics.extra.scanner_rules_fingerprint`、`last_scan_pipeline.scanner_rules_fingerprint` 一致。修改 YAML 任意内容后指纹即变。
- **`evaluation_source`**：`FRESH` = 本轮经 `evaluate_pools`（TA-Lib 多池）重算；`CARRYOVER` = 因 **信号冷却** 跳过本轮打分、从 L2 **上一条**仍满足预警/确认的快照行沿用写入当前 `batch_id`（分数与 `entry`/`stop`/`take_profit` 为**上次写入值**，非当日最新行情重算）。详见 §3.8 冷却与 `diting-core/diting/scanner/l2_cooldown_carryover.py`。
- **下游提示**：Module C 若仅消费 `confirmed_passed`，仍应注意 `CARRYOVER` 行在剧烈行情中可能**滞后**；名单完整性优先于价格新鲜度时沿用，与产品约定一致。
- **迁移**：新增列随 `make init-l2-quant-signal-table`（`diting-core/scripts/init_l2_quant_signal_table.py`）及部署仓 `diting-infra/schemas/sql/07_l2_quant_signal_snapshot.sql`（含 `charts/.../schema-init/sql/` 同名文件）一并维护。

### 3.12 Golden batch 回归（防静默漂移）

- **目的**：在 **mock OHLCV**（`ohlcv_dsn=None`）、**无 L1/L2 DSN**（不读库行业映射）的前提下，对固定标的列表断言 **technical_score 区间**、`sector_strength`、`industry_mapped`、`strategy_source` 及 `metrics.extra` 计数，防止改池子逻辑、YAML 或打分路径时**静默漂移**。
- **环境**：须 **`PYTHONHASHSEED=0`**（mock K 线生成依赖 `hash(symbol)`，解释器默认随机种子会导致分数漂移）；执行前清除 **`PG_L2_DSN`**、**`TIMESCALE_DSN`**（或等价地不传 DSN），与 CI 一致。
- **入口**：`make golden-scanner-batch`（`diting-core`）；或 `PYTHONHASHSEED=0 env -u PG_L2_DSN -u TIMESCALE_DSN python3 scripts/golden_scanner_batch.py`。fixture：`tests/fixtures/golden_scanner_batch.json`；实现：`diting/scanner/golden_batch.py`。**退出码 0** 表示通过，**1** 表示断言失败（stderr 打印差异）。
- **与单测**：`tests/unit/test_golden_scanner_batch.py` 子进程调用上述脚本，与 Makefile 同源。

## 4. 过滤与输出（契约约定）

- **过滤（确认档）**：`technical_score >= confirmed_threshold`（默认等于 `scanner.technical_score_threshold`）且 `sector_strength >= sector_strength_threshold`，并满足可选主池/次池收紧；若启用流动性过滤则同时满足 liquidity 条件。阈值从 YAML 加载。**`passed` / `confirmed_passed` 表示确认档**，供 Module C 等下游默认识别为「强信号」。
- **短线双档（预警 + 确认）**：与 **`long_term` 中长期价值发现分支**区分命名；在 `module_b_quant_engine.a_track_short` 配置（键名保留为历史兼容）。**预警档**：`alert_technical_score_threshold`（或由 `profiles.<signal_profile>` 覆盖）≤ 得分 < 确认阈值且满足板块条件时 `alert_passed=true`、`signal_tier=ALERT`。**确认档**：同原通过逻辑，`confirmed_passed=true`、`signal_tier=CONFIRMED`。`quant_signal_snapshot` 写入 **预警或确认** 任一档为真的行；下游可只消费 `confirmed_passed=true`。
- **输出契约（扩展）**：在原有字段基础上增加 **signal_tier**（`NONE`/`ALERT`/`CONFIRMED`）、**alert_passed**、**confirmed_passed**、**signal_profile**（当前 `a_track_short.signal_profile`）；**建议风控**（仅信号层，执行在 Module E）：**entry_reference_price**（默认最近收盘）、**stop_loss_price**、**take_profit_prices**（或 L2 列 `take_profit_json`）、**risk_rules_json**、**stop_rule_id** / **tp_rule_id**。
- **输出契约（原有）**：symbol, technical_score, strategy_source, sector_strength, **industry_mapped**（§3.4）, correlation_id, second_pool_id, second_pool_score, **technical_score_percentile**，pool_scores；**long_term_score**、**long_term_candidate**；可选 liquidity_score、volatility_ratio；**market_regime**（本批指数环境快照，见 §3.10）。
- **可选收紧**：`scanner.pass_require_lead_pool_min`、`scanner.pass_require_second_pool_min`（未设置或 null 则无额外约束）；**strict** 等 profile 可在 `a_track_short.profiles.<name>.scanner` 覆盖阈值与次池下限，见下 §8。

## 8. 短周期扫描：`a_track_short` 三档模式与建议止损止盈

> [!NOTE] **[TRACEBACK] 战略追溯锚点**
> - **原子能力**: [Stage3-02 Module B 策略实现规约](./02_B模块策略_策略实现规约.md) §8
> - **实现**: `diting-core/config/scanner_rules.yaml` → `module_b_quant_engine.a_track_short`；代码 `diting/scanner/config_loader.get_a_track_short_params`、`diting/scanner/risk_levels.compute_a_track_risk_levels`、`diting/scanner/quant.py`

### 8.1 命名与边界

- **短周期扫描（`a_track_short`）**：本节约定的技术面筛选 + 风控建议，对应 MoE 中 **SHORT_TERM** 使用场景；**不是** `long_term_score` / `long_term_candidate` / VC-Agent **中长期**分支（勿混名）。
- **止损/止盈**：Module B **只输出建议价位与规则 JSON**；实盘触发在 **Module E**（或经纪商适配层）。

### 8.2 `signal_profile`（三档）

| 取值 | 含义 | 典型配置 |
|------|------|----------|
| **alert_sensitive** | 更高早盘/预警灵敏度 | `profiles.alert_sensitive.optimization` 如降低 `trend_confirm_bars`；`alert_technical_score_threshold` 可更低 |
| **balanced** | 默认 | 空 `profiles.balanced` 或省略 |
| **strict** | 更高胜率倾向 | `profiles.strict.scanner` 提高 `technical_score_threshold`、设置 `pass_require_second_pool_min` |

切换方式：仅改 `a_track_short.signal_profile`，合并逻辑见 `get_a_track_short_params`。

### 8.3 双档输出与 L2

- **dual_tier_output**：为 true 时计算 `alert_passed` / `confirmed_passed`；`quant_signal_snapshot` 包含任一档为真的标的，列 **signal_tier**、**alert_passed**、**confirmed_passed**。
- **可观测性列**：**scanner_rules_fingerprint**、**evaluation_source**（见 §3.11），与 `l2_snapshot_writer` 写入一致。
- **schema 迁移**：新增列需执行 `make init-l2-quant-signal-table`（或等价 `ALTER`），见 `diting-core/scripts/init_l2_quant_signal_table.py`；部署仓与 Chart 初始化 SQL 与上述一致：`diting-infra/schemas/sql/07_l2_quant_signal_snapshot.sql`（及 `charts/diting-stack/schema-init/sql/` 同名文件）。

### 8.4 建议风控字段（`a_track_short.risk`）

| 键 | 说明 |
|----|------|
| `enabled` | 是否计算 entry/stop/tp |
| `stop_mode` | `fixed_pct`（默认，对齐 A 轨 2% 量级）或 `atr_multiple` |
| `fixed_stop_pct` | 固定止损比例，如 0.02 |
| `atr_period` / `atr_stop_multiple` | ATR 止损时有效 |
| `take_profit_r_multiples` | 以风险额为 1R，止盈价 = entry + R×m |

输出写入 **risk_rules_json**、**take_profit_json**（数组），供展示与下游解析。

### 8.5 风控优化（实现要点）

- **`strategy_risk_overrides`**：按主策略池 `strategy_source`（1～4）覆盖 `stop_mode`、`fixed_stop_pct`、`atr_stop_multiple` 等，使趋势/反转/突破/动量可采用不同止损逻辑；合并顺序：全局 `risk` → 本表 → `volatility_tier` → `alert_tier`。
- **`volatility_tier`**：在**本批扫描截面**内对 ATR/close 做分位，划低/中/高三档，对 `atr_stop_multiple` 与 `fixed_stop_pct` 施加乘子（与 `filters.volatility_regime` 对 technical_score 的调制**独立**）；`enabled: false` 时不参与。
- **`alert_tier`**：当 `signal_tier=ALERT` 时可选单独 `take_profit_r_multiples`（及可选止损乘子）；`CONFIRMED` 仍用顶层 `take_profit_r_multiples`，便于 Module C 对预警档采用更保守的止盈目标。
- **追溯**：`risk_rules_json` 中含 `strategy_source`、`atr_percentile`、`volatility_tier`（若启用）等，便于回测与审计。

### 8.6 持仓与每日信号复核（执行侧衔接）

每日 B 输出用于 **新开仓** 的 R/止损/止盈锚定，以及 **已持仓** 的复核（技术面是否仍达标、建议止损是否仅收紧）。**不**默认用当日新表整体覆盖已落库的止盈阶梯；边界与合并规则见 [03_研究对象状态与每日复核规约](./03_研究对象状态与每日复核规约.md)，DNA 见 `_System_DNA/core_modules/dna_a_track_position_lifecycle.yaml`，代码见 `diting-core/diting/position_lifecycle.py`。
