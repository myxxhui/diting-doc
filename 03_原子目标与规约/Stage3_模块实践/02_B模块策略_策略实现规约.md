## 3. TA-Lib 指标与策略池映射

### 3.1 指标清单与 TA-Lib API

| 指标 | TA-Lib 函数 | 参数 | 用途 |
|------|-------------|------|------|
| 均线 MA | `talib.MA(close, timeperiod)` | timeperiod=5,10,20,60 | 趋势池：多头排列 |
| MACD | `talib.MACD(close, fastperiod=12, slowperiod=26, signalperiod=9)` | 返回 macd, signal, hist | 趋势池：DIF=macd, DEA=signal；水上金叉：macd>0 且 macd>signal |
| RSI | `talib.RSI(close, timeperiod=14)` | 0–100 | 反转池：RSI<20 或 RSI<30（超卖） |
| 布林带 | `talib.BBANDS(close, timeperiod=20, nbdevup=2, nbdevdn=2)` | 返回 upper, middle, lower | 反转池：价格触及下轨（close ≤ lower 或接近） |
| 最高价 N 日 | `talib.MAX(high, timeperiod=N)` | N=20 | 突破池：close > 前 N 日最高 |
| 成交量均线 | `talib.SMA(volume, timeperiod=20)` | 20 | 突破池：volume > 2×SMA(volume,20) |

### 3.2 策略池条件（可执行布尔表达式）

| 策略池 | 条件（与 dna_module_b 语义一致） | 实现要点 |
|--------|----------------------------------|----------|
| **趋势 Trend** | ① MA5 > MA10 > MA20（多头排列）；② MACD 水上金叉：MACD 线 > 0 且 DIF > DEA（即 macd > signal）。两条件均满足记为趋势命中。 | 取最近一根 K 线（或最近 3 根均满足）的指标值判断。 |
| **反转 Reversion** | ① RSI(14) < 30（超卖）；② 收盘价 ≤ 布林下轨（或 close ≤ lower×1.01）。满足其一即记为反转命中。 | 可放宽 RSI<20 为 RSI<30 以增加样本；下轨可设容差。 |
| **突破 Breakout** | ① 收盘价 > 前 20 日最高价（不含当日）：close > MAX(high, 20)[-2]；② 成交量 > 2×SMA(volume, 20)。两条件均满足记为突破命中。 | 前 20 日最高取 MAX 结果的前一个值。 |

### 3.3 technical_score 计算（0–100）

- **方式一（推荐最小实现）**：每个策略池独立打分；取**三池中最高分**作为该标的 technical_score，strategy_source 取该最高分对应池。
  - 趋势池得分：两条件均满足 80，满足其一 40，否则 0。
  - 反转池得分：两条件均满足 80，满足其一 40，否则 0。
  - 突破池得分：两条件均满足 80，满足其一 40，否则 0。
- **方式二（扩展）**：多池同时命中时取加权和（如每池最多 33，总和 cap 100）；或连续化子分 + 权重，见 [02_B模块策略_设计](./02_B模块策略_设计.md) 打分策略扩展。

### 3.4 sector_strength（板块强度）

- **本阶段最小实现**：无板块信息时固定为 `1.0`；若有 Module A 的 Domain Tag 或板块分类，可定义为「同板块标的的 technical_score 均值 / 全市场 technical_score 均值」，< 1 表示弱于大盘，> 1 表示强于大盘。
- **阈值**：scanner.sector_strength_threshold（如 1.1）仅在有板块强度计算时过滤；本阶段固定 1.0 时默认通过。

## 4. 过滤与输出（契约约定）

- **过滤**：仅输出 `technical_score >= technical_score_threshold` 且 `sector_strength >= sector_strength_threshold` 的标的；阈值从 YAML 加载（与 dna_module_b.scanner 一致）。执行由 Stage3-03 完成。
- **输出契约**：每条 QuantSignal：symbol, technical_score, strategy_source（TREND/REVERSION/BREAKOUT/UNSPECIFIED）, sector_strength, correlation_id。

## 5. 配置与 YAML

- **路径**：`diting-core/config/scanner_rules.yaml`（或与 dna_module_b 语义对齐的路径）。
- **内容**：strategy_pools（三池名称与参数）、scanner.technical_score_threshold、scanner.sector_strength_threshold；禁止在引擎代码中硬编码阈值。
- **与 DNA 对齐**：键名与 `_System_DNA/core_modules/dna_module_b.yaml` 的 strategy_pools、scanner 一致。
