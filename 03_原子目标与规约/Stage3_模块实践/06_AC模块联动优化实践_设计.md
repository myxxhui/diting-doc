| 模块 | 优化项 | 目标 |
|------|--------|------|
| **A** | 展示列 | 大类 / 垂直分列；多主营 Top2-3 标签；标签附营收占比%；主营/次营营收额 |

| **C** | 认知边界 | 「主营细分无利好或利空」拆为：无细分 / 主营利空 / 主营无信号 |

## 二、A 模块设计

### 2.1 输出列结构（L2 快照与 CLI）

| 列名 | 含义 | 数据来源 |
|------|------|----------|
| **大类** | 农业/科技/宏观 | tags 中首个 AGRI/TECH/GEO 或 DOMAIN_CUSTOM 经 tag_to_router_domain 映射 |
| **垂直** | 主营垂直标签 Top2-3，逗号分隔 | domain_label（如 火电,热力）；多 tag 时取前 2-3 个 |
| **垂直占比** | 主营、次营营收占比，如「火电 62%,热力 28%」 | segment_shares.revenue_share |
| **主营营收** | 主营分部绝对营收（万元） | financial_summary.revenue × 主营 revenue_share |
| **次营营收** | 次营分部绝对营收（万元） | 同上，取 Top2 |

### 2.2 营收数据来源

- `symbol_business_profile`：revenue_share（0~1）
- `financial_summary`：revenue（总营收，最新 report_date）
- 分部营收 = total_revenue × revenue_share
- 若 financial_summary 无最新期，显示「—」或「待采集」

### 2.3 电力链披露映射补全

`refine_power_label_from_disclosure()` 与 `classifier_rules.yaml` 补全：

- 风电设备 / 风电运营 区分
- 综合能源服务、电力销售、供电业务等

### 2.4 43 标的规则修正

按本设计文档「43 标的全量检测清单」逐只核对，修正 classifier_rules 与 tag_to_router_domain。执行 scripts/verify_all_43_symbols.py 做全量检测，不做抽检。

## 三、C 模块设计

### 3.1 路由列：改为垂直细分

**现状**：路由 = 农业/科技/宏观（过粗）

**调整**：路由列显示 **垂直细分**（如 火电、锂电、风电设备），与 A 的垂直列对齐。

**实现**：C 读 A 快照时，`路由` = A 的「垂直」列首值（或大类+垂直组合）；若无垂直则回退大类。内部仍用 农业/科技/宏观 选 risk_factor_templates。

### 3.2 短轨「垃圾箱」→「不予支持」

**文案**：`is_supported=False` 时，短轨列显示 **「不予支持」**，不用「垃圾箱」。

### 3.3 风险等级：无则空

**现状**：默认 `risk_level=RISK_LEVEL_MID`

**调整**：仅在以下情况赋值，否则为空字符串或 None：

- **高**：任一细分带「高风险」标签
- **低**：景气强度≥0.5 且无高风险
- **空**：无法判断（如 segment_signals 为空、主营无信号）→ 不虚构「中」

### 3.4 认知边界：独立判断

**现状**：「主营细分无利好或利空」混在一起

**调整**：拆为独立情况：

| 情况 | 文案 |
|------|------|
| segment_list 为空 | 上游无数据：标的主营构成为空 |
| 全部细分无 signal | 上游无数据：全部细分无垂直信号 |
| 主营有利空 | 主营细分利空（认知边界） |
| 主营无利好且无利空 | 主营细分无信号（认知边界） |
