# Module C 明细表字段说明与 A 标签准确性分析

[Ref: 04_研究议会与Agent编排_设计] [Ref: 06_AC模块联动优化实践_设计] 本文档解释 `make run-module-c` 明细表中各列含义、核心判断逻辑，以及 A 标签的准确性分析与修复建议。

**2025-03 更新**：大类/垂直分列；路由=垂直细分；短轨「不予支持」；风险等级无则空；认知边界独立判断。

---

## 一、明细表各列通俗解释

### 1. 大类 / 垂直 / 路由

**大类**：农业/科技/宏观，用于内部选专家模板。

**垂直**：主营垂直细分 Top2-3（如 火电,热力、锂电）， comma 分隔。

**路由**（更具体）：显示 **垂直细分首值**（如 火电、锂电），不再仅 农业/科技/宏观。若无垂直则回退大类。

---

### 2. 长候（long-term candidate）

**含义**：B 模块量化侧判定的「是否具备长期投资候选资格」。

**判断逻辑**：`quant_signal.long_term_candidate`，由 B 模块的 QuantScanner 根据长期收益率、趋势等计算。为「是」时才可能走 **长轨**。

---

### 3. 长轨（long-term track）

**含义**：中长期研究路径 — 专家意见的 `horizon=LONG_TERM`，且 `is_supported=True`。

**核心判断逻辑**：

1. `quant_signal.long_term_candidate == True`
2. `MOE_ENABLE_VC_AGENT` 开启
3. VC-Agent 产出 `horizon=TIME_HORIZON_LONG_TERM` 且 `is_supported=True`

**与短轨区别**：LONG_TERM 分支下决策中枢可套用较长周期的风控参数（如豁免部分短周期硬止损/现金拖累规则）；SHORT_TERM 分支保持更严格的风控默认。

---

### 4. 短轨（short-term track）

**含义**：短周期研究路径 — 专家意见的 `horizon=SHORT_TERM`。

**核心判断逻辑**：

1. 路由命中（非垃圾箱）
2. `unified_opinion()` 跑完
3. **若 `is_supported=True`**：显示「偏多/偏空 + 置信度」
4. **若 `is_supported=False`**：显示「垃圾箱」

**为何是垃圾箱？** 见下文「认知边界」。

---

### 5. 不予支持（原「垃圾箱」）

**含义**：右脑不予支持，不参与决策中枢投票。展示文案为「不予支持」。

**触发条件（任一满足即不予支持）**：

1. **路由失败**：A 标签无法映射到 农业/科技/宏观
2. **认知边界**（独立判断）：
   - **无细分**：主营构成（segment_list）为空 →「上游无数据：标的主营构成为空」
   - **主营无信号**：主营细分在 segment_signal_cache 无信号 →「主营细分无信号（认知边界）」
   - **主营利空**：主营细分方向为 bearish →「主营细分利空（认知边界）」
   - **全部无垂直信号**：「上游无数据：全部细分无垂直信号」
   - **利好与主营未对齐**：alignment_score < 0.3

---

### 6. 摘要与各项数值

**「对齐得分=0.00 景气强度=0.00 利好强度=0.00 主营细分无利好或利空（认知边界）」**

| 字段       | 含义                           | 本批为何为 0 |
|------------|--------------------------------|--------------|
| 对齐得分   | 利好信号与主营细分的匹配程度   | segment_signals 为空，无法计算 |
| 景气强度   | 有利好信号的细分营收占比之和   | 同上 |
| 利好强度   | 利好方向的加权置信度            | 同上 |
| 主营细分无利好或利空 | 认知边界文案           | 主营一票否决：主营细分无 bullish 信号 |

**风险等级**（`alignment.py`，无则空、不虚构）：

- **高**：任一细分带「高风险」标签
- **低**：景气强度 ≥ 0.5 且无高风险
- **中**：有信号可判断但未达「低」条件
- **空**：无法判断（如 segment_signals 为空、主营无信号）→ 不虚构「中」，摘要中不输出风险等级

## 二、A 标签准确性分析

### 2.1 数据来源

- **申万行业**：L2 `industry_revenue_summary`（industry_name）
- **主营披露**：L2 `symbol_business_profile` + `segment_registry`（name_cn，按营收降序）

**分类逻辑**（`classifier_rules.yaml` + `semantic.py`）：

1. 按 categories 顺序匹配 `industry_keywords`，**只取第一个命中**
2. 申万仅为「电力」时：不依赖 YAML，走 `refine_power_label_from_disclosure()`，用主营披露名映射（火电/水电/热力/售电等）
3. 无主营披露时：标「无披露」

**A 标签列**：`domain_tags` 的中文拼接；DOMAIN_CUSTOM 用 `domain_label`（如热力、售电），否则用农业/科技/宏观。

---

### 2.2 潜在问题

| 问题类型     | 说明                                                                 |
|--------------|----------------------------------------------------------------------|
| 大类与垂直混列 | A 标签同时含「宏观」和「热力」，不区分大类与垂直细分                 |
| 只取首个匹配 | 多主营公司只展示第一个命中的标签，可能漏掉更重要的主营               |
| 电力链依赖披露 | 申万「电力」时完全依赖 segment_registry；表空或采集不全则不准       |
| 热力 vs 火电 | 热电联产公司：披露为「供热/热力」→ 热力；披露为「燃煤/火电」→ 火电。若披露表述偏「热力」则易只标热力 |

---

### 2.3 以 000539.SZ（粤电力 A）为例

- **公司**：广东电力发展，主营火电、热电联产、新能源等
- **若 A 标签为「热力」**：说明主营披露 Top1 被映射为热力（如「供热」「热力」等词先匹配）
- **可能偏差**：若火电营收更大，但披露顺序或表述导致先命中热力，则不够贴切
- **更贴切**：可标「火电,热力」或按营收主次标「火电」

---

### 2.4 43 只逐类抽检建议

| 标的       | A 标签示例      | 合理性简述                          |
|------------|-----------------|-------------------------------------|
| 000539.SZ  | 热力            | 粤电力 A，热电联产多，热力可接受但可能缺火电 |
| 601016.SH   | 电力            | 节能风电，应为「风电」更准          |
| 300750.SZ   | 锂电            | 宁德时代，合理                       |
| 002015.SZ   | 售电,热力       | 协鑫能科，售电+热电，合理            |
| 600396.SH   | 热力            | 金山股份，火电+供热，热力可接受      |
| 603507.SH   | 风电设备        | 合理                                |
| 688331.SH   | 生物制品        | 合理                                |
| 600483.SH   | 总部及其他      | 可能是披露原文，偏粗                 |

**需人工核查**：对重点标的查年报/主营构成，对比 `symbol_business_profile` 与 `segment_registry` 的采集质量。

---

### 2.5 最佳修复方案

1. **拆分列**
   - 路由 = 大类（农业/科技/宏观）
   - A 标签 = 仅垂直细分（热力、售电、火电等），或分「大类」「垂直」两列

2. **多标签展示**
   - 有多个主营时，A 标签展示 Top2–3 的垂直标签（如「火电,热力」），不限于首 match

3. **电力链披露映射补全**
   - 完善 `refine_power_label_from_disclosure()` 的关键词
   - 确保 `symbol_business_profile` 与 `segment_registry` 采集完整

4. **规则顺序与优先级**
   - 电力链：披露映射 > 申万行业匹配；细分（火电/水电/热力）优先于宽口径「电力」

5. **人工抽检与修正**
   - 对 universe 高权重标的做抽检，修正 `classifier_rules.yaml` 与披露映射逻辑

---

## 三、小结

| 列/概念           | 通俗理解                           |
|-------------------|------------------------------------|
| 路由              | 归到农业/科技/宏观三大桶，选专家模板 |
| 长候              | B 判定的长期候选资格               |
| 长轨              | 长期路径，VC-Agent 支持，走 B 轨   |
| 短轨              | 短期路径，unified_opinion 支持则显示方向，否则垃圾箱 |
| 垃圾箱            | 右脑不予支持（路由失败或认知边界） |
| 摘要数值全 0      | segment_signal_cache 未建，无细分信号 |
| 风险等级=中       | 默认分支（无利好/利空/高风险时的兜底） |
| A 标签            | classifier 输出的行业/主营标签拼接，可能有精度与覆盖问题 |
