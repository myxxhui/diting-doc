## 一、契约范围

| 契约 | 用途 |
|------|------|
| `ResearchCard` | 研究结论卡片 |
| `Candidate` | 研究候选 |
| `CouncilSession` | 议会会议记录 |
| `AgendaTopic` / `AgendaSubmit` | 议程议题 |
| `ExpectationGap` | 预期差度量 |
| `ToolCall` | 工具调用记录 |

## 二、Proto 定义（v1）

### 2.1 ResearchCard

```proto
syntax = "proto3";
package deep_strike.v1;

import "google/protobuf/timestamp.proto";
import "google/protobuf/struct.proto";

enum ConclusionStance {
  CONCLUSION_STANCE_UNSPECIFIED = 0;
  CONCLUSION_STANCE_BULLISH = 1;
  CONCLUSION_STANCE_BEARISH = 2;
  CONCLUSION_STANCE_NEUTRAL = 3;
  CONCLUSION_STANCE_UNCERTAIN = 4;
}

message EvidencePiece {
  string ref = 1;                       // db://... 或 s3://...
  string source_type = 2;               // announcement / news / research_report / data_metric
  string excerpt = 3;                   // 关键摘录
  google.protobuf.Timestamp source_ts = 4;
  double weight = 5;
}

message ResearchCard {
  string id = 1;                        // UUID v7
  string correlation_id = 2;            // 串到 CouncilSession
  string schema_version = 3;
  string subject_id = 4;                // 标的 / 行业 / segment
  string subject_type = 5;              // symbol / industry / segment
  string title = 6;
  string conclusion = 7;                // 一句话结论
  ConclusionStance stance = 8;
  string applicable_period = 9;         // 适用周期，如 "Q4 2026" / "1-3 months"
  double confidence = 10;               // 0~1
  repeated EvidencePiece evidence_chain = 11;
  string suggested_state_machine_template = 12; // 状态机模板 ID（可空）
  google.protobuf.Struct extra = 13;
  string producer = 14;
  bool degraded = 15;                   // 是否降级路径产出
  google.protobuf.Timestamp created_at = 16;
}
```

## 三、HTTP / gRPC 端点

### 3.1 议会服务

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /v1/sessions` | gRPC + REST | 启动议会会议（同步：返回 session_id；结果异步推送） |
| `GET  /v1/sessions/{id}` | REST | 查询会议详情 + 工具调用列表 |
| `GET  /v1/sessions/{id}/transcript` | REST | 拉取完整对话记录（仅授权用户） |
| `POST /v1/sessions/{id}/cancel` | REST | 中止会议 |

### 3.2 候选注册表

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET  /v1/candidates` | REST | 列出（按 subject / kind / status / priority） |
| `GET  /v1/candidates/{id}` | REST | 单个详情 |
| `POST /v1/candidates/{id}/archive` | REST | 归档 |

### 3.3 议程编排

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /v1/agenda/topics` | REST | 提交议题（用户 / 事件） |
| `GET  /v1/agenda/topics` | REST | 列出排程 |
| `DELETE /v1/agenda/topics/{id}` | REST | 取消议题 |

### 3.4 内容理解

| 端点 | 方法 | 说明 |
|------|------|------|
| `POST /v1/comprehend/documents` | gRPC | 提交文档（异步） |
| `GET  /v1/comprehend/documents/{id}` | REST | 查询解析结果 |

### 3.5 SSE 推送（实时）

| 端点 | 说明 |
|------|------|
| `GET /v1/sessions/{id}/events` | SSE 推送会议进度（agent 思考 / 工具调用 / 阶段切换） |
| `GET /v1/agenda/stream` | SSE 推送议程变化（新议题 / 议题状态变更） |

## 四、关键字段说明

- `correlation_id`：贯穿"议程 → 议会 → 卡片 → 候选 → 极寒防御门禁 → 状态机 → 评测"全链路
- `schema_version`：所有契约必带，按 SemVer
- `evidence_chain`：研究卡片必须非空；空数组的卡片会被极寒防御 reject
- `degraded`：议会失败转规则路径的标记，前端必须显式提示

## 五、错误码

| 错误码 | HTTP | 含义 |
|--------|------|------|
| `DS.AGENDA_FULL` | 503 | 议程队列满 |
| `DS.SESSION_TIMEOUT` | 504 | 会议超时 |
| `DS.NO_TOOL_AVAILABLE` | 503 | 工具不可用（如推理网关熔断） |
| `DS.EVIDENCE_INSUFFICIENT` | 422 | 证据不足，无法形成结论 |
| `DS.SUBJECT_NOT_FOUND` | 404 | 标的 / segment 未识别 |
