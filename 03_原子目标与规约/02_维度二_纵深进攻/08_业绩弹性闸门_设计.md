## 一、本模块定位

The Mapper 业绩弹性闸门是 Lighthouse-Alpha **"产业链图谱与标的映射器"**（PRD §3 阶段二）的工程化实现——把 The Critic 通过的题材簇映射到具体 A 股标的，**核心目的是把"被稀释的大盘组装厂"过滤掉，留下细分龙头**。

**核心理念**（承接 L1 §6.3）：
> 一个标的"营收能不能翻倍"取决于其细分业务的**产能能否短期不可复制地放大**；若产能扩张壁垒低（纯组装/代工/已饱和的成熟产能），即便订单饱满，营收弹性也会被**整体营收基数稀释**。

**模块边界**：
| ✅ 本模块 | ❌ 其他模块 |
|---|---|
| 4 档营收基数稀释判定 | The Critic 物理证伪（前置归 07_主动嗅探层 §3.4）|
| 业绩弹性最低增益（毛利率 ≥ +5pp） | thesis 卡完整 schema（归 02_后端服务子模块）|
| 单题材簇推荐数量上限 | The Architect 监控字典生成（归 07_主动嗅探层 §3.6）|

## 二、子模块设计

### 2.1 The Mapper 核心（`the_mapper_service`）

**主责**：消费 `cluster_critic_results where physical_gate=true`，按 4 档 + 业绩弹性 + 数量上限三关过滤，输出 `mapper_outputs`。

**内部组件**：

```
┌──────────────────────────────────────────────┐
│ ClusterLoader                                │
│   ├─ from DB: cluster_critic_results         │
│   └─ filter: physical_gate=true              │
│         ↓                                     │
│ CandidateExpander                            │
│   ├─ 按 cluster.theme → 调财务数据 API       │
│   ├─ 拿到候选标的列表（10~50 个）            │
│   └─ 富化字段: total_revenue / margins       │
│         ↓                                     │
│ FourTierClassifier ← DNA Y02 4 档配置        │
│   ├─ elite_segment（300亿- + 30%+）          │
│   ├─ qualified（15%+）                        │
│   ├─ diluted（< 15%）→ confidence_cap 0.45   │
│   └─ diluted_largecap（1000亿+ + <5%）→ 硬拒 │
│         ↓                                     │
│ ProfitElasticityChecker                      │
│   └─ 题材毛利率 - 整体毛利率 ≥ 5pp           │
│         ↓                                     │
│ TopKLimiter（每 cluster ≤ 3）                │
│         ↓                                     │
│ Emitter                                       │
│   ├─ 写 DB: mapper_outputs                   │
│   └─ 产事件: ThesisProposedEvent → step_05   │
└──────────────────────────────────────────────┘
```

---

### 2.2 营收基数稀释判定（4 档 · PRD §3 阶段二）

**公式**：`dilution_ratio = predicted_topic_revenue_delta / total_revenue_last_year`

| tier | 触发条件 | action | confidence_cap | tag |
|---|---|---|---|---|
| **elite_segment** | total_rev < 300 亿 且 题材贡献 ≥ 30% | promote | **0.85** | `elite_segment_leader` |
| **qualified** | 题材贡献 ≥ 15% 且不属于上面 | pass | 0.70 | — |
| **diluted** | 题材贡献 < 15% 且不属于下面 | soft_reject | 0.45 | `revenue_diluted` |
| **diluted_largecap** | total_rev ≥ 1000 亿 且 题材贡献 < 5% | **hard_reject** | — | `largecap_diluted` |

> diluted_largecap 是 L1 §6.2 哲学边界 ❌ 第 5 行的工程兑现——**强制拒绝**该标的本题材推荐，不进入推荐池。

---

### 2.3 业绩弹性最低增益

**判据**：题材业务毛利率 ≥ 标的整体毛利率 + **5 个百分点**

| 通过 | 不通过 |
|---|---|
| 进入下一关（数量上限）| 标 `gross_margin_drag` + 置信度上限 0.55 |

> 用毛利率而非净利率：毛利率反映"业务本身的盈利能力"，不受其他业务"管理费用 / 销售费用"稀释。

---

### 2.4 数量上限

- 单题材簇最多映射 **3 个**标的
- 按 `elasticity_ratio`（业绩弹性比）降序
- 超出的标的进入"次日复评队列"

## 三、输入与输出 schema

### 3.1 输入

```yaml
ClusterCriticResult:
  cluster_id: str
  theme: str                       # 如 "智算中心液冷"
  physical_gate: bool              # 必须 true 才入 mapper
  physical_floor_evidence: object
  supply_elasticity_evidence: object
  the_scorer_composite: float      # 0~10
  the_critic_passed_at: datetime
```

### 3.2 输出

```yaml
MapperOutput:
  cluster_id: str
  symbol: str                      # 6 位代码
  company_name: str
  tier: elite_segment | qualified | diluted | diluted_largecap
  action: promote | pass | soft_reject | hard_reject
  tags: list[str]
  confidence_cap: float
  metrics:
    total_revenue_last_year_yuan: float
    predicted_topic_revenue_delta_yuan: float
    topic_contribution_pct: float       # = delta / total
    topic_business_gross_margin: float
    total_gross_margin: float
    elasticity_ratio: float             # 综合排序键
  reasoning: str                        # 中文，引用 PRD §3 阶段二原文
  mapped_at: datetime
```

## 六、失败模式与保护

| 失败 | 保护 |
|---|---|
| 财务 API 不可用 | 走本地小模型解析 thesis 卡内财报附录字段；标 `data_partial`，置信度上限降一档 |
| 标的财务数据 stale（> 90 天）| 标 `data_stale_90d`，置信度上限 0.55；不阻塞但提示用户 |
| 大模型评分结果与本地规则严重冲突 | 以**规则为准**（本模块仅小模型 + 规则；大模型不直接决定 tier）|
| 单 cluster 候选标的 = 0 | 标 `no_candidate_for_theme`，不报错；归档到"未映射 cluster"复审池 |

## 八、关键设计取舍

1. **仅小模型 + 规则**：The Mapper 不调大模型（成本上限 ¥ 0.10/cluster），规则可解释 + 可单测
2. **diluted_largecap 是 hard_reject 不是 soft_reject**：基石⑥ ❌ 哲学边界的工程兑现——大盘组装厂**强制**不推荐
3. **用毛利率不用净利率**：避免被"管理费用/销售费用"稀释，更准确反映业务本身盈利能力
4. **数量上限 3**：与 L2 §三推荐与持仓限额对齐（单 cluster 不能淹没用户带宽）
5. **失败时以规则为准**：The Mapper 不依赖大模型决策，确保结果稳定 + 可审计
