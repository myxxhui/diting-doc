## 一、整体分层

```

│ 1. View (路由 + 页面)                                    │
```

## 三、状态管理策略

### 3.1 状态分类

| 类别 | 工具 | 例子 |
|------|------|------|
| 服务端状态（远程数据 + 缓存） | **TanStack Query** (React Query) | useResearchCards / useAdvisories |
| 实时流（SSE / WebSocket） | 自建客户端 + Query 订阅 | useCouncilSessionStream |
| 表单本地状态 | **React Hook Form** + Zod | 提问表单 / 偏好编辑 |
| 全局 UI 状态（侧边栏 / 主题 / 弹窗） | **Zustand** | useUIStore |
| 鉴权 / 用户信息 | Zustand + 持久化 | useAuthStore |
| URL 状态（筛选 / 分页 / Tab） | URL Search Params | useFilters |

### 3.2 数据获取约定

```ts
// 统一的 query key 命名：[资源, 操作, 参数]
useQuery({
  queryKey: ['research-cards', 'list', { subjectId, status }],
  queryFn: () => api.deepStrike.cards.list({ subjectId, status }),
  staleTime: 30_000,
});

// 实时流：query + SSE 订阅
useEffect(() => {
  const sub = sseClient.subscribe(`/v1/sessions/${sessionId}/events`, (ev) => {
    queryClient.setQueryData(['council-session', sessionId], (old) => merge(old, ev));
  });
  return sub.unsubscribe;
}, [sessionId]);
```

### 3.3 错误处理与降级

- 全局 `<ErrorBoundary>` 捕获 + 上报到超级个体进化反馈采集
- 局部错误：组件内显示"重试"按钮 + 友好降级文案
- 极寒防御 reject 类错误：以"系统拒绝原因"卡片展示，给用户明确"为什么"

## 四、设计系统（UI Kit）

### 4.1 命名风格
- **不直接用 Tailwind 原子类组合到 Feature 层**；Feature 层调 UI Kit 组件
- UI Kit 内部可用 Tailwind / CSS-in-JS（项目内统一）

### 4.2 关键组件
- `<Card>` / `<Section>` / `<Tabs>` / `<EmptyState>` / `<LoadingSpinner>`
- `<EvidenceTrail>`（证据链可展开）
- `<ConfidenceBar>`（置信度条 + 颜色）
- `<DegradedBadge>`（降级标记）
- `<StateMachineDiagram>`（基于 Mermaid 或自建 SVG 渲染）
- `<TimelineMini>`（迷你时间线）
- `<NotificationBudgetMeter>`（通知预算计量条）
- `<AuditTrail>`（审计回放）

### 4.3 主题与可访问性
- 浅色 / 深色双主题
- WCAG 2.1 AA 合规
- 键盘导航全覆盖
- 国际化（中英起步）

## 七、约定与禁止

### 约定
- **Feature 层不直接 fetch；统一走 domain hooks**
- **类型强制**：API 客户端 / Proto 生成的类型必须覆盖；禁用 `any`（除非加注释豁免）
- **可访问性优先**：UI Kit 组件须通过 axe 检查
- **埋点统一**：通过 `useTrack` hook，写入超级个体进化的 `feedback_implicit`

### 禁止
- 在 Feature 层直接组合 Tailwind 原子类（应封装到 UI Kit）
- 在组件内直接调浏览器 fetch（应走 service layer）
- 在前端实现业务规则（应在后端；前端仅展示与汇总）
- 直接读后端数据库或绕过 BFF / API Gateway
