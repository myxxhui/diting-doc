<a id="b-track-data-requirement"></a>
## 一、B 轨数据需求清单

以下为**超额完成** B 轨概念所需的数据范围；「必选」为 B 轨闭环最低要求，「可选」为超额或后续迭代。

| 类别 | 内容 | 必选/可选 | 说明 |
|------|------|-----------|------|
| **财报与经营** | 利润表/资产负债表/现金流量表科目级，季度/年度，同比环比；毛利率、净利率、ROE、负债结构、资本开支、自由现金流、营收/利润增速序列 | 必选 | 细财报；供 VC-Agent 与 B 轨成长质量分 |
| **管理层与治理** | 高管名单、背景、任职与变动、薪酬与股权激励、实控人与一致行动人、董事会构成、关联交易 | 必选 | 供 VC-Agent 与治理分 |
| **业务与竞争** | 主营业务构成、产品与客户结构、产能与产能利用率、市占率、竞争格局、上下游议价能力 | 必选 | 供潜在空间/赔率与共识 |
| **研发与创新** | 研发投入、费用化/资本化、专利与项目、技术路线或标签 | 必选 | 供研发强度分与共识 |
| **行业与政策** | 行业空间与增速、政策与监管要点、技术替代与颠覆风险标签 | 可选 | 供共识与逻辑证伪 |

<a id="b-track-storage"></a>
## 二、存储表与契约

- **B 轨原始/加工数据**：在 L2（或约定层）设专用表或视图，建议命名与用途如下；具体 DDL 与字段以代码仓或部署仓 schema 为准，本文档约定**用途与消费方**。

| 表/视图（建议） | 用途 | 消费方 |
|----------------|------|--------|
| `company_fundamental`（或等价） | 财报与经营指标、时序 | B 轨预筛（成长质量分）、VC-Agent |
| `management`（或等价） | 管理层与治理信息 | VC-Agent、治理分 |
| `b_track_raw_feed`（或等价） | 上述数据的 B 轨专用汇总/视图，按 symbol 可查 | B 轨预筛、VC-Agent |

- **B 轨候选表**：`b_track_candidate_snapshot`（或等价）
  - **用途**：仅存放 B 轨策略与打分**通过**的标的；C 的 Router/VC-Agent 仅或优先从此表读取 B 轨候选。
  - **建议字段**：symbol、batch_id、long_term_score、growth_quality_score、rd_intensity_score、governance_score（可选）、potential_upside（可选）、time_horizon（3Y/5Y/10Y）、industry、lifecycle_stage（来自 A）、created_at；与 [01_B轨系统设计](01_B轨系统设计.md)#2.3 一致。
  - **写入方**：B 模块（或 B 轨预筛）；写入时机为每次 B 轨扫描/预筛完成；去重与覆盖策略（如按 batch_id 覆盖或追加）由实现约定。
  - **消费方**：Module C Router/VC-Agent。

- **采集与更新频率**：财报类随披露周期；管理层与治理、行业政策可为事件驱动或定期拉取；在 11_ 或 Stage2 采集规约中与本文档对齐，并标注「B 轨必采/可选」。

<a id="b-track-consumption"></a>
## 三、消费方与读接口

| 消费方 | 读取内容 | 约定 |
|--------|----------|------|
| B 轨预筛（B 模块内或独立服务） | company_fundamental、management 等，用于 B 轨维度分与通过判断 | 按 symbol 查询；未命中时该 symbol 不写入 b_track_candidate_snapshot 或降级为仅动量分 |
| Module C VC-Agent | b_track_candidate_snapshot 行 + company_fundamental、management 等摘要或主键引用 | 输入契约见 [03_B轨共识与逻辑证伪规约](03_B轨共识与逻辑证伪规约.md)；必选/可选字段与新鲜度由 DNA 与实现约定 |
| 逻辑证伪模块（Module D 或独立） | 共识表（证伪条件）+ 事件流（财报暴雷、管理层变动等） | 见 03_B轨共识与逻辑证伪规约 |
