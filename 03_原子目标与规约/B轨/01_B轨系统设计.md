## 一、设计目标

各模块按**超额完成** [04_B轨系统概念与目标](../../01_顶层概念/04_B轨系统概念与目标.md) 来设计：时间视野 3/5/10 年、回报目标 10×～100×、全关键商业信息、可追溯可证伪、仅逻辑/大周期退出、与 A 轨隔离。

<a id="design-b-track-modules"></a>
## 二、各模块 B 轨设计要点

### 2.1 采集模块

- **数据范围**：B 轨专用清单见 [02_B轨数据与存储规约](02_B轨数据与存储规约.md)#数据需求；包含细财报、管理层与治理、业务与竞争、研发与创新、行业与政策。
- **存储与契约**：B 轨数据在 L2 有专用表/视图，schema、主键、更新频率在 02_ 约定；消费方（B 轨预筛、VC-Agent、逻辑证伪）的读接口在 02_ 约定。
- **超额**：采集项有 B 轨必选/可选标记；至少一条「数据源→存储→B 轨模块读取」端到端可验收。

### 2.2 A 模块（语义分类器）B 轨扩展

- **输出**：ClassifierOutput 增加 B 轨用字段：如 `industry`、`sub_industry`、`lifecycle_stage`（early_growth/growth/mature/decline）、`b_track_eligible`（是否允许进 B 轨管线）、可选 `theme`/`tech_route`。
- **准入规则**：L3 约定「B 轨准入」规则（如 TECH + lifecycle_stage ∈ {early_growth, growth} → b_track_eligible=True）；可配置。
- **传递**：B 模块（或 B 轨预筛）从 L2 或约定接口按 symbol/batch 读取 A 的上述字段；表名/字段/刷新时机在 02_ 与 A 实践文档约定；未命中时降级策略约定。

### 2.3 B 模块（量化扫描）B 轨策略与共享表

- **B 轨专用策略与打分**：在现有 long_term_score（如 120d 动量分位）基础上，增加可配置维度（与采集对齐）：成长质量分、研发强度分、治理/管理层分、长期动量分、可选潜在空间/赔率估计；阈值与权重可配置。
- **准入与通过**：B 轨通过条件 = A 的 b_track_eligible=True 且 B 轨策略与打分满足配置阈值（如 long_term_score≥X、成长质量分≥Y）。
- **B 轨专用共享表**：
  - 表名：`b_track_candidate_snapshot`（或等价）；仅存放 **B 轨策略与打分通过** 的标的及必要字段（symbol、batch_id、long_term_score、各维度分、潜在空间/赔率、时间、可选 A 标签摘要）。
  - 写入方：B 模块（或 B 轨预筛）在每次扫描/预筛后写入；写入时机、去重与覆盖策略在 02_ 约定。
  - 消费方：C 的 B 轨逻辑（Router/VC-Agent）**仅或优先**从该表读取 B 轨候选；不在全量 QuantSignal 中混选。

### 2.4 C 模块（MoE）B 轨逻辑

- **路由**：请求若 symbol 在 `b_track_candidate_snapshot` 或带 b_track_pass 约定，则**仅或优先**走 VC-Agent；与 Agri/Tech/Geo 互斥/并行由配置约定（09_ 引用本文档）。
- **VC-Agent 输入**：至少 symbol、B 轨候选表行、采集层 B 轨数据引用（表/主键或摘要）；必选/可选字段与新鲜度在 02_ 与 [03_B轨共识与逻辑证伪规约](03_B轨共识与逻辑证伪规约.md) 约定。
- **VC-Agent 输出与共识**：ExpertOpinion 含 horizon=LONG_TERM、reasoning_summary 等；**共识**结构化并落库，schema 与消费见 [03_B轨共识与逻辑证伪规约](03_B轨共识与逻辑证伪规约.md)。

### 2.5 D 模块（判官）B 轨逻辑

- **双轨分流**：LONG_TERM → 豁免 2% 与现金拖累，仅施加逻辑证伪与大周期反转（09_ 保留概要，细节在 [03_B轨共识与逻辑证伪规约](03_B轨共识与逻辑证伪规约.md)）。
- **逻辑证伪**：输入 = C 的共识（证伪条件）+ 事件流（财报暴雷、管理层变动、政策/技术颠覆等）；输出 = 平仓/减仓指令，原因码 logic_disproof。
- **大周期反转**：输入 = 标的/行业价格与均线、行业指数关键位、波动率 regime 或 VC-Agent 建议；输出 = 止盈/减仓信号，原因码 major_trend_reversal。

### 2.6 E 模块（风控）B 轨逻辑

- B 轨仓位**不**触发 2% 硬止损与现金拖累；规则引擎按 track=B 走 B 轨分支。
- B 轨专用约束（可配置）：单标的 B 轨仓位上限、B 轨合计敞口占比上限、单行业/主题 B 轨敞口上限、再平衡频率与触发条件。
- 可接收 D 的「逻辑证伪/大周期反转」指令，对对应 B 轨仓位执行或标记，与 F 对接。

### 2.7 F 模块（执行）B 轨逻辑

- 订单与仓位带 `track=A|B`；执行层与经纪侧按轨区分（08_ 与 03_ 分账一致）。
- B 轨平仓仅根据判官/风控的「逻辑证伪」或「大周期反转」指令执行，不因短期波动或 A 轨风控强平。
- B 轨订单与平仓原因（logic_disproof / major_trend_reversal）可追溯，与 L2 共识、事件表可关联。
