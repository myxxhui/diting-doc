## 一、设计目标

各模块按 [04_B轨系统概念与目标](../../01_顶层概念/04_B轨系统概念与目标.md) 与 [00_B轨整体设计](00_B轨整体设计.md) 设计：**时间视野 1～2 月/轮**、**单轮收益目标 30%～100%+**、**年化复利目标 200%**、**中线止损与阶段止盈**、与 A/C 轨隔离。

<a id="design-b-track-modules"></a>
## 二、各模块 B 轨设计要点

### 2.1 采集模块

- **数据范围**：B 轨所需为**标的主营构成**（segment_list）、**细分一手信号**（经适配器+信号理解层写入 segment_signal_cache）；见 [12_右脑数据支撑与Segment规约](../_共享规约/12_右脑数据支撑与Segment规约.md)、[C 模块 B 轨设计](../Stage3_模块实践/04_B轨_C模块右脑与信号设计.md)（或 09_ Module C B 轨小节）。
- **基本数据依赖**：B 轨**复用** [11_数据采集与输入层规约](../_共享规约/11_数据采集与输入层规约.md) 提供的 **L1 行情/技术数据**（OHLCV）、**标的池**（get_current_a_share_universe）、**行业/板块**（与 Module A 同源）；采集侧由 11_ 与 Stage2 A 轨采集落地，本处不重复约定。
- **存储与契约**：标的主营构成表、细分注册表、细分信号缓存表在 12_ 与 02_B轨数据与存储规约 约定；消费方为 A 扩展、信号层、Module C（右脑）。**调度层 / 信号层 / 采集层 / 存储层**的设计规范与实践归属见 [02_B轨数据与存储规约 §四](02_B轨数据与存储规约.md#b-track-layers)。

### 2.2 A 模块（语义分类器）B 轨支持

- **输出**：ClassifierOutput 含领域 Tag、**主营占比列表（segment_shares）**，供 B 轨候选解析细分与 C 右脑消费。
- **准入**：B 轨候选可沿用 A 的 Tag 与 segment_shares；无需单独 b_track_eligible，由 B 模块阶段通过条件决定。

### 2.3 B 模块（量化扫描）B 轨策略与候选表

- **输入**：B 模块做阶段/趋势打分与 B 轨通过判断时，读取 **L1 行情/技术数据**（经 MarketDataFeed，由 11_ 采集写入）、**标的主营构成**（或 A 的 segment_shares）；见 [02_B轨数据与存储规约](02_B轨数据与存储规约.md)#三 消费方与读接口。
- **B 轨专用策略与打分**：**阶段/趋势类信号**（非仅日线短线），例如：周线或月线趋势、突破长期盘整、板块强度与资金持续性；可配置维度：阶段动量分、趋势确认分、板块强度分等；阈值与权重可配置。
- **准入与通过**：B 轨通过条件 = 阶段/趋势得分满足配置阈值（如 phase_score≥X、trend_confirm=True）；通过标的写入 B 轨候选表或输出带 MEDIUM_TERM 标记。
- **B 轨候选表**：`b_track_candidate_snapshot`（或等价）
  - **用途**：仅存放 **B 轨阶段/趋势策略通过** 的标的；字段含 symbol、batch_id、phase_score、trend_confirm、sector_strength、created_at 等。
  - **写入方**：B 模块在每次扫描后写入；**消费方**：Module C（右脑）**仅或优先**从此表读取 B 轨候选，做利好与主营对齐、景气与风险判断后输出 ExpertOpinion（horizon=**MEDIUM_TERM**）。

### 2.4 C 模块（右脑 MoE）B 轨逻辑

- **路由**：请求若 symbol 在 `b_track_candidate_snapshot` 或带 b_track_pass 约定，则走**右脑统一分析**（按股配置+统一分析），输出 ExpertOpinion 且 **horizon = MEDIUM_TERM**；不走 VC-Agent（VC-Agent 为 C 轨长线）。
- **输入**：symbol、quant_signal（B 阶段信号）、segment_list、segment_signals（来自 12_ 与信号层）；见 [04_A轨_MoE议会_设计](../Stage3_模块实践/04_A轨_MoE议会_设计.md) 与 C 模块 B 轨设计文档。
- **输出**：ExpertOpinion 含 is_supported、direction、confidence、reasoning_summary、risk_factors、**horizon=MEDIUM_TERM**；无共识落库（共识为 C 轨）。

### 2.5 D 模块（判官）B 轨逻辑

- **三轨分流**：MEDIUM_TERM → **中线止损**（如 5%～8% 或 ATR/结构，可配置）、**阶段止盈**（如 30%/50%/100% 分级或阶段结束/景气拐点）；不施加 2% 硬止损与现金拖累。
- **投票**：Quant（B 阶段通过）+ Expert（C 支持且看多）→ 有效信号；仓位凯利×确信度；Verdict 带 horizon=MEDIUM_TERM。

### 2.6 E 模块（风控）B 轨逻辑

- B 轨仓位**不**触发 2% 硬止损与现金拖累；按 track=B 走 B 轨分支。
- **中线止损**：检查 B 轨仓位是否触发中线止损（5%～8% 或 ATR/结构），触发则输出平仓指令。
- **阶段止盈**：检查 B 轨仓位是否触发阶段止盈规则（如浮盈达 30%/50%/100% 或阶段结束信号），触发则输出减仓/平仓指令。
- B 轨专用约束（可配置）：单标的 B 轨仓位上限、B 轨合计敞口占比、单行业/主题敞口上限。

### 2.7 F 模块（执行）B 轨逻辑

- 订单与仓位带 `track=B`；执行层与经纪侧按轨区分。
- B 轨平仓根据判官/风控的**中线止损**或**阶段止盈**指令执行；平仓原因（mid_stop / phase_take_profit）可追溯。
