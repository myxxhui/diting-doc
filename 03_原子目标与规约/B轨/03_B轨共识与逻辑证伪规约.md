# B 轨共识与逻辑证伪规约

> [!NOTE] **[TRACEBACK]**
> - **顶层概念**: [04_B轨系统概念与目标](../../01_顶层概念/04_B轨系统概念与目标.md)

> - **本文档**: B 轨**专用**共识 schema、逻辑证伪与大周期反转的输入/输出与接口；与 A 轨风控分离。

<a id="b-track-consensus-schema"></a>
## 一、B 轨共识 schema

- **目的**：VC-Agent 的结论结构化落库，供 D/E/F 做逻辑证伪判定、仓位标识与审计。
- **建议表名**：`b_track_consensus`（或等价）。

| 字段（建议） | 类型 | 说明 |
|--------------|------|------|
| symbol | VARCHAR(32) | 标的 |
| batch_id | VARCHAR(64) | 与当次 VC-Agent 调用批次一致 |
| opinion_id | VARCHAR(64) | 可选，关联 ExpertOpinion |
| key_assumptions | TEXT/JSONB | 关键假设列表，可解析 |
| evidence_summary | TEXT | 证据摘要 |
| disproof_triggers | TEXT/JSONB | 证伪条件（如「毛利率连续两季下滑超 5%」「实控人变更」） |
| potential_upside | VARCHAR(32) | 可选，潜在空间/赔率描述或枚举 |
| time_horizon | VARCHAR(8) | 3Y/5Y/10Y |
| created_at | TIMESTAMPTZ | 写入时间 |

- **写入方**：Module C（VC-Agent 或聚合层）；在产出 ExpertOpinion 后同步或异步写入。
- **消费方**：Module D 逻辑证伪子模块（读取 disproof_triggers 与事件比对）；E/F 可选读用于展示与审计。

<a id="b-track-disproof"></a>
## 二、逻辑证伪

- **输入**：
  - C 的共识（`b_track_consensus` 中 key_assumptions、disproof_triggers）；
  - **事件流**：财报暴雷、管理层重大变动、政策/技术路线颠覆、或 VC-Agent/独立模块的「假设失效」信号；事件来源、格式与写入位置（如 L2 事件表 `b_track_events` 或消息队列）由实现约定。
- **判定逻辑**：判官内子模块或独立服务比对事件与共识中的证伪条件；命中则产出「逻辑证伪」结论。
- **输出**：平仓/减仓指令，原因码 `logic_disproof`，供 E/F 执行并留痕；与 Verdict/执行层接口在 09_ 或协议中约定。

<a id="b-track-reversal"></a>
## 三、大周期反转

- **输入**：标的/行业价格与均线、行业指数关键位、波动率 regime；或 VC-Agent 的「大周期反转」建议；数据来源与更新频率由实现约定。
- **判定逻辑**：可配置规则（如长期均线死叉、行业指数跌破关键支撑、波动率 regime 切换）；判定主体为判官内子模块或 VC-Agent。
- **输出**：止盈/减仓信号，原因码 `major_trend_reversal`，供 E/F 执行并留痕。

<a id="b-track-consensus-exit"></a>
## 四、准出

- 共识 schema 与 dna_b_track 的 `consensus_*`、`disproof_*` 一致。
- 逻辑证伪与大周期反转的输入来源、输出格式在 L4 [Stage4 01_B轨_共识与执行_实践](../../04_阶段规划与实践/Stage4_MoE与执行网关/01_B轨_共识与执行_实践.md) 中可验收（占位实现或真实实现均可，接口固定）。
