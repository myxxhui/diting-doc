<a id="b-track-goal"></a>
## 一、B 轨目标（系统目标）

| 维度 | 目标 |
|------|------|
| **持仓周期** | 1～2 个月/轮 |
| **单轮收益目标** | 捕获具备 **30%～100%+** 阶段涨幅潜力的标的 |
| **年化复利目标** | **200%**（进取目标，需回测与实盘校验） |
| **每年可衔接轮数** | 2～5 轮（视市场环境：熊市 1～2 轮，牛市 3～5 轮） |

<a id="b-track-logic"></a>
## 二、B 轨核心逻辑

### 2.1 识别（谁负责、用什么）

- **Module B（量化扫描）**：输出**阶段/趋势类信号**（非仅日线短线），例如：周线或月线趋势、突破长期盘整、板块强度与资金持续性；通过条件包含「阶段得分」或「中线候选」标记，写入 B 轨候选表或等价输出。
- **Module C（右脑 MoE）**：对 B 轨候选做**利好与主营对齐、细分景气、政策/情绪**判断，输出 ExpertOpinion（`horizon = MEDIUM_TERM`）；采用「按股配置 + 统一分析」、信号来自适配器/信号理解层（见 [12_右脑数据支撑与Segment规约](../_共享规约/12_右脑数据支撑与Segment规约.md) 与 C 模块 B 轨设计文档）。
- **判官 D**：Quant（B 阶段通过）+ Expert（C 支持且看多）→ 有效信号；仓位按凯利×确信度；输出 Verdict 带 `horizon = MEDIUM_TERM`。

### 2.2 买入（何时、多少）

- **何时**：B 阶段信号通过 + C 专家支持且看多 + 风控 E 通过（盈亏比等）。
- **多少**：动态凯利 × Expert 确信度；B 轨可配置单笔上限与总敞口占比。

### 2.3 卖出（核心：止盈与止损）

- **止损**：**中线止损**，**不用** 2% 硬止损。采用 5%～8% 固定或 ATR/结构止损，可配置；触发则市价平仓。
- **止盈**：**阶段止盈**，规则化，例如：
  - 浮盈达 30% → 移止损至成本或移动止盈；
  - 达 50% → 减仓一定比例或移动止盈；
  - 达 100% → 再减或仅留移动止盈；
  - 或按「阶段结束/景气拐点」信号止盈（与 C 或独立阶段模块联动）。
- 止盈/止损规则在 L3 风控规约与 B 轨策略配置中写死，E/F 按轨执行。

<a id="b-track-modules"></a>
## 三、各模块 B 轨分工概要

| 模块 | B 轨职责 |
|------|----------|
| **采集** | 标的主营构成、细分信号源；为 C 提供 segment_list、segment_signals（见 12_、C 模块 B 轨设计）。 |
| **A** | 领域 Tag、主营占比列表（segment_shares）；B 轨候选可沿用或扩展。 |
| **B** | **阶段/趋势**信号、B 轨通过条件、写入 B 轨候选表或输出 MEDIUM_TERM 标记。 |
| **C** | 右脑统一分析、利好与主营对齐、景气与风险；输出 ExpertOpinion（MEDIUM_TERM）；信号适配器+信号理解层见 C 模块 B 轨设计。 |
| **D** | 按 MEDIUM_TERM 分支：中线止损、阶段止盈规则；仓位与复盘分轨。 |
| **E** | B 轨仓位不施 2% 硬止损；施中线止损与止盈检查；盈亏比等可沿用。 |
| **F** | 订单/仓位带 track=B；执行 B 轨平仓指令（止损/止盈）。 |

<a id="b-track-data"></a>
## 四、B 轨关键数据

- **标的细分与信号**：segment_list（标的主营构成）、segment_signals（细分一手信号，经适配器+信号理解层产出 direction/strength/risk_tags）；见 [12_右脑数据支撑与Segment规约](../_共享规约/12_右脑数据支撑与Segment规约.md)、[C 模块 B 轨设计](README.md#c模块b轨设计文档)。
- **B 轨候选**：B 模块输出的阶段通过标的列表（表或流），C 仅或优先消费该集合。
- **配置**：中线止损比例、阶段止盈档位、单笔/总敞口上限等，由 DNA 或 config 驱动。
