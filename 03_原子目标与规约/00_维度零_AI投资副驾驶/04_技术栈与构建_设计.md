## 一、技术栈选型（候选；V1 阶段在 DNA 固化）

### 1.1 核心栈

| 维度 | 选型 | 备注 |
|------|------|------|
| 语言 | **TypeScript 5.x（strict）** | `noUncheckedIndexedAccess` 等严格选项打开 |
| 框架 | **Next.js 14+（App Router）** | SSR + 流式 + 静态优化 |
| 包管理 | **pnpm 9+** | workspace 支持好；磁盘占用低 |
| Monorepo 编排 | **Turborepo** | 增量构建 + 缓存 |
| Node 运行时 | **Node 20 LTS+** | 与生产一致 |

### 1.2 UI / 设计

| 维度 | 选型 | 备注 |
|------|------|------|
| 样式 | **Tailwind CSS 4+** | 用于 ui-kit 内部；不在 Feature 层裸用 |
| 无样式组件库 | **Radix UI** | 无障碍 + 行为正确 |
| 图标 | **Lucide React** | 体积小；可摇树 |
| 字体 | 自托管开源字体（如 Inter / Noto Sans SC） | 不依赖第三方 CDN |
| 主题 | CSS 变量 + light / dark | 系统跟随 |

### 1.3 数据 / 状态

| 维度 | 选型 | 备注 |
|------|------|------|
| 服务端状态 | **TanStack Query v5** | 缓存 / 重试 / 流式 |
| 客户端状态 | **Zustand 4+** | 轻量；持久化用 middleware |
| 表单 | **React Hook Form + Zod** | 校验复用到 BFF |
| 实时 | 自建 SSE / WS 客户端 | 与 Query 集成 |

### 1.4 可视化

| 维度 | 选型 | 备注 |
|------|------|------|
| 图表（时序 / K 线） | **ECharts 5+** | 大量数据点 |
| 通用图表 | Recharts | 简单图 |
| 状态机图 | **React Flow 12+** | 节点 / 边 / 自动布局 |
| 大数据表 | TanStack Table | 虚拟滚动 |

### 1.5 测试

| 维度 | 选型 | 备注 |
|------|------|------|
| 单元 / 组件 | **Vitest + Testing Library** | 快、与 Vite 兼容 |
| E2E | **Playwright** | 多浏览器；trace 强 |
| 视觉回归 | Playwright snapshot 或 Chromatic | 关键页面 |
| 可访问性 | axe-core / Playwright a11y | 关键路径覆盖 |

### 1.6 工程化

| 维度 | 选型 | 备注 |
|------|------|------|
| Linter | **ESLint** + **typescript-eslint** | 严格 |
| 格式化 | **Prettier** | |
| 提交规范 | **commitlint + conventional commits** | |
| Git hooks | **lefthook**（或 husky） | |
| 错误监控 | **Sentry** | 可选自建 |
| 性能监控 | **Web Vitals** + 自建 | |
| 埋点 | 自建 → 写超级个体进化 `feedback_implicit` | |

### 1.7 安全

| 维度 | 选型 | 备注 |
|------|------|------|
| CSP | strict-dynamic | 不允许 eval；inline 仅 nonce |
| Cookie | HttpOnly + SameSite=Lax + CSRF token | |
| XSS 防护 | React + DOMPurify（仅富文本展示） | |
| 依赖扫描 | **Snyk** / pnpm audit | CI 强制 |

## 三、构建与发布

### 3.1 本地开发
- `pnpm install`
- `pnpm dev`：并行启动 console + admin（按需）
- 与本地 BFF 联调通过 `.env.local` 注入 API base URL

### 3.2 CI（构建产物 + 验证）

| 阶段 | 内容 |
|------|------|
| Lint | `pnpm lint` |
| Type Check | `pnpm typecheck` |
| Unit Test | `pnpm test --coverage` |
| Build | `pnpm build`（Next.js standalone） |
| E2E | `pnpm e2e`（against staging-like 环境） |
| 安全扫描 | `pnpm audit --audit-level=high` |

### 3.3 镜像与部署
- 构建 Next.js standalone → Dockerfile（Alpine + Node 20）
- 镜像推送到 deploy-engine / 部署仓的 registry
- Helm Chart 在 `diting-infra/charts/diting-stack/templates/web/`

### 3.4 发布灰度
- 与超级个体进化 `version_manager` 联动
- 灰度策略：按 cookie / header 路由到 canary 容器组

## 四、性能预算（V1 起强约束）

| 指标 | 目标 | 失败处理 |
|------|------|---------|
| LCP（首屏 largest contentful paint） | < 2.5s | CI 阻塞合并 |
| INP | < 200ms | CI 阻塞 |
| CLS | < 0.1 | CI 阻塞 |
| JS 总传输（gzip） | < 250KB（首屏路由） | CI 阻塞 |
| 关键 API 首字节 | < 500ms | 监控告警 |

工具：Lighthouse CI + Web Vitals 上报。

## 五、可访问性（A11y）

- 全应用通过 axe 检查（CI）
- 键盘可达全部交互
- ARIA 角色与标签齐全
- 颜色对比度 ≥ 4.5（正文）/ 3 （大字号）
- 动画支持 `prefers-reduced-motion`

## 六、国际化与本地化

- 默认中文（zh-CN）；英文（en）作为第二语言
- 所有 UI 文案不允许硬编码；走 `useT()` hook
- 数字 / 日期 / 货币按 locale 格式化
- RTL 暂不支持（如有需要再加）

## 七、约定与守则

### 必须
- TypeScript strict + 不允许 `any`（除非 `// eslint-disable-next-line` 加注释）
- 任何外部 API 数据必须用 Zod schema 校验后再用
- 所有交互按钮必须有 loading / disabled / error 三态
- 所有列表必须支持空 / 加载中 / 错误三态

### 禁止
- 在前端写"会改变后端业务状态"的逻辑（前端只发请求）
- 在 production 包中保留 console.log / debugger
- 使用未发布的实验性 React 特性（除非走 feature flag）
- 直接操作 DOM（除非通过 ref + 受控）
