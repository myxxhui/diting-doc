## 一句话定位

**前端工程是四大模块的"产品形态出口"。**

它不是后端的附属层，而是把"研究 / 监控 / 风险 / 进化"四种主轴体验**一致化、可观测化、可被人主导**的产品平台。

## 为什么前端要并列于四大模块

- L1 §"项目要解决的问题"明确指出本项目的产品形态是"系统化认知生产工作台"，前端就是工作台本身
- L1 §"五条根本原则" §1：先让系统对自己有用——本人作为第一用户，前端体验决定项目能否每天被使用
- L1 §"项目不再是什么" §"不把自动下单作为第一阶段核心目标"——所有"决策建议"都需要人来确认，前端是确认入口
- 旧 ABCDEF 体系把前端当作隐式的"展示层"，实际上前端的工程复杂度（实时数据、长连接、SSE / WebSocket、状态机可视化、对话流）等同于一个独立后端

## 八个前端域

每个前端域承接一个或多个后端模块，独立成域、独立验收。

| # | 前端域 | 主要承接模块 | 一句话定位 |
|---|--------|-------------|-----------|
| 1 | 监控驾驶舱 | 状态机监控 + 极寒防御 | 一屏看全所有活跃状态机与风险事件 |
| 2 | 投研对话台 | 纵深进攻（研究议会） | 与议会多轮对话、追问、保存（含工具调用可视化） |
| 3 | 研究候选广场 | 纵深进攻 | 浏览 / 标星 / 加入观察的统一入口 |
| 4 | 状态机时间线视图 | 状态机监控 | 单标的 / 单逻辑的状态轨迹与历史迁移 |
| 5 | 风险熔断面板 | 极寒防御 | 实时风险事件、门禁拒绝原因、熔断状态、人审入口 |
| 6 | 进化复盘工作台 | 超级个体进化 | 评测报告 / 复盘 / 反馈打标 / 标注训练集 |
| 7 | 知识库前端 | 超级个体进化 | RAG 索引 + 检索 + 编辑 + 引用追踪 |
| 8 | 个人成长仪表板 | 超级个体进化 | 四岗位（推理 / Agent / Ops / Runtime）能力成长追踪 |

## 用户主权与"人在回路"

按 L1 §"项目不再是什么" 与 §"五条根本原则" §4：

- **所有研究结论 / 调仓建议 / 外部动作建议都需要"人确认"才能落地**
- 前端必须给出"人审入口"——单击"确认 / 暂缓 / 拒绝"，并在拒绝时收集原因供反馈
- 前端不允许"自动下单 / 自动发邮件 / 自动调用第三方 API"——这些都属超级个体进化的"外部动作边界"，须经极寒防御门禁
