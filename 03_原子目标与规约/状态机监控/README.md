## 一句话定位

**把"研究结论"变成"持续被监控的状态"，让"逻辑何时失效"成为可观测、可预警、可回放的工程事件。**

> 来自 L1 §核心价值 §更稳：把"主观判断"变成"状态机监控"。

## 核心交付物

- 状态机状态变更事件（state_transition_event）
- 调仓 / 退出建议（advisory_signal，**仅建议、不执行**）
- 状态时间线快照
- 状态机健康度指标

## 后端服务子模块（计划，第 2 批展开）

| 子模块 | 英文键 | 主责 |
|--------|--------|------|
| 状态机注册表 | `state_machine_registry` | 模板 / 实例 / 版本管理；模板可由议会在产出研究结论时附带 |
| 探针调度器 | `probe_scheduler` | cron / 事件触发 / 流式触发；与数据层联动 |
| 状态迁移引擎 | `transition_engine` | 触发条件评估 + 迁移执行 + 事件发出 |
| 调仓建议生成器 | `rebalance_advisor` | 把状态变化翻译成建议（**仅建议、不下单**） |
| 失效审计与复盘服务 | `invalidation_auditor` | 每次状态迁移可被回放；为超级个体进化提供反馈数据 |
| 状态变更通知派发 | `notification_dispatcher` | → 前端 / Webhook / 邮件，须经极寒防御门禁 |

## 前端接入

- 监控驾驶舱（所有活跃状态机总览）
- 状态机时间线视图（单标的 / 单逻辑）
- 调仓建议中心
