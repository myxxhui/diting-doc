# L3 · 状态机监控 · 接口契约设计

> [!NOTE] **[TRACEBACK]**
> - **同模块**：[02_后端服务子模块_设计](./02_后端服务子模块_设计.md)、[04_数据契约_设计](./04_数据契约_设计.md)

## 一、契约范围

| 契约 | 用途 |
|------|------|
| `StateMachineTemplate` | 模板定义 |
| `StateMachineInstance` | 实例 |
| `ProbeResult` | 探针结果 |
| `TransitionEvent` | 状态迁移事件 |
| `BoundaryBreach` | 越界事件 |
| `Advisory` | 调仓 / 退出建议 |
| `WatchList` / `WatchListItem` | 关注列表 |

## 二、Proto 定义（v1）

### 2.1 StateMachineTemplate

```proto
syntax = "proto3";
package state_watch.v1;

import "google/protobuf/timestamp.proto";
import "google/protobuf/struct.proto";

message TemplateNode {
  string id = 1;                         // WATCH / PRE_BREAK / ...
  string description = 2;
}

message TemplateProbe {
  string id = 1;
  string type = 2;                       // quote / news / derivative / custom
  string subject_field = 3;              // symbol / segment_id
  int32 interval_sec = 4;                // 默认间隔
  string trigger = 5;                    // periodic / stream
  google.protobuf.Struct config = 6;
}

message TransitionCondition {
  string probe = 1;
  string expr = 2;                       // 表达式 DSL
}

message TemplateTransition {
  string from = 1;
  string to = 2;
  string mode = 3;                       // all / any，默认 all
  repeated TransitionCondition conditions = 4;
}

message TemplateGuard {
  string subject_field = 1;
  string metric = 2;
  double threshold = 3;
  string severity = 4;                   // INFO / WARN / ERROR / CRITICAL
  string action = 5;                     // TRANSITION_TO_<NODE> / EMIT_ADVISORY / NOTIFY
}

message TemplateAdvisory {
  string on_state = 1;                   // 进入哪个状态时触发
  string kind = 2;                       // WATCH / EXIT / REBALANCE / ALERT
  string fallback = 3;
  int32 expires_in_sec = 4;
}

message StateMachineTemplate {
  string template_id = 1;
  string template_version = 2;           // SemVer
  string schema_version = 3;
  string description = 4;
  repeated TemplateNode nodes = 5;
  repeated TemplateProbe probes = 6;
  repeated TemplateTransition transitions = 7;
  repeated TemplateGuard guards = 8;
  repeated TemplateAdvisory advisories = 9;
  google.protobuf.Timestamp created_at = 10;
  google.protobuf.Timestamp updated_at = 11;
}
```

## 三、HTTP / gRPC 端点

### 3.1 模板与实例

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET  /v1/templates` | REST | 列出模板 |
| `GET  /v1/templates/{id}` | REST | 单模板详情 |
| `POST /v1/templates` | gRPC + REST | 创建模板（仅授权用户） |
| `PUT  /v1/templates/{id}` | gRPC + REST | 更新模板（语义化版本） |
| `POST /v1/instances` | gRPC + REST | 实例化（绑定 subject） |
| `GET  /v1/instances` | REST | 列出（按 subject / status / group） |
| `GET  /v1/instances/{id}` | REST | 单实例详情（含 history） |
| `POST /v1/instances/{id}/suspend` | REST | 挂起 |
| `POST /v1/instances/{id}/archive` | REST | 归档 |

### 3.2 探针 / 迁移 / 越界

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET  /v1/probes/results` | REST | 探针结果检索（按 probe_id / instance / time） |
| `GET  /v1/transitions` | REST | 迁移日志 |
| `GET  /v1/breaches` | REST | 越界事件列表 |

### 3.3 Advisory

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET  /v1/advisories` | REST | 列出（按 subject / kind / status） |
| `GET  /v1/advisories/{id}` | REST | 单详情 |
| `POST /v1/advisories/{id}/ack` | REST | 用户确认（已读） |
| `POST /v1/advisories/{id}/dismiss` | REST | 用户忽略 |

### 3.4 关注列表

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET  /v1/watchlists` | REST | 用户的列表 |
| `POST /v1/watchlists` | REST | 创建 |
| `PUT  /v1/watchlists/{id}/items` | REST | 添加 / 删除 item |
| `GET  /v1/watchlists/{id}/states` | REST | 列表内全部实例的当前状态 |

### 3.5 SSE / WebSocket

| 端点 | 说明 |
|------|------|
| `GET /v1/instances/{id}/events` | SSE：状态迁移 / 越界 / Advisory |
| `GET /v1/watchlists/{id}/stream` | SSE：列表内任何实例的状态变化 |

## 四、关键字段说明

- `template_version` 与 `instance.template_version`：实例创建时锁定版本，模板升级不影响已有实例
- `Advisory.expires_at`：必填；过期未执行 → 实例自动 `ARCHIVED`
- `Advisory.evidence`：必填；空证据被门禁 reject

## 五、错误码

| 错误码 | HTTP | 含义 |
|--------|------|------|
| `SW.TEMPLATE_NOT_FOUND` | 404 | 模板不存在 |
| `SW.INSTANCE_CORRUPTED` | 503 | 实例数据损坏，需人工修复 |
| `SW.PROBE_UNHEALTHY` | 503 | 探针不可用 |
| `SW.TEMPLATE_VERSION_MISMATCH` | 409 | 实例与模板版本不一致 |
| `SW.NOTIFICATION_BUDGET_EXCEEDED` | 429 | 用户通知超预算（仅返回，不阻塞业务） |

## 六、版本兼容

- 模板与实例的 `template_version` 严格分离，模板升级不影响旧实例
- 旧实例可主动"升级"到新模板版本（需用户确认）
- 通信契约破坏性变更走同 [极寒防御 §四](../极寒防御/03_接口契约_设计.md) 策略
