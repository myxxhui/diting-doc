## 1. 模板与实例注册器（template_instance_registry）

### 主责
- 维护**状态机模板**（节点、迁移条件、保护带、探针订阅、Advisory 触发条件）
- 维护**状态机实例**（绑定标的 / 主营 / 议题；当前状态；历史轨迹）
- 提供按 `subject / template_id / status` 的查询 API

### 模板结构（YAML 示例）
```yaml
template_id: tmpl_breakout_v1
schema_version: "1.0.0"
description: "突破型标的状态机"
nodes:
  - id: WATCH
    description: "观察"
  - id: PRE_BREAK
    description: "突破前夕"
  - id: BREAKING
    description: "突破中"
  - id: HOLDING
    description: "持有"
  - id: EXIT_PENDING
    description: "等待退出"
  - id: ARCHIVED
    description: "已归档"

probes:
  - id: probe_quote_5min
    type: quote
    subject_field: symbol
    interval_sec: 300

  - id: probe_news_realtime
    type: news
    subject_field: symbol
    trigger: stream

transitions:
  - from: WATCH
    to: PRE_BREAK
    conditions:
      - probe: probe_quote_5min
        expr: "rsi_14 > 65 AND vol_ratio > 1.5"

  - from: PRE_BREAK
    to: BREAKING
    conditions:
      - probe: probe_quote_5min
        expr: "close > resistance_20d"

  - from: BREAKING
    to: HOLDING
    conditions:
      - probe: probe_quote_5min
        expr: "close > entry_price * 1.05"

  - from: HOLDING
    to: EXIT_PENDING
    conditions:
      - any:
        - { probe: probe_quote_5min, expr: "close < trail_stop" }
        - { probe: probe_news_realtime, expr: "negative_event_score > 0.7" }

guards:
  hard_thresholds:
    - subject_field: symbol
      metric: drawdown
      threshold: -0.15
      severity: CRITICAL
      action: TRANSITION_TO_EXIT_PENDING

advisories:
  - on_state: EXIT_PENDING
    kind: EXIT
    fallback: "若 24h 内未执行，自动归档为 ARCHIVED 并产出复盘报告"
```

### 实例数据
```
instance_id, template_id, template_version, subject_id, current_state,
state_entered_at, history (列表), context (键值对), owner_user_id, group_id
```

### 失败模式
| 失败 | 保护 |
|------|------|
| 模板加载失败 | 实例锁定状态，不允许迁移；触发风险事件 |
| 实例数据不一致 | 实例标记 `corrupted`，需人工修复 |

## 2. 探针调度器（probe_scheduler）

### 主责
- 按模板订阅创建探针任务
- 调度 Worker 池执行探针；自适应频率（根据对象活跃度调整 interval）
- 写探针结果 → 触发评估器

### 内部组件
```
┌──────────────────────────────────────────┐
│ Subscription Manager                      │
│   ├─ 模板订阅 → 实例 + 探针组合            │
│   └─ 自适应频率（根据迁移频率 / 用户优先）  │
│         ↓                                 │
│ Scheduler (cron / 流订阅)                 │
│         ↓                                 │
│ WorkerPool                                │
│   ├─ 行情探针                              │
│   ├─ 新闻探针                              │
│   ├─ 衍生探针                              │
│   └─ 自定义探针                            │
│         ↓                                 │
│ ResultDispatcher → 评估器                 │
│ AuditWriter → ProbeAuditLog                │
└──────────────────────────────────────────┘
```

### 自适应频率
- 默认 interval 来自模板；
- 实例近 N 次迁移密集 → 缩短 interval；
- 实例长期无变化 → 拉长 interval；
- 用户高优先实例 → 不低于最小 interval；
- 总成本预算守护：单 Worker 池每 5 分钟成本上限。

### 失败模式
| 失败 | 保护 |
|------|------|
| 探针失败 | 重试 N 次；连续失败标记探针 unhealthy；通知极寒防御 |
| Worker 满载 | 排队 + 降级（跳过低优先实例并告警） |

## 3. 状态迁移评估器（transition_evaluator）

### 主责
- 接收探针结果 + 实例上下文 → 评估迁移条件 + 保护带 → 触发迁移
- 写迁移日志；产出 `TransitionEvent`

### 评估流程
```
ProbeResult
  ↓
LoadInstance(instance_id) → currentState
  ↓
LoadTransitions(template, currentState)
  ↓
EvaluateConditions(transition, probe_result, instance.context)
  ↓ (所有条件满足)
ApplyTransition (currentState → newState)
  ↓
EvaluateGuards (hard_thresholds, soft_thresholds)
  ↓ (越界)
EmitBoundaryBreach
  ↓
WriteTransitionLog
EmitTransitionEvent
TriggerAdvisoryGenerator (如适用)
```

### 关键设计
- **单实例串行**：同一实例的事件按到达顺序串行处理（避免迁移竞态）
- **条件表达式 DSL**：支持 `AND/OR/比较/聚合`；表达式预编译缓存
- **迁移幂等**：同一探针重复触发不会重复迁移

### 失败模式
| 失败 | 保护 |
|------|------|
| 表达式解析失败 | 实例锁定 + 风险事件 |
| 写日志失败 | 阻塞迁移直到日志写入成功 |
| 评估超时 | 该实例此轮跳过；下一探针重试 |

## 4. Advisory 生成器（advisory_generator）

### 主责
- 监听 `TransitionEvent` / `BoundaryBreach`
- 按模板的 `advisories` 配置生成 `Advisory`
- 推送到极寒防御 `decision_gate`

### Advisory 结构
```
{
  id, kind: WATCH/EXIT/REBALANCE/ALERT,
  subject_id, action: 描述,
  conditions: 触发条件快照,
  evidence: { probe_results, transition_event_id, instance_id, current_state, history_ref },
  fallback: 不执行的后果说明,
  expires_at: 失效时间
}
```

### 关键设计
- **必带 evidence**：拒绝无证据的 Advisory
- **必带 fallback**：拒绝无回退的 Advisory
- **expires_at**：超时未执行 → 实例进入 ARCHIVED 并产生复盘事件

## 5. 节奏化通知器（notification_dispatcher）

### 主责
- 接收 `Advisory`（已过门禁）+ `BoundaryBreach`（高严重度）
- 按用户的"通知预算"与"通知通道偏好"派发
- 超预算时合并为"摘要式通知"

### 通道
- Push（移动端）
- Email
- IM（企业微信 / 飞书 / Slack）

### 通知预算
```yaml
default_user_budget:
  daily_max: 30
  hourly_max: 10
  high_severity_bypass: true   # CRITICAL 越限直发
overflow_strategy: digest        # digest / drop / queue
```

### 失败模式
| 失败 | 保护 |
|------|------|
| 通道失败（如邮件 SMTP） | 重试 + 切备用通道 |
| 全部通道失败 | 写"未送达"队列 + 触发风险事件 |

## 6. 关注列表分组服务（watchlist_grouping）

### 主责
- 维护用户的关注分组（按主营 / 行业 / 自定义标签）
- 提供按分组聚合状态机实例的查询
- 分组变更触发探针订阅调整

### 数据模型
- `watchlists`（用户 → 列表）
- `watchlist_items`（列表 → 实例）
- `watchlist_tags`（自定义标签）
