## 三、按 5 大模块的执行重点

### 3.1 共享平台基础（前置项）

- 推荐技术栈：K3s + vLLM + HAMi + LangGraph + LLaMA-Factory（与战略文档一致）
- 输出：底座组件 Helm Chart、推理网关健康端点、配置中心热更新能力
- 验收：`make smoke-platform` 全过；详见 [04_/共享平台基础/01](../../04_阶段规划与实践/共享平台基础/01_本阶段实践与验证.md)

### 3.2 极寒防御

- MVP：Proto v1 + DB 迁移 + 风险事件总线 + 决策门禁器（schema/evidence）+ 审计日志（无链式 hash）+ 风险熔断面板
- V1：7 类污染检测 + 嵌入式熔断 SDK + 7 层门禁全开 + 链式 hash + OpenSearch 索引 + 配置中心接入
- V2：SDK 嵌入全微服务 + 自动合规导出 + 多租户 + Chaos engineering

### 3.3 纵深进攻

- MVP：单 Agent 议会 + 候选注册表 + 周期议程 + content_comprehension（NER/事件/主营对齐）+ 极寒门禁对接
- V1：多 Agent + MoE Router + 特征工程（批/流/嵌入索引/预期差）+ 评测兜底
- V2：browser/computer-use Tool Runtime + 自适应议程 + 跨议题学习 + 多模态证据

### 3.4 状态机监控

- MVP：内置 1~2 模板 + 行情探针（5min cron）+ 转移评估器 + Advisory 基础（无门禁）+ 关注列表中心
- V1：多探针（新闻/衍生）+ 自适应频率 + 越界保护带 + Advisory 必经门禁 + 通知预算 + digest + SSE + OpenSearch 索引
- V2：模板灰度 + 自定义探针沙箱 + 跨用户聚合 + 模板市场

### 3.5 超级个体进化

- MVP：反馈采集 + 数据集治理基础 + 单进程 SFT + 评测 runner local + 手动灰度
- V1：评测中枢（rule + LLM + human）+ 复盘归档 + 知识库 + 模型版本治理 + 自动 canary + 一键回滚
- V2：在线持续学习 + DPO/RLHF + 多模型 MoE 微调 + 安全防护 + 自动数据标注

### 3.6 前端工程与服务

- MVP：Monorepo + console（4 大面板）+ chat-bff + WS/SSE + 基础观测
- V1：admin + 投研工作台 + ws-gateway 多实例 + BFF 完整 + 性能预算 + i18n + WCAG 2.1 AA + 完整可观测
- V2：PWA + 离线 + 移动壳 + 桌面 Tauri + Yjs 协作 + Edge BFF

## 五、失败与回退策略

- **同一问题修复重试上限**：≤ 2 次（按 [系统规则 §7.2 第 6 条](../../00_系统规则_通用项目协议.md#72-ai-守则通用-l4-强化)）
- **超出后回退**：
  - 应用层：`helm rollback diting-stack <last-stable>`
  - 配置层：通过配置中心回滚到上一稳定版本（极寒防御 V1 起）
  - 模型层：`make super-evo-rollback VERSION=<id>`（超级个体进化 V1 起）
  - 数据层：保留上次成功 snapshot；不强行覆盖
- **审批**：critical 失败必须经架构师审批；非 critical 由 Tech Lead 决定
- **登记**：所有回退须登记到 [06_审计与一致性报告](../../06_追溯与审计/03_审计与一致性报告/)
