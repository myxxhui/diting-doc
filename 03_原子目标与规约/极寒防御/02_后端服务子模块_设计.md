## 1. 风险事件总线（risk_event_bus）

### 主责
- 跨模块统一接收 / 分发 / 持久化风险事件
- 提供按 severity / category / time_range 的检索 API
- 触发下游订阅者（监控驾驶舱、超级个体进化的反馈采集）

### 输入
- 任何模块产生的 `RiskEvent`（HTTP POST / gRPC / 消息队列三种入口）

### 内部组件
```
┌─────────────────────────────────────────────┐
│ Ingress (HTTP / gRPC / Kafka consumer)      │
│         ↓                                    │
│ Validator (schema_version, required fields) │
│         ↓                                    │
│ Enricher (补充 ts / correlation_id / actor) │
│         ↓                                    │
│ Persister (主存：PG; 索引：OpenSearch)      │
│         ↓                                    │
│ Dispatcher (Kafka topic / WebSocket push)   │
└─────────────────────────────────────────────┘
```

### 输出
- 持久化记录（PG 表 `risk_events`）
- Kafka topic `risk-events.{severity}` 分发
- WebSocket 推送给在线前端

### 失败模式与保护
| 失败 | 保护 |
|------|------|
| Persister 故障 | 写入本地 disk queue，恢复后回灌；阻塞 Ingress 直到能持久化（不允许丢） |
| Dispatcher 故障 | 仅丢失实时推送，不影响持久化；前端从 PG 拉取 |
| 外部消费者慢 | Kafka 消费组隔离，慢消费不影响主链路 |

## 2. 决策门禁器（decision_gate）

### 主责
- 对所有"出站对象"（研究结论 / 调仓建议 / 外部动作请求）做多层门禁判定
- 产出 `GateDecision { pass | degrade | reject }` 与可回放证据

### 输入
- `GateRequest { object_type, object_payload, context, requester }`

### 内部组件
```
┌──────────────────────────────────────────────┐
│ RuleLoader (从 06_动态配置中心拉规则)         │
│         ↓                                     │
│ ContextBuilder (合并 RuntimeContext)         │
│         ↓                                     │
│ GateChain                                     │
│   ├─ 1. Schema/字段完整性                     │
│   ├─ 2. 证据完整性（evidence_ref 必须可解析）│
│   ├─ 3. 合规规则                              │
│   ├─ 4. 指标门禁（成本/频率/置信度）          │
│   ├─ 5. 输入污染（调 input_sanitizer）        │
│   ├─ 6. 熔断检查（调 circuit_breaker）        │
│   └─ 7. 人审钩子（可选）                       │
│         ↓                                     │
│ DecisionAggregator (按"最严"决议)            │
│         ↓                                     │
│ AuditEmitter (写 audit_log_service)          │
└──────────────────────────────────────────────┘
```

### 多层门禁的合并规则
- **任何一层 reject → 最终 reject**
- **任何一层 degrade（且无 reject）→ degrade**
- **全部 pass → pass**
- **任何层异常（rule 加载失败 / 超时）→ reject + critical event**

### 输出
- `GateDecision`（同步返回 + 异步审计）
- 高严重度 reject 自动触发 RiskEvent

### 失败模式与保护
| 失败 | 保护 |
|------|------|
| 规则加载失败 | 立刻 reject 所有请求 + 触发 critical 风险事件 |
| 单层规则评估异常 | 该层视为 reject；不影响其它层；记录异常 |
| 审计写入失败 | 同步阻塞返回，不允许"放行但没审计" |

## 3. 熔断控制器（circuit_breaker）

### 主责
- 监听服务健康（来自 [08_心跳协议](../_共享规约/08_心跳协议与健康检查规约.md)）+ 错误率 / 延迟
- 按规则迁移服务状态：`CLOSED → OPEN → HALF_OPEN → CLOSED`
- 通知所有调用方"该服务已熔断"

### 部署形态
- **嵌入式 SDK**：所有微服务嵌入同一 SDK，本地缓存熔断状态
- **中心协调服务**：聚合健康信号、广播状态变更

### 状态迁移规则（可热配置）
```
CLOSED:
  错误率 > 50% (60s 窗口) OR P99 延迟 > 阈值
  → OPEN (持续 30s)

OPEN:
  超时 → HALF_OPEN

HALF_OPEN:
  连续 5 次成功 → CLOSED
  任一失败 → OPEN
```

### 输出
- `BreakerStateChange` 事件 → 风险事件总线
- 嵌入式 SDK 本地状态变更

### 失败模式与保护
| 失败 | 保护 |
|------|------|
| 中心协调服务故障 | 嵌入式 SDK 退化为本地决策（按本地观测） |
| 健康信号丢失 | fail-safe：默认 OPEN，宁误杀不放过 |

## 4. 输入污染检测器（input_sanitizer）

### 主责
检测进入 LLM / 决策链的输入是否被污染：

| 检测类别 | 触发条件 |
|---------|---------|
| Prompt Injection | 关键词模式（如 `ignore previous instructions`）+ 嵌入相似度 |
| 数据漂移 | 与基线分布的 KL/JS 距离超阈值 |
| 数值异常 | NaN / Inf / 量级跳变（与最近 N 期对比） |
| 超长 / 编码异常 | tokens 超阈值 / 非 UTF-8 / 控制字符 |
| 时间戳异常 | 未来时间 / 过期时间 |
| 外部 URL 注入 | 检测嵌入的不可信 URL |

### 输入
- `SanitizeRequest { content_type, payload, source }`

### 输出
- `SanitizeResult { passed: bool, findings: [...], sanitized_payload? }`
  - `findings` 记录检测到的污染类型与严重度
  - 可选返回"已脱敏"的 payload（用于自动降级）

### 失败模式与保护
- 任何检测器异常 → 视为 `passed: false`
- 模型类检测器（如嵌入相似度）调用推理网关失败 → 降级到正则规则

## 5. 审计日志服务（audit_log_service）

### 主责
- 不可篡改记录所有门禁判定、人审、熔断、外部动作
- 提供合规导出 + 第三方审计入口

### 输入
- `AuditEntry { decision_id, actor, action, payload, evidence_ref, ts }`

### 不可篡改设计
- **链式 hash**：每条记录的 hash 包含上一条 hash（链上链下双备份）
- **定期签名快照**：每 10 分钟一次签名快照，签名密钥归属安全组
- **WORM 存储**：底层存储采用对象存储 + retention lock

### 输出
- 写入成功响应（业务方阻塞等待）
- 按 `decision_id / actor / time_range` 检索 API
- 合规批量导出（按周期）

### 失败模式与保护
| 失败 | 保护 |
|------|------|
| 主存储不可用 | 写入本地 WAL，恢复后回放；同时阻塞业务（不允许"做事不留痕"） |
| 链式 hash 校验失败 | 触发 critical 风险事件 + 隔离当前节点 + 通知合规组 |
