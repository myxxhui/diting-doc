## 一、存储拓扑

| 数据类 | 主存 | 索引 | 归档 |
|--------|------|------|------|
| 研究卡片 / 候选 / 会议 | PostgreSQL | OpenSearch | OSS（90d 后） |
| 工具调用日志 | PostgreSQL（按月分区）| OpenSearch | OSS |
| 完整对话记录 | 对象存储（s3 / OSS） | OpenSearch（仅元数据） | 同主存 |
| 内容理解派生表 | PostgreSQL | OpenSearch | — |
| 嵌入向量 | Milvus / OpenSearch / pgvector | — | — |
| 特征仓库 | Feast 或自建（hot=Redis、cold=Parquet） | — | — |
| 时间线索引 | PostgreSQL（按 subject 分区） | — | — |

## 二、表结构（PostgreSQL）

### 2.1 research_cards

```sql
CREATE TABLE research_cards (
  id UUID PRIMARY KEY,
  correlation_id UUID NOT NULL,
  schema_version TEXT NOT NULL,
  subject_id TEXT NOT NULL,
  subject_type TEXT NOT NULL,             -- symbol / industry / segment
  title TEXT NOT NULL,
  conclusion TEXT NOT NULL,
  stance SMALLINT NOT NULL,
  applicable_period TEXT,
  confidence DOUBLE PRECISION NOT NULL,
  evidence_chain JSONB NOT NULL,          -- [{ref, source_type, excerpt, source_ts, weight}, ...]
  suggested_state_machine_template TEXT,
  extra JSONB,
  producer TEXT NOT NULL,
  degraded BOOLEAN NOT NULL DEFAULT FALSE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_research_cards_subject_created ON research_cards(subject_id, created_at DESC);
CREATE INDEX idx_research_cards_stance_confidence ON research_cards(stance, confidence DESC);
CREATE INDEX idx_research_cards_correlation ON research_cards(correlation_id);
```

### 2.2 candidates

```sql
CREATE TABLE candidates (
  id UUID PRIMARY KEY,
  correlation_id UUID NOT NULL,
  schema_version TEXT NOT NULL,
  subject_id TEXT NOT NULL,
  subject_type TEXT NOT NULL,
  kind SMALLINT NOT NULL,
  status SMALLINT NOT NULL,
  priority INT NOT NULL,
  confidence DOUBLE PRECISION NOT NULL,
  source_card_id UUID REFERENCES research_cards(id),
  suggested_state_machine_template TEXT,
  suggested_probe_set TEXT,
  extra JSONB,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_candidates_status_priority ON candidates(status, priority);
CREATE INDEX idx_candidates_subject ON candidates(subject_id);
CREATE INDEX idx_candidates_kind_created ON candidates(kind, created_at DESC);
```

### 2.3 council_sessions

```sql
CREATE TABLE council_sessions (
  id UUID PRIMARY KEY,
  topic_id UUID,
  schema_version TEXT NOT NULL,
  topic_title TEXT NOT NULL,
  subject_id TEXT,
  participating_agents TEXT[] NOT NULL,
  votes JSONB NOT NULL,                   -- [{agent_id, stance, confidence, reasoning, evidence, extra}, ...]
  dissents JSONB,
  consensus TEXT,
  transcript_ref TEXT NOT NULL,           -- s3:// 完整对话
  tool_call_ids UUID[] NOT NULL DEFAULT '{}',
  degraded BOOLEAN NOT NULL DEFAULT FALSE,
  degraded_reason TEXT,
  started_at TIMESTAMPTZ NOT NULL,
  ended_at TIMESTAMPTZ
);

CREATE INDEX idx_council_sessions_started ON council_sessions(started_at DESC);
CREATE INDEX idx_council_sessions_subject ON council_sessions(subject_id);
```

### 2.4 agenda_topics

```sql
CREATE TABLE agenda_topics (
  id UUID PRIMARY KEY,
  title TEXT NOT NULL,
  subject_id TEXT,
  source SMALLINT NOT NULL,               -- 1=USER, 2=EVENT, 3=PERIODIC
  priority INT NOT NULL,
  context JSONB,
  scheduled_at TIMESTAMPTZ NOT NULL,
  requested_by TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'pending', -- pending / scheduled / running / done / cancelled
  session_id UUID REFERENCES council_sessions(id)
);

CREATE INDEX idx_agenda_status_scheduled ON agenda_topics(status, scheduled_at);
CREATE INDEX idx_agenda_source_scheduled ON agenda_topics(source, scheduled_at DESC);
```

### 2.5 tool_calls

```sql
CREATE TABLE tool_calls (
  id UUID PRIMARY KEY,
  session_id UUID NOT NULL REFERENCES council_sessions(id),
  agent_id TEXT NOT NULL,
  tool_name TEXT NOT NULL,
  arguments JSONB,
  result JSONB,
  status SMALLINT NOT NULL,               -- 1=SUCCESS, 2=FAILED, 3=TIMEOUT
  elapsed_ms INT NOT NULL,
  ts TIMESTAMPTZ NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (ts);

-- 按月分区，由运维脚本自动创建
CREATE INDEX idx_tool_calls_session_ts ON tool_calls(session_id, ts);
CREATE INDEX idx_tool_calls_tool_status ON tool_calls(tool_name, status);
```

### 2.6 expectation_gaps

```sql
CREATE TABLE expectation_gaps (
  subject_id TEXT NOT NULL,
  source_card_id UUID NOT NULL REFERENCES research_cards(id),
  our_view TEXT NOT NULL,
  market_consensus TEXT NOT NULL,
  gap_score DOUBLE PRECISION NOT NULL,
  momentum_decay DOUBLE PRECISION NOT NULL,
  computed_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (subject_id, source_card_id, computed_at)
);

CREATE INDEX idx_gap_subject_computed ON expectation_gaps(subject_id, computed_at DESC);
```

## 三、内容理解派生表

```sql
-- 实体抽取结果
CREATE TABLE comprehension_entities (
  id UUID PRIMARY KEY,
  doc_id TEXT NOT NULL,
  entity_type TEXT NOT NULL,              -- company / person / product / event
  raw_text TEXT NOT NULL,
  resolved_subject_id TEXT,               -- 对齐到标准 subject_id（可空）
  confidence DOUBLE PRECISION,
  ts TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_entities_doc ON comprehension_entities(doc_id);
CREATE INDEX idx_entities_resolved ON comprehension_entities(resolved_subject_id);

-- 事件抽取结果
CREATE TABLE comprehension_events (
  id UUID PRIMARY KEY,
  doc_id TEXT NOT NULL,
  event_type TEXT NOT NULL,
  subject_ids TEXT[] NOT NULL,            -- 事件涉及的标的
  event_ts TIMESTAMPTZ,                   -- 事件发生时间
  details JSONB,
  ts TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_events_subject ON comprehension_events USING GIN (subject_ids);
CREATE INDEX idx_events_type_ts ON comprehension_events(event_type, event_ts DESC);

-- 主营 segment 映射
CREATE TABLE subject_segment_map (
  subject_id TEXT NOT NULL,
  segment_id TEXT NOT NULL,
  weight DOUBLE PRECISION NOT NULL,       -- 主营占比
  source TEXT NOT NULL,                   -- announcement_id / report_id / 等
  effective_from TIMESTAMPTZ NOT NULL,
  effective_to TIMESTAMPTZ,
  PRIMARY KEY (subject_id, segment_id, effective_from)
);

CREATE INDEX idx_segment_map_subject ON subject_segment_map(subject_id);
CREATE INDEX idx_segment_map_segment ON subject_segment_map(segment_id);

-- 标的时间线
CREATE TABLE subject_timeline (
  subject_id TEXT NOT NULL,
  event_id UUID NOT NULL,
  event_type TEXT NOT NULL,
  event_ts TIMESTAMPTZ NOT NULL,
  weight DOUBLE PRECISION,                -- 在标的时间线上的权重
  PRIMARY KEY (subject_id, event_ts, event_id)
);

CREATE INDEX idx_timeline_subject_ts ON subject_timeline(subject_id, event_ts DESC);
```

## 四、嵌入向量

| 向量类型 | 存储 | 维度 | 用途 |
|---------|------|------|------|
| 文档语义嵌入 | Milvus / OpenSearch（dense_vector）/ pgvector | 与所选模型对齐 | RAG 检索 |
| 标的画像嵌入 | 同上 | 同上 | 相似标的查找 |
| 事件嵌入 | 同上 | 同上 | 事件聚类 / 跨源关联 |

> 向量索引选型由 [05_接口抽象层](../_共享规约/05_接口抽象层规约.md) 的 `Inference Gateway Port` + `Data Source Port` 决定，DNA 中固化。

## 五、Feature Store

| 特征类型 | 主键 | 频率 |
|---------|------|------|
| 行情衍生 | `(subject_id, ts)` | 日 |
| 事件聚合 | `(subject_id, window, ts)` | 日 + 流（实时） |
| 行业 / 产业链聚合 | `(industry_id, ts)` | 日 |
| 嵌入向量 | `(subject_id, model_version, ts)` | 按变更触发 |

> 推荐 [Feast](https://feast.dev/) 或自建；选型在 V1 阶段定。

## 七、保留与归档

| 表 | 主存 | 索引 | 归档 |

| Feature Store hot | 7d | — | — |
| Feature Store cold | 1 年 | — | 永久 |
