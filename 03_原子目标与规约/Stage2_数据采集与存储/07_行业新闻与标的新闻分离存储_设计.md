<a id="design-stage2-07-goal"></a>
## 一、目标与背景

### 1.1 目标

将**行业新闻/公告**与**标的新闻/公告**在 L2 中**显式分离存储**，支持按 scope 精确查询，并衔接 12_ 规约的 segment 三层（domain/板块/业务）信号源选型。

### 1.2 背景与问题

| 现状 | 问题 |
|------|------|
| `news_content` 仅用 `symbol` 区分：`_MARKET_`（全市场）与 `600982.SH`（个股） | 无「行业维度」存储；板块级 segment（如 tech_semiconductor）无法按申万行业查行业新闻 |
| 12_ §2.2 约定第 2 层板块级→板块新闻、第 3 层业务级→标的新闻 | 当前实现仅支持标的级，板块/行业新闻未落地 |
| config/signal_layer.yaml：A 轨=「行业+标的新闻」 | 行业新闻与标的新闻混存，消费方无法按需分别拉取 |

### 1.3 设计原则

- **与 Module A / B 行业口径同源**：行业名来自 `industry_revenue_summary.industry_name`（申万一级：电子、计算机、电力设备、有色金属等），与 classifier_rules、板块强度一致。
- **向后兼容**：现有 `symbol=_MARKET_` 与 `symbol=600982.SH` 的数据与查询逻辑可平滑迁移。
- **消费可扩展**：信号层适配器可按 segment 层级选择「行业新闻」或「标的新闻」；Module C 可按需组合。

<a id="design-stage2-07-schema"></a>
## 二、存储与 Schema 设计

### 2.1 统一表 + scope 维度（推荐）

在现有 `news_content` 上增加 **scope**、**scope_id**，形成三维标识：

| 字段 | 类型 | 说明 |
|------|------|------|
| `scope` | VARCHAR(32) NOT NULL | `market` \| `industry` \| `symbol` |
| `scope_id` | VARCHAR(128) NOT NULL | 见下表 |
| `symbol` | VARCHAR(32) | **冗余**：scope=symbol 时等于 scope_id；scope=industry 时 NULL；scope=market 时为 `_MARKET_`，便于兼容 |

**scope 与 scope_id 取值约定**

| scope | scope_id | 含义 | 采集来源 |
|-------|----------|------|----------|
| `market` | `_MARKET_` | 全市场/宏观新闻 | js_news、OpenBB 等 |
| `industry` | `电力`、`电子`、`计算机` 等 | 申万一级行业新闻 | 见 §3 采集设计 |
| `symbol` | `600982.SH`、`000001.SZ` | 个股新闻/公告 | stock_news_em、JQData 等 |

### 2.2 DDL 变更（迁移脚本）

```sql
-- 新增列（默认值保证存量数据可先按 symbol 推理 scope）
ALTER TABLE news_content ADD COLUMN IF NOT EXISTS scope VARCHAR(32) NOT NULL DEFAULT 'symbol';
ALTER TABLE news_content ADD COLUMN IF NOT EXISTS scope_id VARCHAR(128) NOT NULL DEFAULT '';

-- 回填存量：symbol=_MARKET_ → market；其余 → symbol
UPDATE news_content SET scope='market', scope_id='_MARKET_' WHERE symbol = '_MARKET_' AND (scope IS NULL OR scope = 'symbol');
UPDATE news_content SET scope='symbol', scope_id=symbol WHERE symbol != '_MARKET_' AND symbol IS NOT NULL AND (scope IS NULL OR scope = 'symbol');

-- 新唯一约束：(scope, scope_id, title_hash, published_at)
CREATE UNIQUE INDEX IF NOT EXISTS uq_news_content_scope_dedup
    ON news_content (scope, scope_id, title_hash, published_at);

-- 查询索引
CREATE INDEX IF NOT EXISTS idx_news_content_scope_scope_id ON news_content(scope, scope_id);
CREATE INDEX IF NOT EXISTS idx_news_content_scope_id_pub ON news_content(scope, scope_id, published_at DESC);

COMMENT ON COLUMN news_content.scope IS 'market=全市场, industry=行业, symbol=个股';
COMMENT ON COLUMN news_content.scope_id IS 'market时_MARKET_, industry时申万行业名, symbol时标的代码';
```

**兼容策略**：若暂不迁索引，查询可先按 `(scope, scope_id)` 或按 `symbol`（兼容 path）双写条件；逐步统一到 scope/scope_id。

### 2.3 与现有写入契约的衔接

- `write_news_content_batch` 增加参数 `scope`、`scope_id`；内部计算 `title_hash` 后按 `(scope, scope_id, title_hash, published_at)` 去重。
- 存量调用方：传入 `scope='symbol'`, `scope_id=symbol` 即可保持原有语义。

<a id="design-stage2-07-ingest"></a>
## 三、采集逻辑设计

### 3.1 采集路径与数据源

| 路径 | 数据源 | scope | scope_id | 执行时机 |
|------|--------|-------|----------|----------|
| 全市场 | js_news / stock_news_em("000001") | market | _MARKET_ | 每批首批一次 |
| 行业新闻 | 见 §3.2 | industry | 电力/电子/... | 每批或按行业列表周期拉取 |
| 个股新闻 | stock_news_em(symbol)、JQData | symbol | 600982.SH | 按标的循环，与现有一致 |

### 3.2 行业新闻数据源（分阶段）

**Phase 1：全市场 + 关键词过滤（逻辑填充期）**

- 拉取 `js_news(indicator="最新资讯")` 或等价全市场接口；
- 按 **申万一级行业名**（与 `industry_revenue_summary`、`industry_fallback.csv` 一致）做**关键词匹配**：标题/摘要含「电力」「电子」「半导体」等 → 写入 `scope=industry`, `scope_id=行业名`；
- 未匹配到具体行业的条目保留为 `scope=market`, `scope_id=_MARKET_`。
- **行业名枚举**：从 L2 `industry_revenue_summary` 的 `DISTINCT industry_name` 或 `config/classifier_rules.yaml` 的 industry_keywords 推导；可配置 `INGEST_INDUSTRY_NEWS_INDUSTRIES` 显式列表。

**Phase 2：板块新闻 API（可选扩展）**

- 若 AkShare 或东方财富提供**按板块/行业**的新闻接口，则直接按行业拉取并写入 `scope=industry`；
- 实现时通过 Provider 抽象，Phase 1 与 Phase 2 可并存。

### 3.3 采集任务拆分

- **run_ingest_news** 内部按三步执行：
  1. **全市场**：无 symbol 时拉全市场 → `scope=market`；
  2. **行业**：`run_ingest_industry_news()` 或集成在 `run_ingest_news()` 内，拉行业新闻 → `scope=industry`；
  3. **个股**：`run_ingest_news(symbol=...)` → `scope=symbol`。

- 环境开关：`INGEST_PRODUCTION_NEWS_INDUSTRY=true` 时启用行业新闻采集；默认可与 `INGEST_PRODUCTION_NEWS` 联动。

### 3.4 增量与去重

- 行业新闻：按 `scope_id`（行业名）查 `MAX(published_at)`，远程无更新则跳过；
- 个股新闻：保持现有 `db_max_published_at` 逻辑；
- 去重：统一 `(scope, scope_id, title_hash, published_at)`。

<a id="design-stage2-07-consumption"></a>
## 四、消费与信号层衔接

### 4.1 查询接口约定

| 场景 | 查询条件 | 示例 SQL |
|------|----------|----------|
| 标的相关新闻 | scope=symbol AND scope_id=标的 | `WHERE scope='symbol' AND scope_id='600982.SH'` |
| 行业相关新闻 | scope=industry AND scope_id=行业名 | `WHERE scope='industry' AND scope_id='电力'` |
| 全市场新闻 | scope=market AND scope_id='_MARKET_' | `WHERE scope='market' AND scope_id='_MARKET_'` |
| 标的+所在行业 | (scope=symbol AND scope_id=标的) OR (scope=industry AND scope_id=标的行业) | 需先查 industry_revenue_summary 得标的行业 |

### 4.2 信号层适配器扩展

**SegBpNewsAdapter 按 segment 层级选数据源**（与 12_ §2.2、06_B轨_设计 §二 一致）：

| segment 层级 | 建议数据源 | scope | 说明 |
|--------------|------------|-------|------|
| 第 1 层 domain | 全市场/宏观 | market | tech/agri/geo 大盘 |
| 第 2 层 板块 | 行业新闻 | industry | tech_semiconductor → scope_id=电子/半导体 |
| 第 3 层 业务 | 标的新闻 | symbol | seg_bp_* 具体业务 → 标的新闻 |

- **segment → scope 映射**：通过 `segment_registry` 扩展字段 `news_scope`（market|industry|symbol）及 `industry_name`（板块级时填入申万行业名），或从 segment_id 解析（如 `tech_semiconductor` → industry=半导体/电子）；
- **context 扩展**：`SegBpNewsAdapter` 的 context 增加 `industry_name`（当 segment 为板块级时），用于查 `scope=industry`；
- **标的级**：保持现有「symbol → news_content」逻辑，仅改为按 `(scope, scope_id)` 查询。

### 4.3 向后兼容

- 已有查询 `WHERE symbol='600982.SH'`：迁移后 `symbol` 在 scope=symbol 时仍等于 scope_id，可保留 `WHERE symbol=X` 或统一改为 `WHERE scope='symbol' AND scope_id=X`；
- `_MARKET_`：`WHERE symbol='_MARKET_'` 等价于 `WHERE scope='market' AND scope_id='_MARKET_'`。

<a id="design-stage2-07-config"></a>
## 五、配置与环境变量

| 配置项 | 说明 | 默认 |
|--------|------|------|
| `INGEST_PRODUCTION_NEWS_INDUSTRY` | 是否采集行业新闻 | false（Phase 1 落地后可 true） |
| `INGEST_INDUSTRY_NEWS_INDUSTRIES` | 显式行业名列表（逗号分隔），空则从 L2/配置推导 | 空 |
| `INGEST_INDUSTRY_NEWS_DAYS_BACK` | 行业新闻回溯天数 | 7 |

## 六、验证与准出

### 6.1 设计层验收

| 验收项 | 定义 |
|--------|------|
| Schema 就绪 | news_content 有 scope、scope_id 列及对应索引；迁移脚本可重复执行 |
| 采集路径 | 全市场、行业、个股三条路径均可写入；写入 scope/scope_id 正确 |
| 消费兼容 | 按 scope+scope_id 可正确查行业/标的/全市场新闻；存量 symbol 查询仍可用 |
| 信号层衔接 | SegBpNewsAdapter 可按 segment 层级选择 industry 或 symbol 新闻 |

### 6.2 可执行验证命令（供 L4 实践）

- `make verify-news-scope`（或等价）：检查 news_content 中 scope/scope_id 分布及非空约束；
- `make ingest-test-news-scope`：Mock 或小样本执行行业+标的采集，断言写入行数与 scope 分布。

### 6.3 准出

1. 本设计文档、DNA、L4 实践、L5 验收行 1:1:1 就绪；
2. Schema 迁移脚本可执行且可重复；
3. 采集逻辑支持三条路径写入，消费方可按 scope 查询；
4. 信号层适配器可按 segment 层级选 industry/symbol 新闻（可在后续实践文档分步实现）。

<a id="design-stage2-07-exit"></a>
## 七、实施顺序建议

1. **Schema 迁移**：增加 scope、scope_id，回填存量，建索引；
2. **写入契约**：`write_news_content_batch` 支持 scope/scope_id；
3. **采集扩展**：`run_ingest_news` 拆出行业路径，Phase 1 用全市场+关键词；
4. **消费迁移**：`SegBpNewsAdapter`、query_ingest_data_overview 等改为按 scope 查询；
5. **信号层扩展**：按 segment 层级选 industry 或 symbol 数据源。
