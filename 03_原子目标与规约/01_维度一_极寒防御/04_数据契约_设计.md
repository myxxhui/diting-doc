## 一、存储拓扑

| 数据类 | 主存 | 索引 | 归档 | 保留期 |
|--------|------|------|------|--------|
| 风险事件 | PostgreSQL | OpenSearch | OSS 冷归档 | 主存 90d，索引 30d，归档 ≥ 5 年 |
| 门禁判定 | PostgreSQL | OpenSearch | OSS | 主存 90d，索引 90d，归档 ≥ 5 年 |
| 熔断状态历史 | PostgreSQL | — | OSS | 90d → 归档 |
| 输入污染事件 | PostgreSQL | OpenSearch | OSS | 同风险事件 |
| 审计日志 | PostgreSQL（WORM） + 对象存储（链式 hash 副本） | OpenSearch | OSS（retention lock） | **永久保留**（合规要求） |

## 二、表结构（PostgreSQL）

### 2.1 risk_events

```sql
CREATE TABLE risk_events (
  id UUID PRIMARY KEY,
  correlation_id UUID NOT NULL,
  schema_version TEXT NOT NULL,
  severity SMALLINT NOT NULL,                -- 1=INFO, 2=WARN, 3=ERROR, 4=CRITICAL
  category SMALLINT NOT NULL,                -- enum RiskCategory
  code TEXT NOT NULL,
  message TEXT NOT NULL,
  affected_subjects TEXT[] NOT NULL DEFAULT '{}',
  evidence_ref TEXT,                         -- s3:// 或 db://
  extra JSONB,
  producer TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_risk_events_correlation ON risk_events(correlation_id);
CREATE INDEX idx_risk_events_severity_created ON risk_events(severity, created_at DESC);
CREATE INDEX idx_risk_events_category_created ON risk_events(category, created_at DESC);
CREATE INDEX idx_risk_events_subjects ON risk_events USING GIN (affected_subjects);
```

### 2.2 gate_decisions

```sql
CREATE TABLE gate_decisions (
  decision_id UUID PRIMARY KEY,
  correlation_id UUID NOT NULL,
  object_type SMALLINT NOT NULL,
  object_schema_version TEXT NOT NULL,
  result SMALLINT NOT NULL,                  -- 1=PASS, 2=DEGRADE, 3=REJECT, 4=PENDING_HUMAN
  reason TEXT NOT NULL,
  layer_results JSONB NOT NULL,              -- [{layer_name, result, reason, details, elapsed_ms}, ...]
  evidence_ref TEXT,
  requester TEXT NOT NULL,
  context JSONB,
  decided_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  payload_hash TEXT NOT NULL                 -- payload 的 sha256，便于审计校验
);

CREATE INDEX idx_gate_decisions_correlation ON gate_decisions(correlation_id);
CREATE INDEX idx_gate_decisions_result_decided ON gate_decisions(result, decided_at DESC);
CREATE INDEX idx_gate_decisions_requester ON gate_decisions(requester, decided_at DESC);
```

### 2.3 breaker_states

```sql
CREATE TABLE breaker_states (
  service TEXT NOT NULL,
  caller TEXT NOT NULL DEFAULT '*',
  state SMALLINT NOT NULL,                   -- 1=CLOSED, 2=OPEN, 3=HALF_OPEN
  reason TEXT,
  metrics JSONB,
  changed_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (service, caller, changed_at)
);

CREATE INDEX idx_breaker_states_service_changed ON breaker_states(service, changed_at DESC);
```

### 2.4 sanitize_findings

```sql
CREATE TABLE sanitize_findings (
  id UUID PRIMARY KEY,
  request_id UUID NOT NULL,
  source TEXT NOT NULL,
  content_type TEXT NOT NULL,
  passed BOOLEAN NOT NULL,
  findings JSONB NOT NULL,                   -- [{kind, severity, detail, evidence}, ...]
  payload_hash TEXT NOT NULL,
  context JSONB,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_sanitize_passed_created ON sanitize_findings(passed, created_at DESC);
CREATE INDEX idx_sanitize_source_created ON sanitize_findings(source, created_at DESC);
```

### 2.5 audit_logs（WORM）

```sql
CREATE TABLE audit_logs (
  entry_id UUID PRIMARY KEY,
  decision_id UUID,                          -- 关联 gate_decisions（可空：人审/熔断不一定有）
  actor TEXT NOT NULL,
  action TEXT NOT NULL,                      -- gate_decision / human_review / breaker_change / external_action
  payload BYTEA NOT NULL,                    -- 原始 payload（序列化后）
  payload_hash TEXT NOT NULL,                -- sha256(payload)
  previous_hash TEXT NOT NULL,               -- 上一条 audit_logs 的 payload_hash（链式）
  signature TEXT,                            -- 节点签名 / 快照签名（快照时回填）
  ts TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_audit_decision ON audit_logs(decision_id);
CREATE INDEX idx_audit_actor_ts ON audit_logs(actor, ts DESC);
CREATE INDEX idx_audit_action_ts ON audit_logs(action, ts DESC);

-- WORM：禁止 UPDATE/DELETE
CREATE OR REPLACE RULE no_update_audit_logs AS ON UPDATE TO audit_logs DO INSTEAD NOTHING;
CREATE OR REPLACE RULE no_delete_audit_logs AS ON DELETE TO audit_logs DO INSTEAD NOTHING;
```

> **链式 hash 校验**：定期任务（每 10 min）扫描最新 N 条记录，验证 `previous_hash` 链完整；签名快照写入对象存储（不可变 bucket）。

## 三、配置数据（动态配置中心）

门禁规则不存数据库，存 [06_动态配置中心](../_共享规约/06_动态配置中心规约.md)，支持热更新与版本回滚。

```yaml
# 示例：决策门禁规则（动态配置中心）
cryo_guard:
  decision_gate:
    rules:
      - layer: schema
        version: "1.0.0"
        enabled: true
      - layer: evidence
        version: "1.0.0"
        enabled: true
        config:
          require_evidence_ref: true
          allow_local_only: false
      - layer: compliance
        version: "1.2.0"
        enabled: true
        config:
          forbidden_subjects: ["ST*", "*暂停上市*"]
      - layer: metric
        version: "1.1.0"
        enabled: true
        config:
          min_confidence: 0.6
          max_per_day_per_subject: 3
          max_token_cost: 5000
      - layer: sanitize
        version: "1.0.0"
        enabled: true
      - layer: breaker
        version: "1.0.0"
        enabled: true
      - layer: human
        version: "1.0.0"
        enabled: true
        config:
          required_for_object_types: ["EXTERNAL_ACTION", "MODEL_DEPLOY"]

  circuit_breaker:
    defaults:
      error_rate_threshold: 0.5
      window_sec: 60
      open_duration_sec: 30
      half_open_success_count: 5
    overrides:
      - service: "inference_gateway"
        error_rate_threshold: 0.3
        open_duration_sec: 60
```

## 四、数据保留与归档

| 表 | 主存保留 | 索引保留 | 归档策略 |
|-----|---------|---------|---------|
| risk_events | 90d | 30d | 90d 后批量导出至 OSS（按 month 分区），保留 ≥ 5 年 |
| gate_decisions | 90d | 90d | 同上 |
| breaker_states | 90d | — | 同上 |
| sanitize_findings | 90d | 30d | 同上 |
| audit_logs | **永久**（PG + OSS 双存） | 1 年（OpenSearch 索引最近 1 年） | 全量，永不删除；按 quarter 在 OSS 加 retention lock |

## 五、与外部数据系统的关系

| 系统 | 关系 |
|------|------|
| L1 / L2 数据层（[11_数据采集与输入层规约](../_共享规约/11_数据采集与输入层规约.md)） | 仅消费"数据质量信号"，不直接写业务表 |
| 推理网关 | 仅消费健康信号 |
| 状态机监控 / 纵深进攻 / 超级个体进化 | 通过 API 接收门禁判定，不直接读极寒防御的表 |
| 监控驾驶舱（前端） | 走 [前端工程与服务](../前端工程与服务/README.md) 的 BFF 层，不直接连库 |

## 六、版本演进策略

- 表结构变更须经 [03_架构设计共识](../_共享规约/03_架构设计共识与协作元规则.md) 的 ADR 与 GitFlow 流程
- 索引变更须 staging 验证 + 灰度
