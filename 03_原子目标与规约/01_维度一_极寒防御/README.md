## 一句话定位

**不可信不放过、坏环境进入冻结、坏决策不出门。**

跨整条研究 → 决策 → 动作链路的"风险解构 + 门禁 + 熔断 + 失败保护"统一抽象。任何模块对外的输出都必须先穿过极寒防御。

## 核心交付物

- 风险事件流（severity / category / affected_subjects / evidence_ref）
- 门禁判定结果（pass / degrade / reject + reason + evidence_ref）
- 熔断状态变更事件
- 审计日志条目

## 后端服务子模块（计划，第 2 批展开）

| 子模块 | 英文键 | 主责 |
|--------|--------|------|
| 风险事件总线 | `risk_event_bus` | 收集 / 分发 / 持久化所有风险事件 |
| 决策门禁器 | `decision_gate` | 多层门禁判定（指标 / 合规 / 可追溯 / 人审） |
| 熔断控制器 | `circuit_breaker` | 异常时降级到冻结 / 只读 / 拒绝出站；管理恢复路径 |
| 输入污染检测器 | `input_sanitizer` | LLM 上下文注入 / 数据漂移 / prompt 投毒检测 |
| 审计日志服务 | `audit_log_service` | 不可篡改的判定记录与回放 |
