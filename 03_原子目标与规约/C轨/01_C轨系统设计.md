## 一、设计目标

各模块按 [05_C轨系统概念与目标](../../01_顶层概念/05_C轨系统概念与目标.md) 设计：时间视野 3/5/10 年、回报目标 10×～100×、全关键商业信息、可追溯可证伪、仅逻辑/大周期退出、与 A/B 轨隔离。

<a id="design-c-track-modules"></a>
## 二、各模块 C 轨设计要点

### 2.1 采集模块

- **数据范围**：C 轨专用清单见 [02_C轨数据与存储规约](02_C轨数据与存储规约.md)；包含细财报、管理层与治理、业务与竞争、研发与创新、行业与政策。
- **存储与契约**：C 轨数据在 L2 有专用表/视图；消费方（C 轨预筛、VC-Agent、逻辑证伪）的读接口在 02_ 约定。

### 2.2 A 模块 C 轨扩展

- **输出**：ClassifierOutput 增加 C 轨用字段：如 `industry`、`sub_industry`、`lifecycle_stage`、`c_track_eligible` 等；准入规则可配置。
- **传递**：B 模块（或 C 轨预筛）从 L2 或约定接口读取；C 轨候选表写入与消费见 2.3。

### 2.3 B 模块（量化扫描）C 轨策略与共享表

- **C 轨专用策略与打分**：长期动量、成长质量分、研发强度分、治理分、潜在空间/赔率估计等；阈值与权重可配置。
- **C 轨专用共享表**：`c_track_candidate_snapshot`（或等价）；仅存放 C 轨策略与打分**通过**的标的。
- **写入方**：B 模块（或 C 轨预筛）；**消费方**：Module C Router/VC-Agent 仅或优先从此表读取 C 轨候选。

### 2.4 C 模块（MoE）VC-Agent 与 C 轨逻辑

- **路由**：请求若 symbol 在 `c_track_candidate_snapshot` 或带 c_track_pass 约定，则**仅或优先**走 VC-Agent。
- **VC-Agent 输入**：symbol、C 轨候选表行、采集层 C 轨数据引用；必选/可选与新鲜度在 02_ 与 03_ 约定。
- **VC-Agent 输出与共识**：ExpertOpinion 含 horizon=**LONG_TERM**、reasoning_summary；**共识**结构化落库，schema 见 [03_C轨共识与逻辑证伪规约](03_C轨共识与逻辑证伪规约.md)。

### 2.5 D 模块（判官）C 轨逻辑

- **三轨分流**：LONG_TERM → 豁免 2% 与现金拖累，仅施加逻辑证伪与大周期反转（09_ 概要，细节在 [03_C轨共识与逻辑证伪规约](03_C轨共识与逻辑证伪规约.md)）。
- **逻辑证伪**：输入 = C 的共识（证伪条件）+ 事件流；输出 = 平仓/减仓指令，原因码 logic_disproof。
- **大周期反转**：输入 = 标的/行业价格与均线、行业指数关键位、波动率 regime 等；输出 = 止盈/减仓信号，原因码 major_trend_reversal。

### 2.6 E 模块（风控）C 轨逻辑

- C 轨仓位**不**触发 2% 硬止损与现金拖累；按 track=C 走 C 轨分支。
- C 轨专用约束：单标的 C 轨仓位上限、C 轨合计敞口占比、再平衡频率等，可配置。
- 可接收 D 的「逻辑证伪/大周期反转」指令，对 C 轨仓位执行或标记，与 F 对接。

### 2.7 F 模块（执行）C 轨逻辑

- 订单与仓位带 `track=C`；C 轨平仓仅根据判官/风控的「逻辑证伪」或「大周期反转」执行；原因可追溯。
