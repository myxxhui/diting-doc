<a id="c-track-data-requirement"></a>
## 一、C 轨数据需求清单

| 类别 | 内容 | 必选/可选 | 说明 |
|------|------|-----------|------|
| **财报与经营** | 利润表/资产负债表/现金流量表科目级，季度/年度；毛利率、净利率、ROE、营收/利润增速序列 | 必选 | 供 VC-Agent 与成长质量分 |
| **管理层与治理** | 高管名单、任职与变动、薪酬与股权激励、实控人、董事会构成 | 必选 | 供治理分与共识 |
| **业务与竞争** | 主营业务构成、产品与客户结构、市占率、竞争格局 | 必选 | 供潜在空间/赔率与共识 |
| **研发与创新** | 研发投入、专利与项目、技术路线或标签 | 必选 | 供研发强度分与共识 |
| **行业与政策** | 行业空间与增速、政策与监管要点、技术替代风险 | 可选 | 供共识与逻辑证伪 |

<a id="c-track-storage"></a>
## 二、存储表与契约

- **C 轨原始/加工数据**：在 L2 设专用表或视图（如 `company_fundamental`、`management`、`c_track_raw_feed`）；消费方为 C 轨预筛、VC-Agent。
- **C 轨候选表**：`c_track_candidate_snapshot`（或等价）；仅存放 C 轨策略与打分**通过**的标的；字段含 symbol、batch_id、long_term_score、各维度分、potential_upside、time_horizon（3Y/5Y/10Y）、industry、lifecycle_stage、created_at。**写入方**：B 模块（或 C 轨预筛）；**消费方**：Module C Router/VC-Agent。
- **共识表**：`c_track_consensus`（或等价）；schema 见 [03_C轨共识与逻辑证伪规约](03_C轨共识与逻辑证伪规约.md)。

<a id="c-track-consumption"></a>
## 三、消费方与读接口

| 消费方 | 读取内容 | 约定 |
|--------|----------|------|
| C 轨预筛 | company_fundamental、management 等，用于 C 轨维度分与通过判断 | 按 symbol 查询 |
| Module C VC-Agent | c_track_candidate_snapshot 行 + 基本面数据引用 | 输入契约见 03_ |
| 逻辑证伪模块 | 共识表（证伪条件）+ 事件流 | 见 03_ |
