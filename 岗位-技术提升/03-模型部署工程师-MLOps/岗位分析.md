# 模型部署工程师（MLOps）· 岗位深度分析

## 一、项目是什么？

从岗位描述可以推断：

### 🏥 业务方向：**医疗 AI 应用**
- 公司性质：**中型医疗服务公司**
- 业务可能包括：医学影像诊断、电子病历 NLP、医疗知识库问答、健康管理 AI、药品研发 AI
- 上海浦东上南的中型医疗公司典型候选：**医联、平安医疗、联影智能、推想医疗、卫宁健康、东软医疗、依图医疗、森亿智能**这一类公司

### 🎯 项目定位：**从 0 到 1 或从 1 到 N 的 MLOps 平台建设**

岗位写"**搭建 CI/CD/CT 流水线、制定技术规范**"——这说明公司**目前还没有或者刚起步的 MLOps 体系**，需要你进来从头搭，或者把既有的"作坊式"模型部署升级成平台化。

典型业务场景：
- 训练几十到几百个医学影像模型（CT / MRI / 病理等）
- 模型需要按科室 / 按医院 / 按数据集分版本迭代
- 医疗合规、HIPAA-like 的数据权限管控是硬性需求
- **数据可能不出公司内网**，训练也要在私有化 / 本地 GPU 集群上跑

## 二、这个岗位在项目中的位置

```
        ┌─────────────────────────┐
        │    算法 / 数据科学团队    │
        │   (Train / Eval 模型)    │
        └────────────┬────────────┘
                     │ 要上线！
                     ▼
     ┌───────────────────────────────────┐
     │  ★ 本岗位：MLOps 平台工程师 ★       │  ← 你在这里
     │                                   │
     │  训练流水线 · 评测平台 · 模型注册  │
     │  部署发布 · 推理服务 · 可观测性    │
     │  特征 / 模型资产 · 权限合规        │
     └─────────┬─────────────┬───────────┘
               │             │
               ▼             ▼
       ┌─────────────┐  ┌──────────────┐
       │ K8s / GPU 集群│  │  业务系统 API │
       │ (Ops 团队合作)│  │  (产品团队)   │
       └─────────────┘  └──────────────┘
```

**位置特点**：
- **"算法团队"与"生产系统"之间的桥梁**
- 承担"模型质量最后一公里"——模型再好，部署不了 / 监控不到都白搭
- 相当于医疗 AI 的**"总装车间"**

## 三、重要程度

**⭐⭐⭐⭐ 核心关键岗（但低于"AI 负责人"和"AI Infra"）**

| 维度 | 分析 |
| --- | --- |
| 薪资 | 6-9 万 × 16 薪 = **96-144 万**，上海 MLOps 岗中等偏上 |
| 经验 | 实际要求 **5 年后端 + 3 年 MLOps** 起，资深岗 |
| 职责清晰度 | 非常清晰的"建平台"定位，**有明确 Owner 属性** |
| 业务影响 | 医疗 AI 的所有模型都要经过这套平台，**影响面 100%** |
| 可替代性 | 中偏低（MLOps 专家在国内仍稀缺） |

在医疗 AI 公司中，这通常是**CTO 下面一级的技术专家岗**，有团队管理机会（2-5 人小团队 Lead）。

## 四、项目技术对未来互联网 AI 技术的锻炼与成长

### 🚀 高价值锻炼，但行业特殊性较强

### 1. 技术锻炼维度 —— ⭐⭐⭐⭐

这个岗位的技术栈是**"大厂 MLOps"的平移**：

| 技术 | 锻炼深度 | 未来价值 |
| --- | --- | --- |
| 端到端 MLOps 平台建设 | ⭐⭐⭐⭐⭐ | 红利期技术 |
| CI / CD / CT 流水线 | ⭐⭐⭐⭐⭐ | 核心能力 |
| Kubernetes 生产实战 | ⭐⭐⭐⭐ | 长期刚需 |
| 分布式训练基础 | ⭐⭐⭐ | 涉及但不深 |
| 模型部署 + 灰度 + 弹性伸缩 | ⭐⭐⭐⭐⭐ | 核心能力 |
| 推理性能优化 | ⭐⭐⭐⭐ | 高价值 |
| 特征 / 数据 / 模型资产管理 | ⭐⭐⭐⭐ | MLOps 核心 |
| 可观测性与稳定性治理 | ⭐⭐⭐⭐ | 运维刚需 |
| 医疗合规 / 权限管控 | ⭐⭐⭐ | 行业壁垒 |

### 2. 与「AI Infra 研发工程师」的对比

```
      AI Infra 研发（大厂）      MLOps 工程师（本岗位）
   ┌────────────────────┐    ┌─────────────────────────┐
   │ 规模：超大            │    │ 规模：中                  │
   │ 场景：通用 AI 平台    │    │ 场景：医疗垂直             │
   │ 新技术：vLLM/LLM推理  │    │ 新技术：更偏传统 ML 流水线  │
   │ GPU 深度：★★★★★        │    │ GPU 深度：★★★             │
   │ 创新空间：大           │    │ 创新空间：中               │
   │ 协作：多业务线         │    │ 协作：单业务线 + 医院      │
   └────────────────────┘    └─────────────────────────┘
```

### 3. 成长天花板

- **3 年后**：MLOps 架构师 / 平台负责人（年薪 150-200 万）
- **5 年后**：医疗 AI 公司技术总监 / CTO-2（年薪 200-300 万）
- **跳槽通道**：
  - 医疗 AI 垂直赛道之间横跳（联影、推想、森亿、平安）
  - 跳去互联网大厂做**通用 MLOps 平台**（阿里云 PAI、华为 ModelArts、字节 BytePS / 火山引擎 AI 平台）
  - 跳去 AI 基础设施创业公司（如 MLOps SaaS）

### 4. 行业特殊性

| 优势 | 劣势 |
| --- | --- |
| 医疗是 AI 应用的"深水区"，需求真实稳定 | 受政策监管强，产品推进慢 |
| 数据敏感，技术壁垒高 | 数据脱敏 / 合规工作占比高 |
| 行业周期稳定，不像 Web3 那样波动 | 相对"互联网大厂"增长慢 |
| 不易被替代（医疗知识积累值钱） | 换赛道时医疗经验"折旧"偏快 |

### 5. 技术栈"互联网可迁移性"评估

| 技术 | 迁移到互联网大厂 | 迁移到 AI 独角兽 |
| --- | --- | --- |
| MLOps 平台建设经验 | 100% 可迁移 | 100% 可迁移 |
| K8s 生产实战 | 100% 可迁移 | 100% 可迁移 |
| 模型服务化 / 灰度发布 | 100% 可迁移 | 100% 可迁移 |
| 医疗合规经验 | 20% 迁移（仅金融 AI 有用） | 30% 迁移 |
| 推理优化 | 100% 可迁移 | 100% 可迁移 |

**可迁移性总评：85%** —— 大部分核心能力通用。

## 五、是否是核心 AI / IT 工程师技术岗？

### ✅ **是。这是典型的"AI 时代高价值 IT 工程师"岗位。**

| 判断标准 | 本岗位表现 |
| --- | --- |
| 是否涉及大模型核心工程？ | ⚠️ 部分涉及（医疗 LLM 可能），但不是主战场 |
| 是否涉及分布式训练核心？ | ✅ 基础要求 |
| 是否涉及云原生核心？ | ✅ Kubernetes 精通 |
| 是否涉及系统编程？ | ✅ Python / Go / Java 其一 |
| 是否决定业务生死？ | ✅ 所有模型上线的咽喉 |
| 是否属于"卡脖子"技术？ | ⚠️ MLOps 人才稀缺，但不算前沿黑科技 |

**核心结论**：
- **硬核程度**：高（MLOps 是工程 + 算法 + 运维三位一体）
- **前沿程度**：中（不像大模型推理那样处于技术爆发期）
- **稳定性**：高（医疗行业抗周期，MLOps 是刚需）
- **适合人群**：
  - 有 5 年以上后端 / 平台开发经验
  - 不想太卷、追求稳定成长
  - 对算法有基本理解，但不想做算法本身
  - 愿意长期深耕 MLOps 方向

## 六、面试 / 备战建议

### 优先突击
1. **MLOps 平台全景认知**：至少啃完一遍 [Google MLOps Whitepaper](https://cloud.google.com/resources/mlops-whitepaper)
2. **开源 MLOps 工具链**：
   - 编排：**Kubeflow / Airflow / Prefect**
   - 实验跟踪：**MLflow / W&B**
   - 模型注册：**MLflow Model Registry / Seldon Core**
   - 特征平台：**Feast / Tecton**
   - 部署：**KServe / Triton / BentoML**
3. **K8s 生产级**：与 01-AI Infra 岗位重合度 80%，重点 HPA / GPU 调度 / 多租户
4. **持续训练（CT）设计**：
   - 数据漂移检测（Data Drift / Concept Drift）
   - 自动触发重训练
   - 模型质量门禁
5. **医疗合规知识加分**：
   - HIPAA / 中国医疗数据安全法
   - 联邦学习（Federated Learning）基础

### 简历关键词（必须出现）
- "从 0 到 1 搭建 MLOps 平台，覆盖 XX 个模型 / XX 个团队"
- "CI/CD/CT 流水线建设，模型上线周期从 X 周缩短到 X 天"
- "模型推理 P99 延迟优化 XX%，成本下降 XX%"
- "灰度发布 / 回滚机制建设"
- "特征平台支撑 XX 亿特征 / 天"
- "精通 Kubernetes，管理 XX GPU / CPU 集群"

### 面试重点准备
- **平台设计题**："从零设计一个医学影像 MLOps 平台，给出架构图"
- **运维实战题**："生产环境模型准确率突然下降，排查链路是什么？"
- **合规题**："医疗数据跨院训练怎么保证隐私？"
- **选型题**："Kubeflow vs Airflow，你会选哪个？为什么？"
- **规模题**："100 GB 模型如何做灰度发布？"
