# AI 技术负责人 · 岗位深度分析

## 一、项目是什么？

从职责中密集出现的**"市场解读、信号提炼、链上、新闻、告警智能化、研究工作流自动化"**等词，业务形态指向非常清晰：

### 🎯 最可能的业务形态：**Web3 / 加密量化 / AI 投研平台**

具体可能是：

| 候选类型 | 匹配度 | 依据 |
| --- | --- | --- |
| 加密货币资讯 / 链上分析平台 | ⭐⭐⭐⭐⭐ | "链上""市场解读""信号提炼"——几乎是 Web3 研究平台的原话 |
| AI 驱动的量化基金 / 对冲基金 | ⭐⭐⭐⭐ | "告警智能化""信号提炼" |
| 金融投研 SaaS / 券商研究端 AI | ⭐⭐⭐ | "研究工作流自动化""新闻" |
| 证券 / 资管机构 AI 实验室 | ⭐⭐ | 但不会写"链上" |

**最终判断：这是一家 Web3 / 加密金融类的创业公司或中型 AI 独角兽**，团队规模小但含金量高，base 地全球远程——典型的 Web3 创业公司配置。

## 二、这个岗位在项目中的位置

```
        ┌─────────────────────────┐
        │      CEO / 创始团队       │
        └───────────┬─────────────┘
                    │
        ┌───────────┼─────────────┐
        │           │             │
   ┌────▼───┐  ┌────▼───┐    ┌────▼────┐
   │ 产品 PM │  │  数据  │    │  工程   │
   └────┬───┘  └────┬───┘    └────┬────┘
        │           │             │
        └──────┬────┴─────────────┘
               ▼
      ┌───────────────────┐
      │ ★ AI 技术负责人 ★  │  ← 你在这里
      │  (Player-Coach)    │
      └─────────┬─────────┘
                │
      ┌─────────┼──────────┐
      ▼         ▼          ▼
   AI 算法   AI 工程    Prompt/Agent
   2-5 人    2-4 人     1-2 人
```

**位置特点**：
- **公司 AI 能力的"一号位"**——技术方向 + 产品方向双向决策权
- **Player-Coach 模式**：一半时间亲自撸代码 / 调 prompt / 设计 Agent，一半时间带团队、定标准
- 直接对 CEO / CTO 汇报，话语权极大
- 团队规模通常不大（5-15 人），但是**整个业务的核心引擎**

## 三、重要程度

**⭐⭐⭐⭐⭐ 公司级核心岗位（Founding Engineer / AI Head 级别）**

| 维度 | 分析 |
| --- | --- |
| 岗位定性 | **负责人级别**，不是"工程师" |
| 决策权 | "负责 AI 方向规划与落地"——**有战略决策权** |
| 对业务的影响 | 如果是加密 AI 产品，**AI 就是产品本身**——AI 做好公司活，AI 做不好公司死 |
| 跨职能 | 跨产品、设计、数据、工程——是**T 字型最顶上的横线** |
| 招聘难度 | 3 年经验 + AI 大厂核心经验 + 产品 sense + 中英文——极稀缺 |

在创业公司 / 独角兽的组织结构中，这通常是**前 10 号员工**级别的岗位，**不只是打工，是事业合伙**。

## 四、项目技术对未来互联网 AI 技术的锻炼与成长

### 🚀 成长曲线：最陡的赛道之一，但强度也最高

### 1. 技术锻炼维度 —— ⭐⭐⭐⭐⭐

这个岗位**覆盖 LLM 应用层的全部核心技术**，且全部是**当下最热、未来 5 年最稀缺**的技术：

| 技术 | 未来价值 | 本岗位覆盖度 |
| --- | --- | --- |
| LLM 应用架构 | 未来 10 年互联网的"新 iOS" | 100% |
| Prompt Engineering / Workflow | 当代软件工程学新范式 | 100% |
| RAG / Retrieval | 企业 AI 落地的"水电煤" | 100% |
| Agent / Tool Use / MCP | **2026-2028 爆发期** | 100% |
| Structured Output | 企业应用 LLM 的底层能力 | 100% |
| AI 评测体系（Eval） | AI 时代的"测试驱动开发"新形态 | 100% |
| 生产级 LLM 监控 | LLMOps 核心能力 | 100% |

### 2. 与「AI Infra」岗位的对比

```
      AI Infra 工程师              AI 技术负责人（本岗位）
   ┌────────────────────┐       ┌─────────────────────────┐
   │  底层 · 硬核 · 稳健  │       │  应用层 · 灵活 · 创造   │
   │                    │       │                         │
   │  K8s / GPU / 推理   │       │  LLM / Agent / RAG      │
   │  性能优化 / 成本    │       │  产品落地 / 业务抽象     │
   │                    │       │                         │
   │  10 年后仍稀缺      │       │  3 年内爆发式增长        │

   └────────────────────┘       └─────────────────────────┘
```

本岗位更偏**"应用 AI"和"产品化 AI"**，是离用户价值最近的岗位。

### 3. 成长天花板与回报

- **路径 A（企业通道）**：技术负责人 → CTO / VP of Engineering（年薪 300-600 万+）
- **路径 B（创业通道）**：公司早期员工 → 如果产品做成独角兽，**股权收益可能上千万**
- **路径 C（跳槽通道）**：Web3 / 金融 AI 的专家型人才极其稀缺，**横向跳槽议价能力极强**
- **路径 D（自己创业）**：这个岗位本身就是为创业做准备——干两三年直接可以 0 到 1 开始自己的 AI 产品

### 4. 技能复利

- **产品 sense**：对 C 端 / B 端 AI 产品的真实认知（很多工程师一辈子都不会有）
- **业务抽象能力**：能把业务翻译成 Prompt / Workflow / Agent
- **管理能力**：Player-coach 经历是从工程师转向技术管理的"黄金过渡"
- **英语能力**：Web3 行业基本英文办公——自动打通海外跳槽通道

### 5. 风险 / 不足

| 风险 | 说明 |
| --- | --- |
| 行业波动性 | Web3 / 加密金融受市场周期影响极大，公司倒闭率较高 |
| 技术半衰期短 | LLM 应用层迭代极快（3 个月换一代），焦虑感强 |
| 技术底层偏浅 | 不会深入 K8s / GPU / CUDA，**长期做会失去系统工程能力** |
| 工作强度 | 创业公司节奏，player-coach 意味着又要写代码又要管人 |
| 难"躺平" | 不像大厂有流程保护，这种岗位出了问题是你负责 |

## 五、是否是核心 AI / IT 工程师技术岗？

### ⚠️ **部分是。这是"AI 应用层 + 技术管理"交叉岗位，不是纯硬核 IT 工程岗。**

| 判断标准 | 本岗位表现 |
| --- | --- |
| 是否涉及大模型核心工程？ | ✅ LLM / Agent / RAG 全覆盖（应用层） |
| 是否涉及分布式训练？ | ❌ 基本不涉及（使用 API / 微调） |
| 是否涉及 GPU / 算力 / K8s？ | ❌ 基本不涉及 |
| 是否涉及系统编程？ | ⚠️ 部分涉及（工程能力要求，但不是 C++/Rust 级） |
| 是否属于 "AI 产品" 核心？ | ✅ 是产品的灵魂 |
| 是否需要"做管理"？ | ✅ Player-Coach，约 30-50% 时间在带人 |

**核心结论**：
- 这个岗位属于 **AI 应用 + 产品 + 管理**的复合岗位
- 技术深度：**中（偏应用）**
- 技术宽度：**极高（产品+技术+管理三维）**
- 性价比：**适合已经有 3-5 年工程经验，想转 AI 产品方向 / 想为创业做准备的人**
- **不适合的人**：希望深耕底层系统、追求纯技术深度、不想做管理的纯工程师

## 六、面试 / 备战建议

### 优先突击
1. **Prompt Engineering 实战**：Few-shot / CoT / Self-consistency / Prompt 评测
2. **RAG 生产级实战**：Embedding 选型、Chunking 策略、Hybrid Search（BM25 + Vector）、Rerank
3. **Agent 框架**：LangGraph / LlamaIndex / AutoGen / 原生 Agent 设计模式
4. **Structured Output**：Function Calling、JSON Schema、Pydantic AI、Instructor
5. **Eval 体系**：离线评测（Ragas、DeepEval）、在线 A/B、Human-in-the-loop
6. **业务 sense**：读 AI 产品论坛（Hacker News、LangChain Blog、OpenAI Dev Day）

### 简历关键词（必须出现）
- "从 0 到 1 落地 AI 产品 XXX，月活 / 调用量 XX"
- "设计 Agent Workflow 将 XXX 业务流程自动化 XX%"
- "建立 AI 评测体系，模型准确率从 X% 提升到 Y%"
- "Player-coach 带领 X 人团队"
- "主导 LLM + RAG 方案，成本下降 XX%"

### 面试重点准备
- **业务抽象题**："你会怎么设计一个链上异常检测的 Agent？"
- **架构题**："RAG 检索质量差，你会怎么排查？从 Embedding 到 Rerank 全链路"
- **管理题**："Player-coach 怎么平衡自己写代码和带团队？"
- **产品 sense 题**："如何判断一个 AI 功能值得上线？评估指标有哪些？"
