## 一、核心理念

谛听不是一个单纯的交易系统，也不是一个脱离业务的练手机器。  
它是一个**双目标系统**：

- 对外，是一个 **A 股分析追踪智能平台**
- 对内，是一个 **AI Infra / Agent Infra / MLOps / AI 产品化训练场**

## 三、L1：顶层概念

### 1. 一句话定义与核心价值

定义项目到底是什么，以及为什么这个项目必须同时服务产品价值与职业价值。

### 2. 战略目标与 ROI

把项目回报拆成三类：

- 产品 ROI
- 能力 ROI
- 资产 ROI

### 3. 双目标系统与架构分层

给出统一的五层架构，把岗位 1/4/3/2 映射到系统不同层级。

### 4. 产品目标系统

定义平台对外到底要解决什么问题：

- 防雷
- 预期差挖掘
- 逻辑跟踪
- 自进化

### 5. 个人能力训练系统

定义副业对你本人要产生什么工程训练价值，尤其聚焦岗位 1 与岗位 4。

## 四、L2：战略维度

L2 把 L1 的双目标拆成可以执行的维度。

### 产品设计轴线

回答：

- 做什么
- 为什么现在做
- 什么先做、什么后做

### 开发与交付轴线

回答：

- 按什么顺序做
- 什么时候引入更重的复杂度
- 如何在 8 个月副业窗口内推进

### 其余产品设计维度

作为支撑维度继续存在：

- 技术栈与架构
- 数据架构
- 生产保障与可观测性
- 安全与机密治理
- 研产同构
- 成本治理
- 平台接入与隔离边界

## 五、五层运行时架构

### Layer 5：产品交互与跟踪

- 候选池
- 研究卡片
- thesis 跟踪
- 退出提醒

### Layer 4：Agent 编排与研究工作流

- 财报 Agent
- 新闻 Agent
- 产业链 Agent
- RAG 与证据引用
- 结构化结论生成

### Layer 3：Eval / CT / 发布闭环

- 标注集
- 评测集
- Prompt / Workflow 回归
- 灰度发布与回滚

### Layer 2：Agent Runtime / 沙箱 / 多租户

- 隔离执行
- 长任务管理
- 会话级状态
- 资源配额
- 运行时观测

### Layer 1：推理与基础设施

- 自托管推理
- `vLLM`
- `Kubernetes`
- 资源治理
- FinOps

## 六、岗位映射

| 岗位 | 系统主战场 | 在项目中的体现 |
|---|---|---|
| 岗位 1：AI 基础设施研发 | Layer 1 | 自托管推理、K8s、观测、成本治理 |
| 岗位 4：智能体基础设施工程师 | Layer 2 | 沙箱、长任务、多租户、隔离、安全 |
| 岗位 3：MLOps | Layer 3 | Eval、版本、灰度、回滚、CT |
| 岗位 2：AI 技术负责人 | Layer 4/5 | 业务抽象、Agent 工作流、产品取舍 |
