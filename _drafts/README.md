# `_drafts/` · 临时草稿目录（生命周期严格管控）

> **本目录定位**：跨会话规划任务（如 Lighthouse-Alpha 嗅探层方案、四阶段提问体系升级等）的**中间产物落盘地**，用于**控制 Cursor / Opus 上下文成本**。
> **绝不**作为正式规约引用；**绝不**让 L3/L4 文档反向 link 到本目录；**禁止**长期存放（任务结束即清空）。

---

## 一、为什么存在这个目录

某些规划任务（如新增跨 6 维度复合能力、升级标准提问体系）需要**多轮 AI 会话**串联：

| 风险 | 后果 |
|---|---|
| 单会话上下文 > 200K | Opus 触发 1.5x~2x 倍率计费 |
| 把所有现有 step / 设计粘进 prompt | 上下文爆炸 + AI 无法精准定位 |
| 中间结论只在对话里 | 下个会话必须从头再来，浪费 token |

**解决方案**：每个会话产出"一份精简的草稿 md"，下个会话用 `@_drafts/<file>.md` 引用即可，**不要在对话里复述**。

---

## 二、生命周期（严格四态）

| 状态 | 含义 | 允许动作 |
|---|---|---|
| **draft（草稿）** | 当前会话刚产出 | 仅本任务串联使用；不得被 03_/04_/05_ 反向引用 |
| **review（待评审）** | 任务进入到合并前的最后一会话（如 S4）| 由 AI 在阶段 4 待办清单中列出"_drafts/X.md → 合并到 04_/00_..."的迁移操作 |
| **merged（已合并）** | 草稿已合并到正式路径 | 立刻 **删除** `_drafts/` 下对应文件 |
| **abandoned（放弃）** | 任务终止 / 方案被否 | 立刻 **删除** |

> **强制规则**：任务全部结束后 `_drafts/` 应**为空**或仅含本 README；非空且非当前活跃任务的草稿视为遗留垃圾，须清理。

---

## 三、命名约定

```
_drafts/
├── README.md                                    # 本文件（永久保留）
├── <任务标识>_<阶段标识>_草稿.md                  # 阶段产物
└── <任务标识>_PRD.md                            # 用户原始需求落盘（可选）
```

**示例**：
- `00_四阶段提问体系_v2_草稿.md` — 标准升级任务的 S1 产物
- `lighthouse_alpha_PRD.md` — 用户原始 PRD（避免在对话里反复粘贴）
- `lighthouse_alpha_stage_0_1.md` — Lighthouse 任务的阶段 0+1 产物
- `lighthouse_alpha_stage_2_3.md` — 阶段 2+3 产物
- `lighthouse_alpha_stage_4_待办.md` — 阶段 4 待办清单
