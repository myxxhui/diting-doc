## 一、背景：为什么推翻重做

step_17 完成后用户反馈「**前端各工作区的规划联动烂透了，后端实现也烂透了**」。诊断根因：

| 症状 | 根因 |
|---|---|
| 同一标的（如 601138）满屏重复 | 雷达每次 `promote` **新建 Campaign**，流转单元是 Campaign 而非标的 |
| 路线图自咬撞车（601138 与 601138「窗口重叠/仓位合计 100%」） | 同标的多条时间线两两比较，跟自己撞 |
| 四 Tab 像脏库过滤器，不反映真实状态 | 视图按 Campaign.status 文本过滤，`funnel_stage` 形同虚设 |
| 规划→执行无晋级入口 | 缺标的级晋级按钮 |

**用户决策**：方案A · 标的级漏斗（一个标的 = 一条贯穿四区的 funnel 记录）+ 清空重建生产库（保留 holdings_sot）。

## 二、实际进展（已核验）

### 2.1 数据模型（标的全局唯一 + 单一状态机）

- `CampaignSymbol` 新增 `funnel_stage`（`radar_intake|roadmap|planning|executing|archived`，默认 `planning`，建索引）+ `updated_at`。
- 唯一约束从 `(campaign_id, symbol)` 改为 **`symbol` 全局唯一**（`uq_funnel_symbol`）。
- 迁移 `db/migrate_step18.py`：ALTER 加列 + 回填 + **容错唯一索引**（存量重复时跳过，清空后重建，应用层 find-or-create 兜底）。接入 `db/database.init_db`。

### 2.2 漏斗中枢 `modules/planning/funnel.py`（新建）

| 函数 | 职责 |
|---|---|
| `get_or_create_container` | 唯一容器 Campaign（`theme=行情解析漏斗`） |
| `upsert_funnel_symbol` | 按 symbol 全局 find-or-create + **前向单向**推进 stage |
| `set_stage(allow_backward)` | 推进/回退（归档回流路线图用） |
| `list_funnel_symbols` / `VIEW_STAGES` | 四区按 stage 过滤（radar/planning/executing/archived 互斥） |

### 2.3 服务层（四区改读 funnel_stage · 标的级晋级/归档）

- `radar/service.promote_candidate`：**不再新建 Campaign**，改 `upsert_funnel_symbol(stage=planning)` + 三支柱 + 证伪任务挂容器。
- `radar/service.list_recent_candidates`（新建）：雷达区=扫描候选工作台（按 symbol 去重 + `already_promoted` 标记）。
- `planning/service.import_portfolio_to_campaign`：持仓导入唯一容器，stage=planning，**按 symbol 去重不翻倍**。
- `planning/service.list_workspace_symbols`：四区视图按 `funnel_stage` 过滤。
- `planning/service.promote_campaign_to_executing`：晋级作用于**单只标的**（带 `symbol`）或批量 planning/roadmap。
- `roadmap/service`：时间线编排推进标的→`roadmap`；`archive_campaign_rolling` 标的级——了结→`archived`，`long_multiwave`→回流 `roadmap` + `next_wave_window`，**绝不归档整个容器**。

### 2.4 路由 / 前端（四区联动 + 晋级按钮）

- `/api/campaigns?view=planning|executing` → 渲染**标的卡**（含阶段 chip、证伪面板内联展开、**人工确认·晋级执行**按钮 / 生成仓位建议 + 归档）。
- `/api/radar/symbols` → 扫描候选卡（**晋级到规划区**按钮，HTMX 返回确认片段）。
- `promote / promote-executing / archive` 端点接 `symbol` 参数 + HTMX HTML 片段回执。
- `workbench.html`：雷达列表更名「扫描候选（待晋级）」；移除冗余的 ⑨/⑩ 独立面板（已内联到标的卡）。
