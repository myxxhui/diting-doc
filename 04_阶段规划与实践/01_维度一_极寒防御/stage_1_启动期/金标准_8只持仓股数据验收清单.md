## §5 金标准修复 SoP（数据未达标时的标准流程）

> **触发条件**：§3 任一条 SQL 不达预期 / `validate_quality_matrix.py` 退出码 ≠ 0。
> **修复禁令**：不得伪造数据；不得跳过本 SoP 直接改阈值（除非用户书面同意并在 L3 同步修订）。

| 不达标项 | 排查方向 | 修复脚本 / 命令 | 修复后验证 |
|---|---|---|---|
| `financial_reports` 不足 | akshare 限流 / 网络断 | `CRYO_THROTTLE_SEC=1.0 make cryo-step02-collect` 重跑 | 重跑 §3.1 第 1/2 行 |
| `industry` null | akshare API 故障 | `crawl_industry_category.py` 含 segment 兜底（基于 `my_holdings.yaml` 的 segment 字段映射） | `SELECT industry, COUNT(*) FROM financial_reports GROUP BY industry;` |
| `roe` null | 旧数据无 ROE 计算 | 一次性 SQL：`UPDATE financial_reports SET roe = net_profit*1.0/NULLIF(total_assets-total_liabilities,0) WHERE roe IS NULL AND net_profit IS NOT NULL AND total_assets IS NOT NULL AND total_liabilities IS NOT NULL;` | 重跑 §3.1 第 4 行 |
| 公告 < 8 类 | 关键词缺失 | 编辑 `crawl_announcements.py` 的 `_classify_ann_type` 增补关键词 → 重跑 `crawl_announcements.py` | 重跑 §3.2 |
| `related_party_raw` 噪音 > 55% | OCR 误捞元数据行 | 重跑 `clean_related_party_noise.py`；如逻辑不合适修改 `_classify_noise` | 重跑 N1 行 |
| `pricing_method` 有效行覆盖 < 50% | regex 关键词不全 / 兜底未生效 | 重跑 `backfill_pricing_method.py`（Phase A 正则 + Phase B 兜底） | 重跑 §3.3 第 3 行 |
| `related_party_graph` < 100 | 未跑 / 关联方关键词太窄 | `python training/scripts/build_related_party_graph.py`；如需扩关键词修改 `_ENTITY_KEYWORDS` | 重跑 §3.4 |

> **同一问题修复重试 ≥ 2 次仍失败** → 按 `00_系统规则` §8.4f 停止 + L4 实践记录写明原因 + 通知架构师。

## §6 金标准对低/中模型的执行约束（防跑题清单）

低/中模型在执行后续数据任务时，**必须**：

1. **不修改本文件 §1 标的清单**：替换标的须修改 `my_holdings.yaml`，再重跑 §3，**不**直接改本文件表格；
2. **不修改 §2 金标准门槛**：扩容 / 降级须**先在 L3 step_02 §5.3 修订并提交**，再回写本文件；
3. **不臆造数据**：所有"非 null 率"必须基于真实 SQL 查询（§3 已列），**禁止**估算；
4. **不跳过 §3.5 矩阵**：任何"数据采集已完成"宣言必须附带 `validate_quality_matrix.py` 退出码 0 的输出；
5. **不破坏 Holdout 隔离**：Holdout 50 案例与本金标准 8 只持仓**互斥**，扩量时须 `holdout_guard.py --check-training-data <jsonl>` 排除 Holdout symbol；
6. **不在金标准外做"顺手扩展"**：扩到 30/100 只属 stage_2 任务，**不在本 step 完成**；扩展时严格按 stage_2 step_01 文档执行；
7. **遇异常先 Verify First**（按 `.cursorrules` §7.2 第 6 条）：不臆想原因；先逐项排查实际 DB 状态、配置、API 返回，再下结论；
8. **修改后必须重跑 §3 全部 SQL**：不允许只跑改动相关的部分；
9. **同会话内必须把验证输出贴到 L4 实践记录**（按 `.cursorrules` §7.2 第 10 条 同会话最小验证）。
