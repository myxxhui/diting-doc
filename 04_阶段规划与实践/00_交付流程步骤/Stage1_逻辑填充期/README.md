## 阶段目标

在骨架上按 L3 规约与 5D 填充业务逻辑；本地单测与必要集成测试可绿，为 Docker 统一环境期提供可测代码。

## 准入条件

- Stage0 骨架期已准出：核心接口存在、本地单测通过、目录与占位可运行

## 准出条件

- 业务逻辑按 L3 规约与 5D 填充完毕；本地单测 + 必要集成测试通过（详细准出清单见 [01_本阶段实践与验证](01_本阶段实践与验证.md)#验证与准出）。

## 交付物

- Module A/B/D/E 等逻辑实现；单测+集成测试通过

## 关键最佳实践目标

- 按 L3 规约与 5D（Design-Drive-Decompose-Defense）填充业务逻辑；逻辑密集处使用 Table-Driven Tests 与边界覆盖。
- 本地单测 + 必要集成测试（关键路径与边界）全部通过；与 03_架构设计共识 测试要求一致。
- 具体「做哪些模块」由 01_需求与产品范围与各 Phase 功能步骤定义；本阶段只约定环境与准出。

## AI 实践最佳（性价比）推荐模型

| 用途 | 推荐模型或档次 | 理由 |
|------|----------------|------|
| 设计/架构与 L3 对齐 | Claude 3.5 Sonnet 或 DeepSeek-R1（待项目选定：架构师 Phase0 前） | 业务逻辑与规约、边界条件需长上下文与多轮澄清 |
| 代码生成（逻辑与单测） | DeepSeek-Coder 或 Cursor 默认模型；公式/边界可用 Claude 3.5（待项目选定） | 逻辑密集处需边界与公式正确，可分层使用 |
| 评审/Defense | 人工为主；辅助可用 Claude 3.5 或 GPT-4o 做静态/测试覆盖检查 | 准出前人工确认 5D 与边界覆盖 |
