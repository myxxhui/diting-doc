## 阶段目标

形成可运行的代码骨架（目录、接口、占位实现），本地单测可绿；为逻辑填充期提供可依赖的接口与目录结构。

## 准入条件

- 项目已具备 L3 规约与仓库结构（见 [02_三位一体仓库规约](../../../03_原子目标与规约/02_三位一体仓库规约.md)）。**若尚未具备，请先执行 [Stage0_pre_仓库与L3就绪](../Stage0_pre_仓库与L3就绪/README.md) 或 [Phase0_Infra](../../Phase0_Infra/) 的 01_。**

## 准出条件

- 核心接口存在且本地单测通过；目录与占位实现可运行（详细准出清单见 [01_本阶段实践与验证](01_本阶段实践与验证.md)#验证与准出）。

## 交付物

- 目录结构、核心接口、占位实现；Table-Driven Tests 通过

## 关键最佳实践目标

- 按 02_三位一体仓库规约与 09_核心模块架构规约建立目录与占位，不提前实现业务逻辑。
- 核心接口（Protocol/抽象类）与 04_全链路通信协议矩阵、05_接口抽象层规约对齐。
- 占位实现使「导入 + 最小调用」可运行；Table-Driven Tests 覆盖边界，本地全绿。

## AI 实践最佳（性价比）推荐模型

| 用途 | 推荐模型或档次 | 理由 |
|------|----------------|------|
| 设计/架构讨论 | Claude 3.5 Sonnet 或 DeepSeek-R1（待项目选定：架构师 Phase0 前） | 接口与目录设计需与 L3 规约对齐，适合长上下文与多轮澄清 |
| 代码生成（占位与单测） | DeepSeek-Coder 或 Cursor 默认模型（待项目选定：架构师 Phase0 前） | 骨架期以占位与 Table-Driven 为主，优先性价比 |
| 评审/Defense | 人工为主；辅助可用 Claude 3.5 或 GPT-4o | 准出前人工确认边界覆盖与 01 规约一致 |
