- **Context**：上述必读 YAML 与 L3；禁止明文密钥。
- **Task**：按 delivery_scope 与 exit_criteria 完成 make build、make test-docker 通过，镜像可构建。
- **Constraint**：工作目录 diting-core（可选 diting-infra）；涉及镜像 tag 时见 02_ 或 DNA 版本约定。

```
你是在 diting-core（可选 diting-infra）中执行 Stage2（Docker 统一环境期）的开发者。请先阅读以下内容再动手：
- 必读：dna_dev_workflow.workflow_stages[s2]、global_const.trinity_repos.repo_i.make_targets（见 03_原子目标与规约/_System_DNA/）；

- 禁止在 Prompt 或 01_ 中写明文密钥。

你的任务：
1. 编写 Dockerfile（及可选 docker-compose）覆盖开发与测试所需运行时与依赖（按 01 技术栈与 02 数据架构）。
2. 在容器内执行全部单元测试与集成测试，确保与本地结果一致。
3. 完成 make build、make test-docker 通过；镜像可构建。
4. 文档化构建与运行命令，使新成员或 CI 可一键复现环境。

约束：工作目录 diting-core（可选 diting-infra）；镜像 tag 见 02_ 或 DNA 版本约定。
请输出：Dockerfile 摘要、make build / make test-docker 执行结果（退出码 0）、镜像 tag 或可复现信息。
```

## 验证与准出 <a id="验证与准出"></a>

工作目录：**diting-core** 根目录。与 DNA `workflow_stages[s2].verification_commands` 及代码仓 Makefile/CI 一致（代码仓变更须同步本文档）。

| 命令/检查 | 工作目录 | 期望结果 | 失败时 |
|-----------|----------|----------|--------|
| `make build` | diting-core 根目录 | 退出码 0；镜像构建成功 | 检查 Dockerfile 与依赖，见「本步骤失败时」 |
| `make test-docker` | diting-core 根目录 | 退出码 0；容器内单测+集成通过 | 在容器内运行单测定位失败用例 |

闭环点：单次 make build + make test-docker 通过（[03_工作流详细规划](../../../03_原子目标与规约/开发与交付/03_项目全功能开发测试实践工作流详细规划.md) 第六节）。

**准出检查清单**（与 01 规约阶段 2 准出条件逐条对应；全部勾选后准出）：

- [ ] 所有开发与测试可在容器内完成；Dockerfile/docker-compose 可复现环境
- [ ] `make build`、`make test-docker` 通过
- [ ] 满足 01 规约「阶段 2：Docker 统一环境期」准出条件
- [ ] **已更新 L5 [02_验收标准](../../../05_成功标识与验证/02_验收标准.md) 中本 stage（s2）对应行（状态+验证方式）**

验收完成后再进入 Stage3。

## 本步骤失败时

- **回退**：回退到本阶段第一步或 Stage1/Stage1b 最后一步。
- **重试上限**：建议 3 次；不通过则回退或暂停待人工排查。
- **临时跳过某验证项**：须经架构师或项目负责人审批并更新 L3/DNA 与本文档。
- **验证失败时**：除回退/重试外，须输出**失败项**（未通过的命令/检查点）及**建议修正动作**（如 make build 失败 → 检查 Dockerfile 与依赖；make test-docker 失败 → 在容器内运行单测定位）；若由 CI 执行，须将失败与 01_ 对应步骤链接。自动反馈修正分级见 [03_项目全功能开发测试实践工作流详细规划 八、失败与回退策略](../../../03_原子目标与规约/开发与交付/03_项目全功能开发测试实践工作流详细规划.md#八失败与回退策略)。
- 具体回退操作与检查点定义见 [03_项目全功能开发测试实践工作流详细规划 八、失败与回退策略](../../../03_原子目标与规约/开发与交付/03_项目全功能开发测试实践工作流详细规划.md#八失败与回退策略)。
