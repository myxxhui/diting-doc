## 阶段说明

L4 负责把 L3 规约落成可执行步骤。执行顺序以 [dna_dev_workflow.yaml](../03_原子目标与规约/_System_DNA/dna_dev_workflow.yaml) 为准。

## 五个阶段

| 阶段 | 对应目录 | 目标 |
|------|----------|------|
| **Stage1** | [Stage1_仓库与骨架](Stage1_仓库与骨架/README.md) | 仓库、接口、协议、模板和基础入口统一 |
| **Stage2** | [Stage2_数据采集与存储](Stage2_数据采集与存储/README.md) | 数据底座、采集链路、容器化与部署验证 |
| **Stage3** | [Stage3_模块实践](Stage3_模块实践/README.md) | 内容理解、信号引擎、研究议会、决策主链 |
| **Stage4** | [Stage4_MoE与执行网关](Stage4_MoE与执行网关/README.md) | 评测/仿真、外部动作边界、发布门禁 |
| **Stage5** | [Stage5_优化与扩展](Stage5_优化与扩展/README.md) | 可观测性、成本、稳定性、复盘与优化 |

## 执行要求

- 每个阶段都必须同时产出产品证据和成长证据。
- 每个步骤都必须回写 L5 验收与 L6 追溯。
- 历史 `A轨/B轨/C轨` 叙事已收口，不再作为 L4 的阶段组织方式。
