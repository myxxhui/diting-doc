## 生产数据环境：前半部分（环境与数据继承）

**目标**：在 diting-infra 执行 `make deploy diting prod` 后，得到竞价实例 + 独立数据盘 + K3s + TimescaleDB/L2/Redis，NodePort+公网 IP 暴露，连接信息写入 `prod.conn`；执行 `make down diting prod` **回收资源且磁盘数据保留**；再次执行 Up 时新起竞价实例挂载同一块数据盘，K3s 与数据库继续使用该盘上的数据，实现数据继承。部署哪些服务（PG/Redis 等）及 K3s 存储、PG/Redis 存储配置由 **config/diting-prod.yaml** 控制。设计细节与 C1/C2 约定见 [06_生产级数据要求_设计#生产数据环境与两段实现](../../03_原子目标与规约/Stage2_数据采集与存储/06_生产级数据要求_设计.md#design-stage2-06-prod-data-env)。

**前半部分·实际验证流程**（部署 → 验证 → 回收且保留磁盘）：

| 阶段 | 工作目录 | 命令 | 说明 |
|------|----------|------|------|
| **部署** | diting-infra | `make deploy diting prod` | 创建竞价实例 + 独立数据盘 + K3s + DB；产出 `prod.conn`、`prod.disk_id` |
| **验证** | diting-core | `cp <diting-infra>/prod.conn .env` 后执行 `make verify diting prod` | 验证 L1/L2 连通及 **Redis 连通与缓存/队列效果性能测试**（见 11_ 规约） |
| **回收（磁盘保留）** | diting-infra | `make down diting prod` | 仅回收 ECS/K3s/EIP，**磁盘数据保留**；`prod.disk_id` 保留供再次 Up 挂载同盘 |

**说明**：prod 使用**独立 NAS**（config/terraform-diting-prod.tfvars 中不设或 `nas_use_existing_access_group = false`），与 dev 隔离。`FULL_DESTROY=1 make down diting prod` 会完整销毁 prod 资源（含 prod NAS）。若 state 中仍残留 dev 共享的 NAS 访问组（diting_nas_group_dev），Make 会在 destroy 前自动从 state 移除，避免 `InvalidAccessGroup.AlreadyAttached`。

**前半可执行步骤（按序执行）**：

| 步骤 | 工作目录 | 命令/操作 | 说明 |
|------|----------|-----------|------|
| 1 | **diting-infra** | `make deploy diting prod` | 创建竞价实例 + 独立数据盘 + K3s + DB；连接信息写入 `prod.conn`，数据盘 ID 写入 `prod.disk_id` |
| 2 | **diting-core** | 将 `diting-infra/prod.conn` 中 DSN 写入 `.env`：`cp /path/to/diting-infra/prod.conn .env`（须含 `TIMESCALE_DSN`、`PG_L2_DSN`、`REDIS_URL`） | 供 `make verify diting prod`（L1/L2/Redis 必验+Redis 性能测试）/ ingest-test 使用 |
| 3 | **diting-core** | `make verify diting prod` | 验证实际实现：L1/L2 连通（前半验证 ①） |
| 4 | **diting-infra** | `make down diting prod` | 回收资源且**磁盘数据保留**；`prod.disk_id` 保留（前半验证 ②） |
| 5 | 云控制台或再次 Up | 可选：在云控制台确认数据盘仍存在；或直接再次 Up | 前半验证 ② 可选确认 |
| 6 | **diting-infra** | `make deploy diting prod` | 再次 Up 时自动读取 `prod.disk_id` 挂载同盘 |
| 7 | **diting-core** | 再次复制 `prod.conn` → `.env` 后执行 `make verify diting prod` 或 psql 查表 | 前半验证 ③：数据继承成立 |

**前半核心验证（三项）**：

| 序号 | 验证项 | 命令/方式 | 期望 |
|------|--------|-----------|------|
| ① | 环境可用、连接正确 | Up 后，在 diting-core 使用 `prod.conn` 中的 DSN 配置 `.env`，执行 `make verify diting prod` | 退出码 0，能连通 L1/L2 |
| ② | Down 保留盘 | 执行 `make down diting prod` 后，在云控制台或约定方式确认数据盘未删除；或再次 Up 时可挂载同盘 | 数据盘仍存在 |
| ③ | 再次 Up 数据可访问一致 | 再次执行 `make deploy diting prod` 后，通过 psql 或约定查询确认 L1/L2 表数据与 Down 前一致（如行数、样例） | 数据继承成立 |

**前半部分·目标测试步骤**（功能/验收测试，与上表三项一一对应）：

| 测试项 | 目标 | 执行步骤（可复制执行） | 通过标准 |
|--------|------|------------------------|----------|
| **T1 环境与连接** | 验证 Up 后环境可用、DSN 正确 | 1）diting-infra：`make deploy diting prod`<br>2）diting-core：`cp <diting-infra>/prod.conn .env`<br>3）diting-core：`make verify diting prod` | 退出码 0；终端输出含 L1/L2 verify OK |
| **T2 Down 保留盘** | 验证回收资源且磁盘数据保留 | 1）diting-infra：`make down diting prod`<br>2）确认 `prod.disk_id` 文件仍存在且非空；或云控制台查看该盘未删除 | `prod.disk_id` 存在；或控制台可见数据盘 |
| **T3 再次 Up 数据继承** | 验证再次 Up 挂载同盘、数据可访问一致 | 1）diting-infra：`make deploy diting prod`<br>2）diting-core：再次 `cp prod.conn .env` 后 `make verify diting prod`<br>3）可选：psql 连接 L1/L2 查表行数或样例，与 Down 前对比 | `make verify diting prod` 退出码 0；可选：表行数/样例与 Down 前一致 |

<a id="l4-stage2-06-phase2"></a>
## 生产数据环境：后半部分（采集生产级 + 落库 + 数据继承）

**目标**：先在 diting-core 完成采集逻辑生产级设计目标（F8/F10、单标日线≥5 年、universe、复权）及 `make verify-data-production`；再在**同一套 Up 流程**中，由 **Up 脚本在集群就绪后调用 make ingest-test**（或约定命令），使生产级数据落库；Down 仍只回收 ECS/K3s/EIP、保留数据盘；再次 Up 挂载同盘后，库内已有此前落库的生产级数据。

**后半步骤约定**：后半部分 Up 时，diting-infra 的 `make deploy diting prod` 在 K3s 与 DB 就绪后，**在 diting-core 使用 `prod.conn` 执行 `make ingest-test`**（或约定命令），完成采集落库。**ingest 失败时**：仍输出连接信息到 `prod.conn`，便于人工排查；是否将本次 Up 记为「部分失败」或「Up 成功、ingest 待重试」由项目在 06_ 或 diting-infra 中约定（建议至少输出连接信息并标记 ingest 状态）。

**后半核心验证（三项）**：

| 序号 | 验证项 | 命令/方式 | 期望 |
|------|--------|-----------|------|
| ① | 采集逻辑生产级 | 在 diting-core 执行 `make verify-data-production`（或 02_ 目标数据约定等价验证） | 退出码 0 或按脚本输出判定通过 |
| ② | Up+采集后库内生产级数据 | 首次执行 Up（含脚本内 make ingest-test）后，psql 或 `make verify-data-production` 确认 L1 单标≥5 年、universe 等满足生产级 | 库内具备生产级数据 |
| ③ | Down+再次 Up 数据继承且验证通过 | Down 后再次 Up 挂载同盘，执行 `make verify-data-production` 或等价验证 | 数据仍在且验证通过 |

## 验证路线说明（目标功能）

采集层已按生产级实现（02 步 F8/F10、目标数据约定），无「逻辑填充期少存」；本步统一按**生产级目标功能**验收。

| 验收内容 | 说明 |
|----------|------|
| **规约一致** | 11_ 生产级小节、06_ 设计、06_dna、global_const 中 history_depth_years=5、复权与覆盖表述一致 |
| **可执行验证** | **diting-core** 须实现 `make verify-data-production`（或等价脚本/查询），检查：L1 单标日线最小 bar 数 ≥5 年对应日线数、标的集合与 universe 表一致、复权口径与约定一致；执行退出码 0 或按脚本输出判定通过 |
| **与 02 步闭环** | 02 步准出时目标数据已满足生产级（F8、F10、V-DATA）；本步可复验或首次运行 verify-data-production 并记录 |
