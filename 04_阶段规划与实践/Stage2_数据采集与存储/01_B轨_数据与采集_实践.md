# Stage2 · 01_B轨 数据与采集实践

> - **顶层概念**: [04_B轨系统概念与目标](../../01_顶层概念/04_B轨系统概念与目标.md)

> - **本步**: B 轨数据需求落地、存储表创建、采集写入与消费方可读（**仅 B 轨**；A 轨采集见 02_A轨_采集逻辑与Dockerfile_实践）

<a id="l4-stage2-b01-goal"></a>
## 步骤目标

- 按 [02_B轨数据与存储规约](../../03_原子目标与规约/B轨/02_B轨数据与存储规约.md) 落实 B 轨数据需求清单（必选：细财报、管理层、业务与竞争、研发与创新）。
- L2 存在 B 轨专用表或视图（如 company_fundamental、management、b_track_raw_feed）；建表脚本或 Chart 可执行。
- 采集模块（或扩展）将 B 轨必采数据写入上述表；至少一条「数据源→存储→B 轨模块可读」路径可验收。

<a id="l4-stage2-b01-dor"></a>
## 准入（DoR）

- Stage2 采集基础设施与 L2 可写可读已就绪（见 [01_A轨_基础设施与依赖_实践](01_A轨_基础设施与依赖_实践.md)、[02_A轨_采集逻辑与Dockerfile_实践](02_A轨_采集逻辑与Dockerfile_实践.md)）。
- [02_B轨数据与存储规约](../../03_原子目标与规约/B轨/02_B轨数据与存储规约.md) 与 dna_b_track 的 `data_requirement`、`storage` 已对齐。

<a id="l4-stage2-b01-steps"></a>
## 实施步骤

**工作目录**：diting-core（或部署仓，若表由 Chart 创建）

1. **建表**：在 L2 创建 B 轨所需表（或通过 init 脚本/Helm Chart）；表名与 [02_B轨数据与存储规约](../../03_原子目标与规约/B轨/02_B轨数据与存储规约.md)#二 及 dna_b_track.storage 一致。
2. **采集扩展**：在现有采集管道中增加 B 轨必采项（财报明细、管理层、业务与竞争、研发与创新）；写入 company_fundamental、management 等表；更新频率与 11_ 或 Stage2 约定一致。
3. **消费验证**：B 轨预筛或 VC-Agent 占位能按 symbol 读取上述表；无数据时降级策略符合 02_ 约定。

<a id="l4-stage2-b01-verify"></a>
## 验收

- [ ] B 轨表存在且 schema 与 02_ 一致或兼容。
- [ ] 至少一个数据源（如财报/管理层）写入 B 轨表并可被按 symbol 查询。
- [ ] L5 或本目录验收清单中「01_B轨 数据与采集」已勾选。

<a id="l4-stage2-b01-exit"></a>
## 准出

- 上述验收通过；设计/DNA 无变更或已同步。
