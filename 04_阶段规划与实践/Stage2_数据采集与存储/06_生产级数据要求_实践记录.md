#### 2. 创建 NodePort 服务

由于默认部署使用 ClusterIP，手动创建了 NodePort 服务以暴露数据库：

```bash
# TimescaleDB NodePort (30001)
kubectl apply -f - <<EOF
apiVersion: v1
kind: Service
metadata:
  name: timescaledb-nodeport
  namespace: default
spec:
  type: NodePort
  selector:
    app.kubernetes.io/name: postgresql
    app.kubernetes.io/instance: timescaledb
  ports:
  - port: 5432
    targetPort: 5432
    nodePort: 30001
    protocol: TCP
    name: postgresql
EOF

# PostgreSQL L2 NodePort (30002)
kubectl apply -f - <<EOF
apiVersion: v1
kind: Service
metadata:
  name: postgresql-l2-nodeport
  namespace: default
spec:
  type: NodePort
  selector:
    app.kubernetes.io/name: postgresql
    app.kubernetes.io/instance: postgresql-l2
  ports:
  - port: 5432
    targetPort: 5432
    nodePort: 30002
    protocol: TCP
    name: postgresql
EOF

# Redis NodePort (30379)
kubectl apply -f - <<EOF
apiVersion: v1
kind: Service
metadata:
  name: redis-nodeport
  namespace: default
spec:
  type: NodePort
  selector:
    app.kubernetes.io/name: redis
    app.kubernetes.io/instance: redis
    app.kubernetes.io/component: master
  ports:
  - port: 6379
    targetPort: 6379
    nodePort: 30379
    protocol: TCP
    name: redis
EOF
```

### 遇到的问题

#### 问题1：NodePort 端口无法从外部访问

**现象**：
- 执行 `make verify diting prod` 时连接超时
- 错误信息：`connection to server at "43.99.3.52", port 30001 failed: Operation timed out`

**原因分析**：
- 阿里云安全组规则默认不开放 NodePort 端口范围（30000-32767）
- 虽然 K8s API 端口（6443）可以访问，但 NodePort 端口被防火墙阻止

**验证**：
- kubectl 可以正常连接集群（通过 6443 端口）
- 在集群内部可以访问数据库服务（通过 ClusterIP）

#### 问题2：prod-write-conn.sh 未正确更新 EIP

**现象**：
- 部署完成后，prod.conn 文件中的 EIP 仍为占位符 `<EIP>`

**原因分析**：
- `scripts/prod-write-conn.sh` 脚本可能未能正确从 Terraform 输出中获取公网 IP
- 或者脚本执行时机问题

**解决方案**：
- 手动从部署输出中获取公网 IP（43.99.3.52）
- 手动更新 prod.conn 文件
