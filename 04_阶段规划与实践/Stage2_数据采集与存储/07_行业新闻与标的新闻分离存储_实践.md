<a id="l4-stage2-07-goal"></a>
## 步骤目标

- L2 `news_content` 支持 `scope` / `scope_id`（`market` | `industry` | `symbol`），唯一键 `(scope, scope_id, title_hash, published_at)`。
- 存量库执行 **`make migrate-l2-news-content-scope`**（幂等）；新环境可使用 `diting-infra/schemas/sql/04_l2_news_content.sql`。
- 采集写入经 **`diting.ingestion.l2_writer.write_news_content_batch`**：8 元组默认 `scope=symbol`；10 元组可带 `industry` 等。

<a id="l4-stage2-07-a-track"></a>
## 与 A 轨信号层衔接

- **`make init-l2-a-track-signal-cache`** 建 `a_track_signal_cache`。
- **`DITING_TRACK=a`** 时 `make refresh-segment-signals` 调用 `refresh_a_track_signals_for_symbols`：从 `news_content` 拉 **标的** 与 **申万行业**（`scope=industry`，`scope_id` 与 `industry_revenue_summary.industry_name` 一致）正文，规则/AI 打标后写入缓存。
- **行业正文为空**：行业侧打标失败属预期，直至行业新闻采集落地；标的路径仍独立写入。

<a id="l4-stage2-07-verify"></a>
## 验收

- [ ] `migrate-l2-news-content-scope` 成功；`INSERT news_content` 使用新 ON CONFLICT 不报错。
- [ ] `a_track_signal_cache` 在 refresh 后出现 `sym:` / `ind:` 键（有正文时）。
- [ ] Module C `DITING_TRACK=a` 可合并 A 轨信号（见 `merge_a_track_into_segment_signals`）。
