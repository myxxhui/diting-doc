# Stage3 · 01_B轨 语义与候选实践

> [!NOTE] **[TRACEBACK]**
> - **顶层概念**: [04_B轨系统概念与目标](../../01_顶层概念/04_B轨系统概念与目标.md)
> - **设计**: [01_B轨系统设计](../../03_原子目标与规约/B轨/01_B轨系统设计.md)#2.2、#2.3，[02_B轨数据与存储规约](../../03_原子目标与规约/B轨/02_B轨数据与存储规约.md)
> - **DNA**: [_System_DNA/B轨/dna_b_track.yaml](../../03_原子目标与规约/_System_DNA/B轨/dna_b_track.yaml) `module_a_extension`、`module_b_b_track`
> - **本步**: A 模块 B 轨输出扩展、B 模块 B 轨策略与 b_track_candidate_snapshot 写入（**仅 B 轨**；A 轨语义与 B 模块见 01_A轨_语义分类器_实践、02_A轨_B模块策略_实践）

<a id="l4-stage3-b01-goal"></a>
## 步骤目标

- A 模块输出增加 B 轨用字段（industry、lifecycle_stage、b_track_eligible 等）；B 模块（或 B 轨预筛）可读取 A 的该输出。
- B 模块实现 B 轨专用策略与打分（long_term_score、成长质量分等），通过条件符合 dna_b_track 的 pass_condition。
- B 轨通过的标的写入 `b_track_candidate_snapshot`；C 的 Router/VC-Agent 仅或优先从此表读取 B 轨候选。

<a id="l4-stage3-b01-dor"></a>
## 准入（DoR）

- [Stage2 01_B轨_数据与采集_实践](../Stage2_数据采集与存储/01_B轨_数据与采集_实践.md) 准出（B 轨表与数据可读）。
- Stage3 语义分类器与量化扫描引擎已就绪（见 [01_A轨_语义分类器_实践](01_A轨_语义分类器_实践.md)、[03_A轨_量化扫描引擎_实践](03_A轨_量化扫描引擎_实践.md)），可扩展输出与写入逻辑。

<a id="l4-stage3-b01-steps"></a>
## 实施步骤

**工作目录**：diting-core

1. **A 扩展**：在 ClassifierOutput 或 L2 分类器输出中增加 B 轨字段（见 dna_b_track.module_a_extension）；实现或配置 b_track_eligible 规则。
2. **B 轨策略**：在 B 模块（或独立 B 轨预筛）中实现 B 轨维度分与通过条件（读取 A 的 b_track_eligible 与 02_ 数据表）；配置与 dna_b_track.module_b_b_track 一致。
3. **候选表写入**：B 轨通过标的写入 b_track_candidate_snapshot；字段与 [02_B轨数据与存储规约](../../03_原子目标与规约/B轨/02_B轨数据与存储规约.md)#二 一致。
4. **C 消费**：Router/VC-Agent 在判断「是否 B 轨请求」时，以 symbol 是否在 b_track_candidate_snapshot 为准（或等价接口）。

<a id="l4-stage3-b01-verify"></a>
## 验收

- [ ] A 输出含 b_track_eligible 等 B 轨字段；B 可读取。
- [ ] B 轨策略与打分可运行；通过标的写入 b_track_candidate_snapshot。
- [ ] C 侧能按 b_track_candidate_snapshot 决定是否走 VC-Agent。

<a id="l4-stage3-b01-exit"></a>
## 准出

- 上述验收通过；设计/DNA 无变更或已同步。
