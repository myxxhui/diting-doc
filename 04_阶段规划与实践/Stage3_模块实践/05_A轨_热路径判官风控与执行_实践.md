## 步骤目标

在一个实践步骤内实现**热路径**可部署单元：Module D 判官 + Module E 风控盾 + Module F 执行网关（同进程、同镜像）；完成 D/E/F **四项 100% 验证**及全链路 A→F 验证。

## 本步骤落实的 DNA 键

- `core_modules/dna_module_d.yaml`：投票、凯利、Cash Drag、Defensive、PyPortfolioOpt
- `core_modules/dna_module_e.yaml`：硬止损、盈亏比、相关性
- `core_modules/dna_a_track_position_lifecycle.yaml`：A 轨持仓止损只紧不松、信号复核 streak（`diting-core/diting/position_lifecycle.py`）
- `core_modules/dna_module_f.yaml`：Broker 抽象、Gateway、Vn.py

## 实现部分

**工作目录**：`diting-core`

1. **判官（D）**：建立 `diting/gavel`（或约定包）；投票+Kelly+Cash Drag+Defensive；公式与阈值写 YAML；双轨分流按 09_；PyPortfolioOpt 组合优化与仓位闭环。**已实现**：`diting-core/diting/gavel/voting.py`（vote 读取 ExpertOpinion.horizon，LONG_TERM 时 Verdict.apply_2pct_stop=False、apply_cash_drag=False、track="B"）；`b_track_rules.py`（逻辑证伪止损、大周期反转止盈占位接口）；见 [06_B轨需求与实现缺口分析](../../06_追溯与审计/B轨需求与实现缺口分析.md)。**B 轨专用**逻辑证伪/大周期反转规约见 [03_/B轨/03_B轨共识与逻辑证伪规约](../../03_原子目标与规约/B轨/03_B轨共识与逻辑证伪规约.md)；实践见 [04_/B轨/03_B轨共识与执行_实践](../B轨/03_B轨共识与执行_实践.md)。
2. **风控（E）**：建立 `diting/risk`，RiskShield；硬止损 2%、盈亏比≥1.5、组合相关性；规则写 YAML。A 轨已持仓时合并每日 B 建议止损、累计低于阈值日数等，见 [03_A轨_持仓与每日信号复核规约](../../03_原子目标与规约/Stage3_模块实践/03_A轨_持仓与每日信号复核规约.md) 与 `diting/position_lifecycle.py`。
3. **执行（F）**：建立 `diting/execution` 或 `diting/drivers`；BrokerDriver 与占位/仿真；Redis Streams、OMS Lite、Human-in-the-Loop；Vn.py Gateway 式接口。
4. **同镜像**：热路径 D+E+F 同进程构建，Dockerfile/requirements 含上述依赖；镜像内判官、风控、place_order 单测通过。

<a id="l4-stage3-05-minimal-data"></a>

### 最小数据接口（实践要点）

A 轨已持仓复核与盘中硬止损依赖 **只读持仓** 与 **只读 last price**，规约见 [03_A轨_持仓与每日信号复核规约 · §7](../../03_原子目标与规约/Stage3_模块实践/03_A轨_持仓与每日信号复核规约.md#design-a-track-pl-minimal-data)；09_ Module E [最小数据接口](../../03_原子目标与规约/_共享规约/09_核心模块架构规约.md#module-e-minimal-data-interfaces) 为边界总述。

| 实践项 | 要求 |
|--------|------|
| **只读持仓** | 从 BrokerDriver / OMS / 本地持仓存储提供查询接口，至少返回 `symbol`、`side`、`quantity`、`entry_price`、`current_stop_price`（与 `dna_a_track_position_lifecycle.minimal_data_interfaces.read_only_positions_api.fields_min` 一致）；不在 B 模块内实现持仓真相。 |
| **只读 last price** | 通过 MarketDataFeed（或等价）按标的查询 `last_price`；**日频**复核可仅用当日 B + 持仓快照；**盘中**硬止损必须接入行情。部署层在 `config/a_track_position_lifecycle.yaml` 的 `minimal_data_interfaces.read_only_last_price.semantics` 固定一种报价语义（如 `mark`）。 |
| **与 B 的 join** | 热路径或编排层用持仓 `symbol` 关联当日 B 输出或 L2 快照，再调用 `diting.position_lifecycle` 与 RiskShield。 |

**工作目录**：`diting-core`。代码注释含 `[Ref: 03_A轨_持仓与每日信号复核规约]`、`[Ref: 05_热路径判官风控与执行]`。

## 验证部分

### 四项 100%（D、E、F 各维度）

| 维度 | 验收标准 | 验证方法 |
|------|----------|----------|
| **接口/结构/逻辑/测试** | D/E/F 分别满足 09_ 与设计文档 | 契约单测、Table-Driven、make test |
| **全链路** | A→F 可跑通 | 全链路测试或主路径执行 |

### 三层验证

- **单模块**：判官、风控、Broker 占位各自单测
- **联动**：D→E→F 联调
- **全链路**：A→F 验证在本步准出时执行
