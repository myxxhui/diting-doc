<a id="l4-stage4-b01-goal"></a>
## 步骤目标

- VC-Agent 产出 ExpertOpinion 后，共识（key_assumptions、evidence_summary、disproof_triggers 等）写入 `b_track_consensus`；schema 与 [03_B轨共识与逻辑证伪规约](../../03_原子目标与规约/B轨/03_B轨共识与逻辑证伪规约.md) 一致。
- 逻辑证伪与大周期反转具备**输入/输出接口**（可占位实现）：输入为共识+事件或价格/均线，输出为原因码 logic_disproof / major_trend_reversal 的平仓/减仓指令。
- D/E/F 对 B 轨仓位与订单的差异化（不触发 2%、轨标识、仅按逻辑证伪/大周期反转平仓）可验证或占位可测。

<a id="l4-stage4-b01-dor"></a>
## 准入（DoR）

- [Stage3 01_B轨_语义与候选_实践](../Stage3_模块实践/01_B轨_语义与候选_实践.md) 准出；VC-Agent 可对 B 轨候选产出 ExpertOpinion。
- 判官（gavel）已存在 TimeHorizon 分支与 B 轨占位（见 [06_B轨需求与实现缺口分析](../../06_追溯与审计/B轨需求与实现缺口分析.md)）。

<a id="l4-stage4-b01-steps"></a>
## 实施步骤

**工作目录**：diting-core

1. **共识表与写入**：创建 b_track_consensus 表（或等价）；VC-Agent 或 C 聚合层在产出 LONG_TERM 意见后写入 key_assumptions、evidence_summary、disproof_triggers、time_horizon 等。
2. **逻辑证伪**：实现或占位 check_logic_disproof_stop（或等价）：输入 symbol + 可选 context（共识、事件）；输出布尔或指令；与 D 判官或 E 对接。
3. **大周期反转**：实现或占位 check_major_trend_reversal（或等价）：输入 symbol + 可选 context；输出布尔或止盈指令；与 D 对接。
4. **D/E/F 分支**：确认 Verdict 的 apply_2pct_stop、apply_cash_drag、track 在 LONG_TERM 时正确；E 不对 B 轨仓位施加 2%；F 订单/仓位带 track=B，平仓仅响应逻辑证伪/大周期反转。

<a id="l4-stage4-b01-verify"></a>
## 验收

- [ ] b_track_consensus 存在且 VC-Agent 可写入。
- [ ] 逻辑证伪/大周期反转接口可调用（占位返回 False 亦可）；接口与 03_ 规约一致。
- [ ] D/E/F B 轨分支在单测或集成路径中可验证。

<a id="l4-stage4-b01-exit"></a>
## 准出

- 上述验收通过；设计/DNA 无变更或已同步。
