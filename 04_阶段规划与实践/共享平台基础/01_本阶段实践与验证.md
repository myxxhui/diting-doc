## 一、本阶段目标

- **stage_id**: `shared_platform_baseline`
- **pillar**: shared_platform
- **milestone**: mvp（前置项，所有其它 stage 的 `depends_on` 起点）
- **工作目录**: `diting-infra/`

为四大模块 + 前端的 MVP 提供平台底座：数据库、消息、搜索、缓存、容器编排、推理网关、配置中心、鉴权、CDN。

## 三、实施内容

### 3.1 K3s 集群（阿里云 ECS 起步）
- 1 master + 2 worker（最小可用）
- 安装 metrics-server / cert-manager / ingress-nginx
- 配置 GPU 节点（按需）+ HAMi GPU 切分

### 3.2 数据存储
- PostgreSQL 15+（主从 + WAL 归档）
- TimescaleDB 扩展（probe_results 时序）
- OpenSearch 集群（3 节点）
- Redis 6+（缓存 + 短期幂等）
- Kafka 3+（事件总线 + 风险事件）
- 对象存储（S3 兼容；阿里云 OSS）

### 3.3 推理网关 Port
- vLLM（OpenAI 兼容 API）+ LoRA 多路复用
- 健康检查 + 限流 + cost 监控

### 3.4 配置中心
- 实现 [06_动态配置中心规约](../../03_原子目标与规约/_共享规约/06_动态配置中心规约.md)
- 支持热更新 / 版本回滚 / 审计

### 3.5 鉴权 / API Gateway
- 用户登录 + JWT
- 服务间 mTLS
- 限流 + WAF

### 3.6 静态 CDN
- 前端构建产物 CDN 化

| 命令 | 工作目录 | 期望 | 对应 DNA 键 |

## 七、本步骤失败时

- **回退目标**：保留上一次稳定 Helm Release
- **重试上限**：同一问题修复重试 ≤ 2 次（按 `00_系统规则.md §7.2`）
- **超出后**：`helm rollback diting-stack <last-stable>`；登记到 06_/审计；通知架构师
- **临时跳过须谁审批**：架构师 + 项目负责人
