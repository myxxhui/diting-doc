## ADR 状态说明

- **Proposed**：提案阶段，等待审查
- **Accepted**：已接受，正在实施或已实施
- **Rejected**：已拒绝，不采用
- **Superseded**：已被其他 ADR 替代
- **Deprecated**：已废弃，不再使用

## ADR 创建流程

1. **提出决策**：识别需要做架构决策的场景
2. **编写 ADR**：使用标准模板编写 ADR 文档
3. **Review**：提交 PR，等待审查
4. **状态变更**：审查通过后，状态变更为 Accepted
5. **实施**：按照 ADR 实施
6. **验证**：验证实施效果
