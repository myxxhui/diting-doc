# B 轨（长期捕手轨）需求与实现缺口分析

> [!NOTE] **[TRACEBACK] 战略追溯锚点**
> - **顶层概念**: [01_一句话定义与核心价值](../01_顶层概念/01_一句话定义与核心价值.md)、[03_双轨制与VC-Agent](../01_顶层概念/03_双轨制与VC-Agent.md)、[04_B轨系统概念与目标](../01_顶层概念/04_B轨系统概念与目标.md)
> - **战略维度**: [01_产品设计维度](../02_战略维度/产品设计/01_产品设计维度.md)、[02_技术栈与架构维度](../02_战略维度/产品设计/02_技术栈与架构维度.md)
> - **原子能力**: [09_核心模块架构规约](../03_原子目标与规约/_共享规约/09_核心模块架构规约.md)（Module C VC-Agent、Module D 双轨分流）；**B 轨专用**：[03_/B轨/](../03_原子目标与规约/B轨/README.md)；L4 实践见各 Stage 下 01_B轨_*（[04_/README](../04_阶段规划与实践/README.md)）

**说明**：本文档区分「A 轨 / B 轨」与「Module A / Module B」。**B 轨** = 长期捕手轨（Strategy B，约 20% 资金），由 VC-Agent 与判官 TimeHorizon 分流支撑；**Module B** = 量化扫描引擎（机会丰度），为两轨共用。当前 A 轨相关能力（含 Module B 扫描）已大量落地；B 轨在 L1/L2/L3 有规约与设计，但**实现多为占位或未落地**，本稿做缺口整理与后续建议。

**关键前置**：若 Module B 的策略与打分逻辑**只覆盖短周期、技术面**（趋势/反转/突破），则**具备长期价值的标的可能根本进不了 B 轨视野**——VC-Agent 与判官分流再完善，也无从对「未被扫描到」的标的施加 B 轨规则。因此 B 轨闭环除 C/D/分账外，还依赖 **Module B 或等价入口具备「长期价值候选」的发现与输出能力**。

## 四、Module B 与 B 轨视野：漏斗前置缺口与可选方案

### 4.1 问题

- **Module B** 当前策略与打分为 **A 轨** 设计：趋势/反转/突破均为**短周期、技术面**（日线 MA/MACD/RSI/布林/放量突破等），通过阈值与 technical_score ≥ 70 筛选的是「短线机会丰度」。
- **B 轨** 目标为**时代级牛股、长期价值**，往往依赖**长期动量、基本面质量、营收/研发等**；若某标的技术面不突出（未触发趋势/反转/突破高分），就不会被 Module B 打高分或通过，**不会进入下游 Router/VC-Agent 的输入池**。
- 因此：**仅靠「Router 路由到 VC-Agent + 判官分流」无法解决「谁有机会被 B 轨看到」**——必须先让**具备长期价值的标的进入扫描输出或 B 轨候选池**，B 轨闭环才成立。

### 4.2 可选设计方案（需产品/架构决策）

| 方案 | 思路 | 优点 | 成本/依赖 |
|------|------|------|-----------|
| **方案一：Module B 扩展（双标/双轨输出）** | 在现有 Module B 扫描中**增加 B 轨向策略或标签**：如长期动量（120d/250d 收益分位）、或简单质量过滤（若有基本面数据）；输出中增加 `long_term_candidate` 布尔或 `long_term_score`，或单独「B 轨候选」列表。Router 据此将此类标的路由到 VC-Agent。 | 复用同一套扫描与标的池，一次扫、双轨用；与现有 L2 写入易兼容。 | 需在 Module B 规约与代码中增加长期动量/质量逻辑与输出字段；若用基本面需 Stage2 数据就绪。 |
| **方案二：独立 B 轨预筛** | 在 Module B 之后、Router 之前增加 **B 轨专用预筛**：用不同规则（如 120d 动量分位、营收增速/研发占比阈值）从全市场或子集筛出 B 轨候选池，再与 Module B 的 technical 结果合并；仅 B 轨候选池的标的送 VC-Agent。 | Module B 保持 A 轨专注；B 轨规则可完全独立迭代。 | 多一套管道与配置；需约定 B 轨预筛输入数据（行情长度、基本面表）与输出形态（与 QuantSignal 或标的列表的合并方式）。 |
| **方案三：双轨共扫、双阈值** | Module B 一次扫描，同时计算 **technical_score（A 轨）** 与 **long_term_score（B 轨）**（如 120d 动量分位 + 可选基本面分）；A 轨通过条件为 technical_score ≥ 70；B 轨通过条件为 long_term_score ≥ 某阈值（或技术面不差且 long_term_score 达标）。输出中带轨标签或双轨通过位，Router 按轨分流。 | 单次扫描、双轨可见；长期与短期逻辑在同一处可对比。 | Module B 需扩展策略与打分（长期动量/质量）、输出 schema 与 L2 表需支持 long_term 相关字段。 |

**建议**：在 L3 明确「B 轨候选来源」采用哪一种（或混合：如 Module B 输出 long_term_candidate 标签 + Router 只对带标签的送 VC-Agent），并据此在 Module B 规约（或 B 轨预筛规约）中补充**长期价值向**的输入/输出与数据依赖（如 120/250 日 K 线、基本面表是否必须）。规约可引用 [02_B模块策略_策略实现规约](../03_原子目标与规约/Stage3_模块实践/02_B模块策略_策略实现规约.md) 与 [02_B模块策略_深度审查与优化建议](../03_原子目标与规约/Stage3_模块实践/02_B模块策略_深度审查与优化建议.md) 中「中长期可考虑周线或双周期」的扩展点，与 B 轨视野对齐。

## 五、建议后续动作（优先级供产品/架构决策）

| 优先级 | 动作 | 说明 |
|--------|------|------|
| **P0** | **明确 B 轨候选来源并落地「长期价值发现」** | 决策采用「Module B 扩展（双标/双轨输出）」或「独立 B 轨预筛」或「双轨共扫双阈值」；在 L3 规约与 Module B（或 B 轨预筛）中补充**长期价值向**策略与输出（如 long_term_candidate / long_term_score、所需 120/250 日 K 线或基本面数据），确保具备长期价值的标的**能进入 B 轨视野**。 |
| **P0** | 在判官实现中增加 **TimeHorizon 分支** | 若已有判官占位，则先做「读取 horizon → LONG_TERM 时跳过 2% 与现金拖累」分支，保证协议闭环。 |
| **P0** | **VC-Agent 占位** | Module C 内可先实现「返回 LONG_TERM 的占位专家」，便于与判官联调双轨分流。 |

| **P1** | **B 轨标的池与 Router 条件** | 约定 B 轨标的池来源（配置/表/接口）、Router 路由条件（enable_vc_agent、long_term_candidate 等）、与 Module B 或 B 轨预筛输出的衔接。 |
| **P2** | 基本面数据管道与 VC-Agent 消费 | 确认 Stage2 采集是否落库/落 L2，VC-Agent 读哪些表或接口。 |
| **P2** | 仓位/订单轨标识与分账 | 执行与风控按轨区分；分账与复盘分轨统计。 |

## 六、小结

- **A 轨**：Module B（量化扫描）、判官 A 轨分支、2% 止损、现金拖累、不可能三角等，设计与实现相对完整。
- **B 轨**：L1/L2/L3 有清晰规约与协议（TimeHorizon、VC-Agent、判官分流、逻辑证伪、大周期反转），但**实现上** VC-Agent、判官双轨分支、逻辑证伪/大周期反转模块、B 轨标的池与分账等多为**未落地或占位**。此外存在**漏斗前置缺口**：若 Module B 的策略与打分**只覆盖短周期技术面**，则具备长期价值的标的**无法进入 B 轨视野**，需在 Module B 或独立 B 轨预筛中补充**长期价值发现**能力（见第四节）。
- **本次已实现**：Module B 长期价值发现（long_term_score / long_term_candidate）、VC-Agent 占位、Router 按 long_term_candidate 路由、判官 TimeHorizon 分支（B 轨豁免 2% 与现金拖累）、逻辑证伪/大周期反转占位接口。建议后续按 P1/P2 补齐 L3 细化（逻辑证伪/大周期反转输入输出）、基本面管道、仓位轨标识与分轨复盘。
