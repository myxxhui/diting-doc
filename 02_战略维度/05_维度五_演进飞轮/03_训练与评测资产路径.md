## 一、维度五的"元路径"特殊性

维度五本身**不是被训练的对象**，而是**承载训练流水线的元设施**。所以本维度的"5 阶段路径"不是模型本身的训练路径，而是**飞轮工程能力的逐步成熟路径**。

## 二、维度五·飞轮工程能力 5 阶段成熟路径

| 阶段 | 名称 | 关键交付 | 启动条件 | 完成标志 |
|---|---|---|---|---|
| **A** | 启动期·手动数据 + 手动微调 | Teacher 蒸馏 + DVC + Label Studio + LLaMA-Factory 跑通 | 第 1 个月 | 第一个维度的首引擎能从 0 训练到 1 |
| **B** | 流水线期·CT 触发 + Registry | K8s GPU Job + MLflow + 评测回放 | P0 跑通后 | 训练触发不再需要架构师手动操作 |
| **C** | DPO 期·偏好对齐 | DPO 流水线接入 | 至少有 1 个引擎跑了 3 个月 | 至少 1 个引擎完成 DPO Stage C |
| **D** | 多 LoRA 期·专科细分 | vLLM 多 LoRA 多路复用 | 至少 5 个引擎都到了 Stage C | 多 LoRA 同 GPU 并发服务 |
| **E** | 议会模式·数字分身 | 数字分身 RAG + 议会模式 | 整体系统运行 1 年+ | 数字分身能给出与架构师 80% 一致的判断 |

## 三、维度五的"自身评测"

维度五没有传统意义上的"模型评测"，但有**飞轮工程效能评测**：

| 评测指标 | 目标 | 备注 |
|---|---|---|
| **数据 → 模型 lead time** | < 24h | 从新增 100 条 verified 样本到新版本 LoRA 部署 |
| **Holdout 回放守门覆盖率** | 100% | 任何 LoRA 上线前必须经过 Holdout 回放 |
| **DVC 版本可追溯率** | 100% | 任何 LoRA 必须能查到对应训练数据 commit |
| **MLflow tracking 完整率** | 100% | 任何训练必须有完整 tracking 记录 |
| **Teacher LLM 月度成本** | < ¥3000 | 控制蒸馏成本 |

## 五、维度五的特殊原则

1. **永远不替代人工 verified**：Teacher LLM 蒸馏 + 人工 verified 是不可分割的双步
2. **永远不放弃 Holdout 守门**：哪怕是单引擎的小修复，也必须经过 Holdout 回放
3. **永远不上线没有 MLflow 记录的模型**：模型可追溯是底线
4. **永远不让训练流水线绕过 DVC**：数据可追溯是底线
5. **永远不让 Agent 生成的代码绕过沙箱**（P2 之后）：安全是底线
