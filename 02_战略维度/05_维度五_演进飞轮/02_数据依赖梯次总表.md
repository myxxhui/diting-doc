## 一、设计原则

- **本维度的"数据"是其他 4 维度产生的**：维度五负责消费、组织、训练
- **维度五独有的元数据**：训练运行 metadata、评测结果、模型版本、数据 commit
- **"少而精"原则**：宁可慢一点积累高质量数据，不要快速积累低质量数据

## 五、数据治理铁律

1. **DVC 强制版本化**：每一份训练数据必须 DVC 锁定，每一个 LoRA 必须能回滚到具体训练数据 commit
2. **MLflow 强制记录**：每一次训练必须有 MLflow tracking，超参 + 指标 + artifact 全记录
3. **Holdout 不可污染**：Holdout 集永久锁定，绝不进入训练数据
4. **Teacher LLM 调用记录**：每次蒸馏调用记录 prompt + response + cost，用于审计与成本控制
5. **Label Studio 标注审计**：每条 verified 样本记录标注员、标注时间、复核状态
