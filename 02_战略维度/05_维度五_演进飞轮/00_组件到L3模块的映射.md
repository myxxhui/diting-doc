## 四、关键能力的协作场景

| 场景 | 涉及的 L3 服务 | 说明 |
|---|---|---|
| **决策日志归因 → 8 象限路由** | `eval_replay_service` → `feedback_collector.quadrant_router` → `knowledge_base_service` | 自动入对应库 |
| **B 象限永久隔离** | `quadrant_router` 写 `violation_archive` | 不进任何 LoRA 训练 |
| **LoRA 训练完成 → 6 指标守门人** | `model_registry.gatekeeper` 检查 6 指标 | 任一不通过 → 挂起 |
| **守门通过 → 灰度发布** | `model_registry.canary_release` 10%→50%→100% | 异常自动回滚 |
| **每月 1 次双盲 Kappa** | `kappa_calibrator` 抽 10 条 → frontend 双盲面板 → 计算 Kappa | Kappa < 0.70 暂停训练 |
| **议会模式决策** | `parliament_orchestrator` 调 4 维度 LoRA → Judge LLM → 投票合议 | 跨四大模块协作 |
| **成本超限拦截** | `cost_controller.check_before_training` | 月度 > ¥10000 → 拒绝训练 |

## 五、部署形态建议

| 服务 / 组件 | 部署形态 | 资源 |
|---|---|---|
| `feedback_collector`（含 quadrant_router + kappa_calibrator）| 独立微服务 | 1 vCPU / 2GB RAM |
| `knowledge_base_service`（RAG + 8 库管理）| 独立微服务（PG + 向量数据库）| 2 vCPU / 8GB RAM |
| `model_registry`（含 gatekeeper + canary + cost + parliament + scheduler）| 独立微服务（MLflow 集成）| 2 vCPU / 4GB RAM |
| `eval_replay_service`（评测 + 回放）| 独立微服务 | 2 vCPU / 4GB RAM |
| `growth_dashboard_service` | 独立微服务 | 1 vCPU / 2GB RAM |
| `external_action_boundary` | 独立微服务（含 LLM Gateway + Webhook）| 1 vCPU / 2GB RAM |
| **GPU 训练节点**（LLaMA-Factory）| 独立物理机（4090 24GB）| 8 vCPU / 64GB RAM + GPU |
| **Teacher LLM API 调用** | 外部 API（无本地部署）| — |

## 七、上下游事件流

### 7.1 维度五输出（被其他维度消费）

```yaml
LoRAUpdatedEvent:              消费方: 维度一/二/三/四的 LoRA 推理热更新
TrainingCompletedEvent:        消费方: 维度零月报飞轮训练贡献页
KappaCalibrationCompleted:     消费方: 维度零双盲 Kappa 历史
ParliamentDecisionEvent (Stage3): 消费方: 维度零议会投票面板
```

### 7.2 维度五输入（消费其他维度）

```yaml
RejectAttributionEvent:        维度一 → F/B 归因
ThesisAttributionEvent:        维度二 → A/G/H 归因
SellAttributionEvent:          维度四 → A/G/H 归因（含卖飞豁免）
DPOPreferencePairs:            维度零反馈闭环 → DPO 训练数据
BlindLabelData:                维度零双盲面板 → Kappa 计算
```
