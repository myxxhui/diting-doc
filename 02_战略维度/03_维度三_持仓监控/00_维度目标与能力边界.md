## 一、战略目标

把"投资逻辑"当作一个被持续监控的服务（service），借鉴 SRE/DevOps 的方法论：

> **"持仓逻辑也是被观测对象"**——一个持仓的生死，不取决于股价，而取决于其底层逻辑是否仍然成立。

## 二、SLI 探针思想

| DevOps 概念 | 投资类比 |
|---|---|
| Service | 一笔持仓 |
| SLI（Service Level Indicator） | 这笔持仓建仓时声明的"3–5 个可验证假设" |
| SLO（Service Level Objective） | 当前 SLI 的健康阈值 |
| Alert | SLI 违反时触发的"逻辑健康度告警" |
| Page | 升级到维度四的卖出决策 |

**核心原则**：每一笔持仓在建仓时（维度二的 thesis 卡片），必须显式列出**3–5 条可验证、可量化的 SLI**，否则不允许进入维度三建仓。

**示例**（持有某新能源车标的）：
- SLI-1: 季度交付量同比增速 > 30%（维度三季度评分）
- SLI-2: 毛利率 > 18%（维度三季度评分）
- SLI-3: 经营性现金流为正（维度三季度评分）
- SLI-4: 行业渗透率仍未到 50% 拐点（维度三月度评分）
- SLI-5: 大股东未减持 > 1% 总股本（维度三事件型监控）

## 三、决策机制

| 决策类型 | 判定 | 动作 |
|---|---|---|
| `healthy` | 所有 SLI 都在 SLO 内 | 持仓继续，更新驾驶舱 |
| `warning` | 1–2 个 SLI 边缘偏离 | 推送 review 提示给架构师 |
| `critical` | 多个 SLI 偏离 / 关键 SLI 严重违反 | 升级到维度四的 Exit Engine |

**永远不直接卖出**：本维度只产生信号，不下单、不调仓。

## 四、关键能力清单（8 引擎概要）

1. **叙事一致性评分**：建仓 thesis vs 当前事实的偏离度（用 LLM 做 NLI 判断）
2. **核心 SLI 探针调度器**：维护、调度、执行所有持仓的 SLI 探针
3. **逻辑健康度综合评分**：把多个 SLI 加权汇总成一个 0–100 的健康度分数
4. **预期差余量计算**：当前股价隐含的预期 vs 真实经营预期的差距
5. **拥挤度监测**：交易拥挤度指标（北上资金、机构持仓集中度）
6. **行业 Beta 漂移检测**：标的相对行业 Beta 的漂移信号
7. **机构持仓变化**：基金/QFII/北上资金持仓变化
8. **管理层信号**：管理层增减持、董事会变更、关键人发言

## 五、能力边界（不做什么）

| 边界 | 不做的事 | 归属 |
|---|---|---|
| 卖出决策 | 不做最终卖出/调仓决策 | 维度四 |
| 风险熔断（如造假/暴雷） | 不做造假识别 | 维度一 |
| 新机会发现 | 不主动搜索新标的 | 维度二 |
| 模型训练流水线 | 不维护 CT 流水线 | 维度五 |

## 六、L3 子模块映射

| L3 子模块 | 职责 |
|---|---|
| `position_state_machine` | 持仓状态机（healthy → warning → critical） |
| `sli_probe_scheduler` | SLI 探针调度器 |
| `sli_probe_executor` | SLI 探针执行器（执行每个 SLI 的查询/计算） |
| `health_score_aggregator` | 健康度综合评分 |
| `narrative_consistency_engine` | 叙事一致性评分（NLI） |

## 七、前端接入

- **持仓驾驶舱**：每笔持仓的 SLI 仪表盘 + 健康度趋势图
- **Thesis 摆渡视图**：建仓 thesis 与最新事实的对比
- **告警中心**：所有 warning/critical 告警 + 升级到维度四的记录
