## 主业务链路（标注 5 维度产品设计入口）

```mermaid
flowchart LR
  source[数据源]
  ingest[采集与清洗]
  firewall["维度一·极寒防御<br/>排雷网关"]
  knowledge[知识与检索]
  engine["维度二·纵深进攻<br/>Agent工作流·四大剧本"]
  output[研究卡片与thesis卡片]
  observer["维度三·持仓监控<br/>SLI探针·健康度评分"]
  exit["维度四·卖出决策<br/>三重退出矩阵"]
  feedback[人工反馈与复盘]
  flywheel["维度五·演进飞轮<br/>CT流水线·LoRA热加载"]

  source --> ingest --> firewall --> knowledge --> engine --> output
  output --> observer --> exit
  exit --> feedback
  observer --> feedback
  engine --> feedback
  firewall --> feedback
  feedback --> flywheel
  flywheel -.新模型/新规则.-> firewall
  flywheel -.新模型.-> engine
  flywheel -.新探针.-> observer
  flywheel -.新阈值.-> exit
```

## 工程链路

```mermaid
flowchart LR
  product[产品需求]
  workflow[Prompt与Workflow]
  eval[评测集与回归]
  deploy[灰度发布]
  observe[观测与告警]
  improve[反馈与迭代]

  product --> workflow --> eval --> deploy --> observe --> improve --> workflow
```

## 链路能力映射

- `知识与检索`、`Agent 工作流`：研究编排能力（→ 维度二）
- `灰度发布`、`回归`：评测与发布能力（→ 维度五）
- `观测与告警`、`推理底座`：基础设施治理能力（→ 维度三/五）
- `Runtime / 沙箱 / 长任务`：运行时与隔离能力（→ 维度一/五的外部动作边界）

## 主业务链路 ↔ 5 维度入口对照

| 主业务链路节点 | 对应产品设计维度 | 在 L3 模块中的归属 |
|---|---|---|
