## 维度定义

本维度同时管理两件事：

- 哪些能力真的服务产品价值
- 哪些能力值得在当前阶段投入工程复杂度

## 战略主轴（四维）

为避免“功能堆砌”与“技术炫技”，本维度统一按四个战略主轴收敛：

1. **极寒防御轴**：欺诈解构、风险熔断、治理预警
2. **纵深进攻轴**：产业链拼图、预期差识别、动量折现
3. **状态机监控轴**：持仓逻辑探针、叙事漂移纠偏、机会成本调度
4. **进化反哺轴**：反馈闭环、评测更新、策略与模型持续迭代

> 4 主轴是**工程归属主轴**（与 L3 四大模块 1:1 对齐）。在**产品设计**视角下，状态机监控轴会进一步拆为「持仓监控（Observer）+ 卖出决策（Exit Engine）」两个产品子视角，形成 5 维度产品生命周期视图。
> 5 维度详见 L2 顶层目录：[01_维度一_极寒防御/](../01_维度一_极寒防御/)、[02_维度二_纵深进攻/](../02_维度二_纵深进攻/)、[03_维度三_持仓监控/](../03_维度三_持仓监控/)、[04_维度四_卖出决策/](../04_维度四_卖出决策/)、[05_维度五_演进飞轮/](../05_维度五_演进飞轮/)；总览见 [06_跨维度协作/](../06_跨维度协作/)。

## P0

> 对应 5 维度：**维度一·极寒防御** + **维度五·演进飞轮（最小数据飞轮）** + **维度二·纵深进攻（首个剧本：财务排雷）** + **维度三·持仓监控（最小集）**

- 数据采集与整理（→ 维度二/五的数据底座）
- 财报 / 公告 / 新闻 / 行业研究摘要（→ 维度一/二）
- 候选池与风险池（→ 维度一/二）
- thesis 跟踪与提醒（→ 维度三 SLI 探针最小集）
- 防御规则最小闭环（高危信号一票否决）（→ 维度一）
- 持仓逻辑快照与核心探针绑定（→ 维度三）
- 模型 A/C 的最小可运行链路（可观测、可回放、可复盘）（→ 维度一 + 维度三 + 维度五）

## P1

> 对应 5 维度：**维度二·纵深进攻剩余三大剧本**（S 曲线 / 卖铲人 / 剩者为王） + **维度三·持仓监控完整能力** + **维度五·演进飞轮完整闭环**

- 多 Agent 编排（→ 维度二）
- 自托管推理（→ 维度五的 LoRA 多路复用）
- 证据引用与结构化输出增强（→ 维度二）
- 评测、灰度、回滚与 CT（→ 维度五）
- 产品指标体系与需求优先级框架（→ 全维度）
- 产业链暗网拼图与弱信号提纯(→ 维度二)
- Alpha 衰减识别与阶段性止盈规则（→ 维度三 → 维度四衔接）
- 模型 B 的剧本化执行（至少 2 个剧本可验证）（→ 维度二）
- 模型 D 的最小反馈反哺闭环（人工反馈→评测→更新）（→ 维度五）

## P2

> 对应 5 维度：**维度四·卖出决策（三重退出矩阵）** + **维度二/五的高级能力**

- 沙箱执行（→ 维度一/五的外部动作边界）
- 长任务状态机（→ 维度三/四）
- 多租户与资源配额
- 更强的平台接入边界
- browser / computer-use runtime
- 人机反馈低摩擦反哺与自动化策略更新（→ 维度五数字分身）
- 跨市场、跨语言、跨模态认知融合扩展
- 模型 A/B/C/D 全量协同与平台化输出能力
- **三重退出矩阵**（证伪熔断 + 估值过载止盈 + 机会成本调仓）（→ 维度四）

## 产品治理约束

### 1. 产品指标体系
- 效果指标：摘要质量、检索命中、提醒准确率、误报率
- 效率指标：单次研究耗时、单次跟踪耗时、人工复核时间
- 成本指标：单任务推理成本、单会话 runtime 成本、单功能维护成本
- 风险指标：错误结论率、失败恢复率、异常升级率

### 2. 需求优先级框架
- `P0`：直接提升研究与跟踪价值，且实现复杂度可控
- `P1`：明显增强系统壁垒，且可以形成稳定复用能力
- `P2`：更偏平台化或高复杂能力，建立在 P0/P1 成熟后

### 3. 上线决策
- 没有指标基线，不上线
- 没有回滚方案，不扩量
- 没有人工兜底，不让高风险自动化直接成为默认
- 没有“逻辑证伪条件”，不允许建仓逻辑进入执行层

### 4. 协作抽象
- 产品定义用户问题、场景边界、验收口径
- 工程负责实现、观测、回滚与稳定性
- 数据/评测负责样本、基线、失败案例沉淀
- 以统一门禁接口做协作，不以口头约定替代

### 5. AI 功能商业化判断
- 是否真的减少人工时间
- 是否真的提升研究质量
- 是否值得持续消耗算力和维护成本
- 是否能形成未来可平台化的资产

## 优先级原则

1. 先满足真实研究需求，再考虑假想用户需求。
2. 优先做“产品价值高且训练价值也高”的能力。
3. 避免无业务价值的炫技，也避免低水平重复开发。
4. 任何 AI 功能都必须回答“指标怎么衡量、为什么现在做、做完是否值得继续维护”。
5. 任何进攻策略必须先通过防御网关，任何持仓策略必须具备状态机监控与退出条件。
