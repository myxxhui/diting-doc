## 一、战略目标

彻底根绝本金的**永久性损失**——避免：
- 退市（如 ST 系列、强制退市）
- 财务造假（如康美 300 亿货币资金虚增、康得新 300 亿现金造假）
- 管理层跑路（如部分实控人失联、抽离资金）
- 商业逻辑伪命题（如概念炒作、股价庞氏）

**铁律**：在 A 股市场，能 100% 防住"死局"，就已经在统计意义上战胜了 90% 的散户与中小机构。

## 二、核心使命

充当无情的"安检网关"。所有标的进入维度二（纵深进攻）之前，必须先通过本维度 SFT（监督微调）后的排雷模型验尸：
- 任何触发红线的标的，无论市面概念炒得多疯，**物理切断交易路径**
- 无谈判余地、无人情面子，黑名单写入即生效

## 三、决策机制

| 决策类型 | 判定 | 动作 |
|---|---|---|
| `pass` | 未触发任何红线 | 放行进入维度二 |
| `degrade` | 触发中等风险信号 | 进入观察池但限制仓位上限（如 ≤2%） |
| `reject` | 触发任意红线 | 写入永久黑名单 + 风险事件流 + 通知架构师 |

**一票否决**：任意子引擎判定 `reject` 即整体 `reject`，不允许加权平均拉低判定严苛度。

**永久黑名单**：解除需要架构师**手动**覆盖（前端"覆盖确认"按钮 + 二次确认 + 写入审计日志）。

## 四、关键能力清单

1. **财务造假测谎**：识别现金流造假、存贷双高、研发资本化突变、应收/存货周转异动等典型粉饰手法
2. **大股东诚信验尸**：用 RAG 调取 CEO 过去 5 年公开发言/承诺，与实际 CAPEX 流向、并购/减持记录交叉比对
3. **关联交易/明股实债识别**：通过股权穿透 + 财报附注，识别表外结构、关联方循环交易、隐藏负债
4. **商誉减值预警**：监控跨界并购溢价、商誉/总资产比例、被并购方业绩对赌履约率
5. **质押爆仓与控制权稳定性**：监控大股东质押率、股价距平仓线、质权人纠纷诉讼
6. **审计师与监管问询风险**：监控审计师变更/出具非标意见、交易所问询函密度与回复质量
7. **关键人离职/治理崩塌**：监控董秘/CFO/审计/独董短期内多人辞任、核心研发人员领英状态变化
8. **海外监管风险**（中概股专用）：扫描 SEC/FDA/欧盟监管警告、被列入观察名单、ADR 退市风险
9. **舆情与品牌信任崩盘**：监控雪球/小红书/微博的群体性差评、产品退坑率、维权群激增
10. **行业系统性风险**：监控政策/监管/反垄断/双减式黑天鹅，对持仓行业敞口预警

## 五、能力边界（不做什么）

| 边界类型 | 不做的事 | 归属维度 |
|---|---|---|
| 研究结论 | 不产生研究卡片、不打分推荐 | 维度二·纵深进攻 |
| 持仓状态 | 不维护 SLI 探针、不计算健康度评分 | 维度三·持仓监控 |
| 卖出建议 | 不直接产生卖出/调仓建议 | 维度四·卖出决策 |
| 训练流水线 | 不维护 CT 流水线本身（仅消费它的 LoRA） | 维度五·演进飞轮 |
| 外部动作 | 不直接发邮件/Webhook/API 调用（必须经"外部动作边界"） | 维度五·外部动作边界 |
| 自动下单 | 不做任何自动平仓/自动下单 | 与 L1 项目目标冲突，永久禁止 |

## 六、L3 子模块映射

| L3 子模块 | 职责 |
|---|---|
| `risk_event_bus` | 风险事件总线（统一风险事件 schema） |
| `decision_gate` | 决策门禁器（pass/degrade/reject 判决） |
| `circuit_breaker` | 熔断控制器（异常时降级到只读 / 拒绝出站） |
| `input_sanitizer` | 输入污染检测（防 LLM 上下文污染、prompt injection） |
| `audit_log_service` | 审计日志服务（每一次门禁判决可被审计回放） |

## 七、前端接入

- **风险熔断驾驶舱**：实时风险事件、熔断状态、门禁拒绝原因
- **黑名单管理面板**：架构师手动覆盖入口
- **审计回放视图**：单标的的完整门禁判决历史
