## 四、议会模式效果验证（**本阶段新增最重要**）

### 4.1 Judge LLM 仲裁准确率

```python
def eval_judge_accuracy():
    holdout_arbitration = load_holdout("parliament_holdout_v1.jsonl")  # 30 仲裁案例
    judge_lora = load_lora("judge_lora_v1")
    
    correct = 0
    for case in holdout_arbitration:
        judge_decision = call_judge_lora(case["input"])
        if judge_decision["decision"] == case["ground_truth"]:
            correct += 1
    
    accuracy = correct / len(holdout_arbitration)
    assert accuracy >= 0.85, f"Judge accuracy {accuracy} < 0.85, BLOCK"
    return accuracy
```

### 4.2 议会模式 vs 单 LoRA 对比

| 对比项 | 单 LoRA 平均 | 议会模式 | 通过条件 |
|---|---|---|---|
| FN 率（漏判） | X% | ≤ 0.7X% | FN 率下降 ≥ 30% |
| FP 率（误判） | Y% | ≤ 1.1Y% | FP 率上升 ≤ 10% |
| 综合判决正确率 | A | ≥ A + 5pct | 提升 ≥ 5pct |
| Kappa（与架构师一致性） | K | ≥ K + 5pct | — |

### 4.3 议会模式失效场景测试

| 场景 | 期望行为 |
|---|---|
| 15 LoRA 全部 reject | 议会自动 confirm reject，无误纵 |
| 15 LoRA 全部 pass | 议会自动 confirm pass，无误判 |
| 1 LoRA reject + 14 pass | 议会启动复议，根据该 LoRA 理由强度决定 |
| 7 reject + 8 pass | 议会必须仔细仲裁，不能简单多数决 |

### 4.4 议会模式回退机制

| 失效场景 | 回退方案 |
|---|---|
| Judge LLM 推理超时 > 10s | 退化到第二阶段聚合器（无议会） |
| Judge 给出无效 JSON | 退化到第二阶段聚合器 |
| Judge 与训练集分歧严重（每周 > 5 起）| 自动告警 + 暂停议会 + 架构师介入 |

## 五、流水线·完整 CT 闭环端到端验证

### 5.1 端到端 v3 测试场景

```
1. 模拟新案例到达（如 2026 年某新爆雷案例）
2. CT 流水线自动触发：
   2.1 案例库自动入库 + DVC tag
   2.2 受影响引擎自动 Stage B 增量 SFT
   2.3 受影响 LoRA 自动 push staging
   2.4 议会模式自动重新评测（含新 LoRA 后）
3. Holdout 自动评测 + 守门
4. 通过 → 自动 push production
5. 不通过 → 自动告警 + 自动回滚
6. 实盘投资组合自动收到告警 → 维度三/四 接力
```

### 5.2 端到端 SLO

| SLO | 目标 |
|---|---|
| 数据增量 → 议会决策端到端时长 | ≤ 6 小时 |
| 议会决策响应延迟（推理时） | ≤ 8s |
| 自动回滚成功率 | 100% |
| 议会决策审计完整率 | 100%（每个议会决策必须有完整 LoRA + Judge 日志） |
