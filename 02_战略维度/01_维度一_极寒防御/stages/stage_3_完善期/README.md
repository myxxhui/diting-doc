## 一、本阶段一句话目标

**新增 3 个引擎覆盖海外/舆情/系统性风险 + 启用议会模式（多 LoRA 投票 + Judge LLM）+ 全维度 Stage E 完整态**——把维度一从"防住 7 类主流暴雷"扩展到"防住 10 类全谱系暴雷 + 议会式集体决策"。

| 项 | 内容 |
|---|---|
| **时间盒** | 9–12 个月（约 3 个月） |
| **维度地位** | 维度一第三阶段，与维度二/三/四 P2 引擎并行 |
| **依赖** | **第二阶段全部通关** + 维度五 P2 全组件就绪（Argo / DPO / 议会决策器 / 多 LoRA 完整工具链） |
| **成本预算** | 数据采集 ¥1500/月（增量） + Teacher LLM ¥3000/月（议会模式数据合成） |

## 二、本阶段引擎清单

### 2.1 新增引擎（3 个）

| # | 引擎 | 本阶段实现范围 | 准入条件 | 验收主指标 |
|---|---|---|---|---|
| 8 | **海外监管风险**（中概股专项） | 监控 SEC 8-K / 集体诉讼 / FDA / 欧盟反垄断 / 中美数据合规等 | D13 海外监管源就绪；30 案例（瑞幸/Luckin/某中概暴雷案）标注 | Recall ≥ 0.85；Precision ≥ 0.65 |
| 9 | **舆情与品牌信任崩盘** | 监控雪球/小红书/黑猫投诉 / 知乎质疑帖密度 / 品牌负面 NLP | D14 舆情数据就绪；30 案例（如某品牌信任崩盘案）标注 | Recall ≥ 0.78；Precision ≥ 0.55（弱信号容忍噪声） |
| 10 | **行业系统性风险** | 监控政策突变 / 反垄断调查 / 行业整治（双减/教培/集采等） | D15 政策数据就绪；30 案例（双减/集采/反垄断案）标注 | Recall ≥ 0.80；Precision ≥ 0.60 |

### 2.2 全维度·议会模式启用（**本阶段最重要的能力跃迁**）

| 项 | 内容 |
|---|---|
| **议会成员** | 10 引擎 × 多 LoRA = 总计 ≥ 15 LoRA |
| **决策机制** | (1) 各 LoRA 独立判定 → (2) Judge LLM（GPT-4o）阅读所有判定 → (3) Judge 给出最终判决 + 反对意见摘要 |
| **触发条件** | 任何一个引擎判 reject 时启动议会复议（避免单一 LoRA 误判 / 漏判） |
| **议会决策器** | 维度五 P2 组件（在维度五 stage_3 详细描述） |

## 五、本阶段进入"完整态稳态"的"通关条件"

✅ 必须满足全部：
- [ ] E8/E9/E10 Holdout 全部达标
- [ ] 议会模式 Judge LLM 仲裁准确率 ≥ 0.85（在 30 仲裁场景测试集）
- [ ] 议会模式 vs 单 LoRA 综合判决，FN 率下降 ≥ 30%
- [ ] D13–D15 全部 DVC 锁定
- [ ] 10 引擎 + 议会模式综合判决在 100 案例 Holdout 上准确率 ≥ 0.94
- [ ] 维度五 P2 全组件跑通
- [ ] vLLM 同时加载 ≥ 15 LoRA + 推理延迟 < 8s
- [ ] 风险熔断驾驶舱前端 v3 跑通（含议会模式可视化）

❌ 阻断条件：
- 任一新引擎指标退化 > 5%
- 议会 Judge LLM 仲裁准确率 < 0.80
- 议会模式 FN 率反而上升（误判增多）
- vLLM 推理延迟 > 10s
