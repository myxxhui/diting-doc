## 二、数据质量验证

### 2.1 D1·财务三表 Great Expectations 规则

```yaml
# diting-data/cryo_guard/financial/great_expectations.yaml
expectations:
  - expect_column_to_exist: revenue
  - expect_column_values_to_not_be_null: revenue
  - expect_column_values_to_be_between:
      column: revenue
      min_value: 0
      max_value: 1e13
  - expect_column_values_to_be_between:
      column: gross_margin
      min_value: -1
      max_value: 1
  - expect_compound_columns_to_be_unique: [symbol, period]
```

### 2.2 D3·财报附注 OCR 抽样校验

| 项 | 标准 |
|---|---|
| 抽样规模 | 100 份 OCR 结果 |
| 校验方式 | 与原 PDF 人工对比关联交易明细字段 |
| 通过标准 | 准确率 ≥ 95% |
| 不通过 → 处置 | 调整 OCR 模型参数 + 重新跑批 |

### 2.3 D5·案例库 v1 标注质量

| 项 | 标准 |
|---|---|
| Cohen's Kappa | 架构师 verified vs Teacher LLM 输出，Kappa ≥ 0.85 |
| 复核机制 | 每 50 条随机抽 5 条，架构师 30 天后再次 verified |
| 不通过 → 处置 | 调整 Teacher LLM Prompt + 重新蒸馏 |

### 2.4 数据采集 SLO

| SLO | 目标 |
|---|---|
| 实时采集任务（D2/D4）成功率 | ≥ 99.5% |
| 季度任务（D1）成功率 | 100%（财报披露后 +3 天必须完成） |

### 3.4 评测自动化（CT 流水线雏形）

```python
# 每次训练完成后自动跑
def auto_evaluate(lora_path, engine_id):
    holdout = load_holdout(f"diting-data/cryo_guard/holdout/{engine_id}_v1.jsonl")
    metrics = run_holdout(lora_path, holdout)
    
    # MLflow tracking
    mlflow.log_metrics(metrics)
    
    # 守门
    thresholds = THRESHOLDS[engine_id]
    if any(metrics[k] < thresholds[k] for k in thresholds):
        raise BlockReleaseError(f"{engine_id} fails Holdout: {metrics}")
    
    # 推送 Registry
    push_to_registry(lora_path, metrics)
```
