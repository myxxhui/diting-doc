## 一、本阶段一句话目标

**用 3 个引擎拼出"维度一·极寒防御"的最小防御闭环**——能挡住财务造假、大股东不诚信、关联方资金腾挪三类核心暴雷。

| 项 | 内容 |
|---|---|
| **时间盒** | 0–3 个月（90 天） |
| **维度地位** | 维度一第一阶段，与维度五 P0 4 组件并行 |
| **依赖** | 维度五 P0 4 组件先就绪（Teacher 蒸馏 / 数据湖 / Label Studio / LLaMA-Factory） |
| **成本预算** | 数据采集 ¥2500/月 + Teacher LLM ¥3000/月 + GPU 4090 一次性 ¥15000 |

## 二、本阶段引擎清单（速查表）

| # | 引擎 | 本阶段实现范围（一句话） | 准入条件 | 验收主指标 |
|---|---|---|---|---|
| 1 | **财务造假测谎** | 30 案例 SFT，识别 6 类典型粉饰（存贷双高/现金流背离/应收异常/存货积压/研发资本化突变/毛利率异常） | Holdout v1 30 案例就绪；Tushare 财报全量回溯就绪 | Recall ≥ 0.95；Precision ≥ 0.70 |
| 2 | **大股东诚信验尸** | RAG 检索 5 年公告，识别"承诺-实际行为"对子（5 类典型言行不一） | 巨潮公告全量抓取就绪；30 言行不一案例标注完成 | Recall ≥ 0.90；Cohen's Kappa ≥ 0.80 |
| 3 | **关联交易/明股实债** | 财报附注 OCR + 股权穿透；图算法识别循环交易 | 企查查 API 接入；附注 OCR Pipeline 就绪；30 案例标注完成 | Recall ≥ 0.85；Precision ≥ 0.70 |

> **本阶段不做什么**：商誉减值（→ 第二阶段）、质押爆仓（→ 第二阶段）、审计师/问询（→ 第二阶段）、关键人离职（→ 第二阶段）、海外监管/舆情/行业系统性（→ 第三阶段）

## 三、本阶段数据清单（速查表）

| # | 数据类别 | 主要数据源 | 是否就绪条件 |
|---|---|---|---|
| D1 | 财务三表全量（10 年回溯） | Tushare、AKShare | 已采全且 Great Expectations 通过 |
| D2 | 公司公告全文 | 巨潮资讯网 | 实时抓取流就绪 |
| D3 | 财报附注 OCR | 自建 OCR Pipeline（PaddleOCR） | 全量历史 OCR 完成；样本抽查准确率 ≥ 95% |

## 五、本阶段进入第二阶段的"通关条件"

✅ 必须满足全部：
- [ ] 引擎 1 财务测谎：Holdout Recall ≥ 0.95、Precision ≥ 0.70、F1 ≥ 0.80
- [ ] 引擎 2 大股东诚信：Holdout Recall ≥ 0.90、Cohen's Kappa ≥ 0.80
- [ ] 引擎 3 关联交易：Holdout Recall ≥ 0.85、Precision ≥ 0.70
- [ ] D1–D6 数据全部 DVC 锁定 + Great Expectations 通过
- [ ] 案例库 v1 永久 Holdout 30 案例独立锁库（CI 检查通过）
- [ ] 维度五 P0 4 组件全部跑通（Teacher 蒸馏 / 数据湖 / Label Studio / LLaMA-Factory）
- [ ] MLflow 中所有 LoRA 版本可追溯到 DVC commit
- [ ] vLLM 推理网关能加载 3 个 LoRA 并对外服务

❌ 阻断条件（任一触发 → 不进入第二阶段）：
- 任一引擎 Recall 退化 > 5%
- Holdout 被污染（CI 检测到训练集与 Holdout 重叠案例 ID）
- DVC tag 缺失或不可回滚

## 六、本阶段责任与节奏自检

| 责任 | 内容 |
|---|---|
| **架构师** | 每周 2h Label Studio verified + 每月 1 次 Holdout 评测 review |
| **AI（蒸馏）** | Teacher LLM 蒸馏每个引擎 1500–3000 条 + 推送 Label Studio |
| **CI** | 每次 commit 跑 Holdout 隔离检查 + 数据质量检查 |
| **每月 1 日** | 自检"是否按节奏推进"，超期 > 1 周 → 反思会议 |
