## 一、本阶段一句话目标

**新增 4 个引擎覆盖更全面的暴雷类型 + 给原 3 引擎做 DPO 偏好对齐 + 引入多 LoRA 多路复用**——把维度一从"防住 3 类核心暴雷"扩展到"防住 7 类主流暴雷"。

| 项 | 内容 |
|---|---|
| **时间盒** | 3–9 个月（约 6 个月） |
| **维度地位** | 维度一第二阶段，与维度二/三 P1 引擎并行 |
| **依赖** | **第一阶段全部通关条件已满足** + 维度五 P1 5 组件就绪 |
| **成本预算** | 数据采集 ¥1500/月（增量） + Teacher LLM ¥2500/月（DPO + 新引擎蒸馏） |

## 二、本阶段引擎清单（速查表）

### 2.1 新增引擎（4 个）

| # | 引擎 | 本阶段实现范围 | 准入条件 | 验收主指标 |
|---|---|---|---|---|
| 4 | **商誉减值预警** | 监控并购溢价、对赌履约率，识别"商誉减值前 1–2 季度"早期信号 | D7 商誉/并购历史就绪；30 案例标注完成 | Recall ≥ 0.85；Precision ≥ 0.65 |
| 5 | **质押爆仓与控制权稳定性** | 监控大股东质押率、平仓预警线、质权人纠纷 | D8 质押明细就绪；30 案例标注 | Recall ≥ 0.90；Precision ≥ 0.75 |
| 6 | **审计师与监管问询风险** | 监控审计师变更、出非标意见、问询函密度 | D9 审计变更 + D10 问询函就绪；30 案例标注 | Recall ≥ 0.88；Precision ≥ 0.70 |
| 7 | **关键人离职/治理崩塌** | 监控董秘/CFO/审计/独董短期内多人辞任 | D11 高管离职公告就绪；30 案例标注 | Recall ≥ 0.80（弱信号容忍较高 FN）；Precision ≥ 0.60 |

### 2.2 原 3 引擎升级范围

| # | 引擎 | 本阶段升级动作 | 升级目标 |
|---|---|---|---|
| 1 | 财务测谎 | 引入 DPO + 多 LoRA（行业细分：金融/制造/科技/医药）+ Stage B 增量训练 | Recall 维持 ≥ 0.95，**Precision ↑ 到 ≥ 0.80** |

## 五、本阶段进入第三阶段的"通关条件"

✅ 必须满足全部：
- [ ] 4 新引擎（E4–E7）Holdout 全部达标
- [ ] 原 3 引擎 DPO 后 Precision 提升至少 5pct（Recall 不退化）
- [ ] D7–D12 数据全部 DVC 锁定
- [ ] 多 LoRA 多路复用在 vLLM 中可同时加载 7 个 LoRA
- [ ] 一票否决聚合器升级版（7 引擎）在 Holdout 上的综合判决正确率 ≥ 0.92
- [ ] 维度五 P1 5 组件全部跑通（K8s GPU Job / vLLM 网关 / MLflow Registry / 评测回放器 / DPO 流水线）
- [ ] CT 流水线自动触发（数据增量 → 训练 → Holdout → 灰度部署）能跑通

❌ 阻断条件：
- 任一新引擎 Recall < 阈值
- 任一原引擎 Recall 退化 > 5%
- DPO 训练后出现"灾难性遗忘"（关键能力丧失）
