## 项目特化顺序

### Step 1：确认产品边界
- 当前服务谁
- 当前最关键的问题是什么
- 哪些能力必须先有，哪些可以后补

### Step 2：建立数据底座
- 原始资料采集
- 清洗与结构化
- thesis 与证据数据模型

### Step 3：跑通主业务链路
- 输入 → 检索 → Agent → 研究卡片 → 跟踪提醒

### Step 4：替换推理底座
- 商业 API → 混合推理 → 自托管推理

### Step 5：引入 Runtime 能力
- 结构化动作
- 沙箱
- 长任务
- 资源回收

### Step 6：补齐评测与发布闭环
- Golden set
- 回归测试
- 灰度发布
- 回滚策略

## 协作要求

每一步都必须产出：
- 明确输入与输出
- 可运行结果
- 指标或验证方法
- 为什么这样做的设计说明
