# L2 · 八个月开发与交付节奏

> [!NOTE] **[TRACEBACK] 战略维度锚点**
> - **顶层概念**: [个人AI成长目标与训练节奏](../../01_顶层概念/05_个人AI成长目标与训练节奏.md)

## 阶段 A：Month 1-2

目标：先跑通业务闭环。

- 数据采集
- 研究摘要
- thesis 跟踪
- 提醒机制
- 建立第一版产品指标体系
- 明确需求优先级框架与上线门禁

这一阶段必须开始训练岗位 2，而不是等系统做大后再补：

- 定义效果、效率、成本、风险四类指标
- 定义什么进入 `P0` / `P1` / `P2`
- 为后续灰度、扩量、回滚准备决策口径

### 阶段 A 的前两周（MLOps 起手式）

- Week 1：完成云端 K3s 基础环境（GPU 可调度 + MinIO 三桶）。
- Week 2：完成 `1 基座 + 1 LoRA` 的训练部署调用闭环。
- 验收口径：从本地工作流成功调用云端 LoRA 推理，并可回放完整操作记录。

## 阶段 B：Month 3-5

目标：替换推理底座，主攻岗位 1。

- 自托管推理
- `vLLM`
- `Triton` / `TGI` 对比或预留
- 基础 K8s / K3s
- 观测与成本记录

这一阶段岗位 1 必须覆盖到：

- GPU 调度与利用率治理
- 推理性能优化
- 分布式训练与 checkpoint
- 至少一段 RTC / 低延迟流式交互基础

## 阶段 C：Month 6-7

目标：引入 Agent Runtime，主攻岗位 4。

- 结构化动作或代码生成
- 隔离执行
- 长任务状态管理
- 资源配额与回收

这一阶段岗位 4 必须覆盖到：

- WebSocket 大规模长连接
- 会话恢复
- browser / computer-use runtime
- `seccomp` / `nsjail` / `eBPF` 一类细粒度安全能力

## 阶段 D：Month 8

目标：补齐评测与发布闭环。

- 评测集
- 灰度与回滚
- 回归验证
- 简历与项目故事沉淀

这一阶段岗位 3 必须覆盖到：

- `MLflow`
- `Kubeflow` / `Airflow`
- `KServe` / `BentoML`
- feature store
- drift detection

同时岗位 2 要在真实迭代中完成：

- 基于指标决定是否上线
- 基于成本和效果决定是否继续投入
- 基于组织协作边界拆分产品、工程、评测职责

## 集群演进路线（全阶段约束）

1. 本地研发：以工作流编排和产品验证为主，不做重型本地推理。
2. 云端 K3s：承担 MVP 训练与推理闭环，快速低摩擦迭代。
3. 阿里云 ACK：承接规模化并发、弹性调度与流水线自动化发布。
