## 一、维度四的特殊性

维度四的"训练数据"主要来自**真实持仓的退出归因记录**，而不是 Teacher LLM 蒸馏。这意味着：
- **Stage A 启动期较短**（用规则系统 + 少量人工标注就能 cold start）
- **Stage B 至关重要**（每次卖出后的事后归因都是训练增量）
- **整体训练成本远低于其他维度**（决策类，不是生成类）

## 二、维度四 5 阶段训练范式

| 阶段 | 名称 | 关键动作 | 数据增量来源 | 训练方式 | 预期能力跃升 |
|---|---|---|---|---|---|
| **A** | 启动期·规则 + 少量人工标注 | R1/R2/R3 用规则系统启动，无需 LLM 蒸馏 | 30 个历史卖出案例（人工标注） | 规则引擎 + LightGBM 多因子加权模型 | 能给出基础的退出建议 |
| **B** | 增量·真实退出归因反馈 | 每次卖出后架构师写归因 → 半年累积 50–100 案例 | 真实退出归因案例库 | LightGBM 增量训练 | 加权策略动态优化 |
| **C** | DPO·偏好对齐 | "AI 退出建议 vs 架构师最终决定" 对子 | 架构师在前端的"覆盖确认"记录 | DPO（针对 LLM 解释部分） | LLM 解释更贴合架构师风格 |
| **D** | 多 LoRA 并行 | 不同行业/持仓周期的退出策略独立 LoRA | 各行业独立训练集 | vLLM 多 LoRA | 行业敏感度 |
| **E** | 议会模式 | R1/R2/R3 + LLM 解释 + Judge LLM | 实盘退出归因 | 议会式 ensemble | 综合退出建议置信度 |

## 四、永久 Holdout 评测集

| 项 | 内容 |
|---|---|
| **大小** | 60 案例（30 卖对了 + 30 卖错了） |
| **构成** | 跨剧本、跨行业、跨持仓周期 |
| **主指标** | **建议命中率 ≥ 0.65**（AI 建议方向与"事后正确决策"一致） |
| **副指标** | **R1 escalate 召回率 ≥ 0.90**（关键逻辑破坏不能漏报） |
| **守门规则** | 每次新版本上线前回放 Holdout |

## 五、训练评测的特殊挑战

| 挑战 | 应对 |
|---|---|
| 卖出决策的"对错"需要事后较长时间验证 | 用历史回测代替（找过去 5 年的卖出决策做事后归因） |
| 真实数据天然稀缺（私人项目卖出案例少） | Stage A 用规则启动 + 少量人工标注；Stage B 持续积累真实案例 |
| 不同市场风格对"卖对了"的定义不同 | Holdout 跨牛熊，避免过拟合到单一市场风格 |
| 架构师"覆盖确认"可能本身就有偏差 | 定期与 Holdout 对照检查架构师的偏差 |
