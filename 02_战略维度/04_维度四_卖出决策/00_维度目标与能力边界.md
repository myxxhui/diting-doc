## 一、战略目标

把"卖出决策"从"靠感觉/被市场情绪绑架"变成"由结构化规则触发"。借鉴 SRE 的 PagerDuty 思路：**任何卖出建议都必须有可追溯的规则触发链**。

**铁律**：
- 不允许"心情不好就卖" / "看见股价跌了就慌"
- 不允许"看好就一直拿"——逻辑破坏 → 必须强制 review

## 二、3 重退出矩阵

| 矩阵 | 触发器 | 决策动作 |
|---|---|---|
| **R1·逻辑破坏熔断** | 维度三的核心 SLI 严重违反 | **强制要求架构师 24h 内 review**，给出"否决"或"清仓"决策 |
| **R2·估值过载止盈** | 估值（PE / PB / DCF 隐含 IRR）超过预设上界 | **建议"分批止盈"**，由架构师决定具体节奏 |
| **R3·机会成本调仓** | 当前持仓的预期回报 < 候选标的预期回报 - 风险溢价 | **建议"调仓"**，由架构师决定 |

**任意一条触发 → 进入"退出审议池"**；不会自动卖出。

## 三、决策机制

| 决策类型 | 判定 | 动作 |
|---|---|---|
| `hold` | 三重矩阵都未触发 | 持仓继续 |
| `review` | R2 / R3 触发 | 推 review 提示给架构师 |
| `escalate` | R1 触发 | 强制 24h 内人工 review |
| `auto_pass` | 某些非常明确的逻辑破坏（如造假定性） | 不需人工，直接 escalate 到清仓审议 |

**永远不自动下单**：所有卖出动作必须人工确认。

## 四、关键能力清单（7 引擎）

1. **逻辑破坏熔断**：消费维度三的 critical 信号，触发 R1
2. **估值过载止盈**：基于反向 DCF / 行业历史估值分位，触发 R2
3. **机会成本调仓**：基于全市场扫描的"更好机会"做对比，触发 R3
4. **分批止盈策略生成**：当 R2 触发时，给出"分 N 批、每批 X%"的具体节奏建议
5. **税费成本优化**：考虑 A 股交易成本/印花税/资本利得税（港股/美股）做净收益计算
6. **多因子退出聚合**：把 R1+R2+R3 加权综合
7. **退出回放归因**：每次卖出后，事后归因"这次卖出是对/错的，因为...."（用于训练）

## 五、能力边界（不做什么）

| 边界 | 不做的事 | 归属 |
|---|---|---|
| 自动下单 | 不直接调用券商 API | 永久禁止 |
| 短线判断 | 不做日内/周内技术性卖出 | 与 L1 冲突 |
| 持仓监控 | 不做 SLI 探针执行 | 维度三 |
| 风险熔断（造假） | 不做造假识别 | 维度一 |
| 模型训练 | 不维护 CT 流水线 | 维度五 |

## 六、L3 子模块映射

| L3 子模块 | 职责 |
|---|---|
| `exit_engine` | 卖出决策引擎（3 重退出矩阵的核心） |
| `valuation_calculator` | 估值计算器（反向 DCF / 行业分位） |
| `opportunity_scanner` | 机会扫描器（全市场扫候选标的） |
| `multi_factor_aggregator` | 多因子退出聚合 |
| `exit_attribution_engine` | 退出归因引擎 |
| `human_confirmation_gate` | 卖出确认门禁（前端） |

## 七、前端接入

- **卖出审议池**：所有 review/escalate 状态的持仓
- **卖出确认按钮**：架构师手动确认入口
- **退出归因视图**：每次卖出后的事后归因记录
