## 一、为什么必须有这个维度

### 1.1 没有它，整个项目"看上去很美但用不上"

| 问题 | 后果 |
|---|---|
| 5 维度后端引擎都跑通，但你不会主动登录服务器跑命令 | 价值释放率 < 10% |
| 系统知道你的持仓出问题了，但没有渠道告诉你 | 你"知道"和"被动收到通知"差着一个雷的距离 |
| 用了一年看不到"系统为我赚了/避了多少" | 没信任 → 不持续用 → 投入归零 |
| 新手 vs 老手用法应该不同，但只有"统一 API" | 永远只有架构师本人能用，无法扩展 |

### 1.2 它解决"产品-技术的最后一公里"

**所有的"好技术"都失败在最后一公里**——把 LoRA / vLLM / Agent 转化为"对真实用户有用的产品"。

维度零就是这最后一公里。

## 二、对你·50 万本金的具体价值清单

> 模拟"第一阶段全维度交付后"，这个产品对你日常的具体帮助：

### 2.1 价值 ①·避雷（最高优先级，因为本金永远不可逆）

| 你能得到的具体帮助 | 来自 |
|---|---|
| 你想买的票，5 秒钟告诉你"能不能买"（reject/degrade/pass） + 理由 | 维度一 P0 3 引擎 |
| 你已持仓的票，每日扫描，发现暴雷信号立刻微信告警 | 维度一 + 维度三 P0 |
| 你周末 review 持仓，系统给一份"持仓体检报告"标出风险等级 | 维度零 + 维度一 + 维度三 |
| 系统帮你建立"永久黑名单"——一旦判 reject 永远不会再向你推荐 | 维度一 + 维度零·决策日志 |

**避险价值定量**：按 50 万本金、单标的最多 5%（即 ¥2.5 万）、平均潜在回撤 50%——**避开 1 个雷 = 节省 ¥1.25 万**。一年避 5 个雷 = ¥6.25 万。

### 2.2 价值 ②·找机会（次优先级，提升收益）

| 你能得到的具体帮助 | 来自 |
|---|---|
| 每周一 8 点收到"本周推荐池"邮件 PDF：3-5 只候选 + 每只 thesis 卡 | 维度二 P0 + 维度零·周报 |
| thesis 卡上有：买入逻辑、目标价、止损价、SLI 探针、置信度 | 维度二 P0 |
| 系统优先推荐你历史 verified 偏好风格的标的（保守/平衡/进取） | 维度五 P0 + 维度零·偏好学习 |
| 不看 K 线、不看公众号——一份周报够你做这周决策 | 维度零·主动通道 |

**收益价值定量**：按 50 万本金、平均仓位 80% 投入 thesis 推荐、跑赢沪深 300 5pct/年 = **额外收益 ¥2 万/年**。

### 2.3 价值 ③·盯持仓（最容易被忽视，但最致命）

| 你能得到的具体帮助 | 来自 |
|---|---|
| 你买入时填一份"为什么买"（5 项 thesis 要素），系统持续监控 | 维度三 P0 |
| 任何一项 thesis 假设破裂（如毛利率突降、CEO 减持）→ 30 秒内微信告警 | 维度三 + 维度零·紧急告警 |
| 每日邮件简报你 "持仓 5 只今日异动 + 系统判断"（你忙时一眼扫过即可） | 维度零·日报 |
| 周末邮件提供 "持仓 thesis 偏离评分"，让你判断是否减仓/清仓 | 维度三 + 维度零·周报 |

**盯持仓价值定量**：A 股最大的损失来自"明明该卖却没卖" + "明明出问题却没发现"。系统帮你早 3-5 天发现一次重大利空 = 大概率避免 -15% 到 -30% 的回撤 = 单只标的省 ¥3000 - ¥7500。

### 2.4 价值 ④·学习（长期价值，越用越准）

| 你能得到的具体帮助 | 来自 |
|---|---|
| 你每次 verified 反馈（赞/踩/修正）都进飞轮，系统下周更懂你 | 维度五 P0 + 维度零·反馈闭环 |
| 你不需要"训练它"——只需要"用它+给反馈"，飞轮自己转 | 维度五 |
| 一年后这个系统比通用 GPT 更懂"你的投资风格" | 维度五·DPO + 维度零·决策日志 |

### 2.5 价值 ⑤·价值证明（让你确信这个系统值得继续投入）

| 你能得到的具体帮助 | 来自 |
|---|---|
| 每月初收到"价值账本月报"：本月避险 ¥X、收益 ¥Y、总价值 ¥Z | 维度零·价值账本 |
| 每个决策都可追溯：日期、系统建议、你的操作、T+90 后结果 | 维度零·决策日志 |
| 你可以理性判断"是否继续用 / 何时升级到下一阶段" | 维度零 |

## 三、产品价值主线·三模演进

> 这条主线**贯穿前中后期**，把整个 diting 项目从 IDE 玩具升级为真实可用的副驾驶。

### 3.1 第一阶段·**观察者模式**（0–3 个月）

```
系统建议 ─→ 你看 ─→ 你 100% 自主决定 ─→ 你的操作进决策日志
```

| 项 | 内容 |
|---|---|
| **核心价值** | 让你"知道"——避雷 + 找机会 + 盯持仓的所有信息你都能感知到 |
| **使用强度** | 极低（每天 5 min 看邮件 / 每周 30 min 看周报）|
| **风险** | 0（系统不操作任何资金） |
| **典型场景** | 周一看周报 + 周三紧急告警 + 周日 verified 5 条 |
| **价值释放** | 50%（你必须自己执行才有价值；忙起来可能漏掉建议） |

### 3.2 第二阶段·**副驾驶模式**（3–9 个月）

```
系统生成"焦点列表" + "待执行订单草稿" ─→ 你点【一键确认】或【调整后确认】 ─→ 系统辅助你下单
```

| 项 | 内容 |
|---|---|
| **核心价值** | 把"看建议 → 思考 → 操作"从 30 分钟压缩到 5 分钟 |
| **使用强度** | 中（每周一次"批量决策会"，5 分钟批量执行）|
| **风险** | 极低（每个动作都需要你点击确认）|
| **典型场景** | 周一 9 点打开 Web，看推荐池，3 只标的批量加仓 |
| **价值释放** | 80% |

### 3.3 第三阶段·**自动驾驶模式·限额版**（9–12 个月）

```
你设定"自动驾驶仓位"（如 30%，即 15 万）─→ 系统在该额度内基于议会模式自主调仓 ─→ 重大动作通知你 ─→ 70% 仓位仍手动
```

| 项 | 内容 |
|---|---|
| **核心价值** | 让 AI 24/7 在限定额度内运作，你只管整体策略 |
| **使用强度** | 月度 review 1 次 |
| **风险** | 限额内可控（最多 30% 仓位 = 15 万）|
| **典型场景** | 月底打开"价值账本"，看自动驾驶仓位本月跑赢手动仓位多少 |
| **价值释放** | 95%+ |

## 四、本维度的成功标准（如何证明它"值"）

| 阶段 | 量化目标 |
|---|---|
| **第一阶段** | 你能持续用 ≥ 12 周不断用（不放弃）；月度避险价值 ≥ ¥3000；不漏 1 个紧急告警 |
| **第二阶段** | 你的"批量决策会"时长 ≤ 5 min；副驾驶建议接受率 ≥ 50%；月度收益价值 ≥ ¥2000 |
| **第三阶段** | 自动驾驶 30% 仓位跑赢手动 70% 仓位；总价值（避险+收益）/年 ≥ ¥10 万 |

> **判定标准**：上述目标任一不达成 → 暂停升级，回头优化产品体验，**而不是去开更多后端引擎**。

## 五、本维度对其他维度的"反向需求"

维度零作为产品骨架，会**反向塑造**其他维度的设计：

| 维度 | 维度零的反向需求 |
|---|---|
| 维度一 | 必须输出"reject + 1 句话理由 + 链接到详情"的标准化结构（不能只丢 JSON） |
| 维度二 | 推荐数量必须可控（每周 ≤ 5 个，不能洪水）；thesis 卡必须有 5 项标准要素 |
| 维度三 | 紧急程度必须分级（红/橙/黄/灰），与告警通道一一对应 |
| 维度四 | 卖出建议必须给出"完整止盈/止损价 + 减仓节奏"，不能只说"建议减持" |
| 维度五 | 必须暴露"verified API"和"偏好对 API"给维度零的反馈闭环 |

> 这些反向需求是后端维度第一阶段必须满足的硬约束，否则 MVP 出不了产品。
