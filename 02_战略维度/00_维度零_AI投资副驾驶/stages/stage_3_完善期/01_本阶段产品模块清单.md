### 4.1 电话告警触发条件

| 触发 | 通道 |
|---|---|
| 自动驾驶仓位异常 P&L > 5% | 电话 + 微信 + Telegram |
| 自动驾驶 24h 内连续 2 次 abstain | 电话（询问用户调整偏好）|
| 议会模式无法达成一致超 7 天 | 电话（议会失效预警）|

### 4.2 与自动执行联动

```python
def handle_broken_strong_constraint(holding):
    """自动驾驶仓位的强约束 broken 联动"""
    if holding.is_in_auto_pilot:
        # 立刻发送红色告警
        send_red_alert(holding)
        
        # 缓冲期 5 个交易日
        for day in range(5):
            if user_has_overridden(holding):
                return  # 用户已接管
            wait(1_day)
        
        # 缓冲期到，系统询问后自动执行
        if get_user_consent_via_phone(holding):
            execute_sell(holding, sell_type="logic_break_exit")
        else:
            # 用户拒绝 → 移出自动驾驶仓位
            move_to_manual_pilot(holding)
```

## 七、新增：议会模式投票合议

| 项 | 详细 |
|---|---|
| **触发** | 自动驾驶仓位需要决策时 |
| **投票者** | 4 个维度的独立 LoRA（维度一防御 / 维度二进攻 / 维度三监控 / 维度四卖出）|
| **路由** | Judge LLM 根据决策类型路由到相关维度 |
| **决策规则** | 维度一一票否决；其他维度 4 票一致 → approve；否则 abstain |
| **决策日志** | 每次议会决策入 `parliament_log` 表，含 4 维度投票轨迹 |
| **追溯** | Web 详情页可看议会投票时序 |
