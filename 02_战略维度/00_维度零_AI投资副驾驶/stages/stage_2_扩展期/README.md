## 一、本阶段一句话目标

**从观察者模式升级为副驾驶模式**——把"看建议 → 思考 → 操作"从 30 分钟压缩到 5 分钟，启动反馈闭环让飞轮自转，引入券商 API 自动同步持仓。

| 项 | 内容 |
|---|---|
| **时间盒** | 3–9 个月（180 天） |
| **产品模式** | 副驾驶模式（系统生成订单草稿 → 用户一键确认 → 系统辅助下单）|
| **维度地位** | 与维度三完整化（7.6-7.9 节）+ 维度四完整化（4 类协议）并行 |
| **依赖前置** | 阶段 1 全部 SLO 达标 + 连续 12 周用户活跃 + 维度三/四完整化就绪 |
| **成本预算** | 同阶段 1 + 券商 API 接入 ¥0 (个人版免费) |

|---|---|---|---|---|

| 2 | **推荐池与 thesis 卡** | + 候选池横向比较（多 thesis 并排）+ 历史推荐回测可视化（"上月推 5 只目前表现"）| 阶段 1 决策日志满 3 个月历史 | 横向比较加载 < 2s |
| 3 | **紧急告警** | + 用户偏好的静默时段（夜间/周末橙色合并到次日 9 点）+ 上下文丰富（关联其他持仓影响）| 用户偏好配置就绪 | 静默时段合规率 100% |

## 三、本阶段新增的"批量决策会"

```
副驾驶模式核心交互·每周一上午:
  
08:00  📧 收到周报 (与阶段 1 同)

09:00  打开 Web → "本周决策中心" (新增页面)
       系统已准备好:
         - 推荐池待决策 5 条 (thesis 卡 + 系统建议)
         - 持仓调仓 4×4 矩阵建议 3 条 (减仓/加仓建议)
         - verified 待标注 5-10 条 (上周决策)
       
09:05  你在 Web 上批量操作:
         - 推荐池: 勾选 AAA / BBB 加入持仓 (订单草稿生成)
         - 持仓调仓: 勾选 XX 减仓 50% (订单草稿生成)
         - verified: 5 条赞 + 2 条踩 + 1 条修正
       
09:08  确认后系统生成"批量执行清单":
         "本次批量决策清单:
          1. 买 AAA ¥3万
          2. 买 BBB ¥3万
          3. 卖 XX 50% (约 ¥1.6 万)
          总计 5 笔操作, 预估耗时 5 分钟"
       
09:10  你点击 [一键打开券商 App 批量执行]
       (券商 API 半自动协助下单)

09:13  执行完成, 系统自动回写决策日志
```

**目标**：从原来"周一花 30 分钟做决策"压缩到"周一花 5 分钟做决策"。

## 五、本阶段进入第三阶段的"通关条件"

✅ 必须满足全部：
- [ ] 反馈闭环 + 双盲 Kappa 标定上线连续 3 月，Kappa ≥ 0.85
- [ ] 券商 API 自动同步持仓上线，同步延迟 < 1h，连续 4 周无失败
- [ ] 副驾驶批量决策会 ≤ 5 分钟（用户在 10 次连续使用中 ≥ 8 次满足）
- [ ] 副驾驶建议接受率 ≥ 50%（决策日志统计）
- [ ] 月度收益价值 OV ≥ ¥2000
- [ ] 月度 SCS ≥ 60 连续 3 月
- [ ] 4×4 调仓矩阵建议命中率 ≥ 60%（建议被采纳后的 T+30 结果验证）

❌ 阻断条件（任一触发 → 不进入第三阶段）：
- Kappa < 0.85（用户与系统不一致 → 训练数据有问题）
- 用户接受率 < 30%（系统建议与用户偏好失配）
- 批量决策会平均 > 10 分钟（产品体验失败）
- 月度 SCS < 50 连续 2 月
- 飞轮训练失败导致维度一/二/三任一引擎降级 > 5%

## 六、本阶段责任与节奏自检

| 责任 | 内容 |
|---|---|
| **架构师** | 每周 1 次 verified（10 条 ≈ 30 min）+ 每月双盲 Kappa 1 次 + 每月调仓矩阵建议命中率审查 |
| **AI（产品实现）** | 5 子模块按周节奏交付 + 批量决策会优化迭代 + 反馈闭环数据周度推送维度五 |
| **CI** | + Kappa 自动计算 + 调仓矩阵命中率自动统计 |
| **每月 1 日** | 自检"批量决策会用时 + Kappa 趋势 + 接受率" |

## 七、阶段 2 的核心风险

| 风险 | 缓解 |
|---|---|
| 用户对"飞轮越训越偏"的担心 | 双盲 Kappa < 0.70 自动暂停训练；月报展示训练数据来源 + 路由 |
| 券商 API 自动同步出错（如错过买卖）| 同步失败 → 立即邮件告警 + 回退到手动维护 |
| 4×4 调仓矩阵建议过于激进 | 矩阵单元的 action 与维度三 04_ §五完整对齐；架构师月度审查 |
| 批量决策会变"无脑点击" | 监控用户的"取消勾选率"；过低 → 提示"你是否在认真审查" |
