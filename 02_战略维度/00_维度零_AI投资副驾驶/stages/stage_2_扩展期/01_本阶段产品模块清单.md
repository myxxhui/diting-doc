## 七、批量决策会（**全新引入·产品形态升级**）

| 项 | 详细 |
|---|---|
| **触发频率** | 每周一 09:00（与周报同步触发）|
| **目标耗时** | ≤ 5 分钟 |
| **页面位置** | Web `localhost:8080/weekly-decision` |
| **核心组件** | (1) 推荐池待决策 (2) 持仓调仓建议 (3) verified 待标注 |
| **半自动下单** | 生成"批量执行清单" → 用户一键打开券商 App + 系统协助 |
| **取消勾选率监控** | 监控用户的 deliberation；过低（< 10%）提示"是否在认真审查"|

### 7.1 批量决策会三组件

```
组件 A·推荐池待决策:
  - 复选框列表，每行 1 个 thesis 卡 + 简介
  - 勾选 → 进入"订单草稿"
  - 默认顺序：confidence 降序
  - 上限：5 条/周

组件 B·持仓调仓建议:
  - 复选框列表，每行 1 个标的 + 调仓动作 + 理由
  - 勾选 → 进入"订单草稿"
  - 优先级：broken_any > weakening_on_target > strong_high > ...

组件 C·verified 待标注:
  - 5-10 条上周决策 (含八象限初判)
  - 用户标注 ✅/❌/⚠️ + 可选评论
  - 进 DPO 偏好对池
```
