## 一、本阶段一句话目标

**用 4 个子模块（不含反馈闭环）拼出"AI 投资副驾驶"的最小产品闭环**——让一个 50 万本金的个人投资者第一天打开就有用、3 个月后能感知到具体的避险与收益价值。

| 项 | 内容 |
|---|---|
| **时间盒** | 0–3 个月（90 天） |
| **产品模式** | 观察者模式（系统建议 → 用户 100% 自主决定 → 系统记录）|
| **维度地位** | 与维度一 P0 / 维度五 P0 并行（消费维度一/二/三/四的事件流）|
| **依赖前置** | 维度一 P0 三引擎就绪 + 维度二 P0 一引擎（利润截留）就绪 + 维度三 P0 健康度就绪 + 维度五 P0 4 组件就绪 |
| **成本预算** | Web 服务 ¥0 (本地) + 邮件 ¥50/月 + 微信/Telegram ¥0 + SQLite ¥0 |

## 二、本阶段子模块清单（速查表）

| # | 子模块 | 本阶段实现范围（一句话） | 准入条件 | 验收主指标 |
|---|---|---|---|---|
| 1 | **持仓体检报告** | Web 首屏 4 色卡片 + 单持仓详情（节点 4 态 + 健康度曲线 30 天）| 维度三 health_change 事件流就绪；用户持仓维护页就绪 | 首屏加载 < 1s；4 色与 push_level 严格一致 |
| 2 | **推荐池与 thesis 卡** | Web 推荐池页 + 单 thesis 卡详情（5 必填元素）+ 3 操作（加入/待考虑/不感兴趣）+ PDF 导出 | 维度二 thesis_proposed 事件流就绪；维度一 PassEvent 关联就绪 | 单周推荐数 ≤ 5；5 必填元素完整性 100% |
| 3 | **紧急告警系统** | 4 红 + 2 橙告警 + 微信群机器人 + Telegram Bot + 邮件 | 推送通道（微信 webhook + Telegram bot token）配置就绪 | 红色 5 分钟到达率 ≥ 99.5%；单日 ≤ 3 红 |
| 4 | **价值账本** | SCS + EV 双指标 + 8 象限归因 + 月报 PDF + 自我熔断 | 维度三逻辑链状态 API 就绪；归因调度器就绪 | T+30 归因延迟 ≤ 1h；月报准时率 100% |
| 5 | 反馈闭环 | （**本阶段不做**，第二阶段启动）| — | — |

> **本阶段不做什么**：反馈闭环 verified（→ 第二阶段）、券商 API 自动同步（→ 第二阶段）、电话告警（→ 第三阶段）、自动驾驶限额（→ 第三阶段）、议会模式（→ 第三阶段）

## 三、本阶段数据/事件流清单（速查表）

| # | 数据/事件流 | 主要来源 | 是否就绪条件 |
|---|---|---|---|
| D1 | `events:cryo_guard:reject` / `degrade` / `pass` | 维度一 P0 引擎 | 维度一 P0 三引擎首次跑通 + Stream 配置就绪 |
| D2 | `events:thrust:thesis_proposed` | 维度二 P0（利润截留扫描仪） | 维度二 P0 引擎首次跑通 + thesis 卡 5 必填元素 schema 落地 |
| D3 | `events:monitor:health_change` | 维度三 P0（叙事一致性 + SLI 调度器）| 维度三 P0 跑通 + 节点 4 态状态机 API 就绪 |
| D4 | `events:monitor:rebalance_advice` | 维度三 P0 | （阶段 1 仅记录，不在 Web 突出显示，阶段 2 启用）|
| D5 | `events:exit:sell_signal` | 维度四 P0 | 维度四 P0 跑通；4 类卖出协议 schema 落地 |
| D6 | `events:flywheel:lora_updated` | 维度五 P0 | 维度五 P0 跑通；首次 LoRA 训练成功（阶段 1 中后期）|
| D7 | 价格行情（T+0 收盘价）| Tushare / AKShare | 每日 17:00 后抓取 |
| D8 | 用户持仓数据 | **手动维护**（Web 维护页 + Excel 导入）| 维护页 + 导入工具就绪 |

> **数据治理铁律**：所有决策日志写入立刻入 SQLite + 永久保留；事件流 trace_id 全链路保留用于审计。

## 五、本阶段进入第二阶段的"通关条件"

✅ 必须满足全部：
- [ ] 子模块 1·持仓体检：首屏加载 < 1s；4 色与 push_level 严格一致连续 4 周
- [ ] 子模块 2·推荐池：单周推荐数 ≤ 5；5 必填元素完整性 100%；PDF 导出成功率 ≥ 99%
- [ ] 子模块 3·紧急告警：红色 5 分钟到达率 ≥ 99.5%；单日 ≤ 3 红
- [ ] 子模块 4·价值账本：T+30 归因延迟 ≤ 1h；月报准时率 100%；连续 2 月 SCS ≥ 60
- [ ] 用户连续 12 周持续使用不放弃（用户活跃留存 verified）
- [ ] 月度避险价值 ≥ ¥3000（基于真实持仓与 reject 事件）
- [ ] 不漏 1 个紧急告警（事件触发到投递成功的 5 分钟到达率 ≥ 99.5%）

❌ 阻断条件（任一触发 → 不进入第二阶段）：
- 任一 SLO 退化 > 20%
- 月度 SCS < 30 连续 2 月
- 用户主动暂停使用（说明产品体验失败）
- 5 必填元素被规避（任一 thesis 缺元素进入推荐池）
