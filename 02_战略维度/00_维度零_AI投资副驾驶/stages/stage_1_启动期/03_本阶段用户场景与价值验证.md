## 一、本阶段用户场景全景

> 第一阶段是**观察者模式**——系统建议 + 用户 100% 自主决定 + 系统记录。
> 验证的核心是：**"我（50 万本金用户）能否被系统主动找到，看懂建议，做出决策，并在月报上看到具体价值？"**

## 二、典型一周完整时间线（端到端验证）

### 周一·一周开局（30 min）

```
08:00  📧 收到周报邮件 (PDF 一页)
       打开手机 30 秒扫读:
         [持仓体检] 5 只里 1 只 reject (XX), 1 只 degrade (YY), 3 只 pass
         [推荐池] 本周新增 3 只候选: AAA / BBB / CCC + 各自 thesis 卡
         [上周价值账本] 避险 ¥1500 / 收益 ¥1200 / 总 ¥2700

09:30  打开 localhost:8080 (10 min 决策时间)
       - 看 XX (reject) 详细理由 → 决定清仓
       - 看 AAA / BBB / CCC 的 thesis 卡（5 必填元素）→ 决定买 AAA + BBB
       - 在 Web 上点 [加入持仓] → 决策日志自动记录

10:00  手动到券商 App 下单 (清 XX, 买 AAA / BBB)
       回 Web 点 [已执行] (5 sec)
       → 决策日志写入 user_action: taken + 实际买入数量/价格
```

**验证点**：
- ✅ 周报 PDF 8:00 准时收到
- ✅ Web 首屏 1 秒内加载完成
- ✅ thesis 卡 5 必填元素清晰可读
- ✅ 决策日志记录完整 + 可追溯

### 周二·正常工作日（10 sec）

```
09:00  📧 日报邮件 (10 秒扫一眼)
       "持仓 5 只今日异动: AAA +2.3%, BBB -1.1%, ZZ -0.5%
        系统判断: 全部正常, 无操作建议"
       不需要打开 Web, 不需要任何动作
```

**验证点**：
- ✅ 日报 9:00 准时
- ✅ 内容简洁（≤ 30 行）
- ✅ 不打扰用户工作

### 周三·突发红色告警（5 min 决策）

```
14:32  📱 微信紧急告警 (30 秒看完)
       "⚠️ XX (002xxx) 触发维度一 reject
        理由: 大股东今日公告减持 3% (3 月内第二次)
        建议: 立刻评估减持
        你当前持仓: ¥3.2 万
        详情 → http://localhost:8080/alert/A-2026-001"

14:35  打开链接看完整理由 (1 min)
14:40  手动到券商 App 减持
14:45  回 Web 点 [已执行]
```

**验证点**：
- ✅ 红色告警 5 分钟到达率 ≥ 99.5%
- ✅ 微信失败时 Telegram 兜底 < 30s
- ✅ 详情链接打开 < 1s
- ✅ 决策日志含 user_action: executed + 30 分钟内的操作记录

### 周四·verified 反馈日（**阶段 1 不做，阶段 2 启动**）

> 本阶段不启用 verified 面板。改为：
> ```
> 决策日志静默记录所有"系统建议 → 用户操作"对子，
> 阶段 2 开启 verified 后批量回填 verified 状态。
> ```

### 周日晚·复盘（5 min）

```
20:00  打开价值账本 Web 页 (5 min)
       看本周决策列表:
         - 周一 [加入 AAA] taken (¥3 万) → T+7 +5%
         - 周一 [加入 BBB] taken (¥3 万) → T+7 -1.1%
         - 周三 [减持 XX] taken (¥3.2 万) → T+5 -8% (避雷成功)
       系统建议: "AAA 接近 thesis 第 1 节点验证, 继续持有"
```

**验证点**：
- ✅ 决策列表展示完整 + 可下钻
- ✅ T+30/60/90 归因调度器按时跑

### 月初·价值证明（10 min）

```
每月 1 日 08:00  📧 月报邮件 (PDF 5 页)
                  
P.1·5 句话价值证明:
  1️⃣  本月 SCS: 65/100 → 强（首月达标）
  2️⃣  本月做了 8 个决策: A+F 占比 50%
  3️⃣  EV: +¥3500（避雷 ¥2200 + 收益 ¥1300）
  4️⃣  价值三角: 安全 ✅ / 确定 ✅ / 收益 ✅
  5️⃣  系统建议: 继续使用
```

**验证点**：
- ✅ 月报 1 日 08:00 准时
- ✅ 5 句话价值证明清晰
- ✅ SCS ≥ 60（首月达标）+ EV > 0

## 五、可能的失败场景与应对

### 5.1 用户嫌"什么都不能买" → 防御过敏

```
原因: 维度一 reject 配额未管控好
应对:
  1. 检查 reject 配额（基石⑤ §5.3 "防御的不过度原则"）
     - 日/周/月 reject 占总候选 ≤ 50%
     - 认知边界 reject 占比 ≤ 30%
  2. 超出 → 触发引擎过敏审查
  3. 调整阈值
```

### 5.2 用户主动不打开 Web → 主动通道失效

```
原因: 主动通道推送内容质量低 / 频次失控
应对:
  1. 检查日报内容（≤ 30 行 + 关键信息）
  2. 检查推送频次（单日红色 ≤ 3 条）
  3. 用户访谈：哪些推送是"必看"哪些是"骚扰"
  4. 调整告警阈值
```

### 5.3 用户对"系统建议"不信任 → 价值证明不足

```
原因: SCS 不稳定 / EV 长期为负
应对:
  1. 检查月报 5 句话是否如实呈现
  2. 检查自我熔断是否触发
  3. 不掩饰失败：H 象限案例月报明示 + 复盘
  4. 信任建立靠"诚实"而非"美化"
```

### 5.4 红色告警漏到达 → SLO 失败

```
原因: 微信/Telegram/邮件三通道全部故障
应对:
  1. 单一通道失败 → 30s 内切换
  2. 三通道全失败 → 紧急邮件 + 短信通知架构师
  3. 月报附"通道可靠性"页
  4. 阶段 3 引入电话兜底
```

## 六、本阶段对用户的"承诺清单"

| 承诺 | 怎么实现 |
|---|---|
| **我每天打开就有用** | 日报每日 9:00 + 紧急时随时推 |
| **我每周末复盘 10 分钟就够** | 周报 PDF 一页 + Web 决策日志可下钻 |
| **我永远不会被系统伤害** | 自我熔断机制 + 不强制执行 |
| **我能看到具体的避险与收益** | 月报 5 句话价值证明 + EV 累计趋势 |
| **不漏 1 个紧急告警** | 三通道冗余 + 5 分钟 SLO 99.5% |
| **不打扰我** | 单日红色 ≤ 3 + 日报可关 + 静默时段（阶段 2）|
| **每条建议都可解释** | thesis 卡 5 必填元素 + 节点 SLI 探针可下钻 |
| **系统会承认失败** | H 象限案例月报明示 + 不掩饰 |

## 七、阶段 1 验收的"5 必看"

> 阶段验收时，请用户必看以下 5 个场景：

1. **看一次完整周报**（10 min）：用户独立判断 thesis 卡是否清晰、价值账本是否可信
2. **看一次红色告警的端到端**（5 min）：从微信到达到用户做决策
3. **看一次月报 5 句话**（5 min）：用户独立判断"是否说服我继续使用"
4. **看一次决策详情下钻**（5 min）：看八象限归因 + 节点状态时序
5. **看一次自我熔断**（如未触发，模拟一次）：用户判断系统是否"诚实"

> 5 个场景中任一让用户"感觉不对" → 暂停阶段 2，回头优化产品体验。
