## 二、子模块 1·持仓体检报告

| 项 | 详细 |
|---|---|
| **本阶段范围** | Web 首屏 4 色卡片 + 单持仓详情卡（节点 4 态展示 + 30 天健康度曲线）|
| **不做什么** | 自动券商 API 同步（→ 阶段 2）、战场分配饼图（→ 阶段 3）|
| **关键 UI 元素** | 总览栏：SCS + EV；4 色风险卡片列表；详情下钻 |
| **数据需求** | Redis 订阅维度一/三事件流 + 用户手动维护持仓表 |
| **关键交互** | 点击红色 → 跳转告警详情；点击单持仓 → 节点状态详情 |
| **SLO** | 首屏 < 1s；事件触发到 Web 显示 < 10s |
| **承接哲学** | ①价值三角 + ⑤防御 + ⑦持仓监控 |

### 2.1 4 色路由规则

| 颜色 | 触发 push_level | UI 表现 |
|---|---|---|
| 🔴 | `emergency_red` 或 `regular_red` | 红色卡片置顶 + 闪烁 |
| 🟠 | `orange` | 橙色卡片次之 |
| 🟡 | `yellow` | 黄色卡片 |
| 🟢 | 健康度 ≥ 0.80 + 全节点正常 | 绿色卡片折叠展示 |

## 三、子模块 2·推荐池与 thesis 卡

| 项 | 详细 |
|---|---|
| **本阶段范围** | Web 推荐池页 + 单 thesis 卡详情（5 必填元素强制展示）+ 3 操作（加入/待考虑/不感兴趣）+ PDF 导出 |
| **不做什么** | 候选池横向比较（→ 阶段 2）、历史回测可视化（→ 阶段 2）、自动下单草稿（→ 阶段 3）|
| **5 必填元素**（严格展示） | ①逻辑链节点（≥3，1 个强约束）②SLI 探针映射 ③战场窗口期 ④收益门槛+目标/止损价 ⑤认知边界检查 |
| **三门槛检查** | confidence ≥ 0.70 / payoff_ratio ≥ 2.0 / win_rate ≥ 0.55，缺一不进 |
| **SLO** | 单周推荐数 ≤ 5；PDF 导出成功率 ≥ 99% |
| **承接哲学** | ②工程化 + ③时间边界 + ⑥进攻 |

### 3.1 推荐池准入闸门

```
ThesisProposedEvent 进入 →
  ① 5 必填元素完整性检查 (任一缺失 → 退回事件源 + 告警)
  ② 三门槛检查 (confidence/payoff/win_rate)
  ③ 维度一 PassEvent 关联检查 (必须有有效的 defense_check_event_ref)
  ④ 通过 → 进推荐池
```

## 四、子模块 3·紧急告警系统

| 项 | 详细 |
|---|---|
| **本阶段范围** | 4 类红色 + 2 类橙色告警 + 三通道（微信群机器人 / Telegram bot / 邮件）+ 用户回执（3 按钮）|
| **不做什么** | 静默时段配置（→ 阶段 2）、电话告警（→ 阶段 3）、与自动执行联动（→ 阶段 3）|
| **过载防护** | 单日红色 ≤ 3 条（超出合并）+ 同标的 24h 同类去重 |
| **通道冗余** | 微信失败 → Telegram 兜底 → 邮件兜底（< 30 秒切换）|
| **SLO** | 红色 5 分钟到达率 ≥ 99.5%；误告警率 < 0.1% |
| **承接哲学** | ①价值三角 + ⑤防御 + ⑦持仓监控 + ⑧卖出决策 |

### 4.1 告警内容模板（4 红 + 2 橙）

| 颜色·类型 | 触发 Stream | 消息要素 |
|---|---|---|
| 🔴 红·1 持仓 reject | `cryo_guard:reject` (持仓中) | 标的 + reject_type + 1 句话理由 + 当前持仓金额 + 详情链接 |
| 🔴 红·2 强约束 broken | `monitor:health_change` (broken + 强约束) | 标的 + 哪个节点 broken + 缓冲期到期日 + 详情链接 |
| 🔴 红·3 暴雷前兆 | `cryo_guard:degrade` (多源汇聚→reject) | 标的 + 多源证据简述 + 详情链接 |
| 🔴 红·4 自动驾驶异常 | 自动驾驶 P&L 异常 | （本阶段不启用）|
| 🟠 橙·1 限时机会 | `thrust:thesis_proposed` (orange) | 标的 + thesis 摘要 + 时间窗口 + 详情链接 |
| 🟠 橙·2 thesis 偏离 | `monitor:health_change` (orange) | 标的 + 偏离评分 + 详情链接 |

## 五、子模块 4·价值账本

## 七、本阶段不做的产品功能（明示边界）

| 不做 | 理由 |
|---|---|
| ❌ 反馈闭环 verified 面板 | → 阶段 2（先保证产品骨架稳定再开飞轮回路）|
| ❌ 双盲 Kappa 标定 | → 阶段 2 |
| ❌ 自动从券商 API 同步持仓 | → 阶段 2（先用手动维护降低集成复杂度）|
| ❌ 战场分配饼图 | → 阶段 2（持仓数据满 3 个月才有意义）|
| ❌ 4×4 调仓矩阵建议 UI | → 阶段 2（先记录建议，不在 UI 主推）|
| ❌ 移动 App | 永不（基石⑥ 不做技术派；邮件 + 微信够）|
| ❌ K 线图 / 技术分析 | 永不（基石⑥ 不做技术派）|
| ❌ 多用户系统 | → 阶段 3 之后 |
| ❌ 电话告警 | → 阶段 3（仅自动驾驶仓位异常）|
| ❌ 自动驾驶限额执行 | → 阶段 3 |

## 八、本阶段产品演进的"哲学激活度"自检

| 哲学基石 | 本阶段激活度 | 主要载体 |

| ① 价值三角 | **完整** | Web 首屏先安全后收益；月报核心页 |

| ③ 时间边界 | **基础** | thesis 显式战场（但战场审计需 3 月数据积累，阶段 2 启用）|

| ⑤ 防御 | **完整** | 持仓 reject 4 色 + 紧急告警 |
| ⑥ 进攻 | **核心** | 推荐池 ≤ 5 + 5 必填 |
| ⑦ 持仓监控 | **核心** | 节点 4 态 + 健康度（4×4 调仓矩阵 UI 推到阶段 2）|

| ⑨ 演进 | **基础** | 首次 LoRA 训练 + 月度 SCS 展示（无 verified 闭环，等阶段 2）|
