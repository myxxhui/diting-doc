## 一、契约总览

### 1.1 事件流总线

| 中间件 | 选型 | 用途 |
|---|---|---|
| **事件流总线** | Redis Streams | 5 后端维度 → 维度零的所有事件 |
| **持久化** | SQLite（决策日志）+ PG（业务数据）| 维度零内部 |
| **降级缓存** | Redis Pub/Sub | Stream 不可用时的备份 |

### 1.2 事件 Stream 命名规范

```
events:{dimension}:{event_type}

示例:
  events:cryo_guard:reject           # 维度一 reject 事件
  events:cryo_guard:degrade          # 维度一 degrade 事件
  events:thrust:thesis_proposed      # 维度二推荐 thesis
  events:monitor:health_change       # 维度三健康度变更
  events:monitor:rebalance_advice    # 维度三调仓建议
  events:exit:sell_signal            # 维度四卖出信号
  events:flywheel:lora_updated       # 维度五 LoRA 更新
```

### 1.3 通用事件 envelope

```yaml
event_envelope:
  event_id: str               # UUID v4
  event_type: str             # 见各维度章节
  source_dimension: enum      # cryo_guard | thrust | monitor | exit | flywheel
  timestamp: ISO8601
  payload: object             # 各事件类型独有
  trace_id: str               # 跨维度追踪
  schema_version: str         # 兼容性管理
```

## 二、维度一 → 维度零（reject/degrade/pass 事件）

### 2.1 RejectEvent

> **来源 L1 基石⑤**：防御哲学边界——宁可错杀。

**Stream**: `events:cryo_guard:reject`

```yaml
RejectEvent:
  event_id: str
  symbol: str                       # 标的代码（如 002xxx）
  name: str                         # 标的名称
  reject_type: enum                 # financial_fraud | governance | related_party | goodwill | pledge | regulatory | overseas | sentiment | systemic | cognitive_boundary
  severity: enum                    # high | medium | low
  
  # 哲学要求（来自基石⑤）
  cognitive_boundary_passed: bool   # 是否在能力圈内
  multi_source_evidence:            # 多源汇聚证据
    - source_engine: str            # 触发的引擎名
      signal_strength: float        # 0-1
      evidence_link: str
  
  triggered_engines: list[str]      # 触发的引擎清单（多源汇聚证据）
  
  # 维度零展示用
  human_readable_reason: str        # 一句话理由（中文，≤ 30 字）
  detail_url: str                   # Web 详情页链接
  
  # 持仓相关
  is_in_holdings: bool              # 用户当前是否持仓
  current_position_value: float | None  # 当前持仓金额
  
  # 推送路由
  push_level: enum                  # emergency_red | regular_red | orange | yellow
  push_channels: list[str]          # ["wechat", "telegram", "email"]
```

### 2.2 DegradeEvent

> 防御未达 reject 但触发关注。

**Stream**: `events:cryo_guard:degrade`

```yaml
DegradeEvent:
  event_id: str
  symbol: str
  name: str
  degrade_reasons: list[str]
  recommended_action: enum          # reduce_position | watch | wait
  push_level: enum                  # 通常为 orange | yellow
```

### 2.3 PassEvent

> 通过防御检查，**不主动推送**，仅用于"用户主动询问"或"周报体检"。

**Stream**: `events:cryo_guard:pass`（仅记录，不触发推送）

```yaml
PassEvent:
  event_id: str
  symbol: str
  cognitive_boundary_passed: bool
  defense_score: float
```

### 2.4 维度一的 SLO

| SLO | 目标 |
|---|---|
| reject 事件生成延迟（自数据可用） | ≤ 5 分钟 |
| 持仓中标的 reject 推送到达 | ≤ 5 分钟（用户感知 < 1 小时）|
| 多源汇聚证据完整性 | reject 事件须含 ≥ 2 个引擎证据（除"认知边界 reject"外）|

## 三、维度二 → 维度零（thesis 卡推荐事件）

> **来源 L1 基石⑥**：进攻哲学边界——5 元素必填、宁少不滥。

### 3.1 ThesisProposedEvent

**Stream**: `events:thrust:thesis_proposed`

```yaml
ThesisProposedEvent:
  event_id: str
  thesis_card_id: str
  symbol: str
  name: str
  
  # === L1 基石⑥ 强制 5 必填元素 ===
  logic_chain:                      # 元素 1: 逻辑链节点
    nodes:
      - node_id: str
        description: str            # 中文，≤ 50 字
        is_strong_constraint: bool
        sli_probes: list[str]       # 元素 2: SLI 探针映射
        weight: float               # 0-1
  
  battlefield: enum                 # 元素 3: 战场（short/main/mid/long）
  window_days: int                  # 元素 3: 窗口期
  min_return_threshold: float       # 元素 4: 最低收益门槛
  target_price: float
  stop_loss_price: float
  
  cognitive_boundary_check:         # 元素 5: 认知边界检查
    passed: bool
    rationale: str
  
  # === 哲学量化要求（来自基石⑥）===
  confidence: float                 # ≥ 0.70
  expected_payoff_ratio: float      # ≥ 2.0
  historical_win_rate: float        # ≥ 0.55
  
  # 维度零展示用
  thesis_summary: str               # 中文一段话（≤ 200 字）
  thesis_pdf_url: str               # 完整 thesis PDF
  
  # 推送路由（通常进周报，限时机会进橙色告警）
  push_level: enum                  # weekly_batch | orange | yellow
  
  # 校验
  must_pass_defense_check: bool     # 必须先过维度一安检
  defense_check_event_ref: str      # 关联的 PassEvent ID
```

### 3.2 维度二的 SLO

| SLO | 目标 |
|---|---|
| 单周新增推荐数 | ≤ 5 个（基石⑥ 强约束）|
| thesis 卡的 5 必填元素完整性 | 100%（缺失任一 → 拒绝接收）|
| thesis 卡的 confidence ≥ 0.70 | 100% |
| thesis 卡必须先过维度一 PassEvent | 100%（维度一 reject 的标的不进入推荐池）|

## 四、维度三 → 维度零（监控/调仓事件）

> **来源 L1 基石⑦**：持仓监控哲学边界——逻辑链 + 持仓策略 + 动态调仓三柱。

### 4.1 HealthChangeEvent

**Stream**: `events:monitor:health_change`

```yaml
HealthChangeEvent:
  event_id: str
  thesis_card_id: str
  symbol: str
  
  # 健康度状态
  previous_health: float
  current_health: float
  health_level: enum                # strong | normal | weakening | broken
  
  # 节点状态变更
  changed_nodes:
    - node_id: str
      previous_state: enum          # active | validated | weakened | broken
      current_state: enum
      evidence: str
  
  # 推送路由
  push_level: enum                  # emergency_red | regular_red | orange | yellow
  # 映射规则:
  #   broken + strong_constraint  → emergency_red
  #   health < 0.30               → regular_red
  #   health [0.30, 0.50)         → orange
  #   health [0.50, 0.80)         → yellow
```

### 4.2 RebalanceAdviceEvent

**Stream**: `events:monitor:rebalance_advice`

```yaml
RebalanceAdviceEvent:
  event_id: str
  thesis_card_id: str
  symbol: str
  
  # 来自维度三 4×4 调仓矩阵
  matrix_cell: str                  # 如 "strong_high" | "weakening_on_target" | ...
  action: enum                      # hold_or_add | hold | partial_take_profit | reduce_30 | reduce_30_50 | reduce_50_plus | exit
  suggested_ratio: float            # 减仓比例（0-1）
  reason: str                       # 中文解释
  
  # 上下文
  current_health: float
  current_return: float
  battlefield: enum
  window_remaining_days: int
  
  push_level: enum                  # 通常为 yellow（进周报）或 orange（紧急减仓）
```

### 4.3 BattlefieldAuditEvent（月度）

**Stream**: `events:monitor:battlefield_audit`（每月 1 日触发）

```yaml
BattlefieldAuditEvent:
  event_id: str
  audit_date: date
  
  distribution:                     # 当前战场分配
    short: float
    main: float
    mid: float
    long: float
  
  healthy: bool                     # 整体是否健康
  alerts:                           # 超配/欠配预警
    - battlefield: enum
      current: float
      healthy_range: [float, float]
      direction: enum               # overweight | underweight
      severity: enum
  
  # 收益仓库状态（来自基石⑦·收益仓库控制边界）
  gain_vault:
    value: float
    status: enum                    # negative | positive_below_safety | at_safety | above_2x_safety
    experimental_position_cap: float
  
  push_level: orange                # 进月报；硬上限超配进紧急告警
```

### 4.4 维度三的 SLO

| SLO | 目标 |
|---|---|
| 强约束 broken 事件生成延迟 | < 1 小时（实盘事件 < 5 分钟）|
| 健康度计算频率 | 节点 SLI 自带 schedule（事件/日/周/季）|
| 月度战场审计准时率 | 100%（每月 1 日 06:00 前）|
| 调仓建议生成延迟（自健康度变更） | < 10 分钟 |

## 五、维度四 → 维度零（卖出建议事件）

> **来源 L1 基石⑧**：卖出决策哲学边界——4 类正确卖出 + 1 类错误卖出。

### 5.1 SellSignalEvent

**Stream**: `events:exit:sell_signal`

```yaml
SellSignalEvent:
  event_id: str
  thesis_card_id: str
  symbol: str
  
  # === 卖出类型（4 类正确卖出之一，来自基石⑧）===
  sell_type: enum                   # take_profit | logic_break_exit | opportunity_cost_reset | battlefield_failure_exit
  
  # 触发上下文
  trigger:
    health: float
    return_pct: float
    quadrant: enum                  # A | B | C | D | E | F | G | H
    in_window: bool
    strong_constraint_broken: bool
  
  # 卖出建议
  recommendation:
    action: enum                    # full_exit | partial_exit_50 | partial_exit_30
    urgency: enum                   # immediate | within_24h | within_week
    reason: str
  
  # 卖飞豁免（哲学要求）
  sell_fly_immunity_window_days: int  # 默认 180
  
  # 上游引用
  health_change_event_ref: str      # 关联的 HealthChangeEvent
  
  push_level: enum                  # emergency_red | regular_red | orange
```

### 5.2 维度四的 SLO

| SLO | 目标 |
|---|---|
| 强约束 broken 后缓冲期到达 → SellSignal 生成 | 5 个交易日内 |
| 健康度 < 0.3 触发 SellSignal | < 1 小时 |
| 4 类卖出标注完整性 | 100% |
| 卖出建议附"卖飞豁免说明" | 100% |

## 七、降级与失败处理

### 7.1 后端事件流不可达时

| 故障 | 维度零响应 |
|---|---|
| Redis Stream 不可用 | 切换 Redis Pub/Sub 备份；超 5 分钟 → 邮件告警架构师 |
| 维度一离线 | 暂停推荐池；持仓监控继续；Web 显示"防御引擎离线，仅紧急告警可用" |
| 维度二离线 | 周报无新推荐；其他正常 |
| 维度三离线 | **不允许**——这是核心；触发紧急告警 + 暂停所有卖出建议 |
| 维度四离线 | 卖出建议由维度三直接接管（降级）|
| 维度五离线 | 飞轮训练暂停；其他正常 |

### 7.2 事件 schema 不兼容

```
schema_version 不匹配时:
  1. 维度零内部转换器尝试 N-1 兼容
  2. 转换失败 → 进入 dead_letter_queue
  3. 每日 1 次合并通知架构师
  4. 严重情况触发紧急告警
```

### 7.3 推送通道失败

| 失败 | 自动切换 |
|---|---|
| 微信 → Telegram → 邮件 | ≤ 30 秒 |
| 全部通道失败 | Web 显示"未投递告警" + 累计 ≥ 3 条触发紧急邮件 |
