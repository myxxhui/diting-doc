## §3 具体规则 / 阈值 / 矩阵

### §3.1 fact_gate 4 档矩阵（与 L3 _共享规约/22 §2 一致）

| tier | status | 条件 | 置信度上限 | 后续动作 |
|---|---|---|---|---|
| 1 | verified_strong | ≥ 2 个 independence ≥ 0.7 独立源 + raw_excerpt | 1.0 | 写库 + thesis 可引 |
| 2 | verified_weak | 1 个 primary 或 2 个同集团 | 0.55 | 写库但 thesis 不可锚定 valuation |
| 3 | pending_more_evidence | 0 primary 或 LLM 无 URL | 0.20 | 暂存 + 24h 重爬 |
| 4 | dlq_rejected | 多源矛盾 或 URL 失效 | 0 | DLQ + 告警 |

### §3.2 戴维斯 4 档判定（与 L3 D2 step_11 §3.5.2 一致）

| phase | 触发（AND） | 业务含义 |
|---|---|---|
| davis_double_click | pe_percentile_180d ≥ 0.8 + pe_trend_30d>0 + eps_growth ≥ +5% | 业绩 + 估值共振主升浪 |
| single_click_eps | eps_growth ≥ +5% + pe_percentile_180d ≤ 0.3 | 业绩没被认可的估值修复机会 |
| double_kill | eps_growth ≤ -5% + pe_percentile_180d ≤ 0.3 + pe_trend_30d<0 | 戴维斯双杀冬天 |
| neutral | 其他 | 中性观望 |

### §3.3 market_phase 4 档判定（与 L3 D3 step_09 §3.5.1 一致）

| phase | 优先级 | 触发（AND） | 业务含义 |
|---|---|---|---|
| exhaustion | 1（最先判） | (pct60>80% + vol_ratio>2.5 + pct5<0) 或 (media>30/7d + pct3<0 + 跌 MA10) 或 (realization 后 10d + 无新利好 + pct5<-5%) | 量价背离 / 媒体高潮 / 后续乏力 → 资金出货 |
| realization | 2 | has_q_report 或 has_pre_announce 或 has_major_contract（窗口 -3 ~ +5 trading_day） | 公告兑现窗口主升浪共振 |
| expectation | 3 | (phys_probe alert ≥ 1 或 pct30>30%) + 无公告兑现 + vol_ratio>1.5 | 资金博弈预期等公告 |
| concept | 4 | pct60<20% + vol_ratio<1.5 + phys_probe alert=0 + 无公告利好 | 仅概念叙事，资金未实质进场 |

### §3.4 thesis 卡 schema 4 新字段 + break_signals[]（与 L3 D2 step_05 §3.5.5 一致）

| 字段 | 类型 | 必填 | 来源 |
|---|---|---|---|
| bottleneck_node | object（node_type/node_name/industry_chain_position）| ✅ | LLM 起草 + fact_gate.verify |
| business_elasticity_score | int[0,100] | ✅ | 引 08_业绩弹性闸门 输出 |
| technology_curve_stage | enum 5 档 | ✅ | LLM 起草 + fact_gate.verify |
| valuation_dynamics | object | 启动期 nullable，step_11 落地后必填 | step_11 写入 |
| break_signals[] | list ≥ 1 | ✅ | LLM 起草 + 用户审 |

### §3.5 4 档 × 4 档协作矩阵（thesis 健康度 × market_phase）

| thesis 4 态 \ market_phase | concept | expectation | realization | exhaustion |
|---|---|---|---|---|
| INITIAL | 观察 | 加速尽调 | 已上车晚到 | 退出（叙事+资金双失） |
| GROWING | 持有 | 持有 + 等公告 | **主升浪持有 / 加仓** | **优先减仓**（即使逻辑还在）|
| STABLE | 持有 | 持有 | 持有 | **优先减仓**（资金已撤）|
| EXIT | 已退 | 已退 | 已退（错失主升浪） | 已退（最佳决策）|

> **关键发现**：第三列「STABLE × exhaustion」 是 D4 SP6 协议要覆盖的卖出场景—— **逻辑还在但资金已撤**——这是当前系统在没有本规划前漏掉的最大卖点。
