## 四、关键能力的协作场景

| 场景 | 涉及的 L3 服务 | 说明 |
|---|---|---|
| **5 必填元素 thesis 卡 finalize** | `research_council_service.finalize_thesis` | 三门槛强制 + 5 必填强制 |
| **战场 × thesis 类型分类** | `signal_feature_engine.classify_battlefield` | 自动推荐战场 |
| **5 步生成工作流** | `agenda_orchestrator` 协调 5 服务 | 议程 → 内容 → 特征 → 议会 → finalize |
| **认知边界检查集成** | `research_council_service` 调 `decision_gate.cognitive_boundary_check`（cryo_guard）| 跨模块协作 |
| **与维度一 PassEvent 关联** | `research_council_service` 强制 30 天内 PassEvent | 闸门集成 |
| **节点 4 态状态机注册** | `candidate_registry` 通过后 → state_watch `state_machine_registry` 注册 | 跨模块协作 |
| **A/G/H 归因** | super_evo `eval_replay_service` 触发 | T+window 后归因 |

## 五、部署形态建议

| 服务 | 部署形态 | 资源 |
|---|---|---|
| `content_comprehension_service` | 独立微服务（含 LLM 调用）| 2 vCPU / 8GB RAM |
| `signal_feature_engine` | 独立微服务（批 + 流）| 2 vCPU / 4GB RAM |
| `research_council_service` | 独立微服务（MoE / Agent 编排）| 4 vCPU / 16GB RAM（含 LoRA 推理）|
| `agenda_orchestrator` | 独立微服务（cron/事件触发）| 1 vCPU / 2GB RAM |
| `expectation_gap_quantifier` | 独立微服务 | 1 vCPU / 2GB RAM |
| `candidate_registry` | 独立微服务（PG 后端）| 1 vCPU / 2GB RAM |

## 七、上下游事件流

### 7.1 维度二输出（被其他维度消费）

```yaml
ThesisProposedEvent:        消费方: 维度零推荐池 + 维度三注册节点状态机
ThesisAttributionEvent (T+window):  消费方: 维度五 A/G/H 路由
```

### 7.2 维度二输入（消费其他维度）

```yaml
events:cryo_guard:pass:     维度一 PassEvent → thesis finalize 闸门
events:flywheel:lora_updated:  维度五 LoRA 守门人通过后更新
events:user:verified:        维度零反馈闭环 → 影响下次训练
```
