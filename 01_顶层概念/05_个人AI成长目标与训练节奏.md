## 成长目标

你不是在做一个普通 side project，而是在用副业为年底跳槽构建真实工程证据。

## 训练优先级

### 主线
1. 岗位 1：AI 基础设施研发工程师
2. 岗位 4：智能体基础设施工程师

### 辅线
3. 岗位 3：MLOps
4. 岗位 2：AI 技术负责人

## 四岗位必须覆盖能力

### 岗位 1：必须补齐到底层
- GPU 调度与资源治理
- 推理性能优化
- `Triton` / `TGI` / `vLLM`
- 分布式训练与 checkpoint
- K8s 多租户与成本治理
- RTC / 低延迟实时链路基础

### 岗位 4：必须补齐到运行时与安全细节
- WebSocket 大规模长连接
- 会话恢复与长任务状态管理
- browser / computer-use runtime
- `seccomp` / `nsjail` / `gVisor` / `Firecracker`
- `namespace` / `cgroup` / `eBPF`
- 多租户隔离、配额、回收

### 岗位 3：必须补齐标准 MLOps 工具链
- `MLflow`
- `Kubeflow` / `Airflow`
- `KServe` / `BentoML`
- feature store
- drift detection
- 灰度、回滚、CT、质量门禁

### 岗位 2：必须补齐负责人能力
- 产品指标体系
- 需求优先级框架
- 上线决策与风险门禁
- 组织协作与跨角色对齐
- AI 功能商业化判断
- 从业务问题到 Prompt / Workflow / Eval 的闭环拆解

## 8 个月训练节奏

### Month 1-2：业务闭环
- 跑通数据采集、研究摘要、跟踪提醒
- 目标：先让系统真正可用
- 同时开始补岗位 2：定义产品指标、优先级框架、上线门禁与功能 ROI 判断

### Month 3-5：推理基础设施
- 从商业 API 逐步迁到自托管推理
- 引入 `vLLM`、`Triton` / `TGI`、基础观测与 K8s
- 开始 GPU 调度、性能优化、分布式训练与 checkpoint 训练
- 视产品形态补一段低延迟链路 / RTC 基础能力
- 主练岗位 1

### Month 6-7：Agent Runtime
- 让模型产出结构化动作或安全代码
- 引入隔离执行、长任务状态管理、资源配额
- 补齐 WebSocket 大规模长连接、会话恢复、browser runtime
- 进入 `seccomp` / `nsjail` / `eBPF` 这一层的细粒度安全能力
- 主练岗位 4

### Month 8：评测与发布闭环
- 建立评测集、灰度、回滚、回归验证
- 引入 `MLflow`、`Kubeflow` / `Airflow`、`KServe` / `BentoML`、feature store、drift detection
- 用产品指标与上线决策框架，把岗位 2 的负责人能力落到真实迭代里
- 补齐岗位 3，强化岗位 2

## 每阶段必须有的输出物

- 可运行代码
- 文档与架构说明
- 指标或验证结果
- 能写进简历的工程案例
- 能在面试中复述的设计取舍
- 能分别对应岗位 1 / 2 / 3 / 4 的能力证据
