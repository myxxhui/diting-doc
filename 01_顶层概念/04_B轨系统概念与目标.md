## 一、产品是什么

谛听的产品侧，不再定义为“某个收益目标驱动的交易系统”，而是定义为：

**一个面向个人投研与后续可产品化演进的 A 股分析追踪智能平台。**

它的第一阶段不是替你全自动交易，而是替你完成以下高价值工作：

- 把海量非结构化信息整理成可消费的研究素材
- 帮你先做“防雷”和“预筛”
- 帮你追踪核心假设是否仍成立
- 在需要关注或退出时及时提醒

## 二、产品四大战略维度

### 1. 确定性防御战略（The Firewall）

目标：优先降低本金永久损失的概率。

这一维度解决：

- 财报粉饰、逻辑自相矛盾、业务恶化
- 关键负面新闻或监管事件漏看
- 把本不该进入候选池的标的挡在外面

产品能力表现为：

- 财报 / 公告 / 新闻风险扫描
- 一票否决条件
- 风险证据卡片
- 风险优先级排序

### 2. 产业爆发力战略（The Engine）

目标：从“信息噪声”里识别真正值得持续跟踪的赔率来源。

这一维度解决：

- 产业链核心环节判断
- 需求、渗透率、产能、订单、政策共振识别
- 市场认知与真实产业变化之间的预期差发现

产品能力表现为：

- 行业 / 产业链穿透分析
- 新闻与财报联合解读
- 主题与标的映射
- 候选标的的逻辑归因

### 3. 全生命周期跟踪战略（The Observer）

目标：解决“买完以后没人持续跟”的问题。

这一维度解决：

- 初始买入逻辑与后续现实变化脱节
- 缺少持续观测，结果变成拍脑袋持有
- 卖出原因不清晰，复盘无法积累

产品能力表现为：

- 为每个标的定义 thesis 与 SLI
- 自动跟踪关键指标
- 逻辑破坏 / 风险事件 / 估值过热 / 机会成本替换提醒
- 退出矩阵与复盘记录

### 4. 自进化战略（The Flywheel）

目标：让产品随着使用不断变强，而不是始终停留在第一版。

这一维度解决：

- Prompt、工作流、RAG 效果无法稳定提升
- 复盘结果无法反哺系统
- “做了一堆 Agent，但不知道变好没有”

产品能力表现为：

- 人工纠错沉淀
- 标注集与评测集积累
- Prompt / Workflow 版本对比
- 微调、灰度、回滚和回归验证

## 三、产品边界

当前产品阶段明确不做以下事情：

- 不把“全自动下单”作为第一阶段核心目标
- 不追求一开始就覆盖所有投资风格
- 不为了复杂而先做重型 SaaS 后台
- 不为了平台化而放弃你自己作为第一用户的反馈速度

换句话说：**先把自己服务好，再为未来平台化留接口。**

## 四、MVP 到平台化的演进

### Phase A：个人研究助手

- 数据采集
- 研究摘要
- 风险与机会卡片
- 手动跟踪清单

### Phase B：结构化跟踪系统

- thesis 建档
- 跟踪指标自动更新
- 事件驱动提醒
- 退出建议记录

### Phase C：多 Agent 工作流平台

- 财报 Agent
- 新闻 Agent
- 产业链 Agent
- 统一的编排、审阅与证据引用

### Phase D：可产品化平台

- 多租户
- 权限与隔离
- 自托管推理
- 发布、评测、成本治理

## 五、产品成功标准

### 对你本人

- 系统真的能节省分析时间
- 系统真的能发现肉眼容易漏掉的风险
- 系统真的能帮你持续追踪逻辑，而不是只在买入前有用

### 对未来产品化

- 有清晰的用户价值主线
- 有清晰的数据资产沉淀
- 有清晰的工作流可复制
- 有清晰的技术护城河可以继续下沉
