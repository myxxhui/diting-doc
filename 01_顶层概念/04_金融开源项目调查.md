## 文档目的

本调查覆盖 **数据聚合、定价与协议、策略与回测、交易执行与运维、资产配置与风控、个人/财富管理、区块链与结算** 等维度，筛选 **工程化程度高、社区活跃** 的顶级开源项目，供技术选型与架构借鉴。每个项目均补充 **2024–2025 年最新动态** 及与谛听模块的映射关系。

**与战略的关系**：本调查为**技术选型与架构借鉴**的参考输入；选型结果应在 L2 [02_技术栈与架构维度](../02_战略维度/产品设计/02_技术栈与架构维度.md)、[03_数据架构与分层存储维度](../02_战略维度/产品设计/03_数据架构与分层存储维度.md) 等维度中体现，并与 L1 不可能三角、双轨制、可追溯目标一致。本调查不替代 L2/L3 的决策，仅支撑可追溯的选型依据。

## 一、数据与信息聚合

### 1. OpenBB (原 OpenBB Terminal / Gamestonk Terminal)

| 维度 | 内容 |
|------|------|
| **一句话定义** | “开源届的彭博终端”——现代化投资研究一体化平台，聚合股票、加密货币、外汇与宏观经济数据。 |
| **核心功能** | CLI + Python SDK；复杂图表与报告生成；上百个免费/付费数据源对接。 |
| **技术栈** | Python；支持 Windows / macOS / Linux / Raspberry Pi。 |
| **核心价值** | **信息聚合与清洗**，解决“数据哪里找”的问题。 |
| **最新动态 (2024–2025)** | 已从 Terminal 演进为 **OpenBB Platform** 为主：**v4.6.0**（2025-01）移除 Python 3.9、Account Module 与 HubService，新增大宗商品与经济数据集；GitHub 约 59.8k stars，推荐以 Platform 方式安装而非旧版 Terminal 源码。 |
| **对谛听的价值** | Module A（语义分类器）与知识图谱所需宏观/基本面数据的最佳“连接器”；可与 AkShare 等配合覆盖国内外数据。 |

### 2. AkShare

| 维度 | 内容 |
|------|------|
| **一句话定义** | 专为**中国金融市场**（股票、期货、基金、宏观）而生的 Python 金融数据接口库。 |
| **核心功能** | 爬取并清洗公开数据；接口统一、免费开源；覆盖东方财富、同花顺、腾讯、新浪、雪球等。 |
| **技术栈** | Python；截至 2026 年初约 16k+ stars，145+ 版本。 |
| **核心价值** | **本土化数据的首选**，弥补国际开源项目对 A 股支持的不足。 |
| **最新动态 (2024–2025)** | 最新 **v1.18.24**；2025 年深交所地区交易排序数据新增优先股/期权交易额字段；生态上出现 **openbb_akshare** 扩展，可将 AkShare 数据无缝接入 OpenBB。 |
| **对谛听的价值** | L2 领域知识库所需的“猪肉价格”“一号文件”“龙虎榜”等数据，AkShare 是最便捷且统一的来源。 |

### 3. ccxt (CryptoCurrency eXchange Trading Library)

| 维度 | 内容 |
|------|------|
| **一句话定义** | 抹平全球 100+ 加密货币交易所 API 差异的**统一适配器**。 |
| **核心功能** | 标准化 `fetch_ticker`、`create_order`、`fetch_balance` 等；REST + CCXT Pro WebSocket 实时行情与下单。 |
| **技术栈** | JavaScript / TypeScript / Python / C# / PHP / Go。 |
| **核心价值** | 跨所套利与聚合交易终端开发时大幅降低对接成本。 |
| **最新动态 (2024–2025)** | 支持 **107+ 交易所**；约 40.9k stars、8.5k+ forks，至 2026 年初仍活跃维护；CCXT Pro 提供 WebSocket 流式接口。 |
| **对谛听的价值** | 若涉足加密资产或跨市场数据，可作为统一行情/交易接口层。 |

## 二、定价、协议与基础设施

### 4. QuantLib

| 维度 | 内容 |
|------|------|
| **一句话定义** | 金融衍生品定价与风险管理的**工业级“物理定律”**库。 |
| **核心功能** | 期权、债券、互换、CDS 等主流工具定价；蒙特卡洛、有限差分等数学方法。 |
| **技术栈** | C++ 核心；官方 Python / C# / Java / R 绑定；修改版 BSD 许可。 |
| **核心价值** | 华尔街与顶级投行的事实标准；复杂奇异期权定价、债券久期/凸性的权威基准。 |
| **最新动态 (2024–2025)** | **v1.41**（2025-01）：移除 1.36 已弃用项；弃用部分实验头文件及旧兼容头文件；修复 GsrProcessCore 崩溃与 observer 生命周期问题；Python 绑定可通过 PyPI 安装无需自编译。 |
| **对谛听的价值** | 若涉及期权/债券等定价或风控指标，可作为数值与定价基准。 |

### 5. QuickFIX

| 维度 | 内容 |
|------|------|
| **一句话定义** | 全球机构间电子交易通信的**FIX 协议“通用语言引擎”**。 |
| **核心功能** | 完整实现 FIX (Financial Information eXchange)；报价、订单、结算等消息标准。 |
| **技术栈** | C++ (QuickFIX) / Java (QuickFIX/J) / .NET (QuickFIX/n) / Go。 |
| **核心价值** | 机构对接（交易所撮合、暗池等）时难以绕开的基础设施。 |
| **最新动态 (2024–2025)** | FIX 规范更新至 EP284（2023-11）；QuickFIX 支持 XML 规范校验；GitHub 约 1.9k stars、823 forks，持续维护。 |
| **对谛听的价值** | 若未来对接机构柜台或交易所，可参考其 FIX 实现与配置模式。 |

### 6. TA-Lib (Technical Analysis Library)

| 维度 | 内容 |
|------|------|
| **一句话定义** | 技术分析指标计算的**标准计算器**。 |
| **核心功能** | 150+ 指标（MACD、RSI、Bollinger Bands、ADX 等）及 K 线形态识别。 |

| **最新动态 (2024–2025)** | Python 版 **v0.6.8**（2024-01）；支持 Pandas 与 Polars；约 11.6k stars、1.9k+ forks；BSD 许可。 |
| **对谛听的价值** | Module B（左脑-量化扫描）与 Scanner 技术面指标可直接或间接依赖，保证口径统一。 |

## 三、策略研发、回测与因子分析

### 7. Qlib (Microsoft)

| 维度 | 内容 |
|------|------|
| **一句话定义** | 微软推出的、面向 **AI 的量化投资平台**，解决“AI 模型在金融数据上的训练与回测”难题。 |
| **核心功能** | 数据管道、ML 模型库（LightGBM/LSTM/Transformer）、回测引擎；支持监督学习、强化学习与市场动态建模。 |
| **技术栈** | Python；MIT 许可；约 37k+ stars，140+ 贡献者。 |
| **核心价值** | **AI 工作流标准化**：从数据 ETL 到 Alpha 挖掘的标准流程，擅长非结构化数据与复杂 ML 模型。 |
| **最新动态 (2024–2025)** | **v0.9.7**（2025-08）；引入 **RD-Agent**——基于 LLM 的自治系统，自动化因子挖掘与模型优化；支持 Python 3.7–3.8，Windows/Linux/macOS。 |
| **对谛听的价值** | Module B 可直接借鉴 **YAML 配置驱动** 的训练与回测模式；其基于 NumPy 的存储格式比 CSV 快一个数量级，适合大规模因子与特征工程。 |

### 8. VectorBT (VectorBT Pro)

| 维度 | 内容 |
|------|------|
| **一句话定义** | 基于 Numba 与 Pandas 的**超高性能向量化回测**库，速度较传统逐笔回测可快两个数量级。 |
| **核心功能** | 矩阵运算实现全市场/多参数扫描；网格搜索与参数优化；不做逐笔模拟，侧重“粗筛”。 |
| **技术栈** | Python + Numba + Pandas。 |
| **核心价值** | **极速扫描与参数优化**，适合全市场技术面/因子初筛。 |
| **最新动态** | 社区持续使用；商业版 VectorBT Pro 提供更多功能；与谛听 Scanner 的“粗筛 + 精算”分工高度契合。 |
| **对谛听的价值** | 用于 Scanner 全市场粗筛（如技术面得分 > 70），再交给 DeepSeek 等做“精算”，效率最优。 |

### 9. Zipline (原 Quantopian 核心引擎)

| 维度 | 内容 |
|------|------|
| **一句话定义** | Python 事件驱动回测的**老牌经典**，定义 initialize / handle_data 等标准范式。 |
| **核心功能** | 事件驱动回测、避免前视偏差；Pipeline API 处理横截面数据。 |
| **技术栈** | Python + Cython；Apache 2.0。 |
| **核心价值** | 许多国内量化平台（聚宽、米筐等）的 API 设计深受其影响，是学习回测逻辑的经典教材。 |
| **最新动态 (2024–2025)** | 最新 release **v1.4.1**（2020-10）；2024-02 仍有代码更新；约 19.4k stars、4.9k forks；存在兼容性与数据依赖（如 treasury）问题，新项目可优先考虑更活跃的框架，但架构思想仍可借鉴。 |
| **对谛听的价值** | 回测框架设计与 Pipeline 式横截面数据处理可作架构参考。 |

### 10. Lean (QuantConnect)

| 维度 | 内容 |
|------|------|
| **一句话定义** | 架构最严谨、支持多资产混合回测的**云原生算法交易引擎**。 |
| **核心功能** | 股票、期货、外汇、期权、加密货币混合回测；Tick 级、滑点/手续费/公司行动（分红拆股）建模严谨。 |
| **技术栈** | C# 核心；支持 Python 3.6 / C# / F#；Docker 镜像 quantconnect/lean；LEAN CLI 本地开发与实盘。 |
| **核心价值** | **机构级仿真度**与**分层架构**：Universe Selection → Alpha Model → Portfolio Construction → Execution Model 四层解耦。 |
| **最新动态 (2024–2025)** | 持续 CI/CD 与版本标签；Docker 镜像近期仍有更新；贡献者可通过 PR 获得 QuantConnect Prime 订阅奖励。 |
| **对谛听的价值** | Repo-I 与整体引擎设计可模仿其四层解耦与仿真度要求，不限于语言。 |

### 11. Alphalens (原 Quantopian)

| 维度 | 内容 |

## 七、区块链与结算

### 19. Hyperledger Fabric

| 维度 | 内容 |
|------|------|
| **一句话定义** | 企业级、**许可制 (Permissioned)** 的分布式账本 (DLT) 框架。 |
| **核心功能** | 私有链与身份准入；智能合约 (Chaincode)；共享账本与隐私/合规平衡。 |
| **技术栈** | Go、Java、Node.js。 |
| **核心价值** | 银行结算、供应链金融、贸易融资等领域首选，满足“共享账本 + 隐私 + 准入”需求。 |
| **最新动态 (2024–2025)** | **v2.5** 为当前 LTS，季度补丁；**v3.0**（2024-09）引入 **SmartBFT 拜占庭容错共识**，排序服务可多机构参与、无需单一信任方；Fabric 已迁至 LF Decentralized Trust 基金会；IBM 仍为主要贡献者。 |
| **对谛听的价值** | 若涉及机构结算、合规账本或多方协作清结算，可评估 Fabric 的链码与网络模型。 |

## 八、总结与谛听选型建议

| 层次 | 建议 |
|------|------|
| **数据层** | **AkShare**（中国数据）+ **OpenBB**（国际/宏观）；加密资产可选 **ccxt**。 |
| **策略研发层** | **Qlib**（AI 流程与配置驱动）+ **VectorBT**（快速全市场扫描）；因子质检 **Alphalens**。 |
| **核心引擎层** | 参考 **Vn.py**（Gateway 设计）与 **Lean**（四层架构与仿真度）。 |
| **定价与指标** | **QuantLib**（衍生品/债券）；**TA-Lib**（技术指标统一口径）。 |
| **风控与仓位** | 直接集成 **PyPortfolioOpt** 于 Module D（判官）。 |
| **运维与交互** | 参考 **Freqtrade** 的 Telegram/WebUI 与 Dry-Run 体验，用于守夜人与运营治理。 |
| **协议与机构** | 机构对接时评估 **QuickFIX**；高性能/高频方向参考 **NautilusTrader**。 |

以上项目均具备高工程化程度与活跃社区，可与谛听的 Python/K8s/AI 栈形成**数据—研发—执行—风控—运维**的完整互补；选型时以“数据与研发先行，执行与运维对齐成熟实践”为原则，逐步引入并固化到 Repo 与部署体系中。
