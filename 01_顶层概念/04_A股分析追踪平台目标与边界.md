## 产品定义

谛听的产品侧，是一个面向个人投研与后续平台化演进的 `A股分析追踪智能平台`。

它的第一阶段重点不是“全自动交易”，而是“把高价值研究和持续跟踪做成系统”。

产品方法论不是价格预测，而是**认识论套利**：

- 市场对高维非结构化信息存在认知摩擦
- 系统通过语义压缩和跨源拼图提前形成结论
- 结论必须经状态机监控与门禁约束进入执行层

## 产品四维主模型（定义、边界、目标）

### 1. 维度 A：极寒防御模型（Fraud Deconstruction Engine）

- **模型定义**：对财务、治理、供应链、叙事做交叉验尸，输出高危熔断信号
- **边界**：
  - 做：风险识别、熔断建议、拉黑策略
  - 不做：自动交易执行
- **目标**：优先规避地基腐烂资产，降低组合尾部风险
- **代表案例目标**：识别存贷背离、商誉陷阱、关联交易空转、实控人逃逸

### 2. 维度 B：纵深进攻模型（Deep-Chain Momentum Harvester）

- **模型定义**：通过产业链暗网信息与弱信号提纯，构建预期差机会池
- **边界**：
  - 做：证据拼图、剧本评分、机会排序
  - 不做：单源情绪叙事追涨
- **目标**：在市场共识前形成高赔率候选
- **代表案例目标**：剪刀差、S 曲线拐点、卖铲人瓶颈、供给出清反转

### 3. 维度 C：状态机监控模型（Thesis State Machine）

- **模型定义**：把持仓逻辑转成可观测状态机（而不是主观叙事）
- **边界**：
  - 做：逻辑快照、探针监控、状态迁移、动作建议
  - 不做：事后解释型“复盘文学”
- **目标**：对抗情绪交易，提升持仓纪律
- **代表案例目标**：逻辑证伪熔断、Alpha 衰减止盈、叙事漂移纠偏、机会成本重调度

### 4. 维度 D：超级个体进化模型（Cognitive Cloner）

- **模型定义**：将人工复盘、纠错、经验沉淀为可迭代系统能力
- **边界**：
  - 做：反馈采集、规则更新、评测验证、灰度发布
  - 不做：只留文档不更新系统
- **目标**：让系统能力随使用时间复利增长
- **代表案例目标**：失败样本回灌、规则版本化、周期性效果提升

## MVP 边界

当前阶段必须做：

- A/B/C 三个维度最小闭环（防御、进攻、状态机）
- 数据采集与整理、结构化研究卡片、候选池与风险池
- thesis 快照、关键探针、状态迁移与提醒
- 每条建议具备证据链与复盘入口

当前阶段不优先做：
- 自动交易闭环
- 面向公众的大型 SaaS 后台
- 为假想用户做复杂权限和结算系统
- 多市场扩张（先完成 A 股单市场高质量闭环）

## 演进路径

| 阶段 | 产品形态 | 主模型目标 |
|---|---|---|
| Phase A | 个人研究助手 | A 模型可用，C 模型可记录逻辑快照 |
| Phase B | 结构化跟踪系统 | B 模型可产出候选，C 模型可执行状态迁移 |
| Phase C | 多 Agent 研究工作流平台 | A/B/C 协同运行，D 模型开始反哺 |
| Phase D | 可平台化的双目标系统 | 四维模型稳定迭代并形成资产复利 |
