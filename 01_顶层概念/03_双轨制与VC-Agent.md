## 一、双轨制定义与资金比例

双轨制定义、资金比例与两轨协同价值见 [01_一句话定义与核心价值](./01_一句话定义与核心价值.md)#双轨制与两轨分工。下文仅给出 TimeHorizon 语义、判官豁免、VC-Agent 与分账/复盘分轨等实现与协议细节。

## 二、TimeHorizon 语义与使用场景

协议定义见 `diting-core/design/protocols/brain/expert.proto`：

- **SHORT_TERM（或未设置）**：现金奶牛轨；看几天；判官施加 2% 硬止损、现金拖累、不可能三角约束。
- **LONG_TERM**：长期捕手轨；看几年；判官豁免 2% 硬止损与现金拖累，仅施加逻辑证伪与大周期反转规则。

Router 或专家在输出 `ExpertOpinion` 时设置 `horizon`；判官（Module D）根据该字段分支执行。

## 三、判官对 LONG_TERM 的豁免规则

当信号或专家意见携带 `TimeHorizon = LONG_TERM` 时：

- **豁免**：不施加 2% 硬止损（单笔亏损 > 2% 不触发市价斩仓）；不施加现金拖累监控（Cash Drag Monitor 不作用于该轨仓位）。
- **仍施加**：
  - **逻辑证伪止损**：产业趋势或基本面逻辑被证伪时止损（例如：核心假设不再成立、政策/技术路线颠覆）。
  - **大周期反转止盈（Major Trend Reversal）**：识别到标的或行业进入长期下行周期时止盈/止仓（可执行定义见下）。

**大周期反转（Major Trend Reversal）可执行定义**（最小可落地版本）：
- **输入**：标的或行业维度的价格/均线/波动率等趋势指标（如长期均线死叉、行业指数跌破关键支撑、波动率 regime 切换）。
- **判定主体**：VC-Agent 或判官（Module D）内独立子模块；具体实现与接口见 [09_ 核心模块架构规约](../03_原子目标与规约/_共享规约/09_核心模块架构规约.md) Module D。
- **输出**：止盈/减仓信号，供执行层平仓或减仓；与判官/执行层接口由 L3 规约约定。
- **逻辑证伪止损可执行定义**：**触发来源**：基本面事件（如财报暴雷、核心假设被推翻）、政策/技术路线颠覆、或 VC-Agent/独立逻辑证伪模块的判定。**输入/输出**：输入为事件或模块判定结果，输出为「逻辑证伪止损」指令；判官（Module D）接收后生成平仓指令，详见 L3 Module D 规约。

详见 [09_ 核心模块架构规约 Module D 双轨分流](../03_原子目标与规约/_共享规约/09_核心模块架构规约.md)。

## 四、VC-Agent 在 MoE 中的位置与 Repo-I

- **位置**：Module C 专家池中的**信仰专家（VC-Agent）**；Router 可根据标的或资金账户类型将请求路由至 VC-Agent，与 Agri/Tech/Geo 等短轨专家并行或按配置选用。
- **Router 与 VC-Agent 衔接**：路由条件（标的类型、账户/资金池、配置开关）、与 Agri/Tech/Geo 的互斥或并行策略、VC-Agent 输出 `LONG_TERM` 与判官消费方式，由 L3 [09_ 核心模块架构规约](../03_原子目标与规约/_共享规约/09_核心模块架构规约.md) Module C 约定。
- **输出**：VC-Agent 输出的 `ExpertOpinion` 设置 `horizon = TimeHorizon.LONG_TERM`，以便判官对该轨施加差异化规则。
- **数据源**：Repo-I 中 VC-Agent 接入**基本面数据**（如 AkShare 财报、营收增速、研发投入占比），与 A 轨技术面信号区分；不依赖 RSI/MACD 等短轨指标。
- **逻辑**：基本面驱动、买入即锁仓、仅在大周期反转或逻辑证伪时平仓。

## 五、A/B 轨分账与复盘分轨

- **分账**：资金账户或仓位按轨标识（A 轨 / B 轨）分账管理，避免 B 轨仓位被 A 轨风控误伤。
- **B 轨资金与分账规则**：**初始划拨**：B 轨约 20% 资金由配置或账户分账约定确定。**A 轨利润输血**：可选；触发条件与上限（如年度/季度、比例上限）由策略或运营约定，执行主体为资金调度或人工确认，具体由 L3 与 08_经纪商解耦维度 细化。**再平衡**：再平衡频率（如半年度）与执行主体（系统或人工）由 L3 规约约定；与 03 分账、08 账户/仓位标识一致。
- **复盘分轨**：复盘与归因时 A 轨与 B 轨分开统计；A 轨用 WinRate/CAGR/MaxDD 等现有指标，B 轨可单独统计 B 轨 MaxDD、平均持仓周期、VC-Agent 信号数与执行率；不用 A 轨标准误评 B 轨设计。

## 六、A/B 轨规范边界

以下规则、模块与指标按轨适用；01/02 中涉及「仅 A 轨」「仅 B 轨」的表述均以本表为准。

| 规则/模块/指标 | 仅 A 轨 | 仅 B 轨 | 共用 |
|----------------|--------|--------|------|
| 2% 硬止损 | ✓ | — | — |
| 现金拖累监控（Cash Drag Monitor） | ✓ | — | — |
| 不可能三角（胜率/年化/回撤 KPI） | ✓ | — | — |
| 逻辑证伪止损 | — | ✓ | — |
| 大周期反转止盈（Major Trend Reversal） | — | ✓ | — |
| 判官分流（按 TimeHorizon 分支） | SHORT_TERM 分支 | LONG_TERM 分支 | Module D 统一入口 |
| 数据防火墙 / 标的池 | 可区分 | 可区分 | 按账户或配置 |
| 账户/仓位标识 | A 轨 | B 轨 | 分账管理 |
| 复盘指标 | WinRate、CAGR、MaxDD、Risk Shield 拦截率 | B 轨 MaxDD、持仓周期、VC-Agent 信号数与执行率、逻辑证伪/反转触发 | 分轨统计，不混用 |

## 合规与风险说明（可选）

当实盘涉及机构或合规要求时，建议：**披露双轨设计**（明确 A 轨与 B 轨的资金比例与规则差异）；**披露 B 轨风险更高**（B 轨豁免 2% 硬止损，MaxDD 可高于 12%）；**留痕与审计**（B 轨仓位、逻辑证伪/大周期反转触发记录可追溯，分轨复盘不与 A 轨混用）。具体合规策略由运营或法务按属地要求细化。
